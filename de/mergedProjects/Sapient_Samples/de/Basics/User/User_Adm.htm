Begriffserklärung

Begriff

Definition

Sinnvolle Aktionen

Person

  • Personen haben einen Vornamen, Nachnamen und weitere persönliche Daten.

  • Eine Person ist jemand, die im System angelegt ist. Diese Person entspricht einer natürlichen Person.

  • Es muss nicht notwendig sein, dass sich jede Person am System anmelden kann.

  • Personen können deaktiviert werden.

  • Durch das Deaktivieren von Personen werden alle persönlichen Daten gesichert, gleichzeitig jedoch auch anonymisiert.

  • Durch das Deaktivieren von Personen werden alle Benutzer/User dieser Person werden gesperrt.

  • Durch das Deaktivieren von Personen wird der Loginname jedoch auch gesichert und durch eine bedeutungslose Kennung ersetzt.

  • Personen können auch wieder reaktiviert werden. In diesem Falle werden die gesicherten Daten wieder hergestellt.

  • Zusätzlich kann man auch den Zeitpunkt planen, an dem die Person deaktiviert werden soll. Für diesen Zeitplan wird der Systemparameter -270 verwendet.

  • Personen können auch gelöscht werden. Dies ist sowohl für eine inaktive oder aktive Person möglich. Hierbei werden die Daten ohne Möglichkeit der Wiederherstellung gelöscht.

  • Beim Löschen von Personen werden auch alle sonstigen Verweise, z.B. Audit Logs von Aktionen eines Benutzers/Users auf eine Systemperson übertragen.

Benutzer/User

  • Benutzer und User können synonym verwendet werden.

  • Benutzer/User haben Login-Accounts (Benutzername, Kennwort, Berechtigungen) und können sich am System anmelden.

  • Benutzer/User werden Benutzergruppen zugeordnet, um Rechte in der Applikation zu erlangen.

  • Benutzer/User können gesperrt werden.

  • Die Sperrung ist entweder zum aktuellen Zeitpunkt möglich und tritt sofort in Kraft oder die Sperrung kann für die Zukunft geplant werden (Setzen eines Ablaufdatum des Benutzers/Users).

  • Eine geplante zukünftige Sperrung eines Benutzers/Users ist z.B. für Saisonarbeiter geeignet.

Person reaktivieren

Neben dem Deaktivieren einer Person ist auch eine Reaktivierung möglich. Die Deaktivierung (inkl. temporäre Anonymisierung) wird hierdurch rückgängig gemacht.

 

Führen Sie folgende Schritte aus, um die Person wieder zu reaktivieren:

1. Öffnen Sie den LC2.

2. Navigieren Sie zum Abschnitt Stammdatenverwaltung und klicken Sie auf die Tabelle Personen.

2.1 Deaktivierte Personen lassen sich anhand des Namens (z.B. Vorname, Nachname) nicht unterscheiden. Für das Reaktivieren ist es jedoch notwendig, zwischen den deaktivierten Personen zu unterscheiden, damit die richtige Person reaktiviert werden kann.

Beachten Sie hierzu die Spalte Deaktivierte Person. In dieser Spalte werden die deaktivierten Personen anhand der früheren Nutzerkennung der anonymisierten Person angezeigt.

 

3. Wählen Sie in der Tabelle Personen die Person aus, die Sie reaktivieren möchten, indem Sie mit der rechten Maustaste am Anfang der Tabelle klicken.

4. Im Kontextmenü haben Sie die Möglichkeit, die Person zu reaktivieren, indem Sie auf den Eintrag Person reaktivieren klicken.

5. Klicken Sie auf die Schaltfläche Aktualisieren, um die Änderungen zu sehen.

6. Die Person wird wieder aktiviert.

7. In der Spalte Deaktiviert ID erscheint wieder der Eintrag NULL.

 

Person löschen (GDPR)

Die Person wird aus dem System entfernt und die personenbezogenen Daten werden irreversibel anonymisiert.

Beachten Sie außerdem:

  • Die zugeordneten Benutzer werden ebenfalls gelöscht.

  • Das Löschen von Personen (Delte Person (GDPR)) ist für aktive und inaktive Personen möglich. Es besteht keine Möglichkeit, die Daten wieder herzustellen.

  • Wenn Personen gelöscht werden, werden zusätzlich weitere Verweise (z.B. Audit Logs von Aktionen eines Benutzers/Users) auf eine Systemperson übertragen.

 

Login sperren

Beim Sperren des Logins eines Benutzers/Users gilt Folgendes:

 

  • Durch das Sperren eines Logins werden keine Daten gelöscht, d.h. die Datensicherheit des Logins ist weiterhin gewährleistet.

  • Der Benutzer/User kann sich nicht mehr am System anmelden.

  • Es kann nach dem Benutzer/User des gesperrten Logins gefiltert werden.

  • Dem Benutzer/User können Aufgaben für die Zukunft zugeordnet werden.
