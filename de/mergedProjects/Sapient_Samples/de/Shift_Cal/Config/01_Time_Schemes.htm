Funktion

Übergeordnete Muster (Schemata) für Arbeitszeitmodelle. Je Arbeitsbereich mit unterschiedlichem Arbeitszeitmodell wird ein Datensatz angelegt, z.B. <Rohbau> <Lackiererei> und <Montage>.

Ein Zeitschema ist durch einen Kalender und die darin zugewiesenen Schichtmodelle (in diesen sind die Schicht- und Pausenzeiten festgelegt) definiert. Für ein Legato-Leitsystem können nun mehrere unabhängige Zeitschemata festgelegt und unter einem wählbaren Namen gespeichert werden.

Die Verknüpfung von Fertigungsbereichen, d.h., Produktionslinien oder Anlagen, mit einem Zeitschema erfolgt in der Definition der Fertigungsstruktur. Dabei kann jedem Knoten eines der vorhandenen Zeitschemata zugewiesen werden.

ID der Java Zeitzone, zu der das Zeitschema gehört. Welche Einträge möglich sind, ist in Java Zeitzonen beschrieben. Es muss die ID eingetragen werden, z.B. Europe/Berlin

 

Beispiel

Die im Legato-Projekt abzubildende Fertigungsstruktur umfasst die Produktionsbereiche Vormontage, Montage und Logistikhalle. Vormontage und Montage arbeiten im 3-Schicht-Betrieb, der Versand in der Logistikhalle arbeitet in Normalschicht.

Es werden zwei Zeitschemata parametriert, die als Montagebereich und Logistikbereich bezeichnet werden. Später werden diese Zeitschemata in der Fertigungsstruktur den Hallenknoten Vormontage und Montage (Zeitschema Montagebereich), sowie der Logistikhalle (Zeitschema Logistikbereich) zugeordnet.
