Script

Diese Ansicht ermöglicht die Eingabe von JavaScript.

Neben internen Skripten können auch externe Skripte betrachtet und verändert werden.

Der Kopfbereich dieses Tabs bietet eine Drop-Down-Box an, die sämtliche JavaScript-Quellen auflistet. Prinzipiell kann eine SVG-Datei beliebig viele externe und interne Skripte referenzieren.

Ist keine Quelle in dem Bild vorhanden, steht nur der Eintrag "Main" zur Verfügung. Dort eingetragener Code wird in den Header des SVG-Quellcodes geschrieben.

Je nach Auswahl wird im unteren Bereich der Inhalt der Skript-Quelle angezeigt. Änderungen werden in den jeweiligen Quellen nachgezogen, sobald diese Ansicht verlassen wird, oder der Speichern-Button betätigt wird. Bei externen Quellen wird nochmals explizit nachgefragt.

Die Verwaltung (löschen, neu anlegen) der Quellen muss noch von Hand im Sourcecode erfolgen.

Externe Quellen werden dadurch gekennzeichnet, dass die Checkbox "External Script" aktiviert ist. Wird der Haken entfernt, wird die externe Referenzierung entfernt und der JavaScript-Code in die Datei eingefügt.

Andersherum kann aus einer lokalen Quelle durch Anhaken dieser Option eine externe Quelle gemacht werden. Jedoch geht dabei der Code der lokalen Quelle verloren, da nur bereits bestehende Quellen eingelesen werden können.

Bei externen Quellen kann es vorkommen, dass die Datei vom Designer nicht gefunden werden kann. Dann wird die Code-Ansicht ausgeblendet, der Link bleibt jedoch erhalten.

Durch den Designer wird kein Debugging durchgeführt, das heißt etwaige Fehler treten frühestens in der Preview auf.

Als Fenster wird in diesem Dialog nur der DOM angeboten, jedoch ohne die Möglichkeit zur Interaktivität.

Tastaturabkürzungen entsprechen den angeführten Abkürzungen des Source-Editors.

Die Syntax der Java-Skripten kann immer mittels Syntax Check (F8) geprüft werden (siehe auch Seite 15).
