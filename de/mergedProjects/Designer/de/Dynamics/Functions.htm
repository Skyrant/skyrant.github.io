SetDbVal

Mittels dieser Funktion kann eine Variable in der Datenbank geändert werden. Durch Definition eines Ereignisses (z.B. Mausklick) wird an die ausgewählte Variable ein festgelegter Wert übergeben.

In dem Feld "Event" kann ein Ereignis ausgewählt werden, bei dessen Eintreffen die Wertänderung ausgelöst wird. Folgende Ereignisse stehen zur Auswahl:

Event

Beschreibung

OnClick

Tritt ein, wenn ein Objekt angeklickt wird.

OnMouseOver

Tritt ein, wenn ein Objekt mit der Maus berührt wird.

OnMouseMove

Tritt ein, wenn die Maus bewegt wird.

OnMouseOut

Tritt ein, wenn ein Objekt mit der Maus berührt wurde und wieder verlassen wird.

OnMouseDown

Tritt ein, wenn die Maustaste gedrückt bleibt.

OnFocusIn

Tritt ein, wenn ein Objekt den Fokus bekommt.

OnFocusOut

Tritt ein, wenn ein Objekt den Fokus verliert.

OnActivate

Tritt ein, wenn das Element durch einen Zeiger aktiviert wird.

 

Im Feld "Action" kann die beim Event auszuführende Aktion gewählt werden.

Option

Beschreibung

DbSetVal

Wert wird beim Event direkt gesetzt.

DbEditNum/Bool/Str

Eingabedialog im Webclient ermöglicht Werteingabe zur Laufzeit für numerische, Boolesche oder String-Werte..

 

In "Value" kann der Wert eingetragen werden, der bei Eintritt des Ereignisses übergeben werden soll.

Die "Position" gibt an, wie die Wertübergabe gehandhabt werden soll.

Option

Beschreibung

Replace

Überschreibt den alten Wert (ABC -> D).

Beginning

Wird vor den alten Wert gesetzt (ABC -> DABC).

End

Wird an das Ende des alten Werts gesetzt (ABC -> ABCD).

 

Eine Variable kann erst ausgewählt werden, wenn die vorangegangenen Optionen eingetragen wurden. Zu diesem Zweck mit dem Button "Add" die Einstellungen aktivieren.

Die getroffenen Parametrierungen werden in die Liste eingetragen. Anschließend kann über den Button bei Variable der "Select Variable" Dialog geöffnet werden.

"Delete" löscht einen Eintrag aus der Tabelle.

Mit "Script" kann das erzeugte Skript betrachtet und verändert werden, dies geschieht im Script-Dialog. Die Dynamic-Funktion wird mit OK aktiv.

SetLink

Über diese Funktion kann eine Verlinkung von Elementen mit Knoten in der Fertigungsstruktur oder mit Ansichten (z.B. Messwerte, Top10, usw.) der Webvisualisierung von Legato erzeugt werden. Diese Dynamik ist nur für <a> Tags verfügbar. Dazu muss zunächst dem Objekt / den Objekten ein Link zugewiesen werden, für den dann diese Dynamik zur Verfügung steht.

Mit "Link to…" kann der Link-Typ bestimmt werden.

Link Typ

Beschreibung

No Link

Kein Link – kann zum Löschen eines Links genutzt werden.

SvgLinktoNode

Bleibt in derselben Ansicht, springt zu einem definierten Knoten der Fertigungsstruktur.

SvgLinktoView

Bleibt auf demselben Knoten, springt in eine definierte Ansicht.

SvgLinktoSubview

Bleibt auf demselben Knoten, springt aber zu einem definierten SVG Bild, das nicht mit einem Knoten verlinkt sein muss.

SvgLinktoNodeAndView

Springt zu einem definierten Knoten und zu einer definierten Ansicht.

SvgLinktoNodeViewAndParams

Springt zu einem definierten Knoten und zu einer definierten Ansicht und öffnet diese gemäß den übergebenen Parametern (z.B. mit einem bestimmten Filter).

Beispiel für node, view, params:

SvgLinktoNodeViewAndParams(5,'ansichtname.html','param1=5&param2=7')

Beispiel für subview, params:

SvgLinktoSubviewAndParams('CntAggrTrend.html', 'filterName=test'

 

Je nach Auswahl des Link-Typs können die anderen Eingabefelder gefüllt werden.

Mit "View" kann eine andere Ansicht des Webclients gewählt werden. Ansichten sind in diesem Zusammenhang alle Auswahlpunkte im oberen Frame der Visualisierung. Nach Anklicken des Buttons öffnet sich ein Dialog mit einer Auswahl aller möglichen Ansichten. Es besteht auch die Möglichkeit, die ID der Ansicht direkt in das Textfeld einzutragen. Eine getroffene Auswahl kann über den Button "Set to empty value" rückgängig gemacht werden.

Das Feld "Node" bietet eine Auswahl über die Knoten der Fertigungsstruktur. Wie bei "View" kann auch hier in dem Textfeld die ID des Knotens direkt eingegeben werden.

Das Feld "Subview" ist ein reines Eingabefeld. Hier kann ein anderes SVG-Bild eingetragen werden, auf das mit diesem Link gesprungen werden soll. Dabei muss dieses Bild nicht mit einem Fertigungsknoten verbunden sein. Es muss nur wie alle SVG Bilder über die Webapplikation hochgeladen worden sein. Zur korrekten Verlinkung muss das Bild folgendermaßen angegeben werden:

Visu/fs/<Bildname>.svg

Für einige Ansichten können in das Feld "Params" Parameter eingegeben werden, wie beispielsweise ein Filter.

Parameter

Beschreibung

filterName=Name des Filters

Spezifiziert den aufzurufenden Filter. Dieser muss in der WebApp gespeichert sein und für den angemeldeten Benutzer zugänglich sein.

 

Dieser Parameter kann für folgende Ansichten verwendet werden:

Auswertungen -> Meldungen -> Archiv

Auswertungen -> Meldungen -> Pareto

Auswertungen -> Meldungen -> Verlauf

Auswertungen -> Meldungen -> Top-Meldungen

Auswertungen -> Meldungen -> Kreuztabelle

Auswertungen -> KPIs -> KPI-Knoten

Auswertungen -> KPIs -> KPI-Schicht

Auswertungen -> KPIs -> KPI-Zeit

Auswertungen -> KPIs -> Verfügbarkeitsverlauf

Auswertungen -> KPIs -> Verfügbarkeitsanteile

 

Außerdem ist die Verlinkung in der Chart-Ansicht Messwerte möglich. In die Ansicht "KPI-Charts" kann nicht verlinkt würden, hierfür müsste man beispielsweise einen Benutzerreport erstellen, der dann mittels Link aufgerufen wird.

Für die Ansicht "Kreuztabelle" können außerdem folgende Parameter eingegeben werden. Werden mehrere Parameter eingegeben, müssen diese durch ein kaufmännisches Und (&) getrennt werden. Mögliche Parameter sind:

Parameter

Beschreibung

saveStatus=true

Sorgt dafür, dass der Zurück-Button in der Tabelle etc. angezeigt wird.

reportName=Name des Reports

Spezifiziert den aufzurufenden Report. Hier muss der Dateiname der Report-Definition ohne die Erweiterung .properties eingegeben werden.

reportFmt=Format des Reports

Hier kann das Format eingegeben werden, in dem der Report aufgerufen werden soll, z.B. pdf oder csv.

Beispiele

Das genaue Vorgehen bei der Parametrierung hängt von vielen Faktoren ab, die hier den Rahmen sprengen würden. Für detaillierte Beispiele siehe bitte das Workshop-Dokument GefasoftDesigner sowie die Dokumentation zum BIRT-Reporting (insbesondere für das Aufrufen von Benutzerreports). Zwei allgemeine Beispiele sind hier aufgeführt:

SvgLinktoSubviewAndParams(subview, params)

Beispiel: SvgLinktoSubviewAndParams('CntAggrTrend.html', 'filterName=test')

Konkret kann diese Funktion zur Darstellung einer gespeicherten, erweiterten Messwertansicht aus der Visualisierung heraus genutzt werden. Hier wird z.B. die Messwertansicht mit dem Namen "test" aufgerufen. Der Parameter filterName muss dabei dem Namen entsprechen, unter dem die Ansicht in der Filterverwaltung gespeichert wurde (wichtig: die gespeicherte Ansicht muss entweder "für alle Benutzer" oder für den aktuell angemeldeten Benutzer gespeichert worden sein). Wird die gespeicherte Ansicht unter dem angegebenen Namen nicht gefunden oder hat der aktuelle Benutzer keine Berechtigung für diese gespeicherte Ansicht, so wird beim Aufruf eine Fehlermeldung ausgegeben.

Aufruf von Benutzerreports aus SVG

SvgOpenAdhocReportWindow(report, repparams, repformat): Öffnet ein neues Browserfenster, das den spezifischen Benutzerreport ad-hoc ausführt und anzeigt.

Parameter

Beschreibung

report

Dateiname der Reportvorlage, z.B. visutest.rptdesign.

repparams

Reportparameter als Javascript-Objekt, z.B. {par_int: 4711, par_datetime: '12-08-1965 14:10:30'} Datum/Zeit-Parameter müssen immer als Strings im Format dd-MM-yyyy HH:mm:ss übergeben werden. Auch wenn nur der Datum- oder Zeitanteil benötigt werden, muss stets ein kompletter, gültiger Datums/Zeit-String übergeben werden! Für Parameter vom Typ Float oder Decimal muss als Dezimaltrenner immer ein Punkt '.' angegeben werden.

repformat

Format des Reports. PDF und HTML sind möglich.

DynBlink

Mit dieser Funktion kann ein Objekt zum Blinken gebracht werden. Die Sichtbarkeit wird im Halbsekunden-Takt umgeschaltet.

Über den Button neben "TDynBlink" kann die Parametrierung durchgeführt werden. Der Dialog "Dynamic Blink" wird im nächsten Kapitel beschrieben, da er von mehreren Dynamiken verwendet wird.

"Delete" löscht die Dynamik.

DynText

Durch diese Funktion können Werte von Variablen an Text-Elemente übergeben werden.

Der Dialog "Dynamic Text" bietet umfassende Einstellmöglichkeiten, um eine Formatierung der Werte zu erreichen.

Unter "Variable" kann wie in den anderen Funktionen die referenzierte Variable selektiert werden oder mittels "Expression" eine Berechnung von Variablen erstellt werden.

In "Predefined Format" sind verschiedene Standard-Formatierungen definiert. Nur mit der Option "UserDefined" sind die anderen Optionen veränderbar.

Option

Beschreibung

UserDefined

Die Formatierung kann frei editiert werden. Siehe weiter unten.

Text

Darstellung als Text. Der Wert der Variable wird unverändert übernommen.

Boolean

Der Wert wird als Ja/Nein Wert ausgegeben.

Hex

Der Wert wird als Hex-Zahl ausgegeben (Bsp.: FF3E09).

Dec

Der Wert wird als Dezimalzahl (Natürliche Zahl – Integer) ausgegeben.

Float

Der Wert wird als Fließkommazahl dargestellt (2 Nachkommastellen mit gerundeter letzter Stelle).

Binary

Der Wert wird als Binärzahl dargestellt (Bsp.: 10110001).

 

Wurde "UserDefined" als Option gewählt, kann im Feld "ValueType" der Grundtyp der Variable gewählt werden. Nach dieser Auswahl richten sich die anderen Parametriermöglichkeiten.

Option

Beschreibung

Text

Der Wert der Variable wird unverändert dargestellt. Keine weiteren Einstellmöglichkeiten.

Integer

Der Wert wird als Ganzzahl dargestellt.
Über Integer Base kann die Basis der Darstellung gewählt werden.

2 = Binärzahl

8 = Oktalzahl

10 = Dezimalzahl

16 = Hexadezimalzahl

Float

Die Zahl wird als Fließkommazahl dargestellt.

 

Über "Decimal places" kann die Anzahl der Nachkommastellen angegeben werden. Die Checkbox "Round" rundet auf die letzte Nachkommastelle.

Soll zusätzlich zur Formatierung noch eine Berechnung einer Variablen durchgeführt werden, kann dies entweder direkt im Scriptcode erfolgen, oder man verwendet die Expression-Funktion des Designers. Diese Funktion ist weiter unten beschrieben.

"Delete" löscht die Dynamik.   

DynHeight

Mit "DynHeight" kann das height-Attribut (Höhe) durch eine Variable oder eine Berechnung von Variablen beeinflusst werden.

Der "Scale Range" Dialog, der zur Parametrierung dient, wird im nächsten Kapitel näher beschrieben.

Die zu referenzierende Variable kann über den "Select Variable-Dialog" ausgewählt werden. Zusätzlich kann über die "Expression-Funktion" eine Berechnung von Variablen parametriert werden. Diese Dialoge sind im nächsten Kapitel beschrieben.

Der "Reference Point" ist in dieser Hinsicht ebenfalls von Bedeutung. Hier kann der Ort gewählt werden, der als Fixpunkt dienen soll. Außerdem kann die Einheit selektiert werden, in der die Änderung erfolgen soll.

Option

Beschreibung

Top

Die obere Kante wird als Fixpunkt genutzt, d.h., die untere Kante des Objektes wird je nach Variablenwert verändert (wandert nach unten je größer der Wert).

Bottom

Die untere Kante wird als Fixpunkt genutzt, d.h. die obere Kante des Objektes wird je nach Variablenwert verändert (wandert nach oben je größer der Wert).

Center

Die Mitte des Objektes bleibt fix, beide Kanten bewegen sich je nach Wert nach außen.

 

"Delete" löscht die Dynamik.

DynWidth

Mit "DynWidth" kann das width-Attribut (Breite) durch eine Variable beeinflusst werden. Die Funktionalität ist analog der DynHeight-Funktion.

Nur der "Reference Point" unterscheidet sich in seiner Funktionalität und Auswahlmöglichkeit bezüglich des Fixpunktes.

Option

Beschreibung

Left

Die linke Kante des Objektes bleibt unverändert, die rechte Kante wandert je nach Größe des Variablenwerts nach rechts.

Right

Die rechte Kante des Objektes bleibt unverändert, die linke Kante wandert je nach Größe des Variablenwerts nach links.

Center

Die Mitte des Objektes ist fix, beide Kanten wandern gleichmäßig nach außen.

 

"Delete" löscht die Dynamik.

DynFillColors

Diese Funktion bewirkt eine Änderung der Füllfarbe eines Objektes.

 Über "TDynFillColors" kann der Parametrierungsdialog geöffnet werden, dieser wird im folgenden Kapitel beschrieben (Dialog Select Colors).

"Delete" löscht die Dynamik.

DynFillOpacity

Mit DynOpacity kann die Deckkraft einer Füllung gesteuert werden. Der "Scale Range" Dialog wird nachfolgend beschrieben.

"Delete" löscht die Dynamik.

DynGradient

Mit dieser Funktion kann ein Gradient beeinflusst werden. Voraussetzung ist, dass dem Objekt ein Gradient zugeordnet wurde, ansonsten ist diese Funktion nicht verfügbar. Die Parametrierung der Funktion erfolgt durch den "Scale Range" Dialog, der im nächsten Kapitel beschrieben wird.

"Delete" löscht die Dynamik.

DynOpacity

Diese Funktion arbeitet analog zu "DynFillOpacity", jedoch wird hier für das gesamte Objekt die Deckkraft geregelt. Der Parametrierungsdialog (Scale Range) wird im nächsten Kapitel beschrieben.

"Delete" löscht die Dynamik.

DynStrokeColors

Mit dieser Funktion kann die Farbe der Umrandung durch einen Variablenwert geändert werden. Der zur Parametrierung benötigte "Select Colors" Dialog wird im nachfolgenden Kapitel beschrieben.

"Delete" löscht die Dynamik.

DynStrokeOpacity

Eine Veränderung der Deckkraft der Umrandung kann mit dieser Funktion durchgeführt werden. Der Parametrierdialog (Scale Range) wird später genauer beleuchtet.

"Delete" löscht die Dynamik.

DynStrokeWidth

Mit "DynStrokeWidth" kann die Breite der Umrandung geändert werden. Der Parametrierungsdialog (Scale Range) wird im nächsten Kapitel beschrieben.

"Delete" löscht die Dynamik.

DynVisibility

Mit dieser Funktion kann die Sichtbarkeit eines Objektes geregelt werden. Anders als "DynOpacity" hat dieses Attribut nur zwei Werte (sichtbar oder nicht sichtbar). Der Dialog zur Parametrierung (Select Visibility) wird nachfolgend beschrieben.

"Delete" löscht die Dynamik.

DynXPos

Eine Änderung der X-Koordinaten kann durch diese Funktion bewerkstelligt werden. Die Anwendung der Dialoge wird später beschrieben. Der "Reference Point" gibt an, wo die Verschiebung starten soll. Unter "ResultUnit" wird eingegeben, in welcher Einheit die Verschiebung erfolgen soll.

Option

Beschreibung

Original

Beginnt Verschiebung am Ursprungspunkt, auf dem das Objekt gezeichnet wurde.

Edge

Beginnt Verschiebung an der linken oberen Ecke des Koordinatensystems.

 

"Delete" löscht die Dynamik.

DynYPos

Diese Funktion ändert die Y-Koordinate eines Objektes. Benötigte Dialoge werden später beschrieben. Der "Reference Point" gibt an, wo die Verschiebung starten soll. Unter "ResultUnit" wird eingegeben, in welcher Einheit die Verschiebung erfolgen soll.

Option

Beschreibung

Original

Beginnt Verschiebung am Ursprungspunkt, auf dem das Objekt gezeichnet wurde.

Edge

Beginnt Verschiebung an der linken oberen Ecke des Koordinatensystems.

 

"Delete" löscht die Dynamik.
