Inspector / Properties

Das Fenster Properties bietet eine Möglichkeit, für das aktuell selektierte Objekt Einstellungen vorzunehmen.

Ist kein Objekt selektiert, werden die Properties der Grafik angezeigt und können geändert werden.

Sind mehrere Objekte selektiert, werden nur gemeinsam genutzte Attribute in den Properties angezeigt. Attribute, die nur von einem Element genutzt werden können, sind in dieser Liste nicht aufgeführt. Vorgenommene Änderungen werden an alle selektierten Objekte weitergereicht.

Properties

Dieser Karteireiter listet die für ein Element verfügbaren Attribute auf. Diese können direkt in der Liste bearbeitet werden. Der Name der Attribute richtet sich nach der w3c-Spezifikation von SVG und Einträge sind entsprechend den Wertvorgaben zu treffen. Eine Beschreibung der Attribute ist unter "Allgemein/Das SVG-Format/Attribute" ersichtlich.

IDs werden beim Zeichnen von Elementen automatisch generiert. Diese Bezeichnung kann jedoch manuell beeinflusst werden. Dabei ist zu beachten, dass eine ID nur einmal in einem SVG-Dokument vorkommen darf. Das Programm vermeidet doppelte Einträge automatisch durch Anhängen von"_1". Ist die Option "Ensure element IDs conforming to the XML standard" in den "Preferences" aktiviert, so wird die ID entsprechend geändert, wenn ein ungültiges Zeichen eingegeben wird (z.B. Prüfung à Pr_fung). Es erscheint ein Logeintrag mit einem entsprechenden Hinweis. Erlaubt sind 'A'..'Z', 'a'..'z', '0'..'9', '_', '-', '.'. Nicht erlaubt sind Umlaute, Sonderzeichen, Leerzeichen. Außerdem darf die ID nicht mit einer Zahl beginnen.

Das Attribut "Display" steuert die Sichtbarkeit eines Elementes im späteren Dokument. Im Designer bleiben Elemente, die auf nicht sichtbar geschaltet wurden, weiterhin sichtbar.

Farbattribute wie "Fill" und "Stroke" werden standardmäßig in RGB geschrieben. Es ist ebenfalls möglich, für eine Farbe das entsprechende Farbwort bzw. den hexadezimalen Wert einzugeben. Der GefasoftDesigner setzt, sofern vorhanden, diesen Wert in das RGB-Format um.

Da nicht alle möglichen Attribute des SVG Standards unterstützt werden, gibt es zu jedem Objekt das Style-Attribut. Dort können dem GefasoftDesigner unbekannte Attribute geschrieben werden. Diese werden im Designer ignoriert, in der Preview bzw. IE aber umgesetzt. Attribute, die interpretiert werden können (z.B. fill) werden automatisch umgesetzt und aus dem Style entfernt.

Es können auch Style-Attribute für das Root-Element definiert werden. Wird hier eine Hintergrundfarbe festgelegt (background: rgb(00,128,00)), so wird der Hintergrund der Zeichenfläche entsprechend eingefärbt. Es werden hierfür nur Farbangaben in der Form "rgb(00,128,00)" berücksichtigt.

Ein Sonderfall ist mit StrokeDashArray implementiert. Dieses Attribut ermöglicht eine Strichelung von Linien. Diese Eigenschaft wird als Standardattribut angeführt, kann jedoch vom Designer nicht korrekt interpretiert werden. Der Designer stellt bei einer Eingabe eine Linie gestrichelt dar, wobei diese Strichelung nicht der tatsächlichen Darstellung in SVG entspricht, sondern nur zur Informationen dient.

Im Attribut "Transform" sind zurzeit nur Matrix-Transformationen erlaubt. Sind andere Transformationen gewünscht, müssen diese direkt im Sourcecode eingegeben werden.

Für die Elemente Line und Polyline werden unter "Other" zusätzlich die Attribute MarkerEnd und MarkerStart angeboten. Hier kann aus einer Dropdown-Liste ein Kreis (url(#markerCircle)) oder ein Pfeil (url(#markerArrow)) selektiert werden. Dieses Element wird dann als End- oder Startpunkt der Linie angezeigt. Dieser Marker wird in der Zeichenoberfläche nicht angezeigt, sondern nur in der Vorschau und natürlich im Browser.

Text Attribute

Der Eintrag "Text" ermöglicht es, den eingegebenen Text zu ändern.

Über den "TextAnchor" wird festgelegt, wo der Text aufgehängt wird.

Pie Attribute

Ein Pie wird durch Nutzung des Path Elements realisiert. Jedoch wird die volle Komplexität nicht durch den Designer unterstützt.

Dynamics

In diesem Reiter können dem selektierten Objekt Dynamiken für das Legato-System zugeordnet werden. Dieser Punkt ist unter dem Hauptkapitel Dynamics erläutert.

Events

Die Liste stellt mögliche JavaScript-Events dar. Zu jedem Event können JavaScript-Funktionen eingegeben werden, die ausgeführt werden, wenn das Event eintrifft. Beispiele hierzu sind im Hauptkapitel Events aufgeführt.
