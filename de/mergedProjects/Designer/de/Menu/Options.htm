Options

Toggle Grid

Zurzeit nicht implementiert.

Line Grid

Wechselt zum Line Grid beziehungsweise schaltet es ein. Im Abstand von 50 Pixeln (der Abstand kann im Preferences Dialog geändert werden) werden horizontale und vertikale Linien angezeigt, an denen die Elemente ausgerichtet werden können.

Pixel Grid

Wechselt zum Pixel Grid beziehungsweise schaltet es ein. Im Abstand von 50 Pixeln (der Abstand kann im Preferences Dialog geändert werden) werden Punkte angezeigt, an denen die Elemente ausgerichtet werden können.

Snap to Grid

Bei aktivierter Option werden Elemente an den Grids ausgerichtet.

Snap to Shape

Bei aktivierter Option werden Elemente an den Punkten bzw. Kanten anderer Elemente ausgerichtet.

Syntax Check

Führt eine Syntax-Prüfung der Java-Scripts oder des SVG-Quelltextes im aktiven Reiter aus. Im Reiter "Script" wird die Syntax der angeführten Java-Script-Methoden geprüft. Wird ein Fehler gefunden, so wird die entsprechende Zeile gefärbt und eine Fehlermeldung in der Status-Bar angezeigt. Ansonsten erscheint die Meldung "Syntax check OK." Im Reiter "SVG Source" werden die Syntax des SVG-Quelltextes und die DTD-Validität der <dyndef> Elemente geprüft. Zusätzlich werden auch die Namen der Methoden in Attributen "name" kontrolliert. Eine Syntax-Prüfung der Java-Scripts erfolgt im Reiter "SVG Source" nicht. Im Falle von mehreren Fehlern im Quelltext wird immer nur der erste gefundene Fehler angezeigt.

Preferences

Dieser Menüpunkt erlaubt einige allgemeine Einstellungen.

Designer

 

Option

Beschreibung

Grid Size

Abstand der Gitterlinien bzw. –punkte.

Draw Grid on Top

Grid über alle Elemente legen.

Operate on all Layers

Selektionen (Änderungen) sind unabhängig vom gerade angewählten Layer. Ist dies nicht aktiviert, kann nur im gerade angewählten Layer gearbeitet werden. Weiterhin kann explizit über den DOM auch in anderen Layern selektiert werden.

"Double-Click" Drawing

Zeichnen durch doppelte Klicks anstatt Click&Drag.

Undo Count

Anzahl der maximalen Undo-Schritte.

Legato Var File

In diesem Feld kann das Legato Variablen File angewählt werden, das für die Dynamiken die Variablen liefert. Dieses XML wird aus der WebApp exportiert. Beim Import des Variablen-Files wird eine ProgressBar angezeigt.

Legato Class File

Wenn benutzerdefinierte Variablen angelegt werden müssen (KPI und Schichtprotokollwerte), muss ein XML mit den Klassendefinitionen eingelesen werden. Der Dialog bietet die Auswahl des Files an. Das File selbst kann aus der WebApp heruntergeladen werden (Administration – SVG-Visualisierung – Variablen).

 

Die beiden XML-Dateien werden automatisch beim Start des Designers geladen oder nach erneutem Aufruf des Preferences Dialogs.

:

Bei großen Variablen-Files kann es etwas dauern, bis diese geparst sind.

Common

 

Option

Beschreibung

Auto Save - Interval

Minutenangabe des Intervalls, in dem Dateien automatisch gespeichert werden. 0 schaltet dies aus.

Create Backup File

Erzeugt die Backup-Datei FileName.bak.svg, wenn gespeichert wird. In dieser Datei wird der Zustand vor dem Speichern gesichert. Diese Option ist standardmäßig aktiv.

Create Backup Files in Folder

Erzeugt die Backup-Dateien im gewählten Verzeichnis. Die letzten 10 Zustände werden dort rotierend gespeichert.

Use Default Browser for Preview

Nutzt statt der internen Preview die externe Applikation, die mit dem Dateiformat SVG verknüpft ist. Normalerweise sollte dies der IE sein.

Default Folder for "Save As"

Hier kann ein Ordner ausgewählt werden, in dem standardmäßig die SVGs gespeichert werden bzw. der bei "Save As" vorbelegt ist.

Ensure element IDs conforming to the XML standard

Ist diese Checkbox aktiviert, so werden die IDs von Elementen auf XML-Konformität überprüft. Wird eine ungültige ID entdeckt, erscheinen ein Hinweis und die Abfrage, ob die ID geändert werden soll oder nicht. Wird mit "ja" bestätigt, wird die ungültige ID geändert (z.B. Prüfung à Pr_fung). Es erscheint ein Logeintrag mit einem entsprechenden Hinweis. Erlaubt sind 'A'..'Z', 'a'..'z', '0'..'9', '_', '-', '.'. Nicht erlaubt sind Umlaute, Sonderzeichen, Leerzeichen. Außerdem darf die ID nicht mit einer Zahl beginnen.

Die Prüfung erfolgt beim Laden einer Grafik sowie beim Wechsel von Tab "SVG Source" in den Tab "Designer".

Bei Bibliothekselementen erfolgt diese Abfrage nie, da hier Sonderzeichen nötig sein können, z.B. für Parameter.

Standardmäßig ist diese Checkbox deaktiviert.

Library

 

Option

Beschreibung

Suggest Last Symbol Parameter Values

Ist diese Checkbox aktiviert, werden für Bibliothekssymbole mit Parametern die zuletzt eingetragenen Werte in einer Textdatei gespeichert. Bei der nächsten Verwendung des Symbols werden dann die zuletzt verwendeten Parameter-Werte vorbelegt (und können dann auch geändert werden). Ist diese Checkbox nicht aktiviert, werden die Werte nicht gespeichert.

Scale Window to Size of Preview Image

Bei der Anzeige der Vorschau von Bibliotheks-Elementen wird das Vorschau-Fenster an die Größe des Elements angepasst (bis zur unter Max. Size definierten Größe).

Max Size

Hier kann die maximale Größe des Vorschau-Fensters für Bibliotheks-Elemente definiert werden (in Pixel).
