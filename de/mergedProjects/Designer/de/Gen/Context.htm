Zusammenhang zwischen Datenbasis, Editor und Gateway

 

 

Datenerfassung, Aufbereitung, Archivierung

Diese erfolgt mittels eines sog. Datenbankgateway. Die Daten können einerseits (im Großteil der Anwendungsfälle) Signale aus einer Anlagensteuerung (SPS) sein, welche via Netzwerk (z.B. Ethernet) an den PC mit dem Datenbankgateway gesendet werden. Es können aber auch Daten eines anderen PC sein. Der Gateway kann aber auch eigene Daten generieren.

Unabhängig von der Verursachung erfolgt das Buchen der Daten via ODBC in eine SQL-Datenbank (mittels Stored Procedures) oder direkt zu einem Webserver (mittels Dienst LeWePo).

Datenverwaltung / Datenpflege

Diese erfolgt mittels SQL-Datenbank.

Datenausgabe / Datenanzeige

Dies umfasst den "visuellen" Bereich des gesamten Systems. Während die beiden vorhergehenden Bereiche weitestgehend im Hintergrund ablaufen, ist dies der ständig sichtbare und vom Anwender konsultierte Bereich. Hier werden die Daten aus der Datenbank und / oder direkt vom Gateway in Form von Diagrammen, Meldungsanzeigen und grafischen Anlagenübersichten (als SVG – Bildmasken) dargestellt.
