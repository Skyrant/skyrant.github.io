Allgemein

Die Produktfamilie "Machine Data eXchange (MDX)" dient im Kontext der industriellen Produktion der Implementierung des Datenaustauschs zwischen Maschinensteuerungen (Operational Technology / OT Devices) des Shopfloor und verschiedensten IT Systemen (Information Technology).

Dazu bietet MDX auf den Betriebssystemen Windows und Linux verschiedene OT Connectors (OTC) zur Anbindung der OT Devices und einen IT Connector (ITC) zur Anbindung der IT Systeme an. Die Verbindung zwischen OT Connectors und IT Connectors erfolgt über das Message Queuing Telemetry Transport (MQTT) Protokoll.

Innerhalb der ITCs besteht die Möglichkeit projektspezifische Skripte auszuführen. Das ermöglicht eine dynamische Konfiguration der OTCs, die Vorverarbeitung und Anreicherung von Datenströmen sowie die Implementierung kundenspezifischer Businesslogik auf der Edge.

Für die Projektierung von ITCs und OTCs stellt die Produktfamilie MDX zwei Anwenderprogramme mit grafischer Benutzeroberfläche unter Windows zur Verfügung. Das eine Programm, der OTC Manager, dient der lokalen Installation und initialen Konfiguration von OT Connectors (OTC) auf Windows Betriebssystemen. Das andere Programm, der MDX Manager, erlaubt darüber hinaus auch die lokale Installation von IT Connectors sowie die Erstellung und das Debugging von Konfigurationen und Skripten zur Ausführung in ITCs.

INFO zur Terminologie:

  • Die Bezeichnungen "OT Connector" und "OTC" ersetzen die bisherigen Bezeichnungen "Sapient Maschinenclient" / "Sapient Maschinen-Client" und "SMC".
  • Die Bezeichnungen "IT Connector" und "ITC" ersetzen die bisherigen Bezeichnungen "(Sapient Cloud Connector) Runtime" / "SCC Runtime" und "SCC".
  • Die Bezeichnung MDX Manager löst den Begriff "(Sapient Cloud Connector) Editor" bzw. "SCC Editor" ab.
  • In Teilen der Software aus der Produktfamilie MDX und deren Hilfeseiten werden bis auf Weiteres auch noch die bisherigen Begriffe verwendet.

Systemumgebung

Die Anwendung ist eine 64-Bit-Applikation. Die Anwendung unterstützt aber ausschließlich 32-Bit-Datentypen. Die unterstützten Datentypen können hier nachgelesen werden: Unterstützte SPS-Datentypen.

Der IT Connector

Der Cloud Connector besteht aus den folgenden zentralen Bausteinen, die auf verbreiteten technologischen Standards beruhen und eine maximale Flexibilität und Performance gewährleisten:

  • Amazon Web Services (Linux)

  • OPC UA Schnittstelle (Win64, Linux/Unix)

  • SendRecv Telegrammschnittstelle (Win64, Linux/Unix)

  • Pub/Sub Schnittstelle (Win64, Linux/Unix)

  • HTTP Schnittstelle (Win64, Linux/Unix)

  • Datenbank-Schnittstelle (Win64, Linux/Unix)

  • Script-Engine für Funktionslogik (Win64, Linux/Unix)

  • Lokale Datenbank zur Erhöhung der Datensicherheit (Win64, Linux/Unix)

Amazon Web Services

Amazon Web Services (AWS) ist einer der führenden Anbieter im Cloud Computing. Als erste AWS-Schnittstelle ist der Simple Queue Service in der Linux-Version des IT Connectors integriert. Es stehen Verbindungen zum Senden oder Empfangen zur Verfügung.

OPC UA Schnittstelle

Da der klassische OPC Standard auf den alternden COM/DCOM Technologien von Microsoft basiert, wurde durch die OPC Foundation eine neue Spezifikation unter dem Namen OPC UA eingeführt. Das Hauptziel dabei war eine Umstellung von COM/DCOM auf moderne web-basierte Technologien bei Erhaltung bzw. Verbesserung der OPC Funktionalität.

Der Einsatz von OPC UA beschränkt sich damit - auch im IT Connector - nicht mehr wie der klassische OPC Standard auf Windows-Umgebungen. Es ist nun ebenfalls möglich den Standard unter Unix/Linux sowie auf Controllern und intelligenten Geräten mit entsprechenden Betriebssystemen zu implementieren.

Im Cloud Connector ist eine OPC UA Client Schnittstelle mit folgender Funktionalität integriert:

  • Discovery

  • Sessionverwaltung

  • Browsefunktion mit Import

  • Read/Write (einfach, ohne Registrierung)

  • Subscribe/Unsubscribe

SendRecv Telegrammschnittstelle

Nicht alle Geräte oder Steuerungen können über OPC UA angesprochen werden. Um eine Kommunikation mit solchen Partnern aufzubauen, kann die SendRecv Telegrammschnittstelle verwendet werden. Diese ermöglicht die generelle Kommunikation des IT Connectors über die folgenden Treiber:

  • TCP/IP

  • ISO-on-TCP

  • UDP

  • Serielle Kommunikation mit 3964R oder offenes Protokoll (feste Länge, festes Endezeichen oder Quittungsverzug, nur unter Windows)

Die Verwendung der SendRecv-Schnittstelle kann insbesondere bei großen Projekten sinnvoll sein, da damit ein Telegramm mit nur einem Script über alle SPSen verarbeitet werden kann. Somit wird auch die Wartbarkeit erhöht. Allerdings ist der initiale Aufwand bei der SPS-Programmierung recht hoch.

Pub/Sub Schnittstelle

Zusätzlich zu den Kommunikationsmöglichkeiten von OPC UA oder der SND/RCV Telegrammschnittstelle nimmt der Datenaustausch über Protokolle basierend auf dem Publish/Subscribe-Prinzip aus dem Umfeld des industriellen Internet der Dinge (industrial Internet of Things, iIoT) beständig zu. Für diese Protokolle stellt der IT Connector eine PubSub-Schnittstelle zur Verfügung. Über diese Schnittstelle können mehrere Clients für verschiedene Publish/Subscribe Protokolle instanziiert werden. Derzeit sind die folgenden Protokolle und deren Versionen im IT Connector verfügbar:

  • Message Queuing Telemetry Transport (MQTT)

Im IT Connector können MQTT Clients entsprechend der Version 3.1.1 dieses Protokolls instanziiert werden. Der IT Connector unterstützt dabei sowohl SSL/TLS Verschlüsselung als auch Authentifizierung des Clients beim Broker mit Username/Password.

  • Apache Kafka

Im IT Connector können Consumer Verbindungen zum Lesen von Daten aus einem Kafka-Cluster angelegt werden.

HTTP Schnittstelle

Um auch Webinterfaces von anderen IT Systemen ansprechen zu können, verfügt der IT Connector über eine HTTP Schnittstelle. Über diese Schnittstelle ist es möglich Webservices von anderen IT Systemen aufzurufen.

Die HTTP Schnittstelle des IT Connectors wird hauptsächlich für die Kommunikation mit dem MES Legato Sapient verwendet. Dieses bietet mehrere Webservices zur Kommunikation über eine JSON RPC Schnittstelle an.

Datenbank-Schnittstelle

Die Datenbank-Schnittstelle dient der Anbindung von verschiedenen Datenbanken mittels des IT Connectors. Der Haupt-Anwendungsfall dieser Schnittstelle ist die Anbindung von Maschinen an die Datenbank des MES "Legato Classic".

Für Windows steht der ODBC-Treiber zur Anbindung an Datenbanken zur Verfügung. Die Vorteile von "ODBC" liegen in der hohen Verfügbarkeit für nahezu alle am Markt befindlichen Datenbanken und deren Versionen. Die in den IT Connector unter Windows implementierte Schnittstelle wurde bisher mit folgenden Datenbanken erfolgreich getestet:

  • Oracle

  • MS SQL-Server

  • DB2

Für Linux/Unix können nur Oracle Datenbanken mittels OCI (Oracle Call Interface) angebunden werden.

Script-Engine

Zur Verknüpfung und Vorverarbeitung der Daten, für logische Funktionen oder einfache Ablaufsteuerungen bietet sich der Einsatz von Scripten an. Dabei ist die Auswahl der Scriptsprache ein oft diskutiertes Thema. Der Einsatz einer standardisierten Sprache mit großer Marktverbreitung, wie VB Script oder Javascript, bietet Vorteile in Bezug auf die Akzeptanz bei potentiellen Anwendern. Demgegenüber kann eine speziell für den IT Connector entwickelte Sprache mit einem auf die Aufgabenstellung abgestimmten Befehlssatz aufwarten. Auch das Zeitverhalten des Scriptinterpreters und die Einbindung in den Gesamtablauf kann für das System optimiert werden.

Für die Ausführung von Scripten im IT Connector wurde in der Vergangenheit eine eigenständige Scriptsprache entwickelt, deren Syntax sich an eine standardisierte Sprache ("Structured Text" aus der IEC 61131-3) anlehnt.

Die Bearbeitung der Scripte erfolgt streng ereignisgesteuert durch die Script-Engine innerhalb des IT Connectors. Als Ereignisse können wählbare Timer oder jede Änderung eines Variablenwertes herangezogen werden. Werte von Variablen können Scripten als Parameter übergeben werden können. Komplexe Applikationen können in mehrere, wiederverwendbare Scripte unterteilt und diese in einer Ordnerstruktur abgelegt werden.

Lokale Datenbank

Zuordnungs- oder Nachschlagetabellen können in einer lokalen (integrierten) Datenbasis (Redis) des Cloud Connectors gehalten werden. Wenn z.B. bei einem bestimmten Ereignis aus der Produktion jeweils ein Eintrag in einer Zuordnungstabelle gelesen werden soll, muss eine lokale Kopie dieser Tabelle im Cloud Connector existieren, da die ständige Abfrage der Datenbank zu einem inakzeptablen Zeitverhalten führen würde.

Zur Überbrückung von Verbindungsproblemen können alle Daten über die lokale Datenhaltung in Redis zwischengespeichert werden. Über projektspezifische Scripte können Anwender so die Datensicherheit ihrer Applikation wesentlich verbessern.
