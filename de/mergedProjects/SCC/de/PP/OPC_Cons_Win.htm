OPC-Verbindungen (nur Win64 Cloud Connector)

OPC-Server Überblick

Mit OLE for Process Control, kurz OPC, ist die von vielen Anwendern gewünschte Standardisierung der Automatisierungssoftware einen großen Schritt vorangekommen. Die mühevolle und kostenintensive Treiberentwicklung für spezielle Schnittstellenprotokolle und Feldbussysteme entfällt komplett. Mit OPC, der standardisierten Programmierschnittstelle für Prozessdaten, muss in Zukunft für jede Hardware, z.B. eine spezielle Feldbuskarte, nur ein OPC-Server entwickelt werden. Umgekehrt muss eine Applikation, z.B. eine Prozessvisualisierung, nur über einen Treiber in Form eines OPC-Clients verfügen, um Daten über verschiedenste Hardwareschnittstellen akquirieren zu können.

Die von der Open Control Nutzergruppe des INTERBUS-S Clubs spezifizierte Schnittstelle CALL-R (Control Applikation Link Layer - Runtime) erweitert den OPC-Standard um einige Funktionen speziell für PC-basierte Automatisierungslösungen in der Industrie:

Anlegen mehrerer unterscheidbarer Instanzen des Server-Objekts

Konfigurieren der ansprechbaren Variablen-Objekte (OPC-Items)

Vermeiden von Zugriffskonflikten beim Zugriff mehrerer Clients auf das gleiche Variablen-Objekt (Item)

Lesen bereits vorverarbeiteter Werte (als Alternative zum ausschließlichen Lesen eines Rohwertes), und das Parametrieren der Vorverarbeitung

Anlegen von Programm- und Programmgruppen-Objekten, sowie die Funktionen zum Laden, Starten und Stoppen von Programmen

Ein transparenter Datenkanal zur Übertragung anwendungsspezifischer, nicht allgemein interpretierbarer Daten.

 

Im Cloud Connector ist ein OPC-Client integriert, der sowohl in Verbindung mit einem Standard-OPC-Server, als auch mit einem CALL-R-Server eingesetzt werden kann.

Installation des OPC-Servers

Installieren Sie einen OPC-Server entsprechend den Herstellerangaben. Das Installationsprogramm muss die notwendigen Einträge in der Registrierungsdatenbank vornehmen.

Parametrieren des OPC-Servers

Parametrieren Sie den OPC-Server entsprechend den Herstellerangaben. Die am OPC-Server parametrierten Items können im Cloud Connector importiert werden (sofern der OPC-Server das Interface zum Abfragen der Konfiguration beinhaltet).

Anlegen des Serverelements

Im Kontextmenü des Grundelements OPC im Projektbaum den Befehl "Neuer Server" wählen. Als Server wird die Kennung des OPC-Servers eingegeben. Wurde der Server korrekt installiert, kann mit dem "Browse"-Button neben dem Eingabefeld eine Auswahlliste geöffnet werden, in der alle installierten OPC-Server aufgelistet werden.

Die Angabe einer Konfigurationsdatei ("Konfig-Datei") des OPC-Servers ist nur möglich, wenn der verwendete Server eine entsprechende Konfiguration von OPC-Items verwendet (z.B. der GEFASOFT OPC-Server GOPS).

"Kontrolllesung nach Schreibvorgang" ermöglicht eine Verifizierung der Variablenwerte, die soeben in die SPS geschrieben wurden.

Anlegen von Gruppen

Jede Gruppe kann ein oder mehrere Items enthalten. Diese Gruppen können Items aus verschiedenen Steuerungen enthalten. Sie haben die Eigenschaft, dass alle enthaltenen Items mit der gleichen Aktualisierungsrate von dem OPC-Server angefordert und versorgt werden.

Selektieren Sie im Projektbaum unter "OPC" den Server, dem Sie eine Gruppe zuordnen möchten, und wählen Sie im Kontextmenü den Befehl "Neue Gruppe". Im Dialog für die Gruppe geben Sie die Aktualisierungsrate in Millisekunden an. Diese Aktualisierungsrate definiert die kürzeste Zeit, in der die Werte vom Server an den Cloud Connector übergeben werden dürfen. Werte werden nur bei Änderung übertragen.

OPC-Variable

Anlegen eines OPC-Items

Als "Item-Id" wird die ID des OPC-Items angegeben, wie das Item auch beim OPC-Server bekannt ist. Diese ID kann bei Servern, welche die Abfrageschnittstelle implementiert haben, importiert werden. "Format" und "Zugriff" wird in diesem Fall automatisch vorbelegt.

Außerdem kann der "Zugriffspfad" eingegeben werden. Dieses Feld ist frei editierbar. Bei bestehenden Projekten wird beim Laden der Name der Variablen als Zugriffspfad eingetragen. Bei neu angelegten Variablen bzw. nach dem kompletten Löschen des Wertes wird im Dialog "<NULL>" angezeigt. In diesem Fall wird als Access Path NULL an den OPC Server übergeben.

:

Leerzeichen werden auch als Leerzeichen übergeben, nicht als NULL.

Über die Checkbox "Format vorgeben" kann das gewünschte OPC-Format an den Server vorgegeben werden. Hier werden nur Formate aufgelistet, die vom Cloud Connector gewandelt werden können. Üblicherweise ist hier kein Eintrag vorzunehmen. Lediglich bei der Verwendung eines Bosch/Rexroth Servers muss die Checkbox aktiviert und ein Format gewählt werden. Dabei sind prinzipiell alle Kombinationen zwischen Format und vorgegebenem Format möglich. D.h., das OPC-Item kann als Format "Int" haben und es kann VT_BSTR vorgegeben werden. Sofern der übergebene String eine Zahl enthält, kann diese gewandelt werden und es wird das richtige Ergebnis geliefert.

:

Hier ist anwendungsspezifisch sicherzustellen, dass die Formate bzw. der Inhalt zusammenpassen. Andernfalls wird als Ergebnis 0 geliefert.

Wird im Kontextmenü der Gruppe "Import" gewählt, können mehrere Items gleichzeitig importiert werden. Der Name der angelegten Elemente ergibt sich aus der ID der Items. Diese können beim Import an einer konfigurierbaren Stelle abgeschnitten werden.

Import von OPC-Items

Beim Import wird angegeben, ab dem wievielten Trennzeichen ("/") der Variablenname übernommen werden soll. Beim Einsatz von GOPS als OPC-Server ist das in der Regel das zweite Trennzeichen, da die vorstehenden Elemente des hierarchischen Namens den Treiber und die Verbindung beschreiben.

In der Spalte "OPC-Item" steht die volle ID, wie sie im OPC-Server hinterlegt ist, in der Spalte "Variable" der Name, wie er dem Element im Cloud Connector gegeben wird. Unter "Typ" steht der OPC-Typ, bei "Format" das entsprechende Cloud Connector Format.

Die Spalte "RW" gibt die möglichen Zugriffsarten für das Item an:

RW

Zugriffsart

R

Item kann nur gelesen werden

W

Item kann nur geschrieben werden

RW

Item kann gelesen und geschrieben werden

 

Eine Variable wird über Eigenschaften beschrieben. Beim Import werden diese Eigenschaften bereits automatisch übernommen.

Die Reaktion auf eine Änderung des Variablenwertes kann zusätzlich parametriert werden. Normalerweise wird an dieser Stelle die Ausführung eines Scripts zur weiteren Verarbeitung der Variablen angestoßen.
