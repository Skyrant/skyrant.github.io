Basis

Kommandoschnittstelle zwischen Cloud Connector und Redis. Kann Redis direkt ansprechen und Daten dorthin senden bzw. von dort holen. Außerdem können über Spools Aufgaben auf mehrere Cloud Connectors verteilt werden.

Über das Kontextmenü - Neues Item kann ein neues Element erstellt werden. Es stehen folgende Typen zur Auswahl: Variable, Ordner, Spool, Kommandoschnittstelle.

Eigenschaften von Spools und Kommandoschnittstellen

Über das Kontextmenü - Eigenschaften auf einen Spool oder eine Kommandoschnittstelle kann der Eigenschaftsdialog geöffnet werden:

Zunächst muss ein Name eingegeben werden. Dieser muss auf dem lokalen Redis eindeutig sein. Da jedoch Spools immer noch den jeweiligen Knoten als Präfix bekommen, muss der Name nur innerhalb des Knotens eindeutig sein, z.B. Basis.Spool1. Diese Eindeutigkeit muss allerdings auch bei mehreren Projekten erhalten bleiben (es kann also nicht in Projekt 1 und Projekt 2 einen Spool Basis.Spool1 geben).

Als Typ stehen Inaktiv, Command und Listen zur Verfügung. Bei inaktiv werden Kommandos direkt an Redis gesendet und davon empfangen. Command sendet lediglich Kommandos an Redis, empfängt jedoch nichts. In Redis werden die Daten zwischengespeichert. Die gesendeten Daten müssen dann von einem anderen Spool oder einer anderen Schnittstelle von Redis geholt werden. Listen holt nur Kommandos von Redis, sendet aber nichts.
