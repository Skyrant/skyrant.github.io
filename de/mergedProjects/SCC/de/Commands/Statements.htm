Anweisungen

If ... Then ... Else ... Endif

Der Anweisung IF muss ein boolescher Ausdruck folgen. Die Programmausführung wird in Abhängigkeit des Ergebnisses dieses Ausdruckes fortgeführt. Bei "true" wird der Anweisungsteil zwischen THEN und ELSE bearbeitet, bei "false" der Anweisungsteil zwischen ELSE und ENDIF. Die Verwendung von ELSE ist optional.

:

Bei Abfragen von booleschen Werten dürfen keine Vergleichsoperatoren (=, >, <,...) angegeben werden. Die Abfrage erfolgt direkt auf die Variable (Abfrage auf true) oder mit dem Operator NOT (Abfrage auf false).

Beispiel:

long     a, b

bool     c

if a > 0 then

  b := 1

else

  b := 2

endif

if c then

  b := 3

endif

 

Switch ... Case ... Default ... End

Der Anweisung switch muss ein numerischer Ausdruck vom Typ "Int", "Long" oder "Double" folgen. Für String oder Bool-Werte kann switch nicht ausgeführt werden. Dieser Ausdruck wird mit den Konstanten, die jeweils einem case-Label folgen, verglichen. Ist ein Vergleich "true", wird der Programmteil bis zum nächsten case, default oder end bearbeitet. Will man nach einem erfüllten Vergleich keine weiteren Abfragen mehr, muss am Ende des Anweisungsblocks der Befehl break stehen.

Bei Abfragen von booleschen Werten dürfen wie bei der Anweisung IF...THEN...ENDIF keine Vergleichsoperatoren (=, >, <,..) angegeben werden. Die Abfrage erfolgt direkt auf die Variable (Abfrage auf true) oder mit dem Operator NOT (Abfrage auf false).

Beispiel:

Long     a

string   s

switch a

  case 0:

    s := 'ZERO'

    break

  case 1:

    s := 'ONE'

    break

  default:

    s := 'TOO MANY'

end

 

While ... Do ... Enddo

Der Anweisung while muss ein boolescher Ausdruck folgen. Der Anweisungsteil zwischen do und enddo wird solange ausgeführt, bis das Ergebnis dieses Ausdruckes den Wert false ergibt.

Die Schleife kann durch den Befehl break abgebrochen werden. Der Befehl continue bewirkt, dass die Programmbearbeitung mit der Abfrage fortgesetzt wird.

Bei Abfragen von booleschen Werten dürfen wie bei der Anweisung IF...THEN...ENDIF keine Vergleichsoperatoren (=, >, <,...) angegeben werden. Die Abfrage erfolgt direkt auf die Variable (Abfrage auf true) oder mit dem Operator NOT (Abfrage auf false).

Beispiel:

long     a, b

while a < 10 do

  a := a + 1

  if a mod 2 = 0 then

    continue

  endif  b := b + 3

enddo
