Tabellenfunktionen für die interne Datenbank

Der Cloud Connector verwendet für die interne Datenbank dBase-kompatible Dateistrukturen. Die Tabellen können auf der Festplatte gespeichert werden oder im Hauptspeicher angelegt werden. Wird die Datei im Hauptspeicher angelegt, gehen die Daten nach Programmende verloren. Ob eine Datei im Speicher oder auf der Festplatte angelegt wird, kann durch den Namen bestimmt werden. Eine Datei im Speicher beginnt bei Windows mit dem Pfad '$MEM\', bei Linux/Unix/AIX mit dem Pfad '$MEM/'.

Für die Tabellenfunktionen kann in einem Script eine Variable vom Typ DBTBL deklariert werden. Alle Tabellenfunktionen werden dann mit dem Namen der Variablen, einem Punkt und dem Namen der Funktion gemeinsam angesprochen.

Beispiel:

dbtbl    tbl

tbl.open('c:\\files\\data.dbf')

 

Append

Deklaration: bool dbtbl.Append(dbrec record)

Hängt einen neuen Datensatz ans Ende der Datei und schreibt den Inhalt des Recordpuffers record in den Datensatz.

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbtbl    tbl

dbrec    rec

bool     ok

tbl.open('c:\\files\\data.dbf')

tbl.bufopen(rec)

rec.empty

ok := tbl.append(rec)

rec.close

tbl.close

Deklaration: bool dbtbl.close

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

bool     ok

rec.close

tbl.close

 

Create

Deklaration: bool dbtbl.Create(string name, string descr)

Erzeugt eine neue Datei. Beginnt der name mit '$MEM\'  wird die Datei nur im Hauptspeicher angelegt.

Die Struktur der Datei wird durch descr festgelegt. Dabei werden die Feldbeschreibungen durch Semikolon getrennt. Eine Feldbeschreibung besteht aus Feldname (max. 10 Zeichen), Feldtyp und optional Nachkommastellen bei numerischen Feldern. Die möglichen Feldtypen sind:

Feldtyp

Beschreibung

Feldlänge

Nachkomma

C

Alphanumerisch

1 - 255

-

N

Numerisch

1 - 19

max. Feldlänge - 2

D

Datum

8

-

L

Logikfeld

1

-

 

Die Funktion liefert true, wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbtbl    tbl

dbrec    rec

bool     ok

tbl.create('$MEM\\List.dbf', 'Name,C,10;Value,N,6,2;Date,D,8')

tbl.bufopen(rec)

rec.empty

ok := tbl.append(rec)

rec.close

tbl.close

 

Delete

Deklaration: bool dbtbl.delete

Kennzeichnet den aktuellen Satz als gelöscht. Der Satzzeiger muss zuvor mit "Goto", "First", "Last", "Next" oder "Prev" richtig positioniert werden. Ein endgültiges Löschen des Datensatzes wird erst durch Aufruf der Funktion "Pack" durchgeführt.

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbtbl    tbl

bool     ok

tbl.open('c:\\files\\data.dbf')

tbl.goto(4)

ok := tbl.delete

tbl.pack

tbl.close

Get

Deklaration: bool dbtbl.Get(dbrec record)

Liest den aktuellen Satz record aus der Datei in den Recordpuffer ein. Der Satzzeiger muss zuvor mit "Goto", "First", "Last", "Next" oder "Prev" richtig positioniert werden.

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbrec    rec

string   s

rec.close

tbl.close

 

GetNumIdx

Deklaration: bool dbtbl.GetNumIdx(dbrec record,
                                  long  field,
                                  long  num)

Sucht den ersten Satz, dessen numerischer Wert im Feld field dem Wert num entspricht. Wurde ein Satz gefunden, wird er aus der Datei in den Recordpuffer record eingelesen. Da es sich dabei um eine Indexfunktion handelt, kann sie nur ausgeführt werden, wenn zuvor die Funktion "SetIndex" für das Feld aufgerufen wurde.

Es wird empfohlen, "SetIndex" vor jedem Aufruf von "GetNumIdx" auszuführen (siehe Beispiel).

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

:

Diese Funktion sollte nur verwendet werden, wenn die Daten sich nicht oft ändern. Ansonsten wird die Abfrage zu häufig durchgeführt, was zu Performance-Einschränkungen führen kann. In diesen Fällen sollte besser "GetNumKey" verwendet werden.

Beispiel:

dbtbl    tbl

dbrec    rec

string   s

tbl.open('c:\\files\\data.dbf')

tbl.bufopen(rec)

tbl.setindex(1)

tbl.getnumidx(rec, 1, 23)

s := rec.ascii(2)

rec.close

tbl.close

Import

Deklaration: bool dbtbl.Import(string file,
                               string delim,
                               bool   skip)

Importiert die CSV-Datei file in die Tabelle. delim gibt das gültige Trennzeichen an, üblicherweise Komma oder Semikolon. Hat skip den Wert "true", wird die Spaltenbeschreibung der CSV-Datei ignoriert und die erste Spalte der CSV-Datei dem ersten Feld zugeordnet usw.

Wird die Spaltenbeschreibung verwendet, wird nach gleichnamigen Feldern gesucht und nur die Daten übernommen, deren Spaltenbeschreibung einem Feldnamen entspricht.

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbtbl    tbl

bool     ok

tbl.create('$MEM\\alist1.dbf',

           'Operand,C,10;'   +

           'Symbolik,C,10;'  +

           'Text,C,40;'      +

           'Klasse,N,3;'     +

           'Art,C,40;'       +

           'Funk,C,1')

ok := tbl.import('C:\\files\\alarms1.csv', ',', false)

tbl.close

RecIsDeleted

Deklaration: bool dbtbl.recisdeleted

Beim Löschen eines Datensatzes mit Delete wird dieser nur als gelöscht gekennzeichnet. Ein endgültiges Löschen des Datensatzes wird erst durch Aufruf der Funktion "Pack" durchgeführt. Durch die Funktion RecIsDeleted kann abgefagt werden, ob der betreffende Satz schon als gelöscht gekennzeichnet wurde.

Der Satzzeiger muss zuvor mit "Goto", "First", "Last", "Next" oder "Prev" richtig positioniert werden.

Die Funktion liefert "true", wenn sie korrekt ausgeführt werden konnte.

Beispiel:

dbtbl    tbl

bool     ok

tbl.open('c:\\files\\data.dbf')

tbl.goto(4)

if tbl.reisdeleted then

  debug('Dieser Satz wurde als gelöscht gekennzeichnet')

endif

tbl.close

Resident

Deklaration: bool dbtbl.resident

Lässt eine im Hauptspeicher angelegte Datei auch nach dem Schließen im Speicher. Sie kann danach auch von anderen Scripten wieder geöffnet werden. Die Daten bleiben bis Beendigung des Cloud Connectors oder bis zum Schließen des Projekts erhalten.

Beispiel:

dbtbl    tbl

long     anz

bool     ok

tbl.create('$MEM\\alist1.dbf',

           'Operand,C,10;'   +

           'Symbolik,C,10;'  +

           'Text,C,40;'      +

           'Klasse,N,3;'     +

           'Art,C,40;'       +

           'Funk,C,1')

tbl.resident

tbl.zap

ok := tbl.import('C:\\files\\alarms1.csv', ',', false)

tbl.close

tbl.open('$MEM\\alist1.dbf')

anz := tbl.nRecs

tbl.close

Deklaration: bool dbtbl.ResidentEx(long type)

Lässt eine im Hauptspeicher angelegte Datei auch nach dem Schließen im Speicher. Sie kann danach auch von anderen Scripten wieder geöffnet werden. Die Daten bleiben je nach type erhalten:

Wert (type)

0

Tabelle nicht resident

1

Tabelle bis Projektende resident

2

Tabelle resident bis Beendigung des Cloud Connectors oder bis zum Schließen des Projekts

 

Beispiel:

dbtbl    tbl

long     anz

bool     ok

tbl.create('$MEM\\alist1.dbf',

           'Operand,C,10;'   +

           'Symbolik,C,10;'  +

           'Klasse,N,3;'     +

           'Art,C,40;'       +

           'Funk,C,1')

tbl.residentEx(1)

tbl.zap

ok := tbl.import('C:\\files\\alarms1.csv', ',', false)

tbl.close

tbl.open('$MEM\\alist1.dbf')

anz := tbl.nRecs
