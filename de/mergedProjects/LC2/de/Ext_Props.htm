Projektspezifische Felder / Erweiterte Eigenschaften

Funktion

Allen Daten-Tabellen können projektspezifische Felder wie erweiterte Eigenschaften zugewiesen werden, z.B. eine Kostenstelle oder eine Farbe. Diese projektspezifischen Felder werden dann als eigene Spalten an die jeweilige Tabelle angehängt und können dort mit Werten versehen werden. Außerdem können diese projektspezifischen Felder auch in der WebApp abgefragt oder als Filter verwendet werden.

Definition von projektspezifischen Feldern

Um die projektspezifischen Felder pflegen zu können, werden Customize-Berechtigungen benötigt und der LC2 muss im System-Developer Mode gestartet werden.

Dies vorausgesetzt, wird bei den Tabellen, die projektspezifische Felder unterstützen, ein neuer Menüpunkt Projektspezifische Felder unter Einstellungen - Anpassen angeboten:

Ein Klick auf diesen Menüpunkt öffnet eine Tabelle, in der die projektspezifischen Felder für die selektierte Datenquelle / Tabelle angelegt werden können.

Feldbeschreibung

Feld

Beschreibung

ID

Vom System vergebene, eindeutige ID. Schreibgeschützt.

Kategorie (optional)

Kategorie, der diese Spalte zugeordnet ist. Default "Erweitert".

Datenquelle

FK - Datenquelle, für die diese erweiterte Eigenschaft / projektspezifische  Spalte erstellt wird. Ist automatisch mit der Tabelle vorbelegt, die gepflegt wird.

Sollen erweiterte Eigenschaften für eine Tabelle gepflegt werden, die nicht im LC2 eingebunden ist, kann dies unter Systemtabellen - Projektspezifische  Felder erfolgen. Hier werden die projektspezifischen Felder für alle Datenquellen angezeigt.

Feldname

Name des projektspezifischen Feldes. Wird nicht angezeigt. Wird für Zuordnung der Eigenschaften dieses Felds benötigt. Der Wert wird immer in Kleinbuchstaben gespeichert. Leerzeichen und Sonderzeichen sind nicht erlaubt.

Angezeigter Name (optional)

Angezeigter Name der Spalte.

Tipp (optional)

Tooltipp, der beim Mouseover angezeigt wird.

Anzeige-Nummer (optional)

Definiert, an welcher Stelle die Spalte im LC2 dargestellt wird. Default ist 100, also am Ende der Tabelle.

Default-Wert (optional)

Default-Wert, mit dem dieses Feld automatisch belegt wird.

Pflichtfeld

Checkbox. 1 = Eingabe verpflichtend; 0 = Eingabe optional.

Sichtbar

Checkbox. 1 = Spalte ist sichtbar; 0 = Spalte ist ausgeblendet.

JSON Eigenschaft

Checkbox. 1 = dies ist eine JSON Eigenschaft. Default ist 1.

Datentyp

Lookup-Liste zur Auswahl des Datentyps der Eigenschaft. Die Feldeingabe wird auf diesen Datentyp geprüft. Wird ein Eintrag in Lookup-Eigenschaften vorgenommen, so muss der passende Datentyp für die Einträge in der Werteliste gewählt werden.

Max. Länge (optional)

Maximale Länge für Werte vom Datentyp STRING.

Lookup-Eigenschaften (optional)

Definition einer Lookup-Tabelle oder einer Werteliste. Bei Lookup-Tabellen muss der Datentyp auf INTEGER gesetzt sein!

Beispiele:

DataSourceSettings.ObjectName="L_NODES";

LookupList = ["xy", "zz"];

Schema-Info

Schema-Info für dieses Feld (Datentyp, Eigenschaft, etc.). Wird automatisch befüllt und dient nur zur Information.

 

Befüllen von projektspezifischen Feldern

Nachdem die projektspezifischen Felder definiert wurden, werden sie als neue Spalten bei der Tabelle angezeigt. Je nach Konfiguration können die Felder nun befüllt werden, z.B. durch Auswahl eines Eintrags aus einer Werteliste oder durch Eingabe eines Strings. Zum Befüllen werden keine Customize-Berechtigungen benötigt.

Die Einträge werden im Feld "Erw. Eigenschaft" (attr_jsonb) automatisch als JSON-String zusammengefasst und können dann von der SE abgefragt werden.

Beispiel:

{"ksta1": 100, "ksta2": "test_attr", "ksta3": 2}
