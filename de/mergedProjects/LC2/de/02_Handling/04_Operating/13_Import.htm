Import von Tabellen

Für den Import muss die Datei ebenfalls wieder im TXT-Format vorliegen. Nach der Auswahl der zu importierenden Datei muss zunächst das korrekte Zeichenkodierungsformat (UTF8 oder ANSI) gewählt werden.

Nach Klick auf OK öffnet sich der Import-Dialog.

Import-Aktion

 

 

Es kann zwischen drei verschiedenen Import-Aktionen gewählt werden:

Zeilen anhängen

Die importierten Daten werden an die bestehende Tabelle angehängt.

Zeilen aktualisieren

Die importierten Daten werden nach einem frei definierbaren Key mit den Daten in der bestehenden Tabelle abgeglichen. Bereits existierende Keys werden aktualisiert, nicht vorhandene Keys werden verworfen.

Zeilen zusammenführen

Die importierten Daten werden nach einem frei definierbaren Key mit den Daten in der bestehenden Tabelle abgeglichen. Bereits existierende Keys werden aktualisiert, nicht vorhandene Keys werden an die bestehende Tabelle angehängt.

Spaltenzuordnung

 

Hier kann die Zuordnung der Spalten der Tabelle auf die importierten Daten definiert werden. Zu jeder Spalte der existierenden Tabelle kann definiert werden, aus welcher Spalte der importierten Tabelle die Daten übernommen werden sollen. So kann z.B. festgelegt werden, dass sowohl in die Spalten "Alarm number" und "Operand" der bestehenden Tabelle die Daten aus der Spalte "Operand" der importierten Tabelle geschrieben werden sollen.

Die Zuordnung ist optional. Wird nichts ausgefüllt, werden die Spalten einfach eins zu eins der bestehenden Tabelle zugewiesen.

Schlüsselspalte

 

 

Für die Import-Aktionen "Aktualisieren" und "Zusammenführen" muss ein Schlüssel definiert werden. Der Schlüssel legt fest, welche Spalten zwischen existierender und importierter Tabelle verglichen werden um festzustellen, welche Datensätze aktualisiert und welche hinzugefügt werden müssen.
