Funktion

Um eine Berechnung zu aktivieren, muss hier mindestens ein Eintrag vorhanden sein. Außerdem können zusätzliche Trigger zur Ausführung der Berechnung ausgewählt werden.

Trigger sind Ereignisse, die die Ausführung einer Funktion anstoßen (triggern). Generell stehen zwei Typen von Triggern zur Verfügung: Zeitgesteuerte Trigger und Ereignisgesteuerte Trigger (Ereignisse). Gültige Triggerparameter sind vom Typ Variable oder Knoten und In. Der Trigger kann auch von mehreren Variablen erfolgen. Mischen von Zeit- und Eventtriggern ist möglich. Es ist darauf zu achten, dass die Ein- und Ausgabeparameter die richtigen Typen haben (String, Num.).

Wenn möglich, sollten Funktionsaufrufe eventgesteuert ausgeführt werden, um unnötige zyklische Berechnungen zu vermeiden, die kein neues Ergebnis zeitigen.

 

Manueller Trigger

Zeitgesteuerte Trigger

zyklisch (5 sec. / 30 sec. / 60 sec. / 120 sec.)

Die Berechnung wird im angegebenen Zeitraster angestoßen und ausgeführt. Diese Trigger werden verwendet, wenn kein Ereignis wie Wertänderung zur Verfügung steht, mit dem eine Neuberechnung durchgeführt werden muss.

Beispiel: Berechnung von Soll aktuell im Zeitraster von 60 sec.

Pausenbeginn /-Ende, Schichtbeginn /-Ende, Tagesbeginn mit/ohne Produktion, Beginn erste Schicht, Ende letzte Schicht, Tagesbeginn, Produktionsbeginn /-Ende

Einmalige Berechnungen mit Auftreten des genannten Ereignisses relativ zum Schicht- Pausenmodell bzw. Kalenderparametrierung. Bezieht sich entweder auf den Knoten, der bei der Berechnung als Trigger markiert wurde oder auf den, der hier eingetragen ist.

 

Eventgesteuerte Trigger

Bei Angabe eventgesteuert muss immer auch der Parameter, dessen Wertänderung die Berechnung anstoßen soll, markiert werden. Dazu wird die Checkbox "Trigger" in der Parameterliste aktiviert. Alternativ kann ein weiterer Wert festgelegt werden, indem das Feld "Variable" hier befüllt wird.

Für Knoten ist ein eventgesteuerter Trigger nicht möglich.

Feldbeschreibung

 

Feld

Beschreibung

ID

Vom System vergebene, eindeutige ID. Schreibgeschützt.

Berechnung

Lookup-Liste zur Auswahl der Berechnung, auf die sich der Trigger bezieht. Wird automatisch befüllt und kann geändert werden.

Trigger

Lookup-Liste zur Auswahl eines Triggers, der die Berechnung anstoßen soll.

Variable

Lookup-Liste zur Auswahl einer Variablen, deren Wertänderung als Auslöser für eventgesteuerte Trigger verwendet werden soll. Kann als zusätzlicher Auslöser definiert werden oder wird vom Berechnungsparameter übernommen.

Knoten

Lookup-Liste zur Auswahl eines Knotens, dessen Schichtereignisse als Trigger verwendet werden sollen. Kann als zusätzlicher Auslöser definiert werden oder wird vom Berechnungsparameter übernommen.
