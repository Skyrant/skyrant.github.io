Überblick

Projektierungsvorlagen stellen eine schnelle Möglichkeit dar, viele gleichartige Daten in Legato Sapient zu parametrieren. Da zum Beispiel häufig gleiche Steuerungen mit gleichem Aufbau vorkommen, kann für so einen Fall eine Vorlage erstellt werden, die dann für die einzelnen Steuerungen instanziiert wird.

Es ist möglich, jede beliebige Tabelle über Projektierungsvorlagen zu befüllen. Dazu wird eine JavaScript-Datei erstellt und im Verzeichnis C:\SapiEng\src\dist\scripts\configurationtemplates abgelegt. Für weitere Details, siehe bitte Skript-Überblick. Dieses Skript wird dann in der Tabelle Projektierungsvorlagen in der Spalte "Eigenschaften" referenziert (z.B. {"script":"create_node"}).

In der Spalte "Parameter" der Projektierungsvorlagen werden dann als JSON-String die einzutragenden Parameter notiert:

{

  "parent_node": {"caption": "Parent node", "dataType": "Int32", "lookupTable": "l_nodes", "required": true},

  "short_name": {"caption": "Short name", "dataType": "String", "maxLength": 80},

  "name": {"caption": "Name", "dataType": "String", "maxLength": 80, "required": true}

}

 

Für jeden hier eingetragenen Parameter wird zur unterliegenden Tabelle "Instanzen" eine Spalte hinzugefügt, in der der Parameter mit dem gewünschten Wert per Instanz versehen werden kann.

Das Anlegen einer neuen Instanz führt noch nicht zum Anlegen irgendwelcher Daten. In der schreibgeschützten Spalte "Parameterwerte" der Instanz wird in einem String angezeigt, welche Werte beim Ausführen des Updates gesetzt werden. Ein Häkchen in der Checkbox "Aktualisierung empf." indiziert, dass nicht ausgeführte Änderungen vorgemerkt sind.

Im Kontextmenü der Instanz kann nun zunächst das Update (oder das Löschen) der Instanz evaluiert werden. Es wird ein Testlauf durchgeführt, ohne die Daten wirklich zu schreiben. Der Dialog zeigt an, ob der Test erfolgreich war und wie viele Änderungen durchgeführt werden. Mouseover über das Ergebnis zeigt genau an, welche Spalten mit welchen Werten geschrieben werden. Diese Funktionen können auch auf eine Mehrfachselektion angewendet werden. Die Aktion wird dann für alle markierten Einträge in eine Warteschlange aufgerufen und ein Ausführungsfenster eingeblendet.

Die Aktionen werden via RPC in der Sapient Engine aufgerufen. Im Normalfall ist das Ergebnis nach Ausführung in diesem Fenster ersichtlich - dauert eine Aktion ungewöhnlich lange, kann hier als Fehler ein Timeout angezeigt werden. Trotzdem läuft die Funktion im Normalfall auf dem Server weiter. In jedem Fall wird für jede Instanz die Instanz-Info (und das Feld "Update Recommended") aktualisiert. Das Ausführungsergebnis ist dann auch nachträglich aus dem Ausführungs-Protokoll (CT_EXECUTIONS) ersichtlich. Das Ausführungsfenster kann auch während der Ausführung geschlossen werden - die Ausführung wird im Hintergrund trotzdem weiter ausgeführt. Mit dem Button "Anstehende Aktionen überspringen" kann die Ausführung aller nicht gestarteten Aufrufe zurückgezogen werden. Der aktuell laufende Aufruf wird dabei nicht abgebrochen!

Nachdem die Funktion ausgeführt wurde, kann das Dialogfenster geschlossen werden. In der Tabelle "Ausführungen" ist nun ein neuer Eintrag mit der eben ausgeführten Aktion zu finden. Hier kann das Ergebnis jederzeit wieder aufgerufen werden.

Einschränkungen:

 

Es können nur Tabellen innerhalb des Legato-Schemas verwendet werden.

Es können nur Tabellen verwendet werden, die einen serverseitig verwalteten numerischen PrimaryKey besitzen (Feldname verbindlich "ID").

 

 

Icon im LC2 zu Projektierungsvorlagen:

  • Ein Icon wird vor jeder Zeile im LC2 angezeigt, wenn der Datensatz über eine Vorlage erstellt wurde.

 

  • Das Icon zeigt beim Mouse Over die Warnung "Vorlageninstanz: Änderungen werden bei der nächsten Aktualisierung der Instanz überschrieben."


 

  • Beachten Sie den Menüeintrag "Vorlage (Proxy) im Navigationsbaum finden" im Kontextmenü. Wenn Datensätze über Projektierungsvorlagen erstellt wurden, kann über diesen Menüeintrag der zugehörige Vorlageninstanz-Proxy im Navigationsbaum auf einfache Art gefunden werden.
