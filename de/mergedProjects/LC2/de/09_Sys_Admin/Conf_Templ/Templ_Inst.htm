System-Administration | Projektierungsvorlagen | Vorlagen-Instanzen

Funktion

Direkt unter der jeweiligen Projektierungsvorlage können Instanzen dafür erstellt werden. Jeder Parameter der Vorlage wird als eigene Spalte angezeigt und kann pro Instanz mit einem Wert versehen werden.

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer.
Vorlage Lookup-Liste zur Auswahl der Vorlage, zu der diese Instanz gehört. Wird automatisch befüllt und kann geändert werden.
Instanz-Schlüssel (optional) Eindeutiger Schlüssel der Instanz innerhalb der Vorlage, z.B. für Instanziierung aus dem SCC heraus.
Name Name der Vorlagen-Instanz.
Beschreibung (optional) Beschreibung der Vorlagen-Instanz.
Parameterwerte

Zusammenfassung der übergebenen Werte für die Parameter der Instanz als JSON-String.

Beispiel:

{"akz":"plc181","plcname":"Anlage 18","varcnt":3,"parent_node":1}

Nur zur Information, schreibgeschützt.
Aktualisierung empf.

Diese Checkbox wird bei jeder Überprüfung der Instanz auf 1 gesetzt, sofern bei der Aktualisierung Änderungen durchgeführt würden. Sind alle Werte der Instanz aktuell, so wird die Checkbox auf 0 gesetzt. Das Setzen der Checkbox erfolgt durch das System, die Spalte ist schreibgeschützt.
Instanz-Info

Kurze Information über den Status der Instanz, z.B. wie viele Änderungen beim Update ausgeführt werden. Nur zur Information, schreibgeschützt.
Letzte Aktualisierung

Zeitstempel, wann die Instanz-Info zuletzt aktualisiert wurde. Nur zur Information, schreibgeschützt.
Parameter

Für jeden Parameter der Vorlage (Spalte "Parameter" der Tabelle "Projektierungsvorlage") wird hier eine eigene Spalte angezeigt, die mit dem entsprechenden Wert befüllt werden kann.

 

Kontextmenü

Durch Klick mit der rechten Maustaste auf eine oder mehrere Vorlagen-Instanzen kann das Kontextmenü geöffnet werden (im Baum oder in der Tabelle). Neben den allgemeinen Optionen finden sich dort die folgenden Aktionen:

"Instanz aktualisieren" evaluieren

Es wird eine Prüfung durchgeführt, ob und falls ja, welche Änderungen bei einer Aktualisierung der Instanz durchgeführt werden.

Instanz aktualisieren

Die anstehenden Änderungen der Instanz werden übertragen.

"Elemente löschen" evaluieren

Es wird eine Prüfung durchgeführt, welche Änderungen beim Löschen der Instanz durchgeführt werden.

Elemente der Instanz löschen

Alle durch die Instanz angelegten Elemente werden gelöscht, sofern möglich. Wurden bereits Datensätze erstellt, die die Instanz-Elemente referenzieren (z.B. Werte für Variablen gebucht), so ist ein Löschen nicht möglich. In diesem Fall wird eine entsprechende Fehlermeldung ausgegeben und der Benutzer hat die Möglichkeit, diese Elemente manuell zu löschen, z.B. über die LC2-Funktion "Referenzen analysieren". Diese Elemente bleiben in der Tabelle Instanz-Proxys stehen und werden mit dem Flag "Gedroppt?" = 1 versehen.

 

Für alle Aktionen öffnet sich ein Fenster, in dem der Fortschritt verfolgt werden kann (Aktionsdialog). Nach Beendigung der Aktion wird das Ergebnis angezeigt. Durch Mouseover werden alle durchgeführten Änderungen eingeblendet. Das Ergebnis wird außerdem als neuer Eintrag in der Tabelle "Ausführungen" eingetragen und kann dort auch später nachgelesen werden. Der Aktionsdialog, der für alle Aktionen angezeigt wird, sieht folgendermaßen aus:

Durch Mouseover über die jeweilige Zeile wird deren Ergebnis komplett eingeblendet.

Durch Klick auf "erledigte Elemente löschen" wird die Liste des Dialogs geleert.

Durch Klick auf "Anstehende Elemente überspringen" wird die Ausführung aller noch nicht gestarteten Aktionen zurückgezogen. Die aktuell laufende Aktion wird fertiggestellt, sie wird nicht abgebrochen.
