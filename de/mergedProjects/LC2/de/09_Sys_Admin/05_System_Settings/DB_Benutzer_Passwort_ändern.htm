DB Benutzer Passwort ändern

Änderungen im LC2

Der LC2 verwendet zwei unabhängige DB-Benutzer:

  • LC2-Login-User (niedrig privilegierter Benutzer)
    Dieser wird gemeinsam mit den Verbindungseinstellungen auf jedem Rechner lokal gespeichert und wird demnach von jedem LC2-Anwender selbst verwaltet.

  • Intern für Schreibzugriffe verwendeter Benutzer (höher privilegierter Benutzer)
    Dieser wird systemweit zentral in einem Systemparameter gespeichert und in der Regel von einem Key User/Administrator festgelegt. Die Anpassung dieses Passwortes sollte unmittelbar nach der Passwortänderung durchgeführt werden.  

 

Änderung des Passworts als LC2-Login-User (niedrig privilegierter Benutzer)

 

  1. Wählen Sie nach dem Starten des LC2 im Database Connections-Dialog (Verbindungsdialog) die gewünschte Datenbank aus.

  2. Klicken Sie auf die Edit-Schaltfläche.

  3. Bearbeiten Sie im Properties-Dialog anschließend die DB-Verbindung unter Connection > Legato Connection String [...].

  4. Ändern Sie im Connection Properties-Dialog anschließend das Passwort.

  5. Klicken Sie abschließend auf die Test Connection-Schaltfläche, um die Verbindung zu testen.

 

Das Passwort für den Login-User wird verschlüsselt lokal auf dem Rechner in der Datei global.config gespeichert.

 

Änderung des Passwortes als höher privilegierter Benutzer

Der LC2 verwendet intern den höher privilegierten Benutzer, um Konfigurationsänderungen in die Datenbank speichern zu können. Die Änderung des Passwortes dieses Benutzers erfolgt über den LC2.

 

  1. Verbinden Sie sich mit der gewünschten Datenbank.

  1. Nach der Anmeldung mit dem Legato-Benutzer erscheint der Database Read/Write User-Dialog. Geben Sie hier nicht den gewöhnlichen Sapient-Anmeldename an. An dieser Stelle muss der höher privilegierte Benutzer eingegeben werden.

Dieser eingegebene Benutzer wird mit verschlüsseltem Passwort in den Systemparametern -202 und -203 hinterlegt.

 

 

Änderungen in Sapient

In regelmäßigen Abständen müssen die Benutzerpasswörter für die Datenbank geändert werden.

 

Einige Teile dieser Beschreibung sind kundenspezifisch. Klären Sie mit Ihrem Gefasoft-Ansprechpartner die genaue Vorgehensweise ab.

 

Gehen Sie wie folgt vor, um diese DB-Änderungen für Benutzer durchzuführen.

 

  1. Stoppen Sie alle Dienste, die sowohl direkt, als auch indirekt auf die DB zugreifen.
    • SapientEngine(s)
    • SCC(s)
    • Applikation(en) zu Drittsystemen, welche Zugriff auf die Datenbank haben.

  1. Ändern Sie die Passwörter der relevanten DB-Benutzer. Diese sind z.B.
    • SAPIENT_CLIENT
    • SAPIENT_OWNER
    • SAPIENT_SERVER
    • SAPIENT_READ (hat nur Leserechte)

 

  1. Passen Sie die Properties-Datei „sapieng.properties“ für alle SapientEngine(s) an.

  2. Tragen Sie mit dem SQL-Developer/SQL-Workbench in der Tabelle “GP_R_CONNECTIONS” unter PROPERTIES das neue Passwort ein. Dies gilt für die Datenbankverbindungen (object_names in der Tabelle "GP_R_CONNECTIONS"):

    • REPOSITORY (Zugriff auf die GP_R-Tabellen)
    • LEGATO (Zugriff auf alle anderen Tabellen)

 

  1. Starten Sie die SapientEngine(s).

 

  1. Starten Sie den LC2 und passen Sie die Verbindungsparameter an. Testen Sie anschließend die Verbindung.

 

  1. Melden Sie sich am LC2 mit dem eigenen User an. Die Anwendung erkennt die Änderungen und fragt nach neuem DB User. Verwenden Sie den User „SAPIENT_OWNER“ und tragen Sie das neue Passwort ein.

 

  1. Starten Sie alle anderen Programme (SCCs, Koppelung zu Drittsysteme) und kontrollieren Sie die Fehlerausgabe.

 

 

  • Die Passwörter müssen an allen Stellen geändert werden, an denen Benutzer existieren (z.B. beim Starten der Sapient Engine).

  • Es gibt für die Sapient Engine eine Properties-Datei (sapieng.properties), die einen Link zur Datenbank enthält. Dieser Link wird benötigt, damit sich die Sapient Engine mit der Datenbank verbinden kann.

  • In der Properties-Datei sind die Passwörter abrufbar. Die Passwörter sind obfuscated (verschleiert/nicht im Klartext angezeigt, jedoch auch nicht verschlüsselt).

  • Die Sapient Engine kann so gestartet werden, dass das Passwort nicht verschleiert dargestellt wird.

  • Für alle Sapient Engines müssen die Passwörter in den Properties-Dateien geändert werden.

 

Verschleierte/obfuscated Passwörter:

Im SQL Workbench ist in der Tabelle gp_r_connections (Liste der Verbindungen im Repository) unter Data und Properties das Passwort abrufbar (entweder obfuscated oder auch im Klartext).
