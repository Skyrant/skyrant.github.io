Read/Write DB Benutzer Einstellungen

Systemadministration | Systemeinstellungen | Read/Write DB Benutzer Einstellungen

Sofern die Authentifizierung des Benutzers nicht gegenüber der Datenbank erfolgt (à Benutzerverwaltung), erfolgt die Anmeldung in drei Schritten

Anmeldung mit einem DB-Benutzer mit Minimalrechten (normalerweise erscheint dieser Dialog nicht, da das Kennwort für diesen Benutzer problemlos gespeichert werden kann)

Es wird in der Datenbank die Authentifizierungsart gelesen (intern, LDAP, Kerberos…) und der entsprechende Anmeldedialog für den Benutzer gezeigt.

Nach erfolgreicher Benutzerprüfung wird der read/write-Benutzer aus der Datenbank gelesen und intern für alle Datenbankzugriffe verwendet (das Passwort ist verschlüsselt im Systemparameter 1103 gespeichert!).

 

Über den DB-Benutzer (read/write)-Dialog wird dieser interne Datenbankbenutzer festgelegt. Dieser Benutzer benötigt uneingeschränkten Zugriff auf alle Legato-Objekte in der Datenbank.
