Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer. Wird vom System vergeben. Schreibgeschützt.

Modul

Lookup-Liste zur Auswahl des zugehörigen Moduls. Wird automatisch befüllt und kann geändert werden.

Gerätetyp (optional)

Lookup-Liste zur Auswahl des Gerätetyps.

Die Auswahl des Gerätetyps dient standardmäßig nur Anzeigezwecken zur Strukturierung v.a. großer Projekte.

ID (SPS Identifier)

Eindeutige Bezeichnung der Steuerung (z.B. SPS) im Projekt, z.B. Ortskennzeichen oder Betriebsmittelkennzeichen. Standard-Gateway-Skripte verwenden diese ID für die Buchung von Verbindungszuständen diese Symbolik für die Adressierung der Verbindung in der Datenbank. Die Verbindungssymbolik im Gateway und hier sollten entsprechend gleichlautend sein.

Name

Verbindungsname. Der Name der Verbindung wird im WebClient an verschiedenen Stellen als Quelle visualisiert.

Hinweis: Im Namen darf die Zeichenfolge "##." nicht verwendet werden. Ansonsten wird eine Fehlermeldung angezeigt.

Beschreibung (optional)

Zusätzliche Beschreibung der Verbindung. Die Beschreibung der Verbindung wird im Webclient an verschiedenen Stellen als Quellenbeschreibung visualisiert.

Knoten (Default)

Lookup-Liste zur Auswahl eines Default-Knotens aus der Fertigungsstruktur. Diesem Knoten werden System-Meldung zur Verbindung und eventuell parametrierte Grenzwertverletzungen zugeordnet.

Beim Import von Meldungstexten für das Meldesystem werden Meldungen standardmäßig diesem Knoten zugeordnet.

Parent (optional)

Lookup-Liste zur Auswahl einer übergeordneten Verbindung zur hierarchischen Darstellung.

Sync (optional)

Checkbox. 0 = Verbindung ist noch nicht synchronisiert; 1 = Verbindung ist synchronisiert. Default ist 0.

Verbindungsstatus Nr.

Anzeige des aktuellen Verbindungsstatus als Nummer. Nur zur Information; schreibgeschützt.

Verbindungsstatus

Anzeige des aktuellen Verbindungsstatus als Text. Nur zur Information; schreibgeschützt.

Importeinstellungen (Meldungen)

Hier werden die Importeinstellungen für den Zuli-Import eingetragen. Für Details, siehe unten.

Auto Import

Die Datei wird für einen automatischen Import durch die Legato Sapient Engine vorgemerkt. Über Jobs wird die angegebene Datei auf Änderungen überwacht und der Import bei Bedarf mit den gewählten Einstellungen automatisch durchgeführt.

Letzter Zuli Import

Datum des letzten Imports von Zuordnungslisten (Zuli). Nur zur Information; schreibgeschützt.

Datum letzte Zuli

Datum der zuletzt importierten Zuli. Nur zur Information; schreibgeschützt.

Letzter Import-Fehler Text

Text des letzten Import-Fehlers. Nur zur Information; schreibgeschützt.

 

 

Importeinstellungen (Meldungen)

In der Spalte "Importeinstellungen (Meldungen)" der Verbindung werden die Einstellungen für den Import festgelegt.

:

Bei den Parametern ist auf Groß-/Kleinschreibung zu achten!

 

Folgende Einstellungen sind verbindlich in jedem Fall einzutragen:

Filename='Pfad zur und Dateiname mit Endung' (Beispiel: C:\Temp\Importdateien\GraphpicS5CSV.CSV)

FileType='Angabe des Typs der Import-Datei' (Beispiel: 5)

Default.AL_CLASS='ID der Default-Meldeklasse' Beispiel: 0)

Default.AL_TYPE='ID der Default-Meldeart' (Beispiel: 0)

 

Folgende Einstellungen können optional eingetragen werden:

Script='Name des auszuführenden Skripts'

:

Bei Verwendung des FileType 20 (Benutzerdefiniertes Skript) ist diese Eingabe verbindlich.

DeleteOldItems='Löschen alter Zuordnungen nach erneutem Import' (Beispiel: 1)

UpdateItems='Aktualisieren bestehender Zuordnungen' (Beispiel: 1)

Charset='Characterset der Import-Datei' (Beispiel: ISO8859_15)

 

Folgende optionale Einstellungen werden nur bei den Datentypen 1 (S5-SEQ-Datei), 2 (S7-SEQ-Datei) und 7 (S7-SDF-Datei) ausgewertet und verarbeitet:

ImportFilterOpType='ImportFilterOperationType' (mögliche Einträge: AB, EB, MB, SY); (Beispiel: SY)

:

SY ist nur bei FileType 1 (S5-SEQ_Datei) zulässig.

ImportFilterFirstByte='ImportFilter von (inklusive) erstem Byte' (Beispiel: 10)

ImportfilterLastByte='ImportFilter bis (inklusive) letztem Byte' (Beispiel: 1000)

 

Die beiden letzten dieser 3 Parameter werden nur ausgewertet, wenn der erste Parameter einen sinnvollen Eintrag enthält.

Mit obigen Beispielwerten werden nur Zuordnungen von SY 10 bis SY 1000 importiert.

 

Standardmäßig werden folgende Datenformate unterstützt:

S5-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S5-Programmes

S7-SDF-Dateien (*.sdf) Empfohlene Quelldatei bei Verwendung eines Simatic S7-Programmes

S7-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S7-Programmes (beschränkt auf 40 Zeichen)

P-DIAG (*.csv) aus Simatic PDiag exportierte Textdateien

Textliste – Meldungsnr. & Text (*.txt) mit tabulatorgetrennter Meldungsnummer und Text

P-DIAG XML (*.xml) Dateien

 

Spezielle Formate für eine einheitliche Textbasis bei Verwendung von GEFASOFT GraphPic NT und Legato in einem Verbundprojekt sind:

S5/S7-GraphPic-Importdateien (*.txt oder *.csv), die von GraphPic NT erzeugt werden können.

Kontextmenü

Im Kontextmenü der Verbindungen (rechte Maustaste auf die Verbindung im Navigationsbaum) werden verschiedene Optionen angeboten.

 

Alle Meldungen zurücksetzen

Setzt alle anstehenden Meldungen der selektierten Verbindung (ohne untergeordnete Verbindungen) auf "Gehend".

 

Alle Meldungen zurücksetzen (auch Unterverbindungen)

Setzt alle anstehenden Meldungen der selektierten Verbindung (inklusive aller untergeordneter Verbindungen) auf "Gehend".

 

 

Meldungsdefinitionen: Import starten

Stößt den Import der Zuordnungsdatei an. Die Einstellungen werden im Feld "Importeinstellungen (Meldungen)" der Verbindung getroffen. Gibt es in diesem Feld keinen Eintrag, so wird eine entsprechende Fehlermeldung ausgegeben. Die entsprechenden Sapient Engine-Jobs sind in der separaten Dokumentation genauer beschrieben.

 

Synchronisationsfunktion der Steuerelemente

Für jede SPS-Verbindung gibt es im LC2 eine Synchronisationsfunktion, mit der Steuerungen synchronisiert/initialisiert werden können.

Diese Funktion können Sie im LC2 im Bereich Anwendungsstruktur und der Datenaustausch-Tabelle (P_CONNECTIONS) finden:

 

 

Durch das Auswählen des Kontrollkästchens in der Spalte Sync aktivieren Sie die Synchronisation.

 

:

Das Aktivieren der Synchronisationsfunktion hat nur eine Auswirkung, wenn diese im entsprechenden ITC-Projekt programmiert und somit festgelegt ist, wie mit der aktivierten Synchronisation weiter verfahren werden soll. Dies ist die Voraussetzung für das Funktionieren der Sync-Funktion.
