Funktion

Die Verdichtung und Aggregation wird von einem Job (AggrPoller) der Sapient Engine anhand der im Folgenden beschriebenen Parametrierung durchgeführt.

:

Der Aggregations-Job muss einer Instanz der Sapient Engine zur Ausführung zugewiesen sein, um zu laufen.

Dies erfolgt unter System Administration - SE Jobs. Stellen Sie sicher, dass der Job AggrPoller korrekt parametriert ist und der richtigen Instanz der SE zugewiesen ist. Weitere Informationen hierzu finden Sie im Kapitel System Administration - SE Jobs sowie in der separaten Dokumentation "Sapient Prozeduren und prozessnahe Funktionen".

Es können für jede Variable folgende Aggregationsdefinitionen angelegt und berechnet werden:

sum: Summe

avg: Durchschnitt

max: Maximum

min: Minimum

first: Erster Wert

last: Letzter Wert

 

Die verfügbaren Aggregationstypen können unter System-Administration | Systemtabellen | Aggregationstypen gepflegt werden.

Während der Aggregation wird ermittelt, ob die angeforderte Aggregation auf Basis einer schon existierenden Aggregation berechnet werden kann. Zum Beispiel für Tagesaggregation können Stundenaggregationen als Vorberechnungen verwendet werden. Wurde keine Stundenaggregation definiert, so wird die Tagesaggregation auf Basis von Rohdaten berechnet.

:

Bei Aggregationen über lange Zeiträume, z.B. Jahresaggregation, kann das Fehlen von Vorberechnungen zu Problemen führen, wenn die Rohdaten nicht so lange aufbewahrt werden (abhängig von Löschjobs). Es wird empfohlen, hierfür sinnvolle Voraggregationen anzulegen (mindestens Monatsaggregationen).

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige ID. Wird vom System vergeben; schreibgeschützt.

Variable

Lookup-Liste zur Auswahl der Variable, deren Werte aggregiert werden sollen. Wird automatisch befüllt und kann geändert werden.

Zeitstempel von

Von-Zeitstempel, wann die letzte Aggregation gestartet wurde. Nur zur Information; schreibgeschützt.

Zeitstempel bis

Bis-Zeitstempel, wann die letzte Aggregation abgeschlossen wurde. Nur zur Information; schreibgeschützt.

Knoten

Für den Aggregationszeitraum "Schicht" ist hier eine Eingabe erforderlich. Lookup-Liste zur Auswahl eines Knotens der Fertigungsstruktur, von dem das Zeitmodell verwendet werden soll.

Ergebnistyp

Lookup-Liste zur Auswahl des gewünschten Ergebnistyps der Aggregation, z.B. Summe oder Durchschnitt.

Beschreibung (optional)

Beschreibung der Aggregation.
