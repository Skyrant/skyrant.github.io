Funktion

Meldungen können automatisch an ein Funkrufsystem (z.B. PageControl der Fa. informel) weitergegeben werden. Legato verfügt über eine offene Schnittstelle, über die ein beliebiges System zur Meldungsweitergabe versorgt werden kann. Nachdem die Parametrierung von Empfängern stark vom verwendeten Rufsystem und dem Übertragungsmedium abhängt, wird in Legato lediglich eine Liste von Weiterleitungszielen (Empfängern) gehalten, die eigentliche Parametrierung der Empfänger erfolgt im Rufsystem. Dort werden Empfängereinstellungen und Funktionen wie Gruppenbildung, Folgerufe oder Eskalation parametriert.

Die Anbindung eines Funkrufservers erfolgt in der Regel im Rahmen der Projekterstellung bzw. des Customizings.

 

:

Detaillierte Informationen zur Verwendung der allgemeinen Legato-Funkrufschnittstelle erhalten Sie von unserem Legato-Service.

 

Mehrere Funkruf-Empfänger

Für einen Funkruf können mehrere unterschiedliche Empfänger im LC2 angegeben werden, d.h. beim Verbuchen einer Meldung ist es nicht nur möglich, komplette Gruppen oder eine einzelne Person mittels eines Funkrufes zu alarmieren, sondern auch ausgewählte (und somit unterschiedliche) Personen.

 

In der Sapient WebApp ist diese Funktion im Boardlet Archivierte Meldungen integriert. Dort kann man mittels einer Dropdown-Liste beispielsweise nach mehreren Funkrufempfängern filtern. Siehe hierzu den Bereich Archivierte Meldungen - Mehrere Funkrufempfänger.

 

Zuordnung von Funkruf-Empfänger

Im LC2 kann das Paging-Empfängerfeld erweitert werden, damit es alle zugewiesenen Empfänger enthält. Beachten Sie Folgendes:

 

  • Die Tabellen zur Konfiguration von Meldedefinition, Knoten je Klasse und der Knoten selbst verfügen über das Feld "Empfänger", das eine durch Kommata getrennte Liste aller zugewiesenen Empfänger enthält. Somit kann die Liste aller zugeordneten Empfängern über ein einziges Feld zugewiesen werden.


  • Die Liste zeigt den eindeutigen Empfängercode jedes Empfängers in der Spalte Code (number) an und kann manuell durch Hinzufügen oder Löschen von Empfängercodes bearbeitet werden.


  • Um auf einfache Art und Weise einen oder mehrere Empfänger hinzuzufügen ist es zusätzlich möglich, Empfänger aus dem Konfigurationsbaum per Drag-and-Drop  in das Empfängerfeld zu ziehen.
    Beispielpfad: Anwendungsstruktur -> Tabelle (z.B. Datenaustausch) ->   flache Zeile/flachen Tabelleneintrag

 

  • Außerdem ist durch Copy-and-Paste das Einfügen von Empfängern im Empfängerfeld möglich.


  • Auch per Texteingabe können neue Empfänger dem Empfängerfeld hinzufügt werden.



  • Wenn nichtexistierende Empfänger der Zeile hinzugefügt werden, erscheint eine Fehlermeldung.


  • Die zugewiesenen Empfänger können weiterhin über die separate Untertabelle, die für Alarmdefinitionen, Knoten und Knoten-/Klassenkonfigurationen verfügbar ist, bearbeitet werden.

 

Zusatzinformationen zur Alarmierung mehrere Empfänger

Beachten Sie im Vorfeld die Informationen im vorangegangenen Abschnitt Zuordnung von Funkruf-Empfänger. Mit der Alarmierung mehrerer Empfänger ist es möglich, mehrere Empfängergruppen mit einer Meldung zu verknüpfen. Dies reduziert die notwendige Anzahl an Empfängergruppen deutlich. Es muss also nicht mehr für jede Konstellation eine neue Empfängergruppe angelegt werden.

Konfiguration

Empfängergruppen: Empfänger können im LC2 über Werksverwaltung →  Meldungsmanagement →  Funkruf-Empfänger konfiguriert werden.

Die zugehörige DB Tabelle lautet m_recipients.

 

Die Konfiguration der Funkrufempfängergruppen kann an vier Stellen erfolgen:

1. Meldungsdefinition

2. Dem Knoten zugeordnete Meldungen

3. Funkruf-Empfänger je Knoten

4. Funkruf-Empfänger je Knoten und Klasse

 

:

Wenn eine Konfiguration an Stelle eins erfolgt, unterbindet diese alle Konfigurationen in Punkt zwei, oder Punkt drei usw.

 

Das System ist hierbei hierarchisch zu verstehen.

 

1. Meldungsdefinition

Pfad im LC2: Anwendungsstruktur →  Datenaustausch →  "Verbindung" →  Meldungsdefinitionen

Für diese Meldungsdefinition werden ein oder mehrere Funkrufempfänger hinterlegt.

Im folgenden Beispiel sind der Meldung mit der Nummer 1 die Empfänger 1 und 3 zugeordnet.

2. Dem Knoten zugeordnete Meldungen

Pfad im LC2: Anwendungsstruktur → Fertigungsstruktur → "Knoten" → Meldungszuordnungen

Für diese Meldungen werden ein oder mehrere Funkrufempfänger hinterlegt.

Im folgenden Beispiel sind der Meldung mit der Nummer 1 die Empfänger 1 und 3 zugeordnet.

3. Funkruf-Empfänger je Knoten

Pfad im LC2: Anwendungsstruktur → Fertigungsstruktur → "Knoten" → Funkruf-Empfänger

Für den Knoten wird für alle ihm zugeordneten Meldungen eine Weiterleitung an die Funkrufempfänger erfolgen.

Im folgenden Beispiel werden alle Meldungen an die Empfänger 3 und ad weitergeleitet.

4. Funkruf-Empfänger je Knoten und Klasse

Pfad im LC2: Anwendungsstruktur → Fertigungsstruktur → "Knoten" → Funkruf-Konfiguration

Für einen Knoten können Funkruf-Empfänger für jede Meldeklasse konfiguriert werden. Dies betrifft nur die Meldungen, die dem jeweiligen Knoten zugeordnet sind.

Im folgenden Beispiel (mit Meldeklassen mit Priorität und Weiterleitungsart) werden alle Meldungen der Meldeklasse Jobfehler an die Empfänger paging_test, ad, mf und tm weitergeleitet.

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer. Kann frei vergeben werden, muss aber eindeutig sein.

Code (Nummer)

Code: Symbolische Empfängerbezeichnung (z.B. Empfänger- oder Gruppenname bei PageControl), bei anderen Servern kann hier beispielsweise auch direkt die Rufnummer hinterlegt sein.

Empfängername

Empfängername: Klartextbezeichnung für den Empfänger.

Anzeige in Liste (man. FR)

Checkbox. Wenn aktiv, erscheint dieser Empfänger in der Ansicht "Manueller Funkruf" der WebApp zur Auswahl.

Funkruf-Server (optional)

Angabe des Funkrufservers, über den die Meldung weitergegeben wird. Die Verteilung auf unterschiedliche Rufserver muss im Funkruf-Gateway berücksichtigt werden.

Sprache (optional)

Lookup-Liste zur Auswahl der Systemsprache, in der die Meldetexte an diesen Funkrufempfänger gesendet werden.
