Task dbtimesync

Beschreibung

Diese Task fragt die Uhrzeit ("absolut", ohne Informationen über dessen Zeitzone) des Datenbank-Servers, auf dem die Sapient Datenbank läuft, ab und stellt den Zeit-Offset zwischen DB-Server und Sapient Engine zur Verfügung. Diese Task muss ausgeführt werden, wenn eine SE-Instanz die Task "jobs" nicht ausführt (siehe auch JOB_POLLER).

:

Für jede Instanz muss entweder die Task "jobs" oder die Task "dbtimesync" aufgerufen werden!

Die Task "dbtimesync" wird durch die folgenden Einträge in der Konfigurationsdatei der Sapient Engine (sapieng.properties) parametriert:

# Db time synchronization task (needed only, if no "jobs" task)

dbtimesync.class = de.gefasoft.sapient.sapieng.main.DbTimeSync

dbtimesync.period = 900

 

Die Einträge haben dabei folgende Bedeutung:

dbtimesync.class - Name der Implementierung

dbtimesync.period - Anzahl Sekunden zwischen den Aktualisierungen. Standardwert 900 = 15 Minuten.
