TemplateAction

Repräsentiert einen Ziel-Datensatz.

TemplateAction-Objekte haben folgende Methoden:

addField(String fieldName, Object value, Boolean preventUpdate):

Fügt ein Feld zu einer Vorlagen-Aktion hinzu. Kann mit folgenden Parametern aufgerufen werden:

fieldName: Name des Zielfeldes

value: Der Wert, der beim Einfügen/Aktualisieren der Zielzeile gesetzt wird

preventUpdate: Wahr --> der Wert wird nur als Default gesetzt, wenn die Zielzeile eingefügt wird. Andernfalls wird der Wert jedesmal aktualisiert wenn die Vorlage aktualisiert wird

 

setSharingOptions(Object options)

Stellt sicher, dass es eine einzelne Zielzeile gibt, die für mehr als eine Vorlageninstanz verwendet werden soll.

 

Beispiel:

Stellt die Existenz eines einzigen Knotens mit einem spezifischen Kurznamen innerhalb eines Parent sicher:

actNode.setSharingOptions({ keyFields: ["parent", "short_name"] });

actNode.addField("short_name", th.getParamValue("nodesym"), false);

actNode.addField("parent", parentNode, false);
