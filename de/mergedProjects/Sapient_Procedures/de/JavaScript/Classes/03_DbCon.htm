DbCon

Objekte dieses Typs bieten jeweils Zugriff auf eine Datenbankverbindung aus einem bestimmten Pool. Sie werden benötigt, um Objekte vom Typ DbQuery anlegen zu können.

DbCon(context, dbconnectionname, readwrite, shared)

Holt eine Verbindung aus dem Verbindungs-Pool namens "dbconnectionname" (üblicherweise "legato") des angegebenen SE-Contexts (Task).

Readwrite(opt): Wenn der erste Parameter ein Sapient Context-Objekt war, legt dieser Parameter fest, ob eine "nur lesen" oder eine "lesen/schreiben" Verbindung zugewiesen wird. Falls "wahr-ähnlich" im Sinne von JavaScript wird eine "lesen/schreiben" Verbindung zugewiesen. Andernfalls vergibt die Verbindung nur lesenden Zugriff auf die Datenbank.

Shared(opt): Wenn <code>truish</code> wird eine geteilte Verbindung vom Sapient Context ausgeliehen. Wenn <code>falsy</code> wird eine nicht-geteilte Verbindung ausgeliehen.

 

DbCon hat nur eine ausführungsrelevante Eigenschaft: autocommit (Boolean, readwrite).

Um die Verarbeitung von Feld- und Parametertypen von SQL-Abfragen zu erleichtern, enthält DbCon noch eine Reihe konstanter (readonly) Eigenschaften. Diese können auch "static" (also ohne Objekt) benutzt werden (z.B. "DbConDOUBLE" statt "var.DOUBLE"):

Die Eigenschaft "dbtype" liefert den GP_R_CONNECTION_TYPE der Datenbank, also "POSTGRESQL" oder "ORACLE".

DbCon-Objekte haben folgende Funktionen:

commit()

Übernimmt die bisher abgesetzten Datenbank-Operationen. Wirkt sich nur aus, wenn autocommit auf false steht, ansonsten wird ja jede Operation sofort übernommen.

rollback()

Verwirft die bisher abgesetzten Datenbank-Operationen. Wirkt sich nur aus, wenn autocommit auf false steht, ansonsten wird ja jede Operation sofort übernommen.

getDbTime()

Gibt die aktuelle DB-Zeit (Date) zurück.
