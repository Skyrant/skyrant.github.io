TemplateHelper

Helper für die Verwendung von Projektierungsvorlagen in SE-Skripten.

 

TemplateHelper-Objekte haben folgende Methoden:

addAction(String templateElementKey, String description, String tableName):

Erstellt eine neue TemplateAction-Instanz und gibt diese zurück. Eine Aktion stellt einen Ziel-Datensatz dar (die Aktion stellt sicher, dass der Ziel-Datensatz mit den gewünschten Feldwerten existiert). Kann mit folgenden Parametern aufgerufen werden:

templateElementKey: Kennung, eindeutig innerhalb eines templateElementKey

description: Beschreibt die Aktion (z.B., um eine Aktion im Serverlog zu identifizieren)

tableName: Name der Zieltabelle

 

getParamValue(String name)

Gibt den Wert eines Vorlagen-Parameters zurück. Kann mit folgenden Parametern aufgerufen werden:

name: Name des Parameters

 

findOrCreateAndUpdateInstance(Object selector, Object paramValues, Boolean evaluationRun)

Findet und aktualisiert oder erstellt eine Vorlageninstanz. Gibt eine Liste mit allen Änderungen zurück, die nötig sind, um die Instanz zu erstellen oder zu aktualisieren. Kann mit folgenden Parametern aufgerufen werden:

selector: Identifziert die Ziel-Instanz. Derzeit ist die einzige Möglichkeit, eine Vorlage auszuwählen die Kombination aus template_symbol + instance key

paramValues: Objekt, das alle Parameter einer Vorlage enthält

evaluationRun: Wahr --> Evaluiert alle Änderungen die nötig sind, um die Instanz zu erstellen oder zu aktualisieren

 

getRowIdByActionKey(String actionKey)

Gibt den Primärschlüssel-Wert einer Zeile zurück, die durch die Aktion mit dem übergebenen Schlüssel betroffen ist.

 

getFieldValue(String tableName, String lookupField, String keyField, Object keyValue)

Gibt den Wert eines Tabellenfeldes zurück. Kann mit folgenden Parametern aufgerufen werden:

tableName: Name der Tabelle

lookupField: Name des zurückgegebenen Feldes

keyField: Name des Schlüsselfeldes

keyValue: Schlüssel
