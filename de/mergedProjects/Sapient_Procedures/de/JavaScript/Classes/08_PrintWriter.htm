PrintWriter

Objekte dieses Typs schreiben Daten bereits stückweise zum Server, während sie noch übertragen werden. Das heißt, die Daten müssen nicht erst komplett in den Speicher geschrieben werden, ehe sie durchsucht werden können. Vielmehr ist es möglich, die bereits geschriebenen Stücke gleich zu durchsuchen.

Objekte werden dabei meist von den Methoden anderer Objekte geliefert, z.B. HttpCon.getRequestPrinter(), können aber auch durch new PrintWriter() erzeugt werden. Dabei sind folgende Kombinationen von Parametern möglich:

new.PrintWriter(file)

Erzeugt einen PrintWriter zum Schreiben in die angegebene Datei (File-Objekt).

newPrintWriter(filename, charsetname)

Erzeugt einen PrintWriter zum Schreiben in die Datei mit dem angegebenen Namen (String). Optional kann auch der Zeichensatz angegeben werden (String).

 

PrintWriter haben die folgenden Funktionen:

checkError():

Boolean; prüft, ob soweit alles korrekt geschrieben werden konnte.

close():

Schließt die Datei oder sonstigen Ausgabekanal. Danach kann nicht mehr geschrieben werden.

flush():

Schreibt die bisherigen Daten in den Ausgabekanal (und leert den Cache).

print(data):

Gibt die unter "data" vorhandenen Daten aus.

println(data):

Gibt die unter "data" vorhandenen Daten aus und beendet die Zeile.
