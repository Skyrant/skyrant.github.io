Einleitung

JavaScripts können in der Sapient Engine verwendet werden, um Jobs, Prozeduren und Datenquellen-Abfragen zu kodieren.

Jobs der Sapient Engine werden komplett in JavaScript geschrieben. Unter "Funktionspfad" ist dann der Pfad der Datei einzutragen.

Es gibt zwei Verzeichnisse:

scripts (für standardmäßig ausgelieferte Scripten)

customscripts (für selbst erstelle Scripten)

 

Es wird immer erst der Ordner "customscripts" und dann der Ordner "scripts" durchsucht. Wird in "customscripts" eine entsprechende Datei gefunden, wird diese ausgeführt und nicht weiter gesucht. Wo genau die jeweiligen Verzeichnisse zu finden sind, kann im Startscript der Sapient Engine eingestellt werden.

Innerhalb dieser Verzeichnisse gibt es Ordner für:

configurationtemplates

jobs

jsonapi

jsonrpc

query

 

Für jobs und jsonrpc können die Scripten grundsätzlich an einem beliebigen Ort abgelegt werden. Der Pfad wird dann beim Job bzw. dem RPC-Aufruf mit angegeben.

Für jsonapi und query müssen die Scripten in den jeweiligen Verzeichnissen gespeichert werden, weil sie beim Laden des Repositorys nur dort gesucht werden.

Über einen Parameter wird angegeben, wie oft nach dem erstmaligen Laden geprüft werden soll, ob ein Script geändert wurde bzw. ob zwischenzeitlich ein neuer Eintrag in customscripts erstellt wurde.

Je nach Art des Scripts muss die in den folgenden Kapiteln beschriebene Struktur berücksichtigt werden.
