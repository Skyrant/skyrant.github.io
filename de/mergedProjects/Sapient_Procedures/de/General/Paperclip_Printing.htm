Paperclip Printing Timeout

Das http-Timeout für das Paperclip Printing ist einstellbar.

 

Beachten Sie hierzu folgendes Szenario:

  • Ein Benutzer möchte ein umfassendes Dashboard ausdrucken.

  • Ein Benutzer klickt auf einen Druckbutton auf der UI (z.B. Adhoc-Druck).

  • Die Druckanforderung wird an die Sapient Engine gesendet.

  • Die Sapient Engine ruft anschließend den Paperclip Printing Service auf. Durch diesen Aufruf ist der Dashboard-Druck erst möglich.

  • Das Dashboard wird dann als Ergebnis des Paperclip-Services als PDF gedruckt.

Ohne die Einstellmöglichkeit des http-Timeouts für den Paperclip-Druck kann es sein, dass PDF-Ausdrucke unterbrochen werden.

Mit dem einstellbaren http-Timeout für das Paperclip Printing ist es möglich, das Timeout auf eine beliebige Zeitspanne einzustellen.

 

Die Standardzeitüberschreitung beträgt 10 Minuten. D.h., wenn der Kunde keine Änderungen in der Zelle vornimmt, beträgt das Timeout 10 Minuten.

 

Beispiel

  • Das http-Timeout ist auf 6 Minuten eingestellt: Wenn ein Dashboard-PDF-Druck länger als 6 Minuten benötigt, wird der Druckauftrag abgebrochen.

  • Wenn das http-Timeout für den Druck verlängert wird, bedeutet dies, dass Dashboard-PDF-Ausdrucke nicht abbrechen.

  • Die http-Zeitüberschreitung kann z. B. auf 30 Minuten eingestellt werden, und der Druck kann garantiert werden.

LC2

  • Im LC2 können Sie das Zeitlimit für das Paperclip Printing unter System-Administration > SE Aufträge > Aufgaben festlegen.

  • Wählen Sie die richtige Instanz (SE-Instanz).

  • Paperclip Printing Timeout kann abschließend im Task type print eingestellt werden.
