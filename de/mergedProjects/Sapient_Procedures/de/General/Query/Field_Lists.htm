Feldlisten abfragen

Die JSON-RPC-Methode um Feldlisten abzufragen ist

de/gefasoft/sapient/data/Query/fields_of

Es gibt drei Parameter:

REPO_CONN (String) - Der Name der Repository-Verbindung, die die abzufragende Datenquelle enthält.

DATASOURCE (String) - Name der Datenquelle, deren Felder abgefragt werden sollen.

INCL_INREFS (Boolean, optional, Default false) - Wenn angegeben und true, enhält die zurückgegebene Feldliste auch eingebundene Referenzen ("to-many"-Beziehungen).

Der Rückgabewert hat zwei Eigenschaften:

queryTime (String Zeitstempel) - Zeitpunkt, wann das Ergebnis erstellt wurde.

data (JSON-Array von JSON-Objekten) - Jedes dieser Elemente enthält Informationen über ein Feld der abgefragten Datenquelle.

name (String) - Der Feldname; innerhalb dieser Datenquelle eindeutig.

type (String) - Der JSON-Type des Feldes: STRING, NUMBER, BOOLEAN oder ANY (wenn nicht bekannt).

keyIndex (String) - Nur verfügbar für Felder die Teil des Primärschlüssels der Datenquelle sind. Gibt die Position an, die dieses Feld in dem Schlüssel hat (1 für die 1., etc.)

localized (Boolean) - Ob die Inhalte dieses Felds lokalisiert sind und sich daher unterscheiden können, abhängig von der Sprachumgebung des Benutzers der die Daten abfragt.

relation (Objekt) - Eine JSON-Objekt-Eigenschaft, die nur verfügbar ist für Felder, die eine Beziehung zu einer anderen Datenquelle sind. Enthält die folgenden Teile:

datasource (String) - Name der Datenquelle, die diese Beziehung referenziert.

field (String) - Name des referenzierten (für abgehende, "to-one"-Beziehungen) bzw. refernzierenden (für eingehende, "to-many"-Beziehungen) Felds in dieser Datenquelle.

many (Boolean) - Ob es sich um eine "to-many", also eingehende Beziehung handelt. Hinweis: Eingehende Beziheungen werden nur aufgeführt, wenn der Parameter  INCL_INREFS im Aufruf dieser Methode auf true gesetzt war.

reffield (String) - Nur für eingehende Beziehungen und nur, wenn bekannt: Der Name des Feldes der Datenquelle, das von der Beziehung referenziert wird.

jsonpart (Object) - Eine JSON-Objekt-Eigenschaft, die nur verfügbar ist für "erweiterte Eigenschaften"-Felder, die in einer JSONB-Spalte gespeichert sind.

field (String) - Name des JSONB-Felds, das die Daten dieses virtuellen Feldes enthält.

path (String) - Pfad in den JSON-Inhalt des "Feldes", das die Daten dieses virtuellen Feldes referenziert.

dbcolname (String) - Name der Datenbank-Tabellenspalte, die diesem Feld entspricht. Nicht verfügbar für "to-many"-Beziehungen; kann ein SQL-Ausdruck sein, wenn dieses Feld lokalisiert wird oder Teil einer JSONB-Spalte ist.

title (String) - Spaltentitel, wie im Repository definiert, übersetzt in die Sprachumgebung des Benutzers, der diese RPC-Methode aufgerufen hat.

hint (String) - Tooltipp-Hinweis für dieses Feld, wie im Repository definiert, übersetzt in die Sprachumgebung des Benutzers, der diese RPC-Methode aufgerufen hat.
