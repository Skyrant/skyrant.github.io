Arbeitsschritte

Allgemein

Das Boardlet Arbeitsschritte zeigt zu Maßnahmen definierte Arbeitsschritte an.

Über den Filter kann dabei zum einen definiert werden, ob man nur Arbeitsschritte anzeigen möchte, für die man selbst verantwortlich ist (own) oder alle. Außerdem kann definiert werden, ob man nur Arbeitsschritte mit einem bestimmten Status anzeigen will oder alle. Außerdem kann festgelegt werden, ob die Anzeige knotenbezogen erfolgen soll. Dann werden alle Maßnahmen angezeigt, die dem selektierten Knoten oder einem seiner Unterknoten zugeordnet sind.

Man kann also beispielsweise das Arbeitsschritt-Boardlet mehrere Male im Dashboard platzieren und jeweils einen anderen Status anzeigen. Per Drag & Drop können die Arbeitsschritte dann einfach organisiert werden, z.B. von "geplant" nach "erledigt" verschoben werden.

Bei Mouseover über den Namen des Verantwortlichen werden dessen Kontaktdaten eingeblendet, z.B. Mail, Skype etc. Durch Klick auf einen Kontakt-Eintrag wird direkt das entsprechende Programm geöffnet, sofern es installiert ist.

Die einzelnen Symbole haben folgende Bedeutung:

Fälligkeitsdatum des Arbeitsschritts.

Arbeitsschritt bearbeiten. Siehe Edit-Modus unten.

Arbeitsschritt löschen.

Tag / Schlagwort für diesen Arbeitsschritt.

Status des Arbeitsschritts.

Fertigstellungsgrad des Arbeitsschritts basierend auf der Anzahl und Reihenfolge der Status.

Edit-Modus

Durch Klick auf das Stift-Icon bei einem Arbeitsschritt kann dieser bearbeitet werden. Es öffnet sich folgender Dialog:

Die folgenden Daten können erfasst bzw. geändert werden: Titel des Arbeitsschritts, Tags, Status, Verantwortlicher, Fälligkeitsdatum, Beschreibung.

Bei den Tags kann nach Eingabe eines oder mehrere Buchstaben ein bereits existierender Tag aus einer Liste selektiert werden. Benutzer mit Administratorenrechten können auch neue Tags anlegen.

Für das Boardlet Arbeitsschritte stehen folgende weitere Einstellungen zur Verfügung.

Reiter Filter

Knoten-ID:

Über diesen Schieberegler kann definiert werden, ob für die Anzeige der selektierte Knoten berücksichtigt werden soll oder ob die Daten aller Knoten angezeigt werden sollen.

Wenn aktiv, werden alle Maßnahmen angezeigt, die dem selektierten Knoten oder einem seiner Unterknoten zugeordnet sind.

Status:

Dropdown-Liste zur Auswahl des Status, für den Arbeitsschritte angezeigt werden sollen oder "alle".

Max. Zeitraum im aktuellen Status (in Tagen):

Hier kann eine Anzahl Tage eingegeben werden, wie lange die angezeigten Arbeitsschritte maximal im selektierten Status gewesen sein dürfen, z.B. um nur Arbeitsschritte anzuzeigen, die letzte Woche erledigt wurden.

Eigentümer:

Dropdown-Liste zur Auswahl des Benutzers, dem die angezeigten Arbeitsschritte zugewiesen sein müssen oder "alle".

Tags:

Dropdown-Liste zur Auswahl eines oder mehrerer Tags, die den angezeigten Arbeitsschritten zugewiesen sein müssen.
