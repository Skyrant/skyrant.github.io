Für das Boardlet Knotenlayout stehen außerdem folgende Einstellungen zur Verfügung.

Reiter Einstellungen

Erweiterter Titel:

Falls zusätzliche Filterparameter festgelegt wurden, können die jeweiligen Filterwerte zusätzlich zum Boardlet-Titel angezeigt werden. Alle zusätzlichen Filter sind in der Dropdown-Liste verfügbar. Der Wert wird hinter dem Boardlet-Titel in [] angezeigt.

Zoom in auf eine Ebene beschränken:

Über diesen Schieberegler wird definiert, ob das Hineinzoomen nur jeweils eine Ebene tiefer führen soll oder zur angeklickten Ebene. Wenn beispielsweise mehrere Unterebenen angezeigt werden, kann man mit dieser Einstellung verhindern, dass man versehentlich zu tief hineinzoomt.

Reiter Layout

Sichtbare Unterebenen:

Hier kann definiert werden, bis zu welcher Tiefe die Knotenstruktur angezeigt werden soll. Die Level sind immer relativ zum aktuell gewählten Level. Beispiel mit einer Eingabe von 2: wenn Level 0 selektiert ist, werden zusätzlich die Level 1 und 2 angezeigt. Ist Level 1 selektiert, werden zusätzlich die Level 2 und 3 angezeigt. Die angezeigten Level sollten immer mindestens gleich hoch sein wie das Detail Level, sonst können die Details natürlich nicht angezeigt werden.

Knoten ohne Layout ausblenden:

Über diesen Schieberegler kann definiert werden, ob Knoten, noch kein Layout haben (z.B. automatisch angelegte) ausgeblendet werden sollen oder nicht.

Anzeige des Knotenstatus:

Legt fest, wie anstehende Störungen / Alarme am Knoten visualisiert werden sollen: aus, Rahmen, Hintergrund. Der entsprechende Knoten wird dann eingefärbt, wenn eine Meldung auf diesem Knoten ansteht. Stehen mehrere Meldungen an, wird die Meldung mit der Klasse der höchsten Priorität visualisiert. Die Farbe wird dabei über den CSS-Style der Meldeklasse definiert.

Server-Datenquelle zur Statusermittlung:

Soll der Knotenstatus nicht auf Meldungen basieren, sondern auf anderen Eigenschaften, kann ein individuelles Skript erstellt werden und hier eingefügt werden. Der Name der Datenquelle muss folgendermaßen eingetragen werden:

Name der Datenquelle: ds_light_status.js

Eintrag hier: light-status

Pflichtfelder der Datenquelle sind NODE und STATE.

Ein Beispielskript kann im Ordner '...\customscripts\jsonapi\legato\node_state' gefunden werden.

JSON-Filter:

Wurde im vorigen Feld eine Datenquelle angegeben, die spezielle Filterkriterien unterstützt, können diese Kriterien hier als JSON-String eingeben werden. Damit wird z.B. ermöglicht, dass Knoten basierend auf einer Produkt-ID unterschiedlich eingefärbt werden. Der Filter muss folgendermaßen eingegeben werden:

[{"parameter": "productId", "filter": { "field": "product", "op": "EQ" }}]

Knotentyp-Details:

Hier können Parameter eingegeben werden, um abhängig vom Knotentyp andere Informationen im Details-Bereich einzublenden. Dies geschieht in folgender Form:

[{"nodeType":2,"detailBoardlet":"node-info-boardlet","detailBoardletParameters":{"valueTypes":[2,3,40]}}]

Im Beispiel werden nur für Knoten des Knotentyps 2 Details angezeigt, und zwar das Node-Info Boardlet mit Variablen der Arten 2, 3 und 40.

:

Werden unten Parameter für das Details-Boardlet eingetragen, so werden die spezifischen Angaben hier überschrieben!

Detaillevel:

Hier wird festgelegt, auf welcher Ebene Details angezeigt werden. 0 = oberste Ebene. Details sind z.B. die Namen der Unterknoten sowie die im nächsten Feld definierten weiteren Informationen wie aktuelle Messwerte etc. Das Detail-Level bezieht sich immer auf die aktuell im Boardlet selektierte Ebene. Detail Boardlet:

Über eine Dropdown-Liste kann ein weiteres Boardlet gewählt werden, dessen Werte als Detail-Informationen zu den Knoten angezeigt werden, beispielsweise die Werte bestimmter Variablen. Im obigen Screenshot sieht man das Aktuelle Werte Boardlet (node-values-boardlet). Außerdem stehen das Knoteninfo Boardlet (node-info-boardlet), das Balkendiagramm Boardlet (barchart-boardlet), das Knotenstatus Boardlet (node-state-boardlet), das Stückzahl-Boardlet (piece-count-boardlet) und das Puffer-Boardlet (buffer-boardlet) zur Verfügung.

Detail-Boardlet-Parameter:

Hier können Parameter für das Detail-Boardlet eingetragen werden, z.B. die zu zeigenden Variablenarten (Beispiel: { "valueTypes": [1,3] } zeigt die Variablenarten mit der ID 1 und 3 an).

:

Diese Parameter gelten für das gesamte Boardlet und überschreiben ggf. die oben getroffenen spezifischen Angaben je Knotentyp!

Detail-Boardlet-Skalierung:

Skalierung des eingebetteten Detail-Boardlets. In der API wird genau definiert, welche eingebetteten Boardlets dies in welcher Form unterstützen. Das Knotenstatus Boardlet beispielsweise zeigt bei einer Skalierung von 2 keine Details mehr an, sondern nur noch das definierte Symbol.

Breite Detail-Boardlet:

Breite, die für das im Detailbereich angezeigte Boardlet bereitgestellt wird. Ändert z.B. die Schriftgröße etc. im Detailbereich. Eine Eingabe hier gilt übergreifend für alle Ebenen. Ausnahmen können dann manuell geändert werden.

Die Logik dahinter ist folgende: Das Detail-Boardlet tut so, als sei es ein eigenes Boardlet mit der eingegebenen Größe. Dieses Boardlet (ohne Kopfzeile etc.) wird dann auf die Größe des Detail-Bereichs skaliert.

Zum Definieren der idealen Größe für ein bestimmtes Detail-Boardlet empfiehlt sich folgendes Vorgehen:

Fügen Sie das gewünschte Boardlet separat in das Dashboard und ziehen Sie es auf die gewünschte Größe, so dass die Daten wunschgemäß angezeigt werden. Öffnen Sie dann die Entwicklerwerkzeuge Ihres Browsers (rechte Maustaste - Element untersuchen - Inspektor). Bewegen Sie den Cursor über die Angaben im Inspektor, bis Ihr Boardlet blau markiert im Browser angezeigt wird. Über dem Element erscheint nun ein Pop-up, in dem auch die Größe des Elements in Pixeln angegeben ist. Diese Angaben können hier eingegeben werden.

Für quadratische Detail-Boardlets wie beispielsweise das Piechart-Boardlet sollten Höhe und Breite gleich sein, sonst wird das Piechart abgeschnitten.

Höhe Detail-Boardlet:

Höhe, die für das im Detailbereich angezeigte Boardlet bereitgestellt wird. Ändert z.B. die Schriftgröße etc. im Detailbereich. Eine Eingabe hier gilt übergreifend für alle Ebenen. Ausnahmen können dann manuell geändert werden.

Koordinaten runden:

Über diesen Schieberegler wird definiert, ob die Koordinaten der einzelnen Elemente gerundet werden sollen.

Reiter Filter

Root-Knoten:

ID des Wurzelknotens. Definiert den Einsprungpunkt des Layout Boardlets (Home).

Zusätzliche Filter:

Wird eine Datenquelle verwendet, die andere Filterkriterien unterstützt, können durch Klick auf + "Parameter hinzufügen" weitere Filterkriterien ausgewählt werden. Dazu muss ein Name für den neuen Filter eingegeben werden und der Typ selektiert werden. Typ kann entweder eine bereits definierte Tabelle sein (z.B. al_classes), oder es wird "LookupTable" ausgewählt und der Name der Nachschlagetabelle eingegeben (z.B. pro_products).
