Werterfassung periodenbezogen

Allgemein

Das Boardlet Werterfassung periodenbezogen (Value Booking Period-Related) zeigt entsprechend parametrierte Variablen des selektierten Knotens in dem Intervall an, in dem Werte gebucht werden müssen (über Filter einstellbar: jede Stunde, jede Schicht, jeden Tag). Es werden immer die Intervalle für einen ganzen Tag bzw. eine ganze Woche angezeigt; der Zeitraum kann über den Zeitfilter selektiert werden. Durch Kombination aus Variablenart und ausgewähltem Knoten werden die entsprechenden und richtigen Werte im Boardlet angezeigt.

 

Das Bordlet Werteerfassung periodenbezogen wird ergänzt durch das eSFM Boardlet Werteerfassung (interval count) boardlet.

 

Nur für manuell gebuchte Variablen innerhalb von eSFM verwendbar.

 

 

In der Menüleiste des Boardlets aktualisiert ein Klick auf die Ansicht, um z.B. Wertänderungen zu visualisieren.

Meldung hinzufügen

Durch Klick auf kann eine neue Meldung hinzugefügt werden, um Stückzahlverlust zu erfassen. Der Dialog wird weiter unten beschrieben.

Die Intervalle (bei Tagen für die gesamte Woche, bei Stunden für den selektierten Tag) werden nebeneinander angezeigt. Bereits gebuchte Werte und ggf. die zugeordneten Sollwerte werden angezeigt. Durch Klick auf das Stift-Symbol können die gebuchten Werte geändert werden oder neue Werte gebucht werden. Dabei wird der Sollwert immer mit dem zuletzt gültigen Sollwert vorbelegt.

 

 

 

Wert buchen und Meldung erfassen

Durch Klick auf das Stift-Symbol wird der Dialog zum Buchen von Werten und Erfassen von Meldungen geöffnet.

 

Die Zeit oben zeigt das aktuelle Intervall an, für das die Werte erfasst werden. Diese Angabe kann nicht geändert werden.

Darunter werden alle Variablen aufgelistet, die für diesen Knoten in diesem Intervall bebucht werden sollen. Wurde zu einer Variable ein Sollwert festgelegt (Spalte Zielwert in "Variablen"), so wird diese Variable ebenfalls zum Ändern angeboten, auch wenn kein Intervall für sie eingetragen wurde. Sofern vorhanden, wird der aktuelle Wert der Variablen angezeigt. Nun können die Werte für das Intervall erfasst werden.

Darunter öffnet sich durch Klick auf "Meldung hinzufügen" eine Eingabemaske zum Erfassen von Stückzahlverlusten. Wurde zum Beispiel der Sollwert in einer Schicht nicht erreicht, kann eine Meldung erfasst werden, in der die Differenz begründet werden kann. Darunter kann die Uhrzeit eingegeben werden, in der die Meldung gültig war (kommend / gehend). Außerdem kann der Meldung eine Meldeart und eine Meldeklasse zugewiesen werden. Die Meldung wird dann unter der Verbindung "Stückzahlverlust" (ID -105) angelegt und dem auslösenden Knoten zugeordnet. Im Boardlet "Meldungen Stückzahlverlust" werden diese Meldungen dann angezeigt.

Für das Werterfassung periodenbezogen stehen folgende weitere Einstellungen zur Verfügung.

Reiter Einstellungen

Einheiten anzeigen:

Über diesen Schieberegler wird definiert, ob zu den Werten die Einheiten eingeblendet werden sollen.

Differenz zum Zielwert anzeigen:

Über diesen Schieberegler wird definiert, ob die Differenz zum Zielwert mit angezeigt werden soll. Dies kann prozentual oder absolut sein.

Reiter Filter

Knoten-ID:

ID des Knotens, für den Werte angezeigt werden. Kann entweder hier geändert werden oder durch Auswahl eines anderen Knotens im Boardlet Knotenlayout oder im Boardlet Knotenbaum (außerhalb des Bearbeitungsmodus).

Variablenarten:

Dropdown-Liste zur Auswahl der angezeigten Variablenarten.

Intervall:

Dropdown-Listen zur Auswahl des angezeigten Intervalls.

Benötigte Parametrierung im LC2

Die Variablen werden im LC2 unter Anwendungsstruktur Datenaustausch [Modul] [Verbindung] Variablen gepflegt. Hier wird auch die Einheit festgelegt.

Unter Anwendungsstruktur Fertigungsstruktur [Knoten] Variablenzuordnungen werden die Variablen dem Knoten zugeordnet, für den sie angezeigt werden sollen. Hier muss das Flag "Anzeigen" auf 1 gesetzt sein, damit die Variable angezeigt wird. Die Sortiernummer legt fest, in welcher Reihenfolge die Variablen angezeigt werden.

Außerdem muss das Flag "Änderbar" auf 1 gesetzt sein, damit die Variable im Client bebucht werden darf.

 

Boardlet-Einstellungen

In den Boardlet-Einstellungen wird die Ist-Variable ausgewählt (Variablenart mit Ist-Werten). Dem Istwert wird ein Sollwert zugewiesen. Und im LC2 wird dieser Variablenart ein Sollwert zugewiesen. D.h. es gibt 2 Variablenarten (Soll- und Ist), und mit Ist-Variable wird dem zugehörigen Sollwert zugewiesen im LC2.
