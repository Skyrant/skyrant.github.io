Wertverlauf

Allgemein

Das Boardlet Wertverlauf zeigt geplante Werteinstellungsereignisse für eine bestimmte Zeitspanne für ausgewählte Variablen an. Bei den Variablen kann es sich um diejenigen handeln, die dem ausgewählten Knoten zugeordnet sind, oder um Variablen, die von ausgewählten Auslösern aus dem Boardlet Geplante Werteinstellungen betroffen sind.

 

Alle bereits erfolgten Planungen des ausgewählten Knotens werden in einer Zeile aufgelistet (Historie).

 

 

Parameter

  • Filter: ausgewählter Knoten, Zeitrahmen, Änderungsart ("Manuell", "Trigger", "Schaltbefehl"), Variable.

  • Zusätzlich kann ein versteckter Parameter "TriggerSymbol" verwendet werden, um nach Änderungen zu filtern, die durch einen bestimmten Job-Trigger verursacht wurden. Die Filterung erfolgt durch Auswahl von Zeilen im Boardlet "Scheduled Value Settings".

  • Wenn der Filter "Variable" angegeben ist, wird der Knotenfilter ignoriert.

  • Wenn der Filter "TriggerSymbol" angegeben ist, werden die Filter "node" und "change_type" ignoriert.

  • Wenn ein Protokolleintrag auf eine nicht existierende Schaltbefehl-ID verweist, dann wird die numerische ID des Schaltbefehls in der Spalte "Schaltbefehl" angezeigt, gefolgt von "(nicht gefunden)" in Kursivschrift.

Benötigte Parametrierung im LC2

Die Variablen werden im LC2 unter Anwendungsstruktur Datenaustausch [Modul] [Verbindung] Variablen gepflegt. Hier wird auch die Einheit festgelegt.

Unter Anwendungsstruktur Fertigungsstruktur [Knoten] Variablenzuordnungen werden die Variablen dem Knoten zugeordnet, für den sie angezeigt werden sollen. Hier muss das Flag "Anzeigen" auf 1 gesetzt sein, damit die Variable angezeigt wird. Die Sortiernummer legt fest, in welcher Reihenfolge die Variablen angezeigt werden.

Außerdem muss das Flag "Änderbar" auf 1 gesetzt sein, damit die Variable im Client bebucht werden darf.
