Datenquellen

In der WebApp wird das Datenquellen-Konzept mithilfe des Boardlets Super-Generic-Table umgesetzt. Für dieses Boardlet steht der zusätzliche Reiter Datenquellen im Bearbeitungsmodus zur Verfügung.

INFO:

In diesem Topic sollen alle Features auf dem Reiter Datenquellen beschrieben werden. Nicht alle Boardlets haben den Reiter Datenquellen. Und für Boardlets mit dem Reiter Datenquellen steht jeweils nur ein Teil der Features zur Verfügung.
Mit den nachfolgend beschriebenen Schritten können Sie in der Legato Sapient WebApp leicht herausfinden, welche Features für ein bestimmtes Boardlet verfügbar sind.

So erhalten Sie einen Überblick über die Features auf dem Reiter Datenquellen für ein bestimmtes Boardlet:

1

Rufen Sie den Bearbeitungsmodus für ein Dashboard auf. [so geht's]

2

Klicken Sie auf das gewünschte Boardlet.

  • Das ausgewählte Boardlet wird durch einen blauen Rahmen hervorgehoben.

  • Wenn für das Boardlet der Reiter Datenquellen verfügbar ist, dann erscheint das folgende Reiter-Symbol in der Seitenleiste rechts:

3

Klicken Sie auf den Reiter Datenquellen.

  • Die Features für das ausgewählte Boardlet werden dynamisch aufgelistet.

  • Die Feature-Beschreibungen finden Sie im folgenden Hilfeabschnitt.

Felder

Hier werden alle Felder angezeigt, die von der Datenquelle abgefragt werden können. Die im Code der Datenquelle als Benötigt festgelegten Einträge sind entsprechend markiert. Diese Felder können nicht deaktiviert werden.

Für alle anderen Felder kann über den Schieberegler rechts festgelegt werden, ob die Daten von der DB abgefragt werden sollen oder nicht.

INFO:

Dies hat KEINEN Einfluss darauf, ob das Feld im Boardlet angezeigt wird. Es geht lediglich darum, ob die Daten abgefragt werden.

Durch Klick auf das in einer Zeile wird der Eintrag aufgeklappt und bietet nun die Möglichkeit, dieses Feld mit einem zuvor angelegten Parameter zu verknüpfen.

  • Erste Dropdown-Liste: Hier werden die möglichen Operatoren angezeigt. Abhängig davon, ob der Schieberegler Array an oder aus ist, unterscheiden sich die Einträge.

An: in, not in.

Aus: greater or equal, less or equal, equal, not equal, greater than, less than.

  • Zweite Dropdown-Liste: Hier werden dann abhängig vom gewählten Operator die zur Verfügung stehenden Boardlet-Parameter angezeigt. Zum genauen Vorgehen beim Anlegen eines neuen Parameters, siehe bitte Reiter Filter / Parameter.

  

 

- oder -

Aktionen

Hier können die im Datenquellen-Model definierte Aktionen für den Endanwender an- bzw. abgewählt werden. Nach Doppelklick auf einen Tabelleneintrag in einem Meldungs-Boardlet hat der Endanwender Zugriff auf diese Aktionen in der Seitenleiste Meldungen.

Im Abschnitt Aktionen haben Sie die folgenden Konfigurationsmöglichkeiten:

  • Neusortieren der Felder mit Drag und Drop.

  • Umbenennen von Feldern

  • Festlegen von Pflichtfeldern und optionalen Feldern

  • Anzeigen von Datenquellenfeldern auf der Seitenleiste Meldungen

Die Beschreibungen der Konfigurationsoptionen finden Sie unten.

Neue Meldung:

  • Blendet die Spalten Verfügbare Felder und Ausgewählte Felder auf dem Reiter Datenquellen ein/aus.

  • Wenn die Spalten verborgen sind, dann ist das Symbol auf der Menüleiste eines Meldungs-Boardlets mit Tabelleneinträgen ebenfalls verborgen. Somit können Endanwender keine neuen Meldungen anlegen.

Nur die Felder in der Spalte Ausgewählte Felder werden in der Seitenleiste Meldungen angezeigt, wenn Endanwender eine neue Meldung anlegen.
Siehe beliebiges Meldungs-Boardlet mit Tabelleneinträgen Symbol Seitenleiste Meldungen.

Verfügbare Felder:

Listet die Felder auf, die auf der Seitenleiste Meldungen für ein Meldungs-Boardlet mit Tabelleneinträgen bereitgestellt werden können.

  • Um ein Feld zur Seitenleiste Meldungen hinzuzufügen, ziehen Sie es von der Spalte Verfügbare Felder in die Spalte Ausgewähle Felder.

  • Um ein Felder von der Seitenleiste Meldungen zu entfernen, ziehen Sie es von der Spalte Ausgewähle Felder in die Spalte Verfügbare Felder.

Ausgewählte Felder:

Nur die Felder in dieser Spalte werden in der Seitenleiste Meldungen eines Meldungs-Boardlets mit Tabelleneinträgen dargestellt. Sie können die Sortierreihenfolge der Felder mit Drag-und-Drop ändern. Mit dem Stift-Symbol können Sie ein Feld umbenennen. Außerdem gibt es zwei Arten von Checkboxen in der Spalte Ausgewählte Felder:

  • Schreibgeschützte Checkboxen für Felder, die im Code der Datenquelle als Benötigt definiert wurden, z.B. das Feld Meldeklasse.

  • Checkboxen, die Sie aktivieren oder deaktivieren können, z.B. das Feld Begründung/Ursache.

Durch das Aktivieren der Checkbox rechts wird ein Feld zum Pflichtfeld. Pflichtfelder werden mit einem Ausrufezeichen gekennzeichnet, das bei Meldungs-Boardlets mit Tabelleneinträgen auf der Seitenleiste Meldungen erscheint.

Beispiel: Boardlet Aktuelle Meldungen Eintrag doppelklicken  die Seitenleiste Meldungen öffnet sich.

 

Felder haben keine Checkboxen, wenn:

  • Das Feld als Schieberegler implementiert ist (an/aus).

  • Das Feld als Dropdown-Liste implementiert ist, über die je nach Auswahl des Endanwender Pflichtfelder auf der Seitenleiste Meldungen vorausgefüllt werden.

In diesen Fällen ist eine Checkbox für die Kennzeichnung als Pflichtfeld nicht sinnvoll und wurde daher weggelassen.

Nur die Felder in der Spalte Ausgewählte Felder werden in der Seitenleiste Meldungen angezeigt, wenn Endanwender eine bestehende Meldung bearbeiten.

Ausgabe, Feld, Parameter

Über die Aktion Ausgabe können Detail-Ansichten definiert werden, die per Klick dann den Wert als Parameter erhalten. Hier stehen alle Felder zur Verfügung, die von der DB geholt werden. Hierzu muss im Reiter Filter / Parameter der entsprechende Eintrag beim Startboardlet auf "out" und beim Zielboardlet auf "in" gesetzt werden.

Anwendungsbeispiel: Master – Detail Dashboard: Selektion einer Zeile in der Master-Tabelle setzt automatisch einen Filter in einer Detailtabelle, um zugehörige Datensätze anzuzeigen.

 

Reiter Filter / Parameter

Hier werden zum einen die im Datasource-Model definierten Filter angezeigt (üblicherweise mindestens Knoten und Zeit).

Durch Klick auf "Parameter hinzufügen" können weitere Parameter oder Filter für dieses Boardlet festgelegt werden. Über den Reiter Datasources – Fields kann der neue Parameter dann mit einem bestimmten Datenbankfeld verknüpft werden. Alternativ ist es natürlich auch möglich, einfach einen bestimmten Wert an ein anderes Boardlet im Dashboard zu übergeben, das einen Parameter mit dem gleichen Namen enthält, ohne eine Referenz auf die Datenbank.

Hier kann ein frei wählbarer Name für den Parameter vergeben werden.

Aus der Dropdown-Liste muss nun die Art des Parameters gewählt werden. Zur Verfügung stehen alle im Datasource-Model definierten Filter sowie:

NumberArray

Ermöglicht die Eingabe mehrerer Zahlen kommagetrennt als Array. Kann z.B. mit der ID-Spalte einer Tabelle verknüpft werden.

StringArray

Ermöglicht die Eingabe mehrerer Strings kommagetrennt als Array. Kann z.B. mit einer Text-Spalte einer Tabelle verknüpft werden.

LookupTable

Ermöglicht eine Mehrfachauswahl aus einer Tabelle. Dazu muss in einem dritten Feld der Name der Tabelle und die zu holenden Spaltennamen angegeben werden, z.B. al_classes, id, name.

Number

Ermöglicht die Eingabe einer Zahl. Kann z.B. mit der ID-Spalte einer Tabelle verknüpft werden.

String

Ermöglicht die Eingabe eines Strings. Kann z.B. mit einer Text-Spalte einer Tabelle verknüpft werden.

Boolean

Ermöglicht das Umschalten eines Schiebereglers zwischen true und false.

INFO:

Für Datenbank-Einträge wird Boolean aktuell nicht unterstützt. Hier muss immer Number verwendet werden (für 1 oder 0).

 

Die neuen Parameter können auf den gewünschten Parametertyp (local, in, out, in/out) gesetzt oder gelöscht werden.

INFO:

Vergessen Sie nicht, den neuen Parameter mit einem Feld zu verknüpfen!

Reiter Tabelleneinstellungen

In diesem Reiter können für alle Spalten, die von der Datenbank geholt werden, Einstellungen hinsichtlich ihres Verhaltens getroffen werden.

Zunächst wird die entsprechende Spalte selektiert. Darunter kann ein individueller Name eingegeben werden, der als Spaltenüberschrift verwendet werden soll.

Die folgenden Schieberegler bestimmen das Verhalten der Spalte in der Tabelle. Sie kann sortierbar, größenveränderbar, verschiebbar und sichtbar sein. Außerdem kann eine bestimmte Breite der Spalte festgelegt werden. Ist der Schieberegler Komponente aktiv, kann die Art der Spalte selektiert werden, z.B. Zeitangabe. Darunter kann die gewünschte Formatierung als JSON-String eingegeben werden. Grüne Häkchen am Ende der Zeilen zeigen an, ob der Inhalt sinnvoll ist.

Für weitere Details, siehe bitte Tabelleneinstellungen für Boardlets.
