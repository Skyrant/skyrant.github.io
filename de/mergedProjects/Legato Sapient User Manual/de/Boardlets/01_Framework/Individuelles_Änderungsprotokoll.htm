Individuelles Änderungsprotokoll

Das Boardlet Individuelles Änderungsprotokoll stellt Informationen zu Änderungen am Fertigungsprozess dar. Es ist möglich, auf ein Protokoll aller Änderungen von z.B. Prozessparametern oder ein Protokoll anderer benutzerdefinierter Daten zuzugreifen.

Bei den Daten kann es sich um Konfigurationsparameter für den Prozess oder um dynamische Daten handeln, die z. B. mit den Produktionseinheiten zusammenhängen.

Im Boardlet kann man nach Änderungen, nach Zeitpunkt, Tabelle und Benutzernamen suchen. Darüberhinaus ist es möglich, nach bestimmten Werten in den Daten zu filtern.

 

Beachten Sie Folgendes:

  • Die neue Audit-Tabelle sys_change_log_prj dient zur Protokollierung von Änderungen an projektspezifischen Tabellen. Alle Spalten von sys_change_log_prj können angezeigt werden.

  • Es kann nach action_tstamp, table_name, user, modified_id gefiltert werden.

  • Ein Dashboard mit den benutzerdefinierten sgt boardlet kann erstellt werden. Darüberhinaus kann eine Zeile ausgewählt und nach dem Element gefiltert werden, um die entsprechenden Änderungen anzuzeigen.

  • Alle Werte und Spalten der benutzerdefinierten Tabelle pro Protokolleintrag können angezeigt werden.

  • Es kann auf einen beliebigen Wert gefiltert werden.

  • Der Zugriff auf diese Daten wird durch ein neues Recht gesteuert.

 

Konfiguration

  • Um Änderungen an der Tabelle PRJ_ABC zu protokollieren, muss ein Dummy-Trigger in dieser Tabelle mit dem Namen PRJ_TL_ABC erstellt werden. Der Triggername muss mit PRJ_TL beginnen.

  • Aufruf der Datenbankprozedur sys_refresh_prj_audit_triggers(): Diese Prozedur erstellt alle Trigger neu, die den Namen PRJ_TL% tragen.

  • Für Tabellen, die Konfigurationsdaten und Live-Daten enthalten, kann es notwendig sein, Ausnahmen für einige Spalten zu definieren, damit Änderungen an diesen Spalten nicht protokolliert werden. Für diese Ausnahmen müssen der Tabellennamen und der Spaltennamen in die Tabelle SYS_EXCEPTIONS_FOR_TRG_COLS eingetragen werden. Diese Ausnahmen werden von der Prozedur sys_refresh_prj_audit_triggers() verarbeitet.

 

Es ist notwendig, dass die Berechtigung auf die Datenquelle (die entsprechende Tabelle) mit der Aktion -110 gesetzt wird.
Es müssen jene Trigger in jeder Kundentabelle konfiguriert werden, welche die Änderungslogs in die Tabelle sys_change_log_prj einfügen.
