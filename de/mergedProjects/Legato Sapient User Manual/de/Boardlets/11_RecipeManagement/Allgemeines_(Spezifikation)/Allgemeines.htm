Allgemeines zu Rezepte und Rezeptmanagement

Allgemeines und Definition

 

  • Ein Rezept ist ein festgelegtes Set von Parametern, die in einer Steuerung (SPS) einen bestimmten Betriebszustand abdecken. Es gibt nur ein Rezept für eine Steuerung mit gleichem Namen und unterschiedlichen Parametern.
  • Ein Rezept kann von der SPS durch ein Flag zu Legato Sapient gesendet werden. Normalerweise ist aber die umgekehrte Richtung der Fall: Das Rezept wird von Legato Sapient zur SPS gesendet.
  • Eine Rezeptdefinition ist eine feste Struktur von Parametern.
  • Eine Rezeptsammlung besteht aus allen Rezepten einer Rezeptdefinition.
  • Ein Rezept und somit ein Parameterset ist eine Zusammenfassung von 1 bis n Datensätzen/Parametern, z.B. Legato DB Variablen und unterschiedliche Datentypen.
  • Ein Rezept ist über eine eindeutige ID identifizierbar und bildet eine versionierbare und archivierbare Einheit in der Legato DB.
  • Es werden bei einer manuellen Änderung von einem oder mehreren Werten im Rezept immer alle Datensätze vom betreffenden Rezept (mit ihrem aktuellen Wert) an die Anlage übertragen.
  • Ein Rezept besteht immer aus Header und Body. Der Header enthält u.a. die Version und einen Kommentarbereich. Der Body beinhaltet die einzelnen Datensätze.
  • Der Aufbau von einem Rezept ist frei parametrierbar.
  • Ein oder mehrere Rezepte können einer Verbindung und einem Fertigungsknoten zugewiesen werden.
  • Die Zuweisung zu einer Verbindung ist notwendig, damit das vollständige Rezept an den SCC übertragen werden kann.

 

  • Das Rezeptmanagement dient dazu, das Set von Parametern, welche für den Prozess relevant sind, zentral zu speichern, zu bearbeiten und zu verteilen.
  • Dabei werden in den Steuerungen der Anlagen alle Rezepte vorgehalten, welche für die Fertigung aller Produktvarianten benötigt werden. Diese Werte können zentral über Legato Sapient Clients bearbeitet und auf die Anlage übertragen werden.
  • Ein Parameter ist ein einzelner Wert, welcher der Anlage vorgegeben wird.
  • Anlageparameter sind die Summe aller Parameter einer Anlage.
  • Das Rezeptmanagement ist eine Parameterverwaltung.
  • Das Rezeptmanagement ist ein Modul zur zentralen Sicherung, Bearbeitung und Verteilung von allen Anlageparametern.

 

  • Generell ist Legato Sapient das bestimmende System. Es kann aber auch in Einzelfällen (z.B. Erstinbetriebnahme) sein, dass die SPS die Rezeptdaten im Speicher hat, diese aber für Legato Sapient noch unbekannt sind.
  • Deshalb gibt es pro Rezept ein sog. Masterflag. Solange das Masterflag nicht gesetzt ist, werden die Parameterwerte aus der Steuerung in Sapient übernommen.
  • Mit gesetztem Masterflag werden keine Änderungen aus der Steuerung mehr übernommen, Legato Sapient nimmt für dieses Rezept die Masterrolle ein.

Notwendigkeit für ein Rezept

  • In einem Rezept werden 1 bis n Parameter mit ihren aktuellen Werten, die zusammengehören, verwaltet.
  • Erfolgt eine Änderung von einem Wert eines Parameters im Rezept, werden trotzdem immer alle Parameter mit ihren aktuellen Werten z.B. an eine Anlagensteuerung übertragen.
  • Dies ist der wesentliche Unterschied zur Änderung einzelner Variablen in der Legato DB. Diese sind immer für sich betrachtet unabhängig von anderen Variablen und deren Werten. Dies gilt auch für die Übertragung an eine Anlagensteuerung.
  • Zielsystem für Rezepte sind Anlagensteuerungen, die für bestimmte Funktionen vordefinierte Parameter und deren Werte benötigen. Hier muss sichergestellt werden, falls sich ein bestimmter Parameter ändern sollte, alle mit diesem Parameter verbundenen Parameter und Werte mitändern bzw. gleich bleiben sollen.
  • Erfolgt ein kompletter Steuerungstausch, d.h., es wird die SPS-Hardware ausgetauscht und auf diese das aktuelle SPS-Programm aufgespielt, so sind nach dem Aufspielen vom Programm alle benötigten Parameter des Programms mit Defaultwerten/Initialwerten versehen. Alle vorher verwendeten Parameterwerte gehen hierbei verloren und müssen nach dem SPS-Tausch ggf. manuell geändert werden. Jedoch besteht hierbei die Gefahr, dass einzelne Werte vergessen werden.
  • Beim Einsatz von Rezepten können nach einem SPS-Tausch vom SPS-Programm die Daten automatisch nach einem SPS-Neustart von Legato angefordert werden. Hierbei werden alle Rezepte der betreffenden SPS mit den zuletzt in der Legato DB gespeicherten Werten an die SPS übertragen.

Arbeit mit einem Rezept

Man kann mit einem Rezept auf verschiedene Art und Weise arbeiten:

 

Variante 1: Manuelles Editieren

  • Die Änderungen an einem Rezept werden über den Sapient Webclient durchgeführt.
  • Alle Parameter eines Rezeptes können manuell geändert werden.
  • Wenn alle Änderungen durchgeführt wurden, erfolgt die Übernahme durch einen Mausklick.
  • Wird eine Änderung im Rezept bestätigt, wird ein Triggersignal in der SE gesetzt, welches dem SCC signalisiert, dass ein Rezept an die SPS zu übertragen ist.
  • Nach der Übertragung an die SPS quittiert der SCC das Rezept in der DB durch das Setzen einer SE-Methode.

 

Variante 2: Anforderung von der SPS bei Neustart

  • Umgekehrt kann eine SPS z.B. bei einem SPS-Neustart alle Rezepturen, die für diese SPS gelten durch das Setzen eines Signals vom SCC anfordern.
  • Der SCC fordert daraufhin alle Rezepte der betreffenden SPS von der SE und somit aus der DB, an.
  • Alle Rezepte der Verbindung werden an den SCC übertragen und je Rezept quittiert.

 

Variante 3: Anforderung von SPS im Programmablauf

  • Das Funktionsprinzip entspricht jenem von Variante 2, es werden in dem Fall aber nur bestimmte Rezepte angefordert.

 

Variante 4: Übertragung der Rezepte aus der SPS

  • Es können Rezepte aus der SPS in die DB übertragen werden.
  • Voraussetzung: Das Rezept muss vorhanden sein und die Struktur des übertragenen Rezeptes muss jenem in der DB entsprechen. Dies gilt nicht für die Werte, sondern nur für die einzelnen Parameter.
  • Beispiel: Das Rezept aus der SPS beinhaltet 100 Parameter, die alle numerisch sind. Das dazugehörige Rezept in der DB muss demnach ebenfalls 100 numerische Parameter enthalten, damit ein Datenabgleich durchgeführt werden darf.
  • Die Übertragung aus der SPS kann sowohl automatisiert über das SPS-Programm, als auch über SCC-Skripte oder manuell über den Webclient erfolgen.
