Datenmodell

Zum Datenmodell des Rezeptmanagements gehören einige Tabellen, bei denen das Präfix RM_ für "Rezeptmanagement" steht:

Tabellenname

  

Informationen/Handhabung von Spalten

P_DATA_TYPES

  • Enthält die SPS-Datentypen, die mit den Rezepturpositionen verknüpft werden müssen.

  • Die Tabelle ist Teil der Legato Sapient Stammdaten und wird mit dem gezeigten vordefinierten Inhalt ausgeliefert.
    Hierbei sollten eine systemrelevante Konfiguration und negative IDs verwendet werden.

  • Änderungen können vom KeyUser über LC2 vorgenommen werden.

------------

RM_RECIPES

In Kombination mit RM_RECIPE_ITEMS:

  • Enthält eine Liste von Rezepten einschließlich der gesamten Struktur von Rezepten und auch die Verknüpfungen zu anderen Objekten (CONNECTIONS, NODES, VALUES usw.).

  • Die Tabelle ist Teil der Konfiguration für die Rezepturverwaltung mittels LC2.

  • Die grundlegenden Informationen und die Struktur können mit Hilfe einer Rezepturvorlage erstellt und ein "Neues Rezept" auf der Grundlage einer ausgewählten Vorlage erstellen werden.

Informationen zur Spalte NUMBER_BYTES:

  • Diese Spalte kann in LC2 nicht bearbeitet werden und der Inhalt sollte über einen Datenbanktrigger berechnet werden:

  • Trigger: Änderung von RM_REC_TEMPLATE_ITEMS. ANZAHL_BYTES_SUMME

  • Berechnung: Summe über alle ITEMS (Datensätze) für die Rezepturvorlage (ID) der Spalte RM_REC_TEMPLATE_ITEMS. ANZAHL_BYTES_GESAMTZAHL

RM_REC_TEMPLATE_ITEMS

In Kombination mit RM_RECIPE_TEMPLATES:

  • Enthält eine Anzahl von Items (als Baum strukturiert), die mit einem bestimmten Rezept ohne eine Verbindung zu anderen Objekten (VALUES etc.) verknüpft sind.

  • Die Tabelle ist Teil der Konfiguration für die Rezepturverwaltung mittels LC2.

 

 

 

Informationen zur Spalte SORT_ID:

  • Definiert die Reihenfolge der Items unterhalb eines bestimmten Parents. Das bedeutet, dass die Reihenfolge der Items unabhängig für jedes übergeordnete Element und auch für Items ohne ein übergeordnetes Element (Ebene 0 oder flache Liste von Items) festgelegt werden kann.

 

Informationen zur Spalte NUMBER_BYTES_VARIABLE:

  • Diese Spalte kann in LC2 nur bearbeitet werden, wenn der DATA_TYPE "STRING" ist. Ansonsten bleibt das Feld leer (oder "0").

Handhabung für die Spalte NUMBER_BYTES_TOTAL:

  • Diese Spalte kann in LC2 nicht bearbeitet werden und der Inhalt wird mit Hilfe eines Datenbanktriggers berechnet:

  • Trigger: Änderung von DATA_TYPE oder Änderung von NUMBER_BYTES_VARIABLE

  • Berechnung: P_DATA_TYPES.NUMBER_BYTES + P_DATA_TYPES.NUMBER_BYTES_ADD_PLC + RM_REC_TEMPLATE_ITEMS. ANZAHL_BYTES_VARIABLE

 

RM_RECIPE_TEMPLATES

In Kombination mit RM_REC_TEMPLATE_ITEMS:

  • Enthält eine Liste von Rezepturvorlagen mit der gesamten Struktur von Rezepten, aber ohne Verbindung zu anderen Objekten (CONNECTIONS, NODES, VALUES usw.).

  • So kann eine Rezepturvorlage für verschiedene PLS (NODES, CONNECTIONS) verwendet werden.

  • Die Tabelle ist Teil der Konfiguration für die Rezepturverwaltung mittels LC2.

Informationen zur Spalte NUMBER_BYTES:

  • Diese Spalte kann in LC2 nicht bearbeitet und der Inhalt wird über einen Datenbanktrigger berechnet.

  • Trigger: Änderung von RM_REC_TEMPLATE_ITEMS. ANZAHL_BYTES_SUMME

  • Berechnung:
    Summe über alle Items (Datensätze) für die Rezepturvorlage (ID) der Spalte RM_REC_TEMPLATE_ITEMS. ANZAHL_BYTES_GESAMTZAHL.

RM_RECIPE_MODES

  • Look-Up-Tabelle, die die verschiedenen Betriebsarten für Rezepte enthält.

  • Die Tabelle ist Teil der Legato Sapient Stammdaten und sollte mit dem gezeigten vordefinierten Inhalt ausgeliefert werden (systemrelevante Konfiguration und negative IDs sollten verwendet werden).

  • Änderungen können vom KeyUser über LC2 vorgenommen werden.

------------

 

RM_UP_DOWNLOAD_RESULTS

  • Look-Up-Tabelle, die die möglichen Ergebnisse für Up-/Download-Prozeduren für Rezepte von/zu Shopfloor-SPSen enthält.

  • Die Tabelle ist Teil der Legato Sapient Stammdaten und sollte mit dem gezeigten vordefinierten Inhalt ausgeliefert werden (systemrelevante Konfiguration, negative IDs sollten verwendet werden).

  • Änderungen können vom KeyUser über LC2 vorgenommen werden.

------------

 

RM_ITEM_VALUES

  • Enthält den VALUE für ein verknüpftes Recipe Item in RM_RECIPE_ITEMS.

  • Neue Datensätze müssen aufgrund von LC2 für jedes entsprechende Item in RM_RECIPE_ITEMS automatisch angelegt werden.

  • Bei der Erstellung eines neuen Rezepts kann ein Standardwert (VALUE) gesetzt werden, in dem eine Rezepturvorlage verwendet wird und ein "Neues Rezept" auf Basis dieser Vorlage erstellt wird.

  • VALUES können manuell über die Web-Applikation geändert werden. Der Änderungsvorgang wird über RMRECIPE_ITEMS.CHANGE_CONTROL gesteuert.

------------

 

RM_RECIPE_DATA

  • Enthält Rezeptpositionen mit Standarddaten.

------------

 

RM_CHANGE_COMMENTS

  • Enthält Kommentare zu Elementänderungen.

------------

 

RM_RECIPE_RESULTS

  • Enthält Upload-Ergebnisse.
