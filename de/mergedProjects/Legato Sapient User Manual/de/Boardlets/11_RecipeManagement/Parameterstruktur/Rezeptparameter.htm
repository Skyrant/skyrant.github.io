Parameterstruktur

Array

Ein Array kann eine komplexe Struktur mit untergeordneten Elementen und weiteren Arrays in mehreren Ebenen (Baumstruktur) umfassen. D.h., ein Array ist eine Liste/Gruppe von (Struktur-) Elementen im LC2. Ein Array im LC2 besteht hierbei aus dem Arrayname, hat eine bestimmte Anzahl an Elementen und verweist auf Parameter.

 

Parameter

  • Legato Sapient gilt als Master für die Rezeptverwaltung.

  • Die SPS speichert alle Rezepte intern.

  • Bei der Initialisierung schickt die SPS alle Rezepte an Legato Sapient für den Abgleich der Datenstände.

  • Typischerweise sind Rezeptparameter z.B. Temperaturwerte

 

Jeder Parameter enthält folgende Daten:

  • Parametername

  • Beschreibung

  • Parametertyp

  • Parameterwert

  • Obere Wertgrenze

  • Untere Wertgrenze

 

 

Beachten Sie das Kapitel Rezeptprozeduren, um weitere Informationen zu den jeweiligen Parametern zu erhalten.

 

Bearbeitung der Parameterwerte

  • Die Bearbeitung der Parameterwerte eines Rezepts erfolgt über den Sapient-Client.

  • Die Änderung der Daten in den Rezepten erfolgt über die Sidebar. Je nach Parametertyp werden die üblichen Eingabeeditoren in der Sidebar bereitgestellt.

 

Für weitere Informationen zur Bearbeitung der Parameterwert siehe das Kapitel Rezeptvorlagen.

 

Sicherung der Parameter

  • Daten werden in der Datenbank von Sapient gesichert.

Versionierung der Parameter

  • Eine Versionierung der Parametersätze ist nicht notwendig.

  • Alte Parametersätze müssen nicht wiederhergestellt werden.

  • Jedoch sind Änderungen über das Änderungsjournal zu dokumentieren und mit Änderungsgrund zu versehen.

  • Der Username des Benutzers, der die Änderung vorgenommen hat, wird im Journal aufgenommen. Dies ist zulässig, wenn parallel nicht nach allen Änderungen einer einzelnen Person gefiltert werden kann.

Übertragung der Parametersets

  • Daten werden an die SPS übertragen. Der Anwender muss erkennen, dass die Parameter erfolgreich in die Anlage übertragen wurden.

  • Dafür ist ein Rückmeldetelegramm für den erfolgreichen Versand in der Schnittstellenbeschreibung spezifiziert.

  • Wenn die Datenübertragung von Legato Sapient zur SPS nicht positiv quittiert wurde, bleibt die Rezeptdarstellung in Legato Sapient erhalten und ermöglicht so einen erneuten Versand, oder auch den Sprung in Diagnosebildschirme mit z.B. dem Kommunikationsstatus, um Maßnahmen einzuleiten.

Template für Parametersets

  • Das Template soll den Anlagen zugeordnet werden können.

  • Wenn ein Aufruf über das HMI erfolgen soll, dann sollen dies nur Rezeptvorschläge für entsprechende Anlagen sein.

Zuordnung von Parametersets

  • Mehrere Rezepte können einer Anlage zugeordnet werden.

  • Damit können auch mehrere Rezeptstrukturen für eine Anlage zugehörig sein.

Kopieren der Parametersets

  • Um die Eingabe zu erleichtern, sollen Rezepte sollen kopiert werden können.

  • Rezepte können damit auch auf andere Anlagen (SPS-Bereiche) übertragen werden.

  • Nach dem Kopier-Funktionsaufruf erscheint ein Menü zur Auswahl des SPS-Bereichs und der Namenseingabe.

Rezeptänderungen

  • Alle Rezeptänderungen in Legato Sapient müssen im Änderungsjournal erfasst werden. D.h. dies umfasst auch das Erfassen der Rezeptänderungen im Änderungsjournal auf der SPS-Seite, wenn dieses an die SPS geschickt wird.

  • Ein Änderungsgrund muss immer angegeben werden. Ebenso wird der auslösende Benutzername mitgespeichert.

Arbeiten vom HMI

  • Wird von einem HMI die Rezeptverwaltung aufgerufen, bekommt man nur eine Auswahl der Rezepte, die vom aufrufenden SPS-Bereich stammen.

  • Ein Rezept identifiziert sich aus SPS-Bereich und Rezeptname.
