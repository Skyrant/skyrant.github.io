Monitoring - Knotenstatus

Allgemein

Das Boardlet Knotenstatus zeigt zu einem selektierten Knoten den Status an. Der Status wird über die Bits eines vom Gateway gesendeten Werts auf ein Symbol gemappt. Das entsprechende Symbol für das erste gefundene Bit wird dann angezeigt.

Die zur Verfügung stehenden Symbole sind im jeweiligen Theme definiert und in der API-Doku beschrieben. Momentan sind folgende Symbole parametriert:

icon-check (default)

icon-stop-hand

icon-robot

icon-driller

icon-tool

 

Alle verfügbaren Symbole (Icons) sind hier zu finden:

[Adresse der Sapient Engine]/app/assets/icons/demo.html

 

Es kann als eigenständiges Boardlet angezeigt werden oder als Detail-Info ins Layout-Boardlet eingebettet werden.

Für das Boardlet Knotenstatus stehen außerdem folgende Einstellungen zur Verfügung.

Reiter Layout

Knotenname anzeigen:

Über diesen Schieberegler wird festgelegt, ob der Knotenname neben dem Symbol angezeigt werden soll oder nicht.

Dargestellter Text:

Hier kann ein beliebiger Text eingegeben werden, der neben dem Symbol und ggf. neben dem Knotennamen angezeigt wird.

Verwendete Icons:

Komma-separierte Liste der verwendeten Symbole. Diese müssen im Theme definiert sein. Welches Symbol angezeigt wird, hängt davon ab, welches Bit vom Gateway gesetzt wird.

Beispiele:

  • Ist das 1. Bit gesetzt (z.B. 01000001 - Bits werden von rechts nach links gelesen), wird das 1. Symbol angezeigt (im Beispiel also icon-stop-hand).

  • Ist das 2. Bit gesetzt (z.B. 01000010), wird das 2. Symbol angezeigt usw.

Ausschlaggebend ist das erste gefundene Bit.

Standard-Symbol:

Symbol, das angezeigt wird, wenn kein Bit gesetzt ist.

Alarmierungsmodus:

Dropdown-Liste zur Auswahl des Alarm-Modus. Legt fest, ob für Warnungen und / oder Alarme nur der Rahmen eingefärbt wird (border) oder der gesamte Hintergrund (background). Die jeweiligen Farben für Warn-Grenzen, Alarm-Grenzen und IO-Zustand werden im Theme definiert. Im Beispiel ist IO-Status grün, Warnung lila, Alarm rot.

Alarmierungsebenen:

Dropdown-Liste zur Auswahl der Alarmstufen (d.h., wann ein Farbumschlag erfolgen soll). Zur Auswahl stehen:

  • Warning and alert: Ist das Alarm-Bit gesetzt, wird der Farbumschlag für die Alarmgrenze angezeigt, ist das Warn-Bit gesetzt, der Farbumschlag für die Warngrenze.

  • Warning: Ist das Warn-Bit gesetzt, wird der Farbumschlag für die Warngrenze angezeigt.

  • Alert: Ist das Alarm-Bit gesetzt, wird der Farbumschlag für die Alarmgrenze angezeigt.

  • None: Es wird nie ein Farbumschlag angezeigt.

Bitmuster Alarm:

Angabe einer Zahl, welches Bit / welche Bits als Alarmgrenze angesehen werden sollen. Beispiel: 0 = keine Alarmgrenze; 1 = erstes Bit; 2 = zweites Bit; 11 = erstes, zweites und viertes Bit etc.

Bitmuster Warnung:

Angabe einer Zahl, welches Bit / welche Bits als Warngrenze angesehen werden sollen. 0 = keine Warngrenze; 1 = erstes Bit; 2 = zweites Bit; 11 = erstes, zweites und viertes Bit etc.

Reiter Filter

Variablenarten:

Filter auf Variablenarten. Für alle Variablen der hier definierten Variablenarten, die dem oben selektierten Knoten zugeordnet sind, werden die Werte im Diagramm angezeigt. Die Variablenarten können aus einer Dropdown-Liste ausgewählt werden. Leer = kein Filter, alle Variablenarten werden angezeigt.

Benötigte Parametrierung im LC2

Die Variablenarten werden im LC2 unter Grundkonfiguration Allgemeine Anwendungen Variablenarten parametriert.

Diese IDs sind im Filter des Boardlets anzugeben, z.B. 2 = Iststückzahl, 3 = Sollstückzahl.

Die Variablen werden im LC2 unter Anwendungsstruktur Datenaustausch [Modul] [Verbindung] Variablen gepflegt. Hier wird auch die Einheit festgelegt.

Unter Anwendungsstruktur Fertigungsstruktur [Knoten] Variablenzuordnungen werden die Variablen dem Knoten zugeordnet, für den sie angezeigt werden sollen. Hier muss das Flag "Anzeigen" auf 1 gesetzt sein, damit die Variable angezeigt wird. Die Sortiernummer legt fest, in welcher Reihenfolge die Variablen angezeigt werden.
