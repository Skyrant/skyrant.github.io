Funktion

Das Meldungsmanagement (näheres siehe eigener Abschnitt) erlaubt die Erfassung von Meldungen aus den Steuerungssystemen über verschiedene Methoden. Die Funktion der Datenbank ist aber für alle Erfassungsmethoden identisch: Aus dem Gateway wird eine eindeutige ID je Meldung mit Zeitstempel je Ereignis (getrennt für "kommend" und "gehend") an die Datenbank übergeben. In der Datenbank werden dann aus einer Zuordnungstabelle der Text der Meldung und andere Informationen zugefügt.

Die Meldungen und deren Eigenschaften können manuell freigegeben werden, oder werden über eine Importfunktion aus anderen Datenquellen übernommen.

ID

Eindeutige Identifikationsnummer. Wird vom System vergeben. Schreibgeschützt.

Verbindung

Lookup-Liste zur Auswahl der Verbindung. Wird automatisch befüllt und kann geändert werden.

Knoten

Lookup-Liste zur Auswahl eines Knotens aus der Fertigungsstruktur. Die Zuordnung der Meldung zu einem Knoten erlaubt die gefilterte Darstellung der Meldungen im Client (bezogen auf einen Knoten und seine Unterelemente) und ist auch die Grundlage für alle knotenbezogenen Statistikfunktionen.

Meldungsnummer

Meldungsnummer. Muss innerhalb dieser Verbindung eindeutig sein (z.B. Code des Störoperanden im Steuerungssystem).

Die Auslösung einer Störmeldung erfolgt mit Übergabe der Meldungsnummer vom Gateway an die Datenbank.

Operand (optional)

SPS-Operand (falls Operandenbezug vorhanden) der Meldung.

Symbol (optional)

SPS-Symbol (falls Operandenbezug vorhanden) der Meldung.

Text

Meldetext bis max. 80 Zeichen Länge.

Meldeklasse

Lookup-Liste zur Auswahl einer vorhandenen Melde-Klasse (siehe Abschnitt Meldungsmanagement) zur Klassifizierung der Meldung. Die Klasse regelt die Verarbeitung und Darstellung einer Meldung. Die Klassen Unterer / Oberer Grenzwert, Untere / Obere Warngrenze, Verbindungsstörung, Modul-Verbindungsstörung und Unbekannte Meldung sind fest im System vorhanden. Andere Klassen können bei der Definition des Meldesystems beliebig durch den Anwender erstellt werden.

Meldeart

Lookup-Liste zur Auswahl einer vorhandenen Melde-Art (siehe Abschnitt Meldungsmanagement). Meldearten sind eine weiteres Merkmal von Meldungen und können entsprechend ausgewertet werden.
Typische Arten sind:

                technisch
                organisatorisch
                systembedingt
                Kalenderpflege

                Zyklische Meldungen
                Unterer/Oberer Grenzwert
                Untere/Obere Warngrenze
                Verbindungsstörung
                Modul-Verbindungsstörung

Weiterleitungsart (optional)

Lookup-Liste zur Auswahl einer Weiterleitungsart für den Funkruf (siehe Kapitel Funkruf).

Typische Weiterleitungsarten sind:

keine Weiterleitung

Erstwerterkennung (1. Meldung wird weitergeleitet, Folgemeldungen werden ausgeblendet)

generelle Weiterleitung (alle Meldungen werden weitergeleitet).

Überschreibt die indirekte (default) Weiterleitungsart der zugewiesenen Meldeklasse (Defaulteinstellung unter Meldungsmanagement | Meldeklassen – Spalte Weiterleitungsart).

Default-Begründung (optional)

Lookup Liste, um eine Default-Begründung für diese Meldungsdefinition auszuwählen. Wird hier ein Eintrag ausgewählt, wird diese Begründung als Default für Meldungen dieser Definition in der WebApp verwendet.

: In der WebApp können nur Einträge ohne weitere Untereinträge als Filter verwendet werden, keine Parents, z.B. in der Pareto-Ansicht!

Funkruf-Empfänger (optional)

Auswahlfeld zur Zuordnung eines im Meldungsmanagement definierten Funkruf-Empfängers (siehe Abschnitt Meldungsmanagement).

Funkruf-Priorität (optional)

Priorität der Meldung für den Funkruf (unabhängig von der Meldungs-Priorität).

Konfig nicht komplett

Checkbox. 1 = Meldung wurde dynamisch angelegt; die Konfiguration ist möglicherweise nicht komplett und muss kontrolliert und ggf. angepasst werden. Die Konfig kann beim Einbuchen von Meldungen dynamisch geändert werden.

0 = Konfiguration ist komplett; keine dynamische Änderung möglich.

Wird z.B. beim automatischen Anlegen von Meldungsdefinitionen gesetzt, um Einträge zu markieren, deren Konfiguration geprüft werden muss. Wenn = 1, dann können die Werte für Meldetext, Meldeklasse und Meldeart beim Einbuchen noch geändert werden.

Nur zur Information; schreibgeschützt.

Vorlagenreferenz

Vorlagenreferenz. Nur zur Information; schreibgeschützt.

 

Kontextmenü

Über die Liste der projektierten Meldungen kann eine Simulation von Meldeeinträgen durchgeführt werden. Selektiert man eine Meldung im Navigationsbaum oder in der Tabelle und betätigt die rechte Maustaste, erscheint ein Kontextmenü mit folgenden Funktionen:

Meldung auslösen: Erzeugt ein Kommend-Ereignis der Meldung.

Meldung zurücksetzen: Erzeugt einen Gehend-Ereignis der Meldung.

 

Siehe auch Platzhalter im Meldetext

Soll eine Meldungsdefinition gelöscht werden, so wird zuerst geprüft, dass keine Meldung dafür mehr ansteht. Nur dann kann die Meldungsdefinition gelöscht werden; andernfalls wird eine Fehlermeldung ausgegeben.

 

ERWEITERTE ZUORDNUNGEN

Jede Alarmdefinition ist einem Knoten zugeordnet (siehe Einstellungen der Anwendungsstruktur | Datenaustausch | Module | Verbindungen | Alarmdefinitionen). Darüber hinaus kann jede Alarmdefinition weiteren Knoten zugewiesen werden. Dies ist möglich unter Anwendungsstruktureinstellungen | Logische Struktur | Knoten | Erweiterte Alarm-/Knotenzuordnungen. Dort kann man einem Knoten eine beliebige Alarmdefinition zuordnen. Wenn ein Alarm gesetzt wird, erscheint dieser nicht nur für den Knoten, der in der Alarmdefinition zugeordnet ist, sondern auch für jeden Knoten, für den eine erweiterte Alarmzuordnung für diesen Alarm existiert.

Die erweiterten Zuweisungen ändern nicht das Verhalten des durch seine Alarmdefinition definierten Alarms, d.h. der Alarm wird auf allen Knoten entsprechend der Vererbungseinstellung seiner Alarmklasse gesetzt.

 

ALARMVERERBUNG FÜR ERWEITERTE ALARM-/KNOTENZUWEISUNGEN

Für jede erweiterte Alarm-/Knotenzuordnung muss der gewünschte Vererbungstyp eingestellt werden. Der Standardtyp ist "keine Vererbung", d.h. der Alarm wird nur für die Knoten gesetzt, denen er zugewiesen ist (und für alle Knoten, die von den Einstellungen der Alarmklasse betroffen sind).

Jeder andere Vererbungstyp führt dazu, dass ein gesetzter Alarm zusätzlich zu der in den Einstellungen der Alarmklasse geforderten Vererbung auch für übergeordnete und/oder untergeordnete Knoten angezeigt wird.

Es ist möglich, dass die bestehende Alarm/Knoten-Zuordnung denselben Zweig der logischen Struktur abdeckt. Obwohl die Vererbungseinstellungen nahelegen könnten, dass ein Alarm mehrfach auf einem einzigen Knoten erscheint, verhindert das System dies.

Mit dem Feld 'Knotenhierarchieebene' ist es möglich, die Vererbung auf eine bestimmte Knotenebene zu beschränken. Die durch die Alarmklasse definierte Vererbung kann durch diese Einstellung nicht beeinflusst werden.

Ausnahme: Wenn der Funktion alarm_coming eine Liste von Knoten übergeben wird, wird für diesen Alarm keine Vererbung durchgeführt!
