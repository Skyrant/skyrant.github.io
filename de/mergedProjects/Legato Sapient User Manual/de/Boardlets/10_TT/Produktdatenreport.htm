Produktdatenreport

Allgemein

Mit dem Boardlet Produktdatenreport können Produktionsanwender Produktdaten, die in Legato Sapient gespeichert sind, im Rahmen von Track and Trace (T&T) analysieren und auswerten. Mit dem Produktdatenreport erhält man z.B. Informationen, ob und welche Engpässe es in einer Produktionslinie gibt.

Die gewonnen Daten dieser Datenanalyse-Funktion können im Boardlet Produktdatendiagramm grafisch dargestellt werden. Eine grafische Darstellung ist in Kreisdiagrammen, Balkendiagrammen und gestapelten Balkendiagrammen möglich. Siehe Produktdatendiagramm für mehr Informationen.

 

Beachten Sie folgende Informationen zum Produktdatenreport:

  • Neben der Analyse der produktbezogenen Daten können über die WebApp durch die PDF-Funktion die Analysedaten als Bericht ausgegeben werden.

  • Die Einstellungen im Boardlet können verschiedene Datentypen (bool, string, numerisch) haben.

  • Es können alle relevanten Produkteigenschaften, die mit Hilfe der T&T-Funktionalität erfasst und gespeichert wurden, und die spezifischen Vorkommen (Anzahl und Prozentsatz) dieser Eigenschaften innerhalb eines Zeitraums abgerufen werden. Beachten Sie hierzu im Boardlet die Zeitbereich-Filter für die Datenanalyse. Beispiele dieser Zeitbereich-Filter sind Individueller Zeitraum, Relative Zeit (z.B. "letzte Woche"), Absolute Zeit oder Klicken zum Wählen.

  • Für num-Werte gibt es die Aggregation in Klassen (Buckets) mit einem vorkonfigurierten Zahlen- und Wertebereich. Bei Klassen (Buckets) handelt es sich um Intervallbreiten mit einer bestimmten Anzahl und einem bestimmten Messwert. Die "Klassen andere" ("Buckets other") sind Bereiche außerhalb der Intervalle.

  • Die dargestellten Inhalte des Boardlets sind abhängig von den ausgewählten Filtern, jedoch unabhängig von den gewählten Knoten.

  • Alle Produkteigenschaften werden für Produktionsschichten für jeden konfigurierten Berichtsbereich aggregiert, d.h. Daten werden gesammelt und in einer zusammengefassten Form ausgedrückt. Außerdem werden alle Produkteigenschaften mittels eines zyklisch getriggerten Verfahrens nach Produkten gruppiert.

  • Es können für die ausgewählten Daten Gruppen bei der Auswahl gebildet werden (für Eigenschaften und auch für Zeitbereiche, z.B. Schicht, Tag, Woche, Monat).

  • Es können die ausgewählten (und auch gruppierten) Daten in einem flexiblen Datengitter dargestellt werden.

  • Das Boardlet Produktdatenreport hat folgende Inhalte:
    (1) Product / Produkt: Festgelegter Produktname
    (2) Attribute / Attribut: Eigenschaft, die durch Filtereinstellungen ausgewählt werden, wie z.B. Farbe oder Zeiteinheit
    (3) Value (from) / Wert (von): Grenzbeginn des Attributwertes einer Klasse (Bucket)
    (4) Value (up to) / Wert (bis zu): Grenzende des Attributwertes eines Buckets
    Zu (3) und (4): Innerhalb der Grenzen [Value from - Value up] wird der Attributwert einer jeweiligen Klasse (Bucket) hinzugezählt, d.h. wenn Value (from) = 200, dann wird der Wert 200 eingeschlossen. Wenn Value (up to) = 4199, dann wird Wert 4199 inkludiert, aber der Wert 4200 exkludiert).
    (5) Count / Anzahl: Anzahl der Produkte, die in der Klasse (Bucket) gezählt werden (Beachten Sie jedoch den Hinweis hierzu unterhalb der Abbildung, da es Produktionseinheiten (Production Unit, PUs) gibt, die keine Attribute haben)
    (6) %: Prozentualer Anteil des Produktes an der Klasse (Bucket), gemessen am gesamten Vorkommen


:

Bei (5) Count / Anzahl gilt: Wenn eine PU, wie z.B. eine Autokarosserie, zugewiesene Attribute (bspw. Farben/Colors) hat, dann werden diese Attribute für das Produktdatenreport-Boardlet aggregiert. Selbst wenn die PU das Attribut (hier: Farbe/Color) hat, aber der Wert leer (EMPTY) oder NULL ist, findet auch eine Aggregation statt.
D.h., in der Spalte Value (from) (3) wird für ein solches Attribut an der betreffenden Stelle eine leere Zelle dargestellt. Wenn aber einer PU keine Attribute (z.B. Farben/Colors) zugewiesen sind, dann wird diese PU nicht gezählt. Wenn die PU zugewiesene Attribute hat, aber der Attributwert leer (EMPTY) oder NULL ist, wird dies nicht aggregiert.

 

Konfiguration und benötigte Parametrierung im LC2

Die Konfiguration des Produktdatenreports findet im LC2 statt. Eine entsprechende Konfiguration im LC2 ist notwendig, da festgelegt werden muss, von welcher Lesestelle Daten gesammelt werden.

Verschiedene Bereiche werden über Lesestellen identifiziert. Es können mehrere Lesestellen angegeben werden. Die zusätzlich angegebenen Lesestellen gehören alle zu einem definierten Bereich.

Grundlegend muss

  • die jeweilige Lesestelle,

  • der Namen der Lesestelle und

  • die Gruppierung

festgelegt werden.

 

:

Beim Aggregieren von Daten werden eine Vielzahl an einzelnen Werten zu einem einzigen Wert zusammengefasst.
Die Datenaggregation erstellt die komplette Datenquelle für den Produktdatenreport und das Produktdatendiagramm.

Die Datenaggregation muss korrekt eingerichtet sein, andernfalls werden im Boardlet Produktdatenreport keine Daten angezeigt.

 

Beachten Sie hierzu die Konfiguration der Aggregierung für Produktdatenreport und Produktdatendiagramm, um mehr Informationen für die Konfiguration für die Boardlets Produktdatenreport und Produktdatendiagramm zu erhalten.

 

Gehen Sie wie folgt vor, um die T&T-Funktionalität im LC2 aufzurufen und notwendige Einstellungen für den Produktdatenreport festzulegen:

  1. Öffnen Sie den LC2 und navigieren Sie in der Navigation (1) auf Production Management / Produktionsmanagement (2) und Tracking & Tracing (3).

  2. Navigieren Sie zu Reporting Area Groups / Report-Bereichsgruppen (4).


  3. Ordnen Sie die Tabelle mit dem Befehl Best Fit (all columns) / Optimale Breite (alle Spalten) (5) über das Kontextmenü der Reporting Area Groups-Tabelle / Report-Bereichsgruppen-Tabelle.

  4. Legen Sie in der Tabelle Reporting Areas / Report-Bereiche Report-Bereiche fest.
    Falls mehr als 50 Auswertungen (= Reporting Areas) angelegt werden, ist das Anlegen von Report-Bereichen empfehlenswert, damit bei diesen Auswertungen noch zusätzliche Gruppierungsbewertungen angegeben werden können. Eine Gruppierung ist hier hilfreich, um eine gezielte Auswertung zu ermöglichen. In der Tabelle Reporting Areas / Report Bereiche kann dies in der Spalte Group / Gruppe gemacht werden.

  5. Durch Klicken auf das Plus-Symbol können Sie im LC2 eine neue Reporting Area Group / Report-Bereichsgruppen anlegen.
    Vergeben Sie einen Namen für die neue Reporting Area Group / Report-Bereichsgruppe.

    EN screenshot

  6.  Anschließend können Sie unter Reporting Areas/Report Bereiche die soeben erstellte Gruppe zuordnen. In der WebApp können Sie diese Gruppen ebenfalls sehen und abrufen.


  7. Wählen Sie im nächsten Schritt in der Reporting Area-Tabelle / Report-Bereiche-Tabelle eine Reporting Area aus, oder erstellen eine neue Reporting Area / Report-Bereich durch Klicken auf das Plus-Symbol). Beim Erstellen einer neuen Reporting Area / Report Bereich vergeben Sie anschließend weitere Werte, wie z.B. den Namen und wählen Sie den entsprechenden Knoten aus. Für diesen Knoten gilt anschließend die ausgewählte Reporting Area.


  8. Vergeben Sie im rechten Bereich im Hauptfenster unter Details die Klassenwerte (Bucketwerte).

 

  • Beachten Sie, dass für flexible Analysen können dynamische Gruppen dieser Eigenschaften im LC2 gebildet werden.

 

  • Wie bei anderen T&T-Funktionen wird eine Durchlaufzeit zwischen zwei Lesestellen oder auch Identifikationspunkten (Identpoints) gemessen. Die Einstellungen hierfür können unter Reporting Areas / Report-Bereiche (1) und Area-Identpoint / Lesestellen (2) im LC2 vorgenommen werden:




  • Damit die Durchlaufzeit zwischen zwei Lesestellen oder auch Identifikationspunkten (Identpoints) gemessen werden kann, werden Identifizierungstypen Einfahrt (Entry ident types) und Identifizierungtypen Ausfahrt (Exit ident types) definiert. Die gewonnen Daten werden auf die gleiche Weise wie andere konfigurierte Attribute analysiert. Beispiele der Identifizierungtypen sind z.B. Abstapeln, Ausfahrt, Einfahrt, usw.

Reiter Filter

Im Filterdialog des Boardlets Produktdatenreport kann alles eingestellt werden, damit der Benutzer eine umfangreiche Auswertung durchführen kann. Eine Voraggregierung im LC2 ist jedoch notwendig.
Neben den bereits im Abschnitt Allgemein beschriebenen Time range / Zeitbereich-Filtern (1) gibt es folgende Filter:

 

(2) Production area / Report-Bereich

(3) Products / Produkte

(4) Attributes to show / Anzuzeigende Attribute
