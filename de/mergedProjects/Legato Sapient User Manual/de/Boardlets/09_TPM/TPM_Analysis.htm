TPM Analyse

Allgemein

Das Boardlet TPM Analyse zeigt die Anzahl der gewählten Begründungen für Wartungstätigkeiten.

Bei Mouseover über einen Balken werden die Begründung und die Häufigkeit eingeblendet.

Durch Klick auf den Namen eines Wertes unten in der Legende kann dieser aus- bzw. wieder eingeblendet werden. Ausgeblendete Werte werden in der Legende hellgrau dargestellt.

Für das Boardlet TPM Analyse stehen folgende weitere Einstellungen zur Verfügung.

Reiter Filter

X-Achse: / Y-Achse:

Über diese Dropdown-Liste können die X- und Y-Achse festgelegt werden. Es stehen Tätigkeiten, Komponenten und Begründungen zur Auswahl.

Auslöser:

Über diese Dropdown-Liste kann selektiert werden, wodurch die Tätigkeit ausgelöst wurde (Wartungsplan, Meldung, Benutzer).

Tätigkeits-Kategorie:

Über diese Dropdown-Liste kann eine Tätigkeits-Kategorie ausgewählt werden.

Top-Ergebnisse:

Hier kann die Anzahl der angezeigten Ergebnisse auf die mit der größten Häufigkeit eingeschränkt werden.

Komponenten:

Über diese Dropdown-Liste kann eine Komponente ausgewählt werden. Hierzu öffnet sich ein Suchfenster, in dem nach Komponenten gesucht werden kann.

Tätigkeiten auf unterster Ebene:

Über diesen Schieberegler wird definiert, ob Tätigkeiten der untersten Ebene angezeigt werden oder nur "Parent"-Einträge.

Benötigte Parametrierung im LC2

  • Tätigkeiten werden im LC2 unter Werksverwaltung TPM Tätigkeiten angelegt.

  • Komponenten werden im LC2 unter Werksverwaltung TPM Komponentenarten angelegt und ein Knotenbezug hergestellt.

  • Begründungen werden im LC2 unter Werksverwaltung TPM Begründungen angelegt.
