TPM Plans (Wartungspläne)

Allgemein

Das Boardlet TPM Plans zeigt alle verfügbaren Wartungspläne zum selektierten Knoten an. Es werden verschiedene Informationen angeboten, beispielsweise die nächste Fälligkeit, die letzte Erledigung und die zuständige Instandhaltungs-Gruppe (IH-Gruppe).

Mittels Drag & Drop lassen sich die Breite oder Position der Spalten einfach verändern. Durch Klick auf eine Zeilenüberschrift wird nach dieser Spalte sortiert.
Nach welcher Spalte sortiert ist, und ob auf- oder absteigend sortiert ist, wird durch die dreieckigen Symbole angezeigt:

 

INFO:

Pflichtfelder in den Seitenleisten sind mit einem Ausrufezeichen gekennzeichnet.

 

Neuen Wartungsplan anlegen

Ist ein Knoten selektiert, erscheinen oben in der Boardlet-Menüleiste weitere Symbole.

Durch Klick auf kann ein neuer Wartungsplan für diesen Knoten angelegt werden. Der Dialog Neuer Wartungsplan erscheint in der Seitenleiste: ------>>

Der Name kann frei vergeben werden. In der Dropdown-Liste Parent kann der Wartungsplan optional einem übergeordneten Plan zugeordnet werden. Dann wird die Fälligkeit unterdrückt, wenn der Parent zur selben Zeit ansteht.

Das Startdatum legt die erste Fälligkeit des Plans fest.

Über die Dropdown-Liste Basiert auf kann festgelegt werden, ob der Wartungsplan basierend auf Zeit, Wert, Meldung oder einer Kombination aus mehreren Kriterien ausgelöst werden soll. Abhängig von der Auswahl hier können im unteren Teil des Dialogs die Trigger konfiguriert werden.

Aus der Dropdown-Liste Priorität kann die Priorität des Wartungsplans selektiert werden (hoch, mittel, niedrig).

Aktiv:

Schieberegler zur Aktivierung/Deaktivierung eines Wartungsplans.

  • an: Der Wartungsplan erzeugt neue Wartungsaufträge entsprechend dem Trigger, der aus der Dropdown-Liste Basiert auf gewählt wurde. Sie werden im Boardlet TPM Wartungsaufträge angezeigt.

Beispiel: Das Startdatum wird auf den 07.07.2020 um 11:00 Uhr festgelegt und in der Dropdown-Liste Basiert auf als Trigger Time gewählt. Zum angegebenen Datum und Zeitpunkt erscheint dann der Wartungsauftrag im Boardlet TPM Wartungsaufträge.

  • aus: Der Wartungsplan erzeugt keine neuen Wartungsaufträge.

Standardmäßig werden nur aktive Wartungspläne im Boardlet angezeigt. Mit folgenden Schritten können Sie auch inaktive Wartungspläne im Boardlet anzeigen:
Wählen Sie im Menü Quick-Edit [] Filter Aktivieren Sie den Schieberegler Deaktivierte anzeigen.

Basiert auf:

Die folgenden Trigger-Typen sind verfügbar: a) Zeit, b) Wert, and c) Meldung. Je nach gewähltem Trigger-Typ werden nun weitere Möglichkeiten zur Parametrierung angeboten:

a) Zeitbasierter Trigger:

Zunächst kann definiert werden, ob der Zeit-Trigger auf einem Kalender- oder einem Cron-Intervall basieren soll.

Bei Kalender-Intervall wird dann zunächst die Größe des Intervalls eingestellt, darunter kann dann die Einheit gewählt werden.

Bei Cron kann ein Cron-Ausdruck in ein Textfeld eingegeben werden.

b) Wertbasierter Trigger:

Zunächst muss eine Variable aus einer Dropdown-Liste selektiert werden, deren Wertänderung triggern soll. Es werden alle dem Knoten zugewiesenen Variablen angeboten, zu dem auch der Wartungsplan gehört.

Darunter wird eingegeben, alle wieviel Stück die Variable einen Wartungsauftrag auslösen soll.

c) Meldungsbasierter Trigger:

Zunächst muss eine Verbindung aus einer Dropdown-Liste ausgewählt werden, deren Meldung den Wartungsauftrag auslösen soll. Darunter kann dann die gewünschte Meldungsdefinition selektiert werden.

  

Wartungsplan bearbeiten

Ist ein Wartungsplan im Boardlet TPM Plans markiert, erscheinen in der Boardlet-Menüleiste weitere Symbole.

Durch Klick auf kann die Konfiguration des Wartungsplans angesehen oder geändert werden, die beim Anlegen getroffen wurde. Der Dialog Wartungsplan bearbeiten erscheint in der Seitenleiste. Siehe Neuen Wartungsplan anlegen.

  

 

Wartungsauftrag hinzufügen/bearbeiten

Ist ein Wartungsplan im Boardlet TPM Plans markiert, erscheinen oben in der Boardlet-Menüleiste weitere Symbole.

Durch Klick auf können Sie in der Seitenleiste (a) einen neuen Wartungsauftrag anlegen oder (b) sich den aktive Wartungsauftrag anzeigen lassen. Der Dialog Wartungsauftrag bearbeiten erscheint in der Seitenleiste: -------->>

  • Für jeden Wartungsplan können Sie nur einen aktiven Wartungsauftrag anlegen. Ein Wartungsauftrag bleibt aktiv, bis er auf den Status erledigt gesetzt wird.
    Siehe Boardlet TPM Wartungsaufträge Status ändern.
    Solange es einen aktiven Wartungsauftrag gibt, können Sie keinen neuen Wartungsauftrag für den markierten Wartungsplan anlegen.

  • Zusätzlich zum aktiven Wartungsauftrag für einen bestimmten Wartungsplan kann es noch mehrere Wartungsaufträge geben, die bereits im Status erledigt vorliegen. Eine Übersicht über alle Wartungsaufträge wird im Boardlet TPM Wartungsaufträge angezeigt.

  

Tätigkeitsverwaltung: Tätigkeit hinzufügen/entfernen

Ist ein Wartungsplan im Boardlet TPM Plans markiert, erscheinen oben in der Boardlet-Menüleiste weitere Symbole.

Durch Klick auf können Sie sich die Liste der für Wartungspläne verfügbaren Tätigkeiten anzeigen lassen. Der Dialog Aktivität Hinzufügen/Entfernen erscheint in der Seitenleiste, siehe Reiter [A]: ---->>

Außerdem können Sie folgende Aktionen ausführen:

  • Hinzufügen von Tätigkeiten zum Wartungsplan, indem Sie die Checkboxen neben den erforderlichen Tätigkeiten aktivieren.

  • Entfernen von Tätigkeiten aus dem Wartungsplan, indem Sie die Checkboxen neben den erforderlichen Tätigkeiten deaktivieren.

Eine Beschreibung von Reiter [B] finden Sie in Tätigkeitsverwaltung: Tätigkeit hinzufügen/bearbeiten weiter unten.

  

Tätigkeitsverwaltung: Tätigkeit hinzufügen/bearbeiten

Ist ein Wartungsplan im Boardlet TPM Plans markiert, erscheinen oben in der Boardlet-Menüleiste weitere Symbole.

Mit Klick auf und auf den zweiten Reiter [B] können Sie den Dialog Aktivität Hinzufügen/Bearbeiten in der Seitenleiste öffnen. Dort können Sie folgende Aktionen ausführen:

  • Anlegen neuer Tätigkeitsbeschreibungen

  • Bearbeiten bestehender Tätigkeitsbeschreibungen

Die neuen bzw. bearbeiteten Tätigkeitsbeschreibungen stehen danach auf dem Reiter [A] zur Auswahl bereit.

So fügen Sie eine neue Tätigkeitsbeschreibung zur Liste der Tätigkeiten hinzu:

1

Schalten Sie den Schieberegler von Bearbeiten auf Hinzufügen.

2

Geben Sie eine Beschreibung für die neue Tätigkeit ein.

3

Weisen Sie die neue Tätigkeitsbeschreibung einer Kategorie zu.

4

Wenn Sie fertig sind, klicken Sie Bestätigen.

Die neue Tätigkeit steht nun auf dem Reiter [A] zur Auswahl bereit.

Für das Boardlet TPM Plans stehen folgende weitere Einstellungen zur Verfügung.

Reiter Filter

Deaktivierte anzeigen:

Über diesen Schieberegler wird definiert, ob nur aktive Wartungspläne angezeigt werden sollen oder auch inaktive Wartungspläne.

Priorität:

Über diese Dropdown-Liste kann nach den Statusarten hoch, mittel und niedrig gefiltert werden.

Benötigte Parametrierung im LC2

Wartungspläne werden im LC2 unter Werksverwaltung TPM Wartungspläne angelegt. Dort können auch Tätigkeiten und Gruppen dem Plan zugeordnet werden.
