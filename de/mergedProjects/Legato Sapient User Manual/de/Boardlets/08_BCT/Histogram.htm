Histogramm

Allgemein

Das Boardlet Histogramm basiert auf dem Balken- und Liniendiagramm. Es dient der Visualisierung der Anzahl von jeweils gleichen Taktzeiten, also z.B. wie oft trat die Taktzeit 7,5 Sekunden auf:

  • X-Achse: Klassen der Taktzeiten (Klassenbreite wird i.d.R. automatisch durch Legato Sapient berechnet, kann aber auch manuell eingestellt werden)

  • Y-Achse: Anzahl der aufgetretenen Taktzeiten

 

Neben der Darstellung der Taktzeiten als Balken können Zusatzinformationen, wie z.B. mittlere Taktzeit, häufigste Taktzeit oder Median in Form von vertikalen Linien in das Diagramm eingeblendet werden.

Zusätzlich wird eine Tabelle mit Zusatzinformationen unter dem Histogramm eingeblendet, hier werden Informationen zur Prozessqualität dargestellt (Standardabweichung, Prozessfähigkeit cp/cpk, etc.)

Bei Mouseover über einen Balken wird die Häufigkeit des Wertes eingeblendet.

Durch Klick auf den Namen eines Wertes unten in der Legende kann dieser aus- bzw. wieder eingeblendet werden. Ausgeblendete Werte werden in der Legende hellgrau dargestellt.

Durch Klicken und Ziehen mit gedrückter linker Maustaste kann in den Zeitbereich hineingezoomt werden. Es wird dann ein Button eingeblendet, über den der Zoom wieder zurückgesetzt werden kann.

Für das Boardlet Histogramm stehen außerdem folgende Einstellungen zur Verfügung.

Layout

Diagrammtyp:

Über die Dropdown-Liste kann der gewünschte Diagrammtyp selektiert werden, z.B. Balken- oder Liniendiagramm.

Hilfslinie:

Aus der Dropdown-Liste können Werte ausgewählt werden, die als vertikale Linien zusätzlich im Histogramm angezeigt werden sollen, z.B. Best Cycle Time, Min und Max.

Klassenbreite manuell setzen:

Über die Klassenbreite wird eingestellt, in welchem Bereich die Taktzeiten zusammengefasst dargestellt werden sollen. Dementsprechend werden die Taktzeiten bei einer Klassenbreite von z.B. 0,1 immer für 0,1 Sekunden zusammengefasst. Üblicherweise wird die Klassenbreite automatisch berechnet; durch Aktivieren dieses Schiebereglers kann sie aber auch manuell gesetzt werden.

Visualisierung der kumulierten Taktzeiten:

Dieser Schieberegler legt fest, ob Taktzeiten, die außerhalb der Grenzen (minimale und maximale Taktzeit ohne Störung) liegen, kumuliert in jeweils einem Balken dargestellt werden oder nicht.

Zu ignorierende Anzahl von Werten am Anfang:

Wieviele Werte am Anfang des Zeitraums für die Berechnung ignoriert werden sollen (weil sie möglicherweise das Ergebnis verfälschen könnten).

Zu ignorierende Anzahl von Werten am Ende:

Wieviele Werte am Ende des Zeitraums für die Berechnung ignoriert werden sollen (weil sie möglicherweise das Ergebnis verfälschen könnten).

Visualisierung der Prozessinformationen:

Über diesen Schieberegler wird gesteuert, ob die Tabelle mit den Prozessinformationen unterhalb des Histogramms angezeigt werden soll oder nicht.

Reiter Filter

Variablenarten:

Filter auf Variablenarten. Für alle Variablen der hier definierten Variablenarten, die dem oben selektierten Knoten zugeordnet sind, werden die Werte im Diagramm angezeigt. Die Variablenarten können aus einer Dropdown-Liste ausgewählt werden. Leer = kein Filter, alle Variablenarten werden angezeigt.

Benötigte Parametrierung im LC2

Die Variablenarten werden im LC2 unter Grundkonfiguration Allgemeine Anwendungen Variablenarten parametriert.

Die Variablen werden im LC2 unter Anwendungsstruktur Datenaustausch [Modul] [Verbindung] Variablen gepflegt. Hier wird auch die Einheit festgelegt.

Hier muss das Flag "Archivieren" auf 1 gesetzt sein, damit die Werte der Variable im Archiv gespeichert werden.

Unter Anwendungsstruktur Fertigungsstruktur [Knoten] Variablenzuordnungen werden die Variablen dem Knoten zugeordnet, für den sie angezeigt werden sollen. Hier muss das Flag "Anzeigen" auf 1 gesetzt sein, damit die Variable angezeigt wird. Die Sortiernummer legt fest, in welcher Reihenfolge die Variablen angezeigt werden.
