Filterstrings

Allgemein

In einigen Boardlets, zum Beispiel in der Generischen Tabelle kann ein Filter als String angegeben werden. In diesem String können auch Platzhalter für Parameter eingetragen werden. Für weitere Details zu Platzhaltern, siehe bitte Platzhalter.

Formatierung der Strings

Die Strings sind immer als Wertepaare formatiert, die einander mit Doppelpunkt gegenübergestellt werden. Die Werte sind dabei in doppelten Hochkommata notiert. Die verschiedenen Wertepaare sind dabei durch Komma voneinander getrennt. Der String darf keine Leerzeichen enthalten. Der gesamte String ist durch geschweifte Klammern eingefasst.

Beispiel:

{"field":"node","op":"in","val":$node$}

Operanden

Als Vergleichsoperanden stehen "in" und "eq" zur Verfügung.

"in" = Wert muss sich innerhalb vorgegebener Wertgrenzen befinden, z.B. zwischen 10-15 oder einer der Werte 1,2,3,6 sein.

"eq" = equal; Wert muss gleich der Vorgabe sein.

Beispiele zu Platzhaltern in Filtern

Das Feld "Node" wird mit einem oder mehreren Knoten-IDs aus einer sendenden Tabelle befüllt (üblicherweise Knotenbaum oder Layout, aber auch jede andere Tabelle mit Knotenbezug möglich).

{"field":"node","op":"in","val":$node$}

Das Feld "Einheit" mit Operand "Equal" soll dem Wert des Platzhalters "Detailinfo" entsprechen.

{"field":"unit","op":"eq","val":"$detailInfo$"}
