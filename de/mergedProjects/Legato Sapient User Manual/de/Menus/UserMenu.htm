Menü Benutzer

Im Menü Benutzer finden sich die Einstellungen, der Ausloggen-Button, die Hilfe und Info (Über). Die Hilfe wird in einem neuen Reiter geöffnet.

Einstellungen

Hier können Einstellungen für den angemeldeten Benutzer getroffen werden. Einstellungen werden in der Datenbank gespeichert.

  

 

Benutzereinstellungen

Hier können übergreifende Benutzereinstellungen getroffen werden.

Theme:

Aus einer Dropdown-Liste kann das zu verwendende Theme gewählt werden. Für Anwender mit einer Rot-Grün-Schwäche steht das Theme Color Filter Red Green bereit. Zusätzlich kann neben dem standardmäßig eingestellten Default-Theme ein Dark-, Mulitcolor- und Custom-Theme ausgewählt werden.

New layout:

Durch Aktivieren von New layout wird das neue UI der WebApp geladen.

 

Das alte Layout ist weiterhin der Standard (default layout).

Der Umschalter für das neue Layout ist nur vorhanden und kann somit nur genutzt werden, wenn der Systemparameter -300 (NEW_LAYOUT_CHOICE_POSSIBLE) gesetzt ist.
Dieser Systemparameter ist nicht auf allen Systemen verfügbar, kann aber manuell hinzugefügt werden.

 


Klicken Sie stets auf den grünen Anwenden-Button am Ende der Seite, damit Änderungen auch übernommen werden:

 

Boardlet Eigenschaften

Hier kann die in den Boardlets zu verwendende Schriftgröße eingestellt werden.

Eigenschaften der Seitenleiste:

Hier können die bevorzugten Einstellungen für die Seitenleiste festgelegt werden.

  • Darstellungsmodus: Es stehen folgende Darstellungsmodi zur Verfügung:
    a) Seitenleiste,
    b) Überlagernd
    c) Tabbed
    Die Auswahl kann später auch bei den einzelnen Dashboards geändert werden.

  • Breite der Seitenleiste: Hier kann die Standardbreite der Seitenleiste festgelegt werden.

  • Kopfzeile anzeigen?: Ist dieser Schieberegler nicht aktiv, wird die Kopfzeile nicht angezeigt. Dann kann auch nicht über die Dashboard-Ordner oder die Einstellungen navigiert werden, sondern es muss direkt die vollständige URL eingegeben werden.

   

 

Sprache:

Hier kann die Sprache selektiert werden, in der die Oberfläche und die Daten angezeigt werden sollen.

Default-Dashboard:

Hier kann ein Default-Dashboard selektiert werden, das nach dem Anmelden des Benutzers automatisch geladen wird.

Hierbei wird zwischen zwei Versionen unterschieden: Desktop- und Tablet und Smartphone.

Beim Klicken auf das linke untere Legato Sapient-Logo wird dieses Dashboard im Anschluss geladen:

Shortcut-Einstellungen:

Hier können verschiedene Dashboards hinterlegt und bei Bedarf auch mit dem entsprechenden Profil versehen ausgewählt werden.

Die Shortcut-Einstellungen beziehen sich auf die Icons im Footer der Hauptseite der WebApp:

Passwort ändern:

Hier kann der angemeldete Benutzer hier sein Passwort ändern. Das neue Passwort muss einmal bestätigt werden.

 

 

 

 

   

 

Anwendungseinstellungen

Hier können allgemeine Einstellungen für die Anwendung getroffen werden.

Obere benutzerdefinierte Bar aktiv?:

Hier kann definiert werden, ob die obere benutzerdefinierte Symbolleiste (Custom Bar) angezeigt werden soll und wenn ja, welche Boardlets darin angezeigt werden sollen. Siehe auch Custom Bar.

Linke benutzerdefinierte Bar aktiv?:

Hier kann definiert werden, ob die linke benutzerdefinierte Symbolleiste (Custom Bar) angezeigt werden soll und wenn ja, welche Boardlets darin angezeigt werden sollen. Siehe auch Custom Bar.

   

 

Filter automatisch im Standardprofil speichern:

Die Filter, die für ein Boardlet über das Menü Quick-Edit [] eingestellt werden, werden automatisch im Standardprofil namens Default (Persönlich) gespeichert.

Verwandte Themen:

    
  • Inhalte von Profilen - Filtereinstellungen
    

Logging-Ebene:

Hier kann die Logging-Ebene festgelegt werden.

Navigationsverzögerung:

Hier kann eine Verzögerung für das Laden der Daten eingestellt werden. Diese Einstellung hat nur Auswirkungen, wenn der Knotenbaum als obere Custom Bar angezeigt wird. Mithilfe dieser Verzögerung ist es möglich, durch alle Ebenen des Knotenbaums zu navigieren, ohne jedes Mal zu warten, bis die Daten des gerade geklickten Knotens geladen sind.

   

 

Druck-Zeitpläne für Dashboards

Voraussetzungen:

  • Damit die Funktion Druck-Zeitplan verfügbar ist, muss ein Druckserver installiert sein (siehe Installation - System-Installation - Druckserver).

  • Es werden nur Boardlets des Dashboards ausgegeben, bei denen die Einstellung Drucken aktiv ist (siehe Boardlet Settings).

Für den angemeldeten Benutzer können beliebig viele Druck-Zeitpläne erstellt werden. Über den Button + Erstellen rechts oben kann ein neuer Zeitplan angelegt werden. Darunter werden bereits erstellte Druck-Zeitpläne aufgelistet.

  •      Rot bedeutet, der Zeitplan ist nicht aktiv.

  •      Grün bedeutet, der Zeitplan ist aktiv.

Außerdem können die generierten Report-Dateien per Email an den gewünschten Empfängerkreis geschickt werden. Informationen dazu finden Sie weiter unten.

Auslösertyp: / Intervall:

Wählen Sie eine der folgenden Zeit-Optionen, um den Druckjob automatisch zu festen Zeiten, Datumsangaben, in sonstigen Intervalle etc. ausführen zu lassen:

  • In einem täglichen Intervall

  • In einem beliebigen anderen zyklischen Intervall

  • Gemäß Kalender

  • Mithilfe des zeitbasierten Job-Schedulers cron

Je nach Auswahl werden möglicherweise weitere übergebene Parameter anders ausgewertet. Siehe hierzu auch die detaillierte Beschreibung unter dem Link "Dokumentation" bei JSON-Code.

Im Modus Einfacher Editiermodus ist für den Auslösertyp Cron die Option Jede bestimmte Sekunde deaktiviert, bis Sie ein größeres Zeitintervall wählen.
Im Experten-Editiermodus können Sie die Sekunden direkt in das Feld JSON-Code eingeben.

Benötigte Parametrierung

Der Mail-Server, von dem die Report-Emails an den oder die Empfänger verschickt werden, muss im LC2 konfiguriert sein.

So konfigurieren Sie den Mail-Server für die Report-Emails:

1

Starten Sie LC2 und loggen Sie sich ein.

2

Wählen Sie System-Administration SE Jobs.

3

Klicken Sie mit der rechten Maustaste auf Tasks und wählen Sie dann Task hinzufügen.

Der Dialog Hinzufügen Task erscheint.

4

Wählen Sie in der Dropdown-Liste Task-Typ den Eintrag print.

5

Klicken Sie und wählen Sie dann die SE-Instanz aus der Tabelle Datenaustausch-Module.

6

Klicken Sie OK, um den Dialog zu schließen.

7

Geben Sie in der Spalte Eigenschaften die Konfiguration für den Mail-Server ein.

Weitere Konfigurationsbeispiele finden Sie unten.

8

Klicken Sie OK, um den Dialog zu schließen.

9

Haken Sie die Checkbox in der Spalte Aktiv an, um die Konfiguration für den Mail-Server zu aktivieren.

 

Beispiel 1: Konfiguration für lokale Mail-Server. Verschlüsselung wird unterstützt, z.B. die TLS-Verschlüsselung (Transport Layer Security).

mailUserName=user

mailPassword=password

mailAddressSender=yes@my.com

mailHost=127.0.0.1

mailPort=25

mailSsl=false

mailAuth=false

mailStartTls=true

mailOptionsTls=<option> // Die verfügbaren Optionen finden Sie in der Tabelle darunter.

<optionen> für mailOptionsTls

Wählen Sie nur eine <option>.

Beschreibung

:

Durch die Auswahl einer <option> geben Sie nur an, dass die TLS-Verschlüsselung verwendet werden soll.
Welche TLS-Version tatsächlich verwendet wird, hängt davon ab, welche TLS-Version auf dem Host installiert ist.

optional

TLS verwenden, sofern der Host TLS unterstützt.

trustAll

TLS anfordern, aber beliebigem Zertifikat vertrauen.

trustHost

TLS anfordern, aber beliebigem Zertifikat vertrauen, das auf den Host-Namen ausgestellt ist.

strict

TLS anfordern und alle Server-Identitätschecks nach RFC 2595 durchführen.

 

Beispiel 2: Herkömmliche Konfiguration für lokale Mail-Server.

mailUserName=notuse

mailPassword=notuse

mailAddressSender=example@my.com

mailHost=127.0.0.1

mailPort=25

mailSsl=false

mailAuth=false

 

Beispiel 3: Kleine Konfiguration für kommerzielle Mail-Server wie Gmail, Outlook etc.

mailUserName=example@gmail.com

mailPassword=examplePassword

mailHost=smtp.gmail.com

mailPort=465
