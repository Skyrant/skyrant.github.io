Parameter

Parametertypen

Für die meisten Filterkriterien der Boardlets (z.B. Knoten-ID oder Variablenarten) kann definiert werden, wie diese als Parameter im Kontext mit anderen Boardlets behandelt werden sollen. Der Parametertyp kann durch Klick auf die Symbole rechts neben dem jeweiligen Wert umgeschaltet werden. Außerdem kann die Verwendung der Parameter auf Container- und Dashboard-Ebene festgelegt werden (im Menü des jeweiligen Elements im Reiter Parameter).

Folgende Parametertypen stehen zur Verfügung:

InOut: Bedeutet, dass dieser Wert sowohl vom selektieren Boardlet empfangen als auch gesendet werden kann.

Local: Bedeutet, dass dieser Wert weder empfangen noch gesendet wird, sondern der eingegebene Wert immer gilt.

In: Bedeutet, dass dieser Wert immer empfangen wird. Empfiehlt sich beispielsweise für die Node ID beim Current Values-Boardlet.

Out: Bedeutet, dass dieser Wert nie empfangen wird, sondern nur gesendet.

Anwendungsbeispiele

Boardlets

Folgende Beispiele sollen die Auswirkung der unterschiedlichen Parametertypen verdeutlichen.

Dashboard mit folgenden Boardlets:

Layout, Aktuelle Werte, Tortendiagramm.

Beispiel 1

Es wurden folgende Einstellungen getroffen:

Layout:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Selektierte Knoten: InOut (ID = 63 = Schweißen 2)

Aktuelle Werte:

Knoten-ID: local (ID = 21 = Komponentenwerk)

Variablenarten: local (leer, d.h. alle Variablenarten werden angezeigt)

Tortendiagramm:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Variablenarten: local (2,40 = Iststückzähler, Ausschusszähler)

 

Die Auswahl eines anderen Knotens im Layout hat keinen Einfluss auf das Aktuelle Werte Boardlet. Hier wird immer der Tickertext vom Komponentenwerk angezeigt (weil das die einzige Variable ist, die dem Knoten zugeordnet ist). Das Tortendiagramm hingegen aktualisiert sich und zeigt immer die Ist- und Ausschussstückzähler des im Layout selektierten Knotens an.

 

Beispiel 2

Es wurden folgende Einstellungen getroffen:

Layout:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Selektierte Knoten: InOut (ID = 63 = Schweißen 2)

Aktuelle Werte:

Knoten-ID: In (ID = 63 = Schweißen )

Variablenarten: local (leer, d.h. alle Variablenarten werden angezeigt)

Tortendiagramm:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Variablenarten: local (2,40 = Iststückzähler, Ausschusszähler)

 

Die Auswahl eines anderen Knotens im Layout beeinflusst sowohl das Aktuelle Werte Boardlet als auch das Tortendiagramm. In den Aktuellen Werten werden jedoch alle Variablen angezeigt, die am selektierten Knoten verfügbar sind, im Tortendiagramm nur die Typen 2 und 40.

Container und Dashboard

Beispiel 3

In diesem Beispiel sollen die Container-übergreifenden Auswirkungen gezeigt werden. Es wird ein Dashboard mit 2 Containern verwendet. Container 1 enthält ein Layout-Boardlet, ein Aktuelle Werte Boardlet und ein Tortendiagramm. Container 2 enthält ein Layout-Boardlet und ein Aktuelle Werte Boardlet.

Container 1

Es wurden die selben Einstellungen getroffen wie in Beispiel 1:

Layout:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Selektierte Knoten: InOut (ID = 63 = Schweißen 2)

Aktuelle Werte:

Knoten-ID: local (ID = 21 = Komponentenwerk)

Variablenarten: local (leer, d.h. alle Variablenarten werden angezeigt)

Tortendiagramm:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Variablenarten: local (2,40 = Iststückzähler, Ausschusszähler)

 

Die Container-Parameter wurden wie folgt eingestellt:

Node ID: InOut

Selected Nodes: Out

Container 2

Es wurden folgende Einstellungen getroffen:

Layout:

Knoten-ID: InOut (ID = 63 = Schweißen 2)

Selektierte Knoten: InOut (ID = 63 = Schweißen 2)

Aktuelle Werte:

Knoten-ID: local (ID = 62 = Schweißen 1)

Variablenarten: local (leer, d.h. alle Variablenarten werden angezeigt)

 

Die Container-Parameter wurden wie folgt eingestellt:

Knoten-ID: InOut

Selektierte Knoten: Out

Ergebnis

Wird in Container 1 im Layout ein anderer Knoten selektiert, so wird diese Auswahl auch auf das Layout von Container 2 übernommen. Außerdem wird die Anzeige des Tortendiagramms aktualisiert. Die Anzeige der Aktuelle Werte Boardlets ändert sich nicht. Diese zeigen weiterhin die Variablenwerte der fest hinterlegten Knoten-ID an.

Wird in Container 2 im Layout ein anderer Knoten selektiert, so wird diese Auswahl auch auf das Layout von Container 1 und das dortige Tortendiagramm übernommen. Die Anzeige der Aktuelle Werte Boardlets ändert sich nicht. Diese zeigen weiterhin die Variablenwerte der fest hinterlegten Knoten-ID an.

Für Dashboards ist die Funktionsweise vergleichbar.
