Allgemein

In Profilen werden Einstellungen gespeichert, die über die Filterkriterien gemacht wurden, und zwar ohne den Bearbeitungsmodus. Durch das erstmalige Öffnen eines Dashboards mit einem Benutzer wird automatisch ein Standardprofil für diesen Benutzer angelegt. In diesem Standardprofil namens Default (Persönlich) sind die Standardeinstellungen des Dashboards hinterlegt (so, wie sie der Ersteller des Dashboards vorgesehen hat).

Für jedes Dashboard wird automatisch ein Standardprofil angelegt. Dieses Standardprofil kann nicht mit anderen Usern geteilt und nicht gelöscht werden. Neben den automatisch angelegten Standardprofilen können Sie benannte Profile für verschiedene Dateninhalte zu einem Dashboard anlegen.

 

Ein Profil beinhaltet für jedes Boardlet:

  • die Filtereinstellungen

  • die Feldeinstellungen von Tabellen (Feldauswahl, Feldreihenfolge und Feldbreite)

  • die Knotenauswahl und

  • den ausgewählten Zeitrahmen für alle Boardlets zu einem spezifischen Dashboard

Ein Profil kann beliebig oft zurückgesetzt (alle gespeicherten Einstellungen werden gelöscht) und mit den aktuell getroffenen Einstellungen gespeichert werden.

Ein Profil kann einem spezifischen User (persönliches Profil für den aktuell eingeloggten User) oder allen Usern (geteiltes Profil) zugewiesen werden.

Inhalte von Profilen - Filtereinstellungen

Filter auf die Daten von Boardlets können im Zuge der Dashboard-Erstellung (Filtereinstellungen im Boardlet) oder zur Laufzeit im Dashboard gesetzt werden.

Sind Filtereinstellungen im geladenen Profil gespeichert, werden ausschließlich diese Filter (nicht die ursprünglichen Einstellungen aus dem Dashboard) angewendet.

Verwandte Themen:

    
  • Filter automatisch im Standardprofil speichern
    

 

Inhalte von Profilen - Feldeinstellungen von Tabellen

Die Feldauswahl (Sichtbarkeit von Tabellenspalten) kann in den Boardlet-Einstellungen ausschließlich beim Bearbeiten eines Dashboards festgelegt werden. Diese Feldauswahl wird beim Setzen eines Profils übernommen.

Sofern die Feldauswahl nach Festlegung der Profile geändert wird, müssen diese Profile zurückgesetzt und erneut gespeichert werden, um die geänderte Auswahl zu zeigen.

Die Feldreihenfolge kann zur Laufzeit in den Boardlets geändert werden (Drag & Drop) und wird im Profil gespeichert.

Die Feldbreiten können zur Laufzeit in den Boardlets geändert werden und werden im Profil gespeichert.

 

Inhalte von Profilen - Ausgewählte Knoten

Die im Knotenbaum des Dashboard getroffene Knotenauswahl (ein Knoten oder mehrere Knoten) wird beim Speichern eines Profils übernommen.

Diese Knotenauswahl wird beim Laden des Profils für die Boardlets im Dashboard angewendet, sofern die Einstellungen des Boardlets das erlauben: Die Einstellung für Knoten muss auf in oder in/out stehen), wie im Folgenden gezeigt.

Inhalte von Profilen - Ausgewählter Zeitrahmen

Filter auf den selektierten Zeitbereich können für manche Boardlets im Zuge der Dashboard-Erstellung relativ oder absolut (z.B. "gestern" oder "12.06.20") oder zur Laufzeit im Boardlet (z.B. "letzte Woche" oder KW21) gesetzt werden.

Ist ein relativer oder absoluter Zeitbereich im geladenen Profil gespeichert, wird ausschließlich dieser Zeitbereich (nicht die ursprünglichen Einstellungen aus dem Dashboard) selektiert.

Für Reports, die auf Dashboards basieren, empfiehlt sich die Angabe eines relativen Zeitbereichs im Profil (z.B. "Tagesreport gestern").

 

 

 

Profile nutzen

Im Menü Profile können Sie folgende Aktionen ausführen:

  • Profile anzeigen und auswählen

  • Profil erstellen

  • Einstellungen im Profil speichern

  • Profil umbenennen

  • Profil zurücksetzen

  • Profil teilen / nicht mehr teilen

  • Profil löschen

  • Um das Menü Profil zu schließen, klicken Sie auf:

 

 

Profile anzeigen und auswählen

Durch Klick auf in der Kopfzeile können die Profile des angemeldeten Benutzers für das gewählte Dashboard geöffnet werden.

In der Dropdown-Liste können bereits gespeicherte Profile ausgewählt werden.

 

Profil erstellen

Erstellen legt ein neues Profil mit einem frei wählbaren Namen und den aktuell aktiven Einstellungen an. Die Einstellungen werden je Benutzer mit Dashboard-Bezug gespeichert.

 

Einstellungen im Profil speichern

Durch Klick auf werden die Einstellungen in das gerade gewählte Profil geschrieben.

 

Profil umbenennen

Mit Umbenennen können Sie dem aktuell ausgewählten Profil einen neuen, aussagekräftigeren Namen geben.

 

Profil zurücksetzen

Zurücksetzen setzt alle Einstellungen auf die Standardeinstellungen des Dashboards zurück. Diese Standardeinstellungen können durchaus bereits Filter- und Layout-Einstellungen beinhalten, z.B. Zeitfilter und Variablenauswahl. Diese Einstellungen wurden jedoch direkt beim Dashboard und ohne Benutzerbezug gespeichert.

:

Wurden benutzerspezifische Einstellungen in das Standardprofil gespeichert, so gehen diese durch Zurücksetzen verloren!

 

Profil teilen / nicht mehr teilen

Mit Teilen können Sie anderen Nutzern das aktuell ausgewählte Profil mit seinen Einstellungen zugänglich machen. Ein solches Profil ist nach dem Profilnamen nicht mehr mit "(Persönlich)", sondern mit "(Geteilt)" gekennzeichnet.

Mit Nicht mehr teilen wandeln Sie das geteilte Profil wieder in ein persönliches Profil um. Es steht anderen Nutzern nun nicht mehr zur Verfügung.

 

Profil löschen

Löschen löscht das aktuell ausgewählte Profil.

 

 

 

Profile fürs Reporting nutzen

In diesem Hilfe-Thema:

  • Use Case: "Link Reporting":

Eine aktuelle Situation wird von User A in einem Dashboard abgebildet und zur Information per Link an User B gesendet. D.h., der von User B geöffnete Link muss dasselbe Dashboard mit den identischen Daten öffnen.

  • Use Case: "Automatischer Report via Druck-Zeitplan":

Über einen konfigurierbaren Zeitplan wird ein Dashboard mit einem vorkonfigurierten Profil geöffnet. Dieses Dashboard mit den selektierten Daten wird dann als PDF-Report gespeichert.

 

Report-Link erstellen

Workflow zur Erstellung eines Report-Links:

1

User A: Das gewünschte Dashboard konfigurieren.

2

User A: Anlegen eines Profils, z.B. "Link_Line_A_200615"

4

User A: Speichern des Profils.

5

User A: Teilen des Profils.

6

User A: Versenden des Links aus der Adresszeile des Browsers, z.B. per E-Mail.

7

User B: Kopieren und einfügen des Links in die Adresszeile des Browsers.

8

User B: Öffnen des Links.

 

Automatischer Report via Druck-Zeitplan

Workflow zur Erstellung eines automatischen Reports via Druck-Zeitplan für Dashboards:

1

Key User: Dashboard für zyklischen Report konfigurieren, z.B. "Production Overview".

2

Key User: Anlegen eines Profils zum Report z.B. "Report_Line_A_weekly".

4

Key User: Speichern des Profils.

5

Key User: Teilen des Profils.

6

Key User: Anlegen eines Druck-Zeitplans mit Angabe von Dashboard und Profil, z.B. "Production Overview" + "Report_Line_A_weekly".

7

Key User - optional: E-Mailversand für den Report konfigurieren.

8

User X: Report aus dem Standardverzeichnis bzw. aus dem Anhang der E-Mail öffnen.

Verwandte Themen:

    
  • Druck-Zeitpläne für Dashboards
