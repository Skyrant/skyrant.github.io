Set Environment Variables

We can now set the JAVA_HOME and CATALINA_HOME in order to put Java and Tomcat into the path of our users.

[root@localhost share]# JAVA_HOME=/usr/java/jdk1.7.0_45  

[root@localhost share]# export JAVA_HOME  

[root@localhost share]# PATH=$JAVA_HOME/bin:$PATH  

[root@localhost share]# export PATH  

[root@localhost share]# CATALINA_HOME=/usr/share/apache-tomcat-7.0.42

[root@localhost share]# export CATALINA_HOME

 

To set the JAVA_HOME and CATALINA_HOME permanently, however, we need to add below to the ~/.bash_profile of the user (in this case, root).

We can also add it /etc/profile and then source it to give to all users.

JAVA_HOME=/usr/java/jdk1.7.0_45  

export JAVA_HOME  

PATH=$JAVA_HOME/bin:$PATH  

export PATH  

CATALINA_HOME=/usr/share/apache-tomcat-7.0.42

export CATALINA_HOME

 

Once you have added the above to ~/.bash_profile, you should log out, then log back in and check that the JAVA_HOME and CATALINA_HOME are set correctly.

[root@localhost ~]#  echo $JAVA_HOME  

/usr/java/jdk1.7.0_45  

[root@localhost ~]#  echo $CATALINA_HOME

/usr/share/apache-tomcat-7.0.42
