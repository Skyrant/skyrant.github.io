Application Engine Service verwalten

Service-Status anzeigen

Um detaillierte Informationen über eine Serviceeinheit (entspricht einem Systemdienst)anzuzeigen, geben Sie folgenden Befehl in die Shell-Eingabeaufforderung ein:

systemctl status Leappeng_LEGATO.service

 

Dieser Befehl zeigt den Namen der gewählten Diensteinheit, gefolgt von der Kurzbeschreibung. Dann folgen ein oder mehrere Felder der Tabelle "Verfügbare Informationen über Diensteinheiten" und, falls der Befehl durch den Root-Benutzer ausgeführt wurde, die letzten Logeinträge.

Verfügbare Informationen über Diensteinheiten

Feld

Beschreibung

Loaded

Geladen. Information, ob die Diensteinheit geladen wurde, der absolute Pfad zur Einheits-Datei und ein Hinweis, ob die Einheit aktiv ist.

Active

Aktiv. Information, ob die Dienst-Einheit läuft, gefolgt von einem Zeitstempel.

Main PID

Haupt-PID. Die PID des entsprechenden Systemdienstes gefolgt von seinem Namen.

Status

Status. Zusätzliche Information über den entsprechenden Systemdienst.

Process

Prozess. Zusätzliche Information über zugehörige Prozesse.

CGroup

C-Gruppe. Zusätzliche Informationen über zugehörige Kontroll-Gruppen (CGroups).

 

Um nur sicherzustellen, dass eine bestimmte Diensteinheit läuft, führen Sie folgenden Befehl aus:

systemctl is-active Leappeng_LEGATO.service

 

Um herauszufinden, ob eine bestimmte Diensteinheit aktiviert ist, geben Sie folgenden Befehl ein:

systemctl is-enabled Leappeng_LEGATO.service

 

Einen Dienst aktivieren

Um eine Diensteinheit (entspricht einem Systemdienst) so zu konfigurieren, dass sie beim Systemstart automatisch startet, geben Sie folgenden Befehl als Root in die Shell-Eingabeaufforderung ein:

systemctl enable Leappeng_LEGATO.service

 

:

Für die Installation der Legato Application Engine empfiehlt es sich, den Dienst zu aktivieren, so dass der Dienst beim Systemstart automatisch gestartet werden kann. Standardmäßig ist der Dienst deaktiviert.

Dieser Befehl liest den [Install] Bereich der gewählten Diensteinheit und erstellt entsprechende symbolische Links zur Datei  /usr/lib/systemd/system/name.service file im Verzeichnis /etc/systemd/system/ und Unterverzeichnissen. Dieser Befehl überschreibt allerdings keine bereits existierenden Links. Um sicherzustellen, dass die symbolischen Links neu erstellt werden, führen Sie folgenden Befehl als Root aus:

systemctl reenable Leappeng_LEGATO.service

 

Dieser Befehl deaktiviert die gewählte Diensteinheit und aktiviert sie sofort wieder.

Einen Dienst deaktivieren

Um zu verhindern, dass eine Diensteinheit (entspricht einem Systemdienst) automatisch beim Systemstart gestartet wird, geben Sie folgenden Befehl als Root in die Shell-Eingabeaufforderung ein:

systemctl disable Leappeng_LEGATO.service

 

Einen Dienst starten

Um eine Diensteinheit (entspricht einem Systemdienst) zu starten, geben Sie folgenden Befehl als Root in die Shell-Eingabeaufforderung ein:

sudo systemctl start Leappeng_LEGATO.service

 

Dieser Befehl startet die gewählte Diensteinheit in der aktuellen Sitzung.

Prüfen Sie, ob der Dienst erfolgreich gestartet wurde, indem Sie den Status überprüfen.

systemctl status Leappeng_LEGATO.service

 

Prüfen Sie, ob die Logdateien leappeng.log und leappeng.err im Logverzeichnis /var/log/leappeng erstellt wurden.

Einen Dienst stoppen

Um eine Diensteinheit  (entspricht einem Systemdienst) zu stoppen, geben Sie folgenden Befehl als Root in die Shell-Eingabeaufforderung ein:

sudo systemctl stop Leappeng_LEGATO.service

 

Dieser Befehl stoppt die gewählte Diensteinheit in der aktuellen Sitzung.

Prüfen Sie, ob der Dienst erfolgreich gestoppt wurde, indem Sie den Status überprüfen.

systemctl status Leappeng_LEGATO.service

 

Einen Dienst neu starten

Um eine Diensteinheit (entspricht einem Systemdienst) neu zu starten, geben Sie folgenden Befehl als Root in die Shell-Eingabeaufforderung ein:

sudo restart Leappeng_LEGATO.service

 

Dieser Befehl stoppt die gewählte Diensteinheit in der aktuellen Sitzung und startet sie sofort wieder.

:

Wenn die gewählte Diensteinheit nicht läuft, wird sie durch diesen Befehl trotzdem gestartet.

Konfiguration aktualisieren

Führen Sie folgende Befehle als Root aus, um die Konfiguration nach den Änderungen an der Datei "Leappeng_LEGATO.service" neu zu laden. So werden die Änderungen an den Einheiten-Dateien übernommen, ohne dass das System neu gestartet werden muss.

sudo systemctl daemon-reload

 

Die Option "daemon-reload" aktualisiert alle Einheiten-Dateien und erstellt den gesamten Abhängigkeitsbaum neu. Dies wird benötigt, um Änderungen an einer Einheiten-Datei sofort zu übernehmen.

Wenn die geänderte Einheiten-Datei zu einem laufenden Dienst gehört, z.B. hier zu "Leappeng_LEGATO.service", muss dieser Dienst ebenfalls neu gestartet werden, um die Änderungen zu übernehmen:

sudo systemctl restart Leappeng_LEGATO.service
