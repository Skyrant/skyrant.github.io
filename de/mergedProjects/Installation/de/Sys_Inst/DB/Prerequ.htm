Systemvoraussetzungen

Zum Betrieb von Legato ist eine Datenbankmanagementsoftware (DBMS) von Oracle nötig.

Ab Legato Version 4.7.3:

Voraussetzung ist Oracle Version 12.2 (alias 18) oder 12.1 (ohne Kompatibilitätsmodus).

Bis Legato Version 4.7.2:

Voraussetzung ist Oracle Version 11gR2. Für Legato zulässig sind die Versionen Oracle Standard Edition One (Oracle SEO), Oracle Standard Edition (Oracle SE) und Oracle Enterprise Edition (Oracle EE). Außerdem ist Oracle 12c (mit Parametern COMPATIBLE='11.2.0' und OPTIMIZER_FEATURES_ENABLE='11.2.0.1') freigegeben.

:

Es ist nicht möglich, den Parameter COMPATIBLE später auf einen niedrigeren Wert zu setzen. Daher ist für die Nutzung von Legato nur ein Upgrade auf Oracle 12c möglich. Eine direkte Installation einer 12c-Instanz wird nicht unterstützt (da dann der Parameterwert automatisch auf 12.1.0 gesetzt ist).

:

Mit Legato kann kein User-basiertes Oracle-Lizenzmodell eingesetzt werden. Es wird eine CPU-basierende Lizenz benötigt!

Es wird empfohlen, die Oracle Enterprise Edition zu installieren, da diese unter anderem die folgenden vorteilhaften Features bietet:

Online Table and Index Rebuild

Partitionierung von Tabellen (optional)

Automatic Workload Repository (AWR) (optional)

 

Einen detaillierten Vergleich der Oracle Versionen findet man auf der Oracle Homepage (http://docs.oracle.com/cd/E11882_01/license.112/e47877/editions.htm#DBLIC116).

Legato ist auch auf Oracle ASM freigegeben. ASM steht für Automatic Storage Management, eine in den Oracle Kernel integrierte Clustersoftware, die die Funktionalität eines Volume Manager und File-System kombiniert und dem DBA die Verwaltung der Datenbankdateien erleichtert. ASM bildet die Schicht zwischen den (physikalischen oder abstrahierten) Festplatten und dem DBMS. Dabei legt Oracle die Datendateien auf nicht formatierten Bereichen unter Umgehung des Betriebssystems ab. Sobald das Oracle-DBMS und die Legato-Instanz (leer, ohne Tabellen, Stored Procedures usw.) installiert sind, verhält sich das System für die Legato-Installation "transparent" genauso wie ein Nicht-ASM-System.

Soll ein Upgrade von Oracle 11gR2 auf Oracle 12cR1 erfolgen, so muss Oracle zunächst mindestens auf Version 11.2.0.2 gepatcht werden. Mit niedrigeren Oracle-Versionen ist kein direktes Upgrade auf Oracle 12c möglich.

:

Die Installation von Legato auf Oracle Real Application Cluster (Oracle RAC) wird nicht unterstützt!

:

Bei der Installation von Oracle Server und OraClient (der z.B. für die Verwendung des LC2 benötigt wird) auf demselben Rechner unter Windows kommt es zu Problemen mit den Pfadangaben in den Umgebungsvariablen. Dort werden der Client- und der Serverpfad in der falschen Reihenfolge eingetragen, weshalb die benötigten Dateien zur Ausführung der Datenbank nicht gefunden werden. Gehen Sie folgendermaßen vor, um die Reihenfolge zu korrigieren: Unter Systemeigenschaften - System - Erweiterte Systemeinstellungen - Umgebungsvariablen - Path mit Doppelklick den Eintrag öffnen. Hier sind alle Pfadangaben als kommaseparierte Liste eingetragen. Am besten kopieren Sie den kompletten Eintrag in einen Editor. Dort kopieren Sie den Pfad des Oracle Servers von hinter dem Client-Pfad VOR den Clientpfad. Meist finden sich die beiden Pfadangaben ganz am Anfang, weil die zugehörigen Programme als letztes installiert wurden. Kopieren Sie nun den geänderten Eintrag wieder in die Umgebungsvariable und speichern Sie die Änderung.

Eine genaue Beschreibung der Installation des DBMS entnehmen Sie bitte je nach Betriebssystem den separaten Dokumenten "Installation_Oracle_Server_de" bzw. "Installation_RHEL_Oracle_Server_de".
