Sapient Cloud Connector

Installation auf einem Windows PC

Systemvoraussetzungen

 

:

Für den Betrieb unter Windows gilt die Einschränkung, dass aus Performancegründen und zur Gewährleistung eines stabilen Betriebsverhaltens, nicht mehr als 50 Verbindungen je Cloud Connector-PC angelegt werden sollen. Das ist bei der Projektierung des Gesamtsystems und Festlegung der erforderlichen Cloud Connector-PCs zu berücksichtigen. Für den Betrieb des Cloud Connectors auf UNIX-Plattformen gelten keine Einschränkungen.

Nur Classic: Zusätzlich

Oracle Client und Provider for OLE DB ab Version 11.2.0.1.0. 64-bit.

Ab SCC 3.1 wird auch ODAC 12.2.0.1.0 (64-bit) unterstützt (nur für Oracle 12.2!). Hierfür gelten folgende Besonderheiten:

Falls der Rechnername einen Unterstrich "_" enthält, kann die Installation nicht abgeschlossen werden. Ändern Sie ggf. den Rechnernamen.

Deaktivieren Sie im Schritt 4 der Installation folgende Default-Option: Oracle Services for Microsoft Transaction Server.

:

Beim Oracle Client 11.2.0.3 kommt es gelegentlich zu Problemen, da der OLE DB Provider nicht korrekt installiert wird. Daher wird die Version 11.2.0.1 empfohlen.

:

Bei Cloud Connectors mit hohen Datendurchsatz wird empfohlen, die Datei sqlnet.ora vom Oracle Treiber zu suchen und dort den Parameter

Sqlnet.outbound_connect_timeout=5

zu setzen, damit auch im Fehlerfall der Ablauf gewährleistet bleibt.

Ab SCC 3.2 wird auch ODAC 12.2.0.1.1 (64-bit) unterstützt (nur für Oracle 12.2!). Es gelten die gleichen Besonderheiten wie für ODAC 12.2.0.1.0.

Installation Sapient Cloud Connector

Die Installation des Sapient Cloud Connectors erfolgt über das Installationsprogramm setupscc.exe. Nach dem Start der Exe kann die Sprache der Installation gewählt werden.

Installation OPC-Server GOPS

Wird die Cloud Connector Installation zur Laufzeit (online) als Win64-Cloud Connector betrieben, sind ggf. der OPC-Server GOPS und die zugehörigen Treiber ebenfalls am Cloud Connector PC zu installieren. Dies erfolgt über die Datei GopsSetup.exe. Es empfiehlt sich, die Installation von Cloud Connector und GOPS gegebenenfalls in einem Schritt durchzuführen.

Zur Kopplung mit TIA (Totally Integrated Automation) steht außerdem der GTOPS von Gefasoft zur Verfügung. Dieser kann mithilfe der Datei SetupGTOPS2.exe installiert werden.

:

Wird im Projekt eine Kopplung über OPC gefordert, kann ausschließlich die Windows-Variante des Cloud Connectors eingesetzt werden. OPC wird von UNIX nicht unterstützt.

Installation für UNIX

Systemvoraussetzungen

Betriebssysteme

Der Cloud Connector ist als 64 Bit Applikation verfügbar.

 

Bibliotheken / Libraries

Der Cloud Connector verwendet folgende Bibliotheken:

Linux 64 Bit

linux-vdso.so.1

librt.so.1

librfccm.so

libACE.so.6.3.0

libACE_SSL.so.6.3.0

libcrypto.so.1.0.0

libssl.so.1.0.0

libxml2.so.2

libstdc++.so.6

libm.so.6

libgcc_s.so.1

libpthread.so.0

libc.so.6

libdl.so.2

libz.so.1

liblzma.so.5

/lib64/ld-linux-x86-64.so.2

 

Zur Installation des Cloud Connectors müssen die Bibliotheken, die nicht auf dem System ohnehin schon vorhanden sind, inkl. dem Cloud Connector Binary, in ein eigenes Verzeichnis kopiert werden (z.B. legato/bin) und die Ausführungsrechte gesetzt werden.

Der Bibliothekspfad muss das o. g. Installationsverzeichnis enthalten.
