Locale Language Configuration

Es gibt die Möglichkeit, verschiedene Sprachen zu konfigurieren. Hierfür ist eine spezielle Umgebung verfügbar, die Locals für die Applikation spezifiziert. Es kann jedoch keine allgemein gültige Aussage zur Konfiguration gemacht werden. Die jeweilige Konfiguration ist immer kundenabhängig.

 

Übersetzungen müssen in das folgende Verzeichnis kopiert werden:

custom-translations/

Beispiel:

custom-translations/pt-br.json

 

Standard-Übersetzungen für andere Sprachen können im Sapient-Translations-Repository gefunden und die gewünschten Locales können als Umgebungsvariable (SAPIENT_LOCALE in einer *.env-Datei) spezifiziert werden:

SAPIENT_LOCALE=pl-pl;pt-br

 

Um Übersetzungen zu überschreiben, muss die Datei einen Namen mit dem dem gleichen Format haben wie "de-de.json" oder "pl-pl.json". In diesem Beispiel sind "de" und "pl" Ländercodes. Es muss sich also um ein einheitliches Format für das Gebietsschema handeln.

Beispielpfad, um Übersetzungen zu überschreiben:

lib/custom/translations
