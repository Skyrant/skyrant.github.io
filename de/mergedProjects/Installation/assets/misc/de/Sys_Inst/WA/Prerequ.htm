Tomcat als Web-Application-Server

 Tomcat gilt als Referenzimplementierung eines Servlet-Containers für J2EE-Webanwendungen. Mit ihm kann die Legato Webanwendung betrieben werden. Er bietet allerdings nicht die umfassenden Möglichkeiten eines Web-Application-Server Kaufprodukts. Nachteile zeigen sich z.B. im Komfort der Konfiguration oder des Monitorings.

:

Für die Verwendung mit Legato ist Tomcat 9  freigegeben.

:

Es wird empfohlen, eine zur Tomcat Version passende JDK Version zu verwenden, also für Tomcat 9 JDK 9.

:

Wird Tomcat installiert, so muss auf einem Windows-Server der standardmäßig aktivierte IIS (Web-Application-Server) deaktiviert werden.

Unter Unix-Servern ist im Allgemeinen der Headless Mode von Java zu nutzen, d.h. im Aufruf der JVM sollte jeweils ein Eintrag -Djava.awt.headless=true vorhanden sein.

Eine genaue Installationsanleitung für Tomcat finden Sie in den separaten Dokumenten Legato_Installation_TomCat_Win_EN und Legato_Installation_TomCat_RHEL_EN.

Empfohlener Speicherplatz für die Java VM

Folgender Speicher sollte zum Betrieb von Legato der Java VM zur Verfügung gestellt werden:

xmx/Maximum memory pool: 4096 MB

bezeichnet den maximalen Speicherplatz, auf den die Java VM zugreifen kann.

PermGen size/initial memory pool: 512 MB

bezeichnet den maximalen Speicherplatz, auf den PermGen zugreifen kann.

 

Der Wert des PermGen sollte insbesondere überprüft werden, wenn java.lang.OutOfMemoryError: PermGen space Fehlermeldungen im WebApp Log auftreten.

Für Tomcat Installationen unter Windows können diese Einstellungen in der Tomcat7w Monitor Application vorgenommen werden:

Für Tomcat Installationen unter Linux kann dies in der Datei /etc/init.d/tomcat unter JAVA_OPTS erfolgen. Dort ist folgender Eintrag vorzunehmen:

JAVA_OPTS="-Xmx4096m -Xms256m -XX:MaxPermSize=512m"

export JAVA_OPTS

 

Oracle JDBC Treiber

Bevor die WebApp installiert werden kann, wird zusätzlich der Oracle JDBC Treiber benötigt. Dieser ist im Installationspaket der ApplicationEngine enthalten.

Kopieren Sie die Dateien ojdbc*.jar und orai18n*.jar (* = aktuelle Version) aus dem Verzeichnis LegatoAppEng\lib in das Verzeichnis tomcat*\lib. Die verwendete Version muss mindestens ojdbc7.jar (für Java 1.8) oder höher sein.

Für den Fall, dass Sie diese Dateien nicht finden können, können Sie sie auch von der Oracle Homepage herunterladen.

Sind diese beiden Dateien nicht im tomcat*\lib Verzeichnis vorhanden, so erscheint eine Fehlermeldung beim Versuch, die WebApp zu starten ("Cannot load JDBC driver class 'oracle.jdbc.driver.OracleDriver'").

Zusatzdateien

Zusätzlich werden weitere Dateien benötigt, um Tomcat als WebApplication Server für Legato zu verwenden. Diese sind die Datei "NoJarScanner-tomcat8.0.jar" und eine Vorlage für die "context.xml". Diese Dateien finden sich im Jenkins-Projektordner "TomcatAddons". Entpacken Sie die Datei "NoJarScanner-tomcat8.0.jar" ins Tomcat lib Verzeichnis ($CATALINA_HOME\lib). Die "context.xml" kann als Beispiel zum Aufbau einer Verbindung zur Datenbank genutzt werden und sollte bei Bedarf ins Verzeichnis META-INF kopiert werden. Findet der Tomcat dort beim Start einer neuen WA eine xml-Datei, so verwendet er diese und benennt sie so, wie die Applikation heißt (z.B. Legato.xml). Legt man die xml-Datei selbst im Verzeichnis conf\catalina\localhost ab, so ist darauf zu achten, sie ebenso wie die jeweilige Applikation zu benennen.
