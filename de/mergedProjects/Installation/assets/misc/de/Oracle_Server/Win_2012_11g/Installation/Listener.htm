Einrichtung des Listeners

Nachdem die DBMS-Software installiert wurde, stehen noch keine Oracle Dienste im System zur Verfügung. Daher sollte nun zunächst ein Listener konfiguriert werden. Der Listener wird benötigt, um die Kommunikation zwischen den Oracle Clients und dem Oracle Server herzustellen.

Im Start-Menü von Windows sollte nun bei den Programmen ein neuer Ordner für das Oracle Home hinzugefügt worden seien. Dort starten wir nun den Net Configuration Assistant. Er kann auch gestartet werden, indem man in einer DOS-Konsole den Befehl "netca" eingibt.

Der Net Configuration Assistent wird gestartet. Hier wählen wir zunächst die Listener Konfiguration und klicken auf "Next".

Wählen Sie hier "Add" aus und klicken Sie auf "Next".

Im nächsten Schritt wird der Name des Listeners eingetragen. Hier kann der Default-Name "Listener" stehen bleiben.

Bei den Protokollen kann ebenfalls die Default-Einstellung "TCP" übernommen werden.

Ganz wichtig ist nun die Vergabe eines sinnvollen Ports für die Kommunikation. Dieser Port muss auch hinterher in Firewall freigeschaltet sein. Der Standard-Port ist hier die 1521, er kann jedoch hier geändert werden. In dieser Beispielinstallation belassen wir diesen Wert jedoch und klicken auf "Next". Mit "Next" kommen wir in das Hauptmenü des Assistenten zurück.

In den Systemdiensten finden wir nun auch den Listener, der schon erfolgreich gestartet wurde.

Als nächstes müssen wir die Methoden benennen, mit denen man sich an den Instanzen anmelden kann. Dazu wählen wir den Punkt "Naming Methods configuration" und klicken dann auf "Next".

Hier sollte man die Methoden "Local Naming" für TNSNAMES und "Easy Connect Naming" auswählen.

Damit ist die Konfiguration des Listeners abgeschlossen. Kontrollieren lässt sich dies zusätzlich in einer DOS-Box mit dem Kommando: lsnrctl status
