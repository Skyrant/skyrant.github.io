Einrichtung einer Datenbankinstanz "LEGATO"

In diesem Schritt wird eine neue Datenbankinstanz für LEGATO eingerichtet. Weitere Hinweise hierzu findet man auch in unserer Legato-Installationsdokumentation.

Um die Datenbankinstanz zu erstellen, brauchen wir den Database Configuration Assistenten. Dieser kann im Windows Start-Menü unter Oracle … / Configuration .. / Database Configuration Assistant gestartet werden oder auch in einer DOS-Konsole mit dem Befehl: dbca

Im Datenbank Assistenten wählen wir nun "Create Database" und klicken auf "Next".

Schritt 2: Der gewünschte "Creation Mode" wird abgefragt. Für Legato sind einige spezielle Einstellungen notwendig. Deshalb wählen wir den "Advanced Mode" und klicken "Next".

Schritt 3: Als Vorlage (template) wählen wir nun eine "Custom Database".

Schritt 4 (Globalen Datenbanknamen wählen): Man kann nun einen Datenbanknamen vergeben. Im Beispiel wählen wir den Namen "LEGATO" und klicken auf "Next". Die neue Datenbank wird nicht als Container Database erstellt.

Schritt 5: Entfernen Sie den Haken in der Checkbox "Configure Enterprise Manager (EM) Database Express".

Schritt 6: Hier werden nun die wichtigsten Passwörter der Systembenutzer vergeben. Es gibt die Möglichkeit, für alle Systembenutzer unterschiedliche Passwörter zu verwenden oder das gleiche. Auf jeden Fall sollten die Passwörter sicher gewählt und auch sicher abgelegt werden.

Oracle fordert auch wieder das Passwort des Oracle Home Users (hier: User "oracle") an.

Wenn dem Assistenten die gewählten Passwörter nicht sicher genug erscheinen, erfolgt der oben gezeigte Warnhinweis unten auf dem Screen.

Man kann nun noch einmal die Passwörter korrigieren oder diesen Warnhinweis ignorieren.. In jedem Fall geht es mit "Next" weiter.

Neben den angezeigten Administrations-Usern SYS und SYSTEM gibt es noch einige weitere. Alle anderen Accounts sind aber per Default gesperrt (LOCKED) und nicht zugänglich.

Schritt 7: Hier wird abgefragt, über welchen Listener die neue Datenbank verbunden werden soll. Wir legen keinen neuen Listener an, sondern wählen den schon laufenden und klicken "Next".

 

Schritt 8 (Auswahl der Speicherorte): Nun kann man eine geeignete Lokation für die Datenbankdateien auswählen. Wenn möglich, sollte diese auf einer separaten Partition liegen, also nicht in der Systempartition. Das Verzeichnis darf auch auf RAID-Systemen abgelegt werden. Dateien, für die das Ablegen auf RAID-Systemen nicht zu empfehlen ist, werden in einem späteren Abschnitt gesondert behandelt. In unserem Beispiel haben wir das Verzeichnis "{ORACLE_BASE}\oradata" (entspricht "c:\oracle\oradata") gewählt. Unterhalb von "{ORACLE_BASE}" (c:\oracle) wird auch die Fast Recovery Area mit einer Größe von 20.000 MByte angelegt.

:

Wählen Sie eine ausreichende Größe, da bei Engpässen die Datenbank keine weiteren Transaktionen durchführen kann. Die Datenbank ist damit nicht mehr verfügbar!

Welche Größe im Einzelfall für den Fast Recovery Bereich anzugeben ist, ist vom Datenbank-Administrator festzulegen.

Mit der zweiten Auswahl in dieser Maske kann man bestimmen, ob überhaupt archivierte REDO-Logdateien angelegt werden oder nicht. Mit diesen Dateien ist auch innerhalb von Backups eine sehr gute Wiederherstellung gewährleistet. Dies sollte man jedoch nur für sensible Produktionsumgebungen wählen. Für reine Testumgebungen kann dies auch ausgeschaltet bleiben.

:

Bei der Installation und dem Import deaktivieren Sie das Archivieren. Sonst läuft der Speicher zu schnell voll. Aktivieren Sie für sensible Produktionsumgebungen das Archivieren, nachdem die Installation abgeschlossen ist (siehe Kapitel "Archiving aktivieren").

Klicken Sie auf "Weiter".

Schritt 9: Auf dieser Seite lassen sich nun die Komponenten auswählen: Wir lassen den Haken bei Oracle JVM, die übrigen können entfernt werden. Im zweiten Reiter (Sample Schema) muss kein Beispiel-Schema angelegt werden.

Schritt 10 (Speicher): Als nächstes wird die Größe des physikalischen Hauptspeichers bestimmt, die für diese Datenbank-Instanz reserviert werden soll. Hier muss man bedenken, dass dem Betriebssystem und anderen Applikationen bzw. auch anderen Datenbank-Instanzen genug Platz gelassen werden sollte (also hier zum Beispiel keine Werte einrichten, die mehr als 80% des gesamten Speichers bedeuten würde).

Es empfiehlt sich jedoch, für jede Instanz mindestens 4 GB zu reservieren. In unserem Beispiel geht dies leider nicht, weil auf dem Testsystem insgesamt nur 4 GB RAM vorhanden sind.

Mit der Einstellung "Use Automatic Memory Management" kümmert sich Oracle selbst darum, wie der Speicher dann intern in der Instanz aufgeteilt wird. Es wird empfohlen, diese Einstellung zu setzen.

Danach bitte auf den Reiter "Character Sets" klicken, um die Zeichensätze auszuwählen.

Schritt 10 (Zeichensätze): Bitte wählen Sie hier den Zeichensatz UTF8 aus. Dazu muss man zunächst auf die manuelle Auswahl "Choose from the list of character sets" klicken und dann noch die Option "Show recommended character sets only" ausschalten, denn sonst findet man den UTF8-Zeichensatz nicht in der Liste.

Schritt 10 (Größenanpassungen): Wählen Sie bitte die Standard-Blockgröße von 8192 Bytes und ca. 200 bis 300 Prozesse auf dieser Seite.

 

Schritt 10 (Verbindungs-Modus): Hier kann die Default-Einstellung übernommen werden.

Öffnen Sie anschließend den Dialog mit den INI-Parametern, indem Sie auf den Button "All Initialization Parameters" klicken. Aktivieren Sie außerdem die Anzeige aller Parameter durch Klick auf den Button "Show Advanced Parameters".

Schritt 10 (Initialisierungs-Parameter): Ändern Sie nun direkt die Oracle Initialisierungs-Parameter, wie zum Beispiel den Parameter: nls_length_semantics auf CHAR. Bitte beachten Sie hierzu die aktuellen Hinweise in der Dokumentation Systeminstallation.

Bitte zusätzlich beachten, dass versteckte Parameter mit _... hier noch nicht eingetragen werden können. Dies kann erst nach Erstellung der Instanz erfolgen.

Schließen Sie den Dialog über "Close" und klicken Sie dann auf "Next".

Schritt 11: Auf diesem Screen wird zunächst die Dateiablage konfiguriert. Durch Drücken des Buttons "Customize Storage Locations" öffnet sich Maske zum Konfigurieren der Datenablage:

Schritt 11: Nachfolgend werden die Angaben für die Tablespaces und Datenbankdateien vorgenommen. Nach Änderungen muss vor dem Verlassen jeder Untermaske der Button "Apply" gedrückt werden, damit die Änderungen angenommen werden.

Schritt 11 (Controlfiles): Damit sind die Steuerdateien der Instanz gemeint. Diese Dateien sind sehr wichtig, denn wenn diese zerstört sind, wird es schwierig sein die Instanz zu starten, wenn nicht gar unmöglich. Hiervon sollten also mehrere Kopien eingetragen werden. Eine Kopie erzeugt man automatisch, indem man hier eine weitere Zeile hinzufügt. In diesem Beispiel sind es drei Kopien. Sicherheitshalber sollten diese sogar auf unterschiedlichen Festplatten oder RAID-Systemen liegen.

Bitte ergänzen Sie auch den Namen der Instanz vor dem Dateinnamen (also LEGATO).

Schritt 11 (Tablespaces): In der Liste der Tablespaces sollten diese fünf erscheinen. Bitte klicken Sie auf "Add", um einen weiteren Tablespace zu definieren. Für weitere Informationen zu den Tablespaces siehe bitte auch die Dokumentation Systeminstallation.

Danach erscheint eine neue Maske zum Anlegen des neuen Tablespaces. Bitte tragen Sie als Namen "LEGATO_DAT" und beim Filenamen "legato_dat.dbf" ein und ändern Sie die Anfangsgröße auf 1000 MB. Klicken Sie auf "Apply", anschließend im Tablespaces-Baum auf LEGATO_DAT und im Fenster "Datafiles" auf den Dateinamen. Ein Klick auf das Stiftsymbol ("Edit Tablespace") öffnet die nachfolgende Maske:

Bitte wechseln Sie hier auf den Reiter "Storage" und aktivieren Sie die Option "Automatically extend datafile when full (AUTOEXTEND)".

Damit wird ein automatisches Anwachsen des Tablespaces aktiviert. Wählen Sie als Erweiterung immer mindestens 64 Megabytes.

Zusätzlich wird hier die Maximalgröße definiert, indem man einen "Maximum Size Value" einträgt. In diesem Beispiel sind dies 20.000 Megabytes.

Das heißt, dieser Tablespace kann zwar wachsen, aber nur bis 20.000 Megabytes.

Klicken Sie anschließend OK und und auf der Maske "Customize Storage" sofort wieder "Apply", um die Eingaben zu sichern.

Wiederholen Sie diese Schritte analog für den Tablespace LEGATO_IDX.

Es sollten danach die folgenden sieben Tablespaces in der Liste erscheinen.

Bitte wechseln Sie nun zu "Datafiles" links im Baum.

Schritt 11 (Datendateien): Klicken Sie auf die Datendateien "datafiles" und ergänzen Sie manuell bei allen den Instanznamen, also in diesem Fall "legato". Die anderen Einstellungen von den System-Tablespaces sollten nicht verändert werden.

Schritt 10 (Redo-Log-Gruppen): Bitte erstellen Sie nun mindestens fünf Redo-Loggruppen mit jeweils zwei Mitgliedern. Ändern Sie die Dateinamen auf "legato_redo01a.rdo". Dies ist sicherer als diese bei ".log" zu belassen, da *.log Dateien gern gelöscht werden.

Bitte beachten Sie, dass alle Log-Gruppen die gleiche Größe haben müssen!

Außerdem wird empfohlen, die Kopie der Logdateien immer auf unterschiedlichen Laufwerken zu speichern.

Wenn Sie alle Einstellungen vorgenommen haben, klicken Sie auf "OK", um zur Screen "Creation Options" zurück zu kommen.

Schritt 11: Hier sollte man immer die Optionen "Save as a Database Template" und "Generate Database CreationScripts" einschalten, damit man ein Template für spätere Verwendung hat und die Skripte für Fehlerfälle. Somit kann man später gegebenenfalls fehlerhafte Installationen noch einmal stückweise nachkorrigieren.

Klicken Sie dann bitte auf "Next". Die Pre Requisite Checks zur Prüfung der Installationsvoraussetzungen werden gestartet:

Im Fehlerfall können Warnungen (nicht-fatale Fehler, hier durch ein Hinweisschild symbolisiert) durch Klicken von "Ignore All" ignoriert werden. (Fehler können nicht ignoriert werden.) Besser ist die Beseitigung der Ursache für die Warnung mit anschließender Wiederholung der Überprüfung ("Check Again").

Nach einem Klick auf "Next" werden alle Einstellungen in einer Zusammenfassung im HTML-Format angezeigt:

Zusammenfassung: Kontrollieren Sie anschließend noch einmal sehr sorgfältig die Zusammenfassung und drücken Sie erst dann "Finish", wenn Sie keine Fehler mehr entdecken konnten. Anschließend beginnt die Einrichtung der Datenbank-Instanz.

Es folgen dann zwei Erfolgsmeldungen für die Erstellung des Templates und der Skripte, die man mit "OK" bestätigt.

Anschließend folgt die Maske mit dem Fortschrittsbalken der Instanz-Erstellung.

Nach der Installation sollte dann obige Erfolgsmeldung erscheinen. Mit Klick auf "Exit" und "Close" wird die Datenbankerstellung abgeschlossen.
