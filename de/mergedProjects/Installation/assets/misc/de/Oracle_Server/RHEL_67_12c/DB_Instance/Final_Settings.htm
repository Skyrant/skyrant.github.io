Abschließende Einstellungen

Parameter anpassen

Nun müssen noch einige Parameter überprüft und angepasst werden.

Öffnen Sie ein Terminal und verbinden sich als Systemadministrator mit der neuen Datenbank-Instanz. In unserem Fall lautet der Befehl hierfür:

sqlplus sys/system@LEGATO as sysdba

 

Anschließend kann man sehen, ob die Instanz hochgefahren wurde. Mit select * from dual; kann man auch einen Test ausführen.

Nun können die Oracle-Initparameter überprüft werden, zum Beispiel mit dem Aufruf: show parameter nls_length_semantics. Bitte richten Sie sich immer nach den aktuellen Hinweisen hierzu in unserer Legato-Systeminstallation.

An dieser Stelle können wir nun auch den versteckten Parameter b_tree_bitmap_plans setzen. Wichtig hierbei ist, dass bei versteckten Parametern immer die doppelten Anführungszeichen benutzt werden müssen. Der komplette Befehl lautet dann:

alter system set "_b_tree_bitmap_plans" = false scope = both;

 

An dieser Stelle muss auch noch der Parameter OPTIMIZER_FEATURES_ENABLE auf 11.2.0.1 gesetzt werden.

:

Wird dieser Wert nicht auf 11.2.0.1 geändert, sondern verbleibt wie voreingestellt auf 12.1.0.2, so kommt es insbesondere bei 1-DB Systemen zu massiven Performance-Einbußen!

Der Befehl zum Ändern des Parameters lautet:

alter system set optimizer_features_enable='11.2.0.1'

 

Autostart für Instanz

Die soeben erstellte Datenbank-Instanz LEGATO ist in der Datei /etc/oratab eingetragen worden. Der User oracle kann diese Datei zum Editieren öffnen:

$ vi /etc/oratab

 

Wie an dem abschließenden N (für "no") in der Zeile, die mit "LEGATO" beginnt (hier: die vorletzte Zeile), erkennbar, wird die Instanz per Default nicht zusammen mit dem Datenbanksystem gestartet. Dieses Verhalten kann geändert werden, indem das N durch ein Y (für "yes") ersetzt wird. Nach dieser letzten Änderung sind Datenbank und Instanz voll funktionsfähig. Allerdings müssten bei einem Neustart des Servers sowohl das Datenbank-System als auch der Listener "von Hand" neu gestartet werden. Nicht zwingend notwendig, aber äußerst sinnvoll ist deshalb die Erstellung einer Autostart-Funktionalität für Datenbank. Diese wird im folgenden Abschnitt beschrieben.
