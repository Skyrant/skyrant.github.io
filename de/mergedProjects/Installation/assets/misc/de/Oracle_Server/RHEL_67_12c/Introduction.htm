Über dieses Dokument

In dieser Dokumentation wird in kurzen Texterläuterungen und anhand von Screenshots beschrieben, wie man einen Oracle Server auf Red Hat Enterprise Linux (im Folgenden kurz RHEL) 6.7 installiert. Diese Installationsanleitung funktioniert identisch für die Linux-Varianten CentOS 6.7 und Oracle Linux 6.7; für Oracle Linux gibt es kleine Unterschiede, die im entsprechenden Kapitel (3.3.5) vermerkt sind. Im Großen und Ganzen ist diese Installation gleich mit der auf früheren Versionen von RHEL 6.x sowie auf Oracle Linux 6.x.

Die Beschreibung richtet sich an Datenbank-Administratoren. Grundlegende Kenntnisse im Umgang mit Datenbank-Servern sowie Kenntnis der Systemumgebung wird vorausgesetzt.

Allgemeiner Überblick

Die folgenden Systembestandteile werden in diesen aufeinander aufbauenden Schritten installiert:

1. Installation des Betriebssystems (RHEL 6.7) inklusive aller aktuellen Patches. Die Betriebssystem-Installation wird in diesem Dokument nicht beschrieben.

2.  Vorinstallation und Installation des RDBMS (Oracle12cR1) und Installation eines Listeners.

3. Installation einer (leeren) LEGATO-Instanz.

Grundvoraussetzung für die Installation ist eine ausreichend dimensionierte Hardware, in etwa wie folgt:

x86_64-Bit-Serverprozessor mit mindestens 2 GHz Taktfrequenz und mindestens zwei Cores (oder einem Core mit Hyperthreading)

4 GB RAM

30 GB Festplattenplatz (entsprechend mehr, wenn eine Fast Recovery Area verwendet werden soll)

Ethernet-Netzwerkkarte

 

Jedem Installationsschritt ist ein eigenes Kapitel gewidmet.

Als Oracle DBMS-Version wird eine Enterprise Edition der Oracle Server verwendet, die für den Betrieb von Legato empfohlen wird. Zu den genauen Systemvoraussetzungen siehe bitte das Dokument "Legato_System_Installation" im Kapitel "Datenbank".

Als Betriebssystem dient ein RHEL 6.7.
