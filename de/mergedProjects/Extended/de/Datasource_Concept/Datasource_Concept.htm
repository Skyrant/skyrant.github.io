Datasource-Konzept

Allgemein

Das Datasource-Konzept (Datenquellen-Konzept) ermöglicht es, jede Art von Abfrage schnell und unkompliziert für den Anwender bereitzustellen. Die Daten können dabei als Tabelle (super-generic-table) oder als Diagramm (super-generic-chart) dargestellt werden.

Grundsätzlich unterscheidet man zwischen gescripteten Datenquellen und Datasource Models.

Gescriptete Datenquellen

Bei gescripteten Datenquellen handelt es sich um JavaScript-Dateien, die sich mit der entsprechenden Select-Methode in gp_r_datasources registrieren. Diese Datenquellen können dann genauso wie Datenbank-Tabellen in Abfragen und Boardlets verwendet werden.

Datasource Models

Ein Datasource Model dient dazu, bestimmte Daten aus mehreren Tabellen darzustellen. Dabei können die anzuzeigenden Spalten sowie bestimmte Filter bereits vorgegeben werden. Weitere Einstellungen können dann vom Dashboard-Entwickler im Client festgelegt werden, ehe der Enduser das neue Modell in einem Boardlet zu sehen bekommt.

Datasource Model (d.h. Einträge in gp_r_datasource_models) referenzieren immer auf Einträge in gp_r_dashboards. Dabei wird das verwendete Modell als Filter bzw. Parameter gespeichert. In gp_r_dashboards können auch weitere Eigenschaften angegeben werden.
