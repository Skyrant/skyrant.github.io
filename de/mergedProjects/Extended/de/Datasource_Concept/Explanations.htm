Erläuterungen

Actions

Hier können Aktionen referenziert werden, die für das Datasource-Model zur Verfügung stehen sollen (im Beispiel die Aktion "archive-alarms"). Dieser Aktion werden Parameter übergeben. Die genaue Beschreibung der Aktionen ist in JavaScript-Dateien ausgelagert, die unter folgendem Pfad gespeichert sind: "\sapient-app\lib\common\app\datasource-actions\archive-alarms.js".

Resources

Hier können weitere Ressourcen referenziert werden, beispielsweise gescriptete Datenquellen.

Required Fields

Required Fields werden IMMER angefordert (abgefragt), sind aber nicht zwingend sichtbar. Es können aber andere Aktionen von ihnen abhängen.

Default Fields

Default-Felder werden standardmäßig angefordert und angezeigt, da ihr Inhalt für die Anzeige relevant ist. Sie können aber über die Feldeinstellungen im Boardlet ausgeblendet werden.

Blacklisted Fields

Blacklisted Fields werden NIE in der WebApp angezeigt, da sie Informationen enthalten, die für den Benutzer nicht relevant sind. Durch diese Vorgehensweise ist sichergestellt, dass Felder, die später zur Tabelle hinzukommen, automatisch auch beim Datasource Model ergänzt werden und nicht manuell hinzugefügt werden müssen.

Static Filter

Hier können statische Filter festgelegt werden, die vom Endanwender in der WebApp nicht verändert werden können. So kann die Ansicht auf bestimmte Einträge eingeschränkt werden (im Beispiel auf Meldungen mit "isPending" = false).

Filter

Hier werden die Standardfilter des Datasource-Models definiert. Für diese Filter muss in der WebApp ein Wert eingegeben werden, ehe die Abfrage erfolgreich gestartet werden kann.

Weitere Filter auf jede beliebige Spalte der Datenquelle können in der WebApp festgelegt werden.
