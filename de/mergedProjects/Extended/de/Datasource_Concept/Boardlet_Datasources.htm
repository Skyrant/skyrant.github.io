Boardlet Datasources

In der WebApp wird das Datasource-Konzept mithilfe des Boardlets "Super-Generic-Table" umgesetzt. Für dieses Boardlet steht ein zusätzlicher Reiter im Bearbeiten-Modus zur Verfügung.

Reiter Datasources

Der Reiter ist in mehrere Abschnitte gegliedert:

Allgemein

Ganz oben kann aus einer Drop-down-Liste das Datasource Model selektiert werden, das verwendet werden soll.

Über den Schieberegler "Aktualisieren" kann festgelegt werden, ob das Boardlet in einem regelmäßigen Intervall aktualisiert werden soll oder nicht. Dies ist z.B. bei anstehenden Meldungen sinnvoll, um immer die aktuellsten Daten anzuzeigen.

Über "Sortieren" wird festgelegt, nach welcher Spalte die angezeigten Daten defaultmäßig sortiert werden.

Standardfilter

Hier werden die Standardfilter des Boardlets angezeigt. Diese sind im Code des Datasource Models unter "filter" zu finden. Jeder dieser Standardfilter muss mit einem Parameter verknüpft werden, über den schließlich ein zu filternder Wert gesetzt wird.

Felder

Hier werden alle Felder angezeigt, die von der Datenquelle abgefragt werden können. Die im Code der Datasource als "Required" festgelegten Einträge sind entsprechend markiert. Diese können nicht deaktiviert werden.

Für alle anderen Felder kann über den Schieberegler rechts festgelegt werden, ob die Daten von der DB abgefragt werden sollen oder nicht.

INFO:

Dies hat KEINEN Einfluss darauf, ob das Feld im Boardlet angezeigt wird. Es geht lediglich darum, ob die Daten abgefragt werden.

Durch Klick auf das + in einer Zeile wird der Eintrag aufgeklappt und bietet nun die Möglichkeit, dieses Feld mit einem zuvor angelegten Parameter zu verknüpfen. Links werden die möglichen Operatoren in einer Drop-Down-Liste angezeigt (abhängig davon, ob der Schieberegler "Array" an oder aus ist, unterscheiden sich die Einträge). Rechts werden dann abhängig vom gewählten Operator die zur Verfügung stehenden Boardlet-Parameter angezeigt. Zum genauen Vorgehen beim Anlegen eines neuen Parameters, siehe bitte Reiter Filter / Parameter.

Aktionen

Hier können im Datasource-Model definierte Aktionen für den Endanwender an- bzw. abgewählt werden. Üblicherweise werden die Aktionen dann nach Doppelklick auf einen Eintrag in der Seitenleiste angeboten.

Über die Aktion "Output" können Detail-Ansichten definiert werden, die per Klick dann den Wert als Parameter erhalten. Hier stehen alle Felder zur Verfügung, die von der DB geholt werden. Hierzu muss im Reiter Filter / Parameter der entsprechende Eintrag beim Startboardlet auf "out" und beim Zielboardlet auf "in" gesetzt werden.

Anwendungsbeispiel: Master – Detail Dashboard: Selektion einer Zeile in der Master-Tabelle setzt automatisch einen Filter in einer Detailtabelle, um zugehörige Datensätze anzuzeigen.

Reiter Filter / Parameter

Hier werden zum einen die im Datasource-Model definierten Filter angezeigt (üblicherweise mindestens Knoten und Zeit).

Durch Klick auf "Parameter hinzufügen" können weitere Parameter oder Filter für dieses Boardlet festgelegt werden. Über den Reiter Datasources – Fields kann der neue Parameter dann mit einem bestimmten Datenbankfeld verknüpft werden. Alternativ ist es natürlich auch möglich, einfach einen bestimmten Wert an ein anderes Boardlet im Dashboard zu übergeben, das einen Parameter mit dem gleichen Namen enthält, ohne eine Referenz auf die Datenbank.

Hier kann ein frei wählbarer Name für den Parameter vergeben werden.

Aus der Dropdown-Liste muss nun die Art des Parameters gewählt werden. Zur Verfügung stehen alle im Datasource-Model definierten Filter sowie:

NumberArray

Ermöglicht die Eingabe mehrerer Zahlen kommagetrennt als Array. Kann z.B. mit der ID-Spalte einer Tabelle verknüpft werden.

StringArray

Ermöglicht die Eingabe mehrerer Strings kommagetrennt als Array. Kann z.B. mit einer Text-Spalte einer Tabelle verknüpft werden.

LookupTable

Ermöglicht eine Mehrfachauswahl aus einer Tabelle. Dazu muss in einem dritten Feld der Name der Tabelle und die zu holenden Spaltennamen angegeben werden, z.B. al_classes, id, name.

Number

Ermöglicht die Eingabe einer Zahl. Kann z.B. mit der ID-Spalte einer Tabelle verknüpft werden.

String

Ermöglicht die Eingabe eines Strings. Kann z.B. mit einer Text-Spalte einer Tabelle verknüpft werden.

Boolean

Ermöglicht das Umschalten eines Schiebereglers zwischen true und false.

INFO:

Für Datenbank-Einträge wird Boolean aktuell nicht unterstützt. Hier muss immer Number verwendet werden (für 1 oder 0).

 

Die neuen Parameter können auf den gewünschten Parametertyp (local, in, out, in/out) gesetzt oder gelöscht werden.

INFO:

Vergessen Sie nicht, den neuen Parameter mit einem Feld zu verknüpfen!

Reiter Tabelleneinstellungen

In diesem Reiter können für alle Spalten, die von der Datenbank geholt werden, Einstellungen hinsichtlich ihres Verhaltens getroffen werden.

Zunächst wird die entsprechende Spalte selektiert. Darunter kann ein individueller Name eingegeben werden, der als Spaltenüberschrift verwendet werden soll.

Die folgenden Schieberegler bestimmen das Verhalten der Spalte in der Tabelle. Sie kann sortierbar, größenveränderbar, verschiebbar und sichtbar sein. Außerdem kann eine bestimmte Breite der Spalte festgelegt werden. Ist der Schieberegler Komponente aktiv, kann die Art der Spalte selektiert werden, z.B. Zeitangabe. Darunter kann die gewünschte Formatierung als JSON-String eingegeben werden. Grüne Häkchen am Ende der Zeilen zeigen an, ob der Inhalt sinnvoll ist.

Für weitere Details, siehe bitte Tabellen-Einstellungen.
