Funktion

Durch die Zuordnung von Meldeklassen wird die Darstellung und Verarbeitung einer Meldung bestimmt. Hier können die Meldeklassen bearbeitet und neue Klassen hinzugefügt werden.

Feldbeschreibung

 

Feld

Eigenschaft

ID

Eindeutige Identifikationsnummer der Klasse.

Siehe auch weiter unten "Reservierte System-Meldeklassen".

Name

Frei wählbarer Name der Meldeklasse.

Beschreibung (optional)

Zusätzliche Beschreibung der Meldeklasse.

Quittierart

Lookup-Liste zur Auswahl einer Quittierart für die Meldeklasse. Die Quittierart legt fest, unter welchen Voraussetzungen die Meldung abgeschlossen wird. Folgende Quittierarten sind standardmäßig vorhanden:

·         Quittiert & Protokoll: die Meldung wird mit der manuellen Quittierung in der Übersicht im WebClient abgeschlossen. Danach wird die Meldung im Archiv angezeigt.

·         Protokoll: die Meldung erscheint nie in der Aktuellansicht, sondern wird automatisch abgeschlossen und im Archiv angezeigt. Die Melde-Dauer ist 0.

·         Gehend: die Meldung wird in der Übersicht angezeigt. Mit dem Gehend-Telegramm wird sie abgeschlossen und ins Archiv verschoben.

·         Quittiert: die Meldung wird mit der manuellen Quittierung im WebClient abgeschlossen und ins Archiv geschoben.

·         Quittiert & Gehend: die Meldung muss im WebClient quittiert (abgeschlossen) werden und das Gateway muss ein Gehend-Ereignis übermitteln. Die Reihenfolge der beiden Ereignisse ist beliebig. Danach wird die Meldung im Archiv angezeigt.

·         Quittiert nach Gehend: für die Meldung muss vom Gateway ein Gehend-Ereignis übermittelt werden. Die zusätzliche manuelle Quittierung durch den Anwender im WebClient ist erst anschließend möglich. Bis diese erfolgt ist, bleibt die Meldung in der Übersicht. Nach der manuellen Quittierung wird die Meldung abgeschlossen und ins Archiv verschoben.

·         Quittierung optional: Meldung kann quittiert werden. Quittierung ist aber nicht zwingend erforderlich. Die Meldung wird mit dem Gehend-Telegramm ins Archiv verschoben und kann ggf. dort quittiert werden.

Priorität (optional)

Priorität der Meldeklasse. Wird hier ein Eintrag vorgenommen, so wird dieser unter "Funkruf" bei der Fertigungsstruktur als "Vererbte Priorität" angezeigt und kann dort knotenspezifisch überschrieben werden. Die Priorität beeinflusst zudem, ob diese Meldung unterdrückt wird, wenn in der Aktuellansicht der Filter auf "Erstwert nach Priorität" gesetzt ist. Je niedriger die Zahl, desto höher die Priorität (d.h. 1 = höchste Prio). Wird nichts eingetragen, wird von niedrigster Prio ausgegangen. Außerdem wird die Prio der Meldeklasse für Berechnete Werte verwendet.

Die Priorität der Meldung beeinflusst auch, wie der Knoten im Layout-Boardlet eingefärbt wird, an dem die Meldung ansteht. Für Prio 1-9 wird die im Theme definierte Farbe für "error" verwendet, von 10-19 die Farbe für "warning", >= 20 die Farbe für "info". Bei 0 oder NULL erfolgt keine Einfärbung des Knotens im Layout.

Vererbungsrichtung

Lookup-Liste zur Auswahl der Richtung, in der Meldungen in der Fertigungsstruktur vererbt werden (Meldungen eines Knotens auch für über- oder untergeordnete Knoten anzeigen).

Hierarchiestufe (optional)

Lookup-Liste zur Auswahl einer Hierarchiestufe, bis zu der eine Vererbung von Meldungen erfolgt (um z.B. Meldungen nicht komplett bis zur Wurzel zu vererben, sondern nur bis zu einer bestimmten Knotenebene).

Kommentar erf. (optional)

Checkbox. Wenn aktiv, wird der Bediener beim Quittieren der Meldung im WebClient aufgefordert, einen zusätzlichen Kommentar zur Meldungsursache einzugeben. Der Kommentar wird zusammen mit der Meldung im Meldearchiv gespeichert (nur sinnvoll bei quittierpflichtigen Meldungen).

Funkruf-Art (optional)

Lookup-Liste zur Auswahl einer Weiterleitungsart für den Funkruf oder eine andere, wie Funkrufsysteme parametrierte weiterführende Behandlung der Meldungen.

Folgende Arten sind standardmäßig wählbar:

·         Keine Weiterleitung: Meldungen werden nicht an unter Funkruf-Empfänger parametrierte Module weitergeleitet.

·         Erstwerterkennung: nur die Erstwert-Meldung wird an einen Funkruf-Empfänger weitergeleitet.

·         generelle Weiterleitung: alle Meldungen dieser Klasse werden an einen Funkruf-Empfänger weitergeleitet.

·         Erstwertverarb. innerhalb der Klasse: nur die Erstwert-Meldung innerhalb der Klasse wird an einen Funkruf-Empfänger weitergeleitet.

·         Erstwertverarbeitung über Priorität: nur die Erstwert-Meldung der Priorität wird an einen Funkruf-Empfänger weitergeleitet.

·         Sofortige Weiterleitung: Meldungen werden sofort an einen Funkruf-Empfänger weitergeleitet.

 

Unter Anwendungsstruktur | Datenaustausch | Datenaustausch-Module | Verbindungen | Meldungsdefinitionen können die einzelnen Meldungen bei Bedarf mit einer anderen Parametrierung überschrieben werden.

KPI-relevant

Checkbox. Wenn aktiv, werden Meldungen dieser Klasse bei der KPI-Berechnung berücksichtigt. Der Verfügbarkeitstyp der Meldung wird über die Meldeart festgelegt.

Diese Spalte wird nur angezeigt, wenn der Systemparameter -43 = 1 ist.

Verfügbarkeits-Typ

Lookup-Liste zur Auswahl eines Verfügbarkeits-Typs. Gibt an, wie Meldungen dieser Klasse bei der KPI-Berechnung berücksichtigt werden.

Diese Spalte wird nur angezeigt, wenn der Systemparameter -43 = 0 ist.

CSS Style

CSS Style zur Formatierung von Meldungen. Für jeden Status einer Meldung kann eine Schriftart, eine Farbe, ein Symbol etc. definiert werden. Alle verfügbaren States, Styles und Icons sind unter "Formatierung" beschrieben.

 

 

Reservierte System-Meldeklassen

Meldeklassen mit negativer ID sind vom System reserviert und dürfen nicht verändert werden. Bei einer Standardinstallation werden auch die IDs 0, 1 und 2 mit ausgeliefert, können aber auch projektspezifisch geändert werden. In diesem Fall ist bei Updates sicherzustellen, dass die projektspezifischen Änderungen nicht ungewollt durch den Standard überschrieben werden.

Negative Meldeklassen-IDs korrespondieren mit den Verbindungs-IDs der Datenquelle System, der die Meldungen beim Auftreten zugeordnet werden.

ID = 0 ist für die Klasse unbekannte Meldung reserviert und darf nicht verändert werden. Diese Klasse wird Meldungen zugeordnet, deren Meldungsnummer nicht in den Meldungsdefinitionen vorhanden ist. Die Meldung wird automatisch als nicht parametriert angelegt
