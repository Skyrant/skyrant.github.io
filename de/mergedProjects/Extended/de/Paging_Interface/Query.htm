Funkrufe mittels Polling abfragen

Es wurde eine neue Java Datenquelle "ds_m_sendlist" implementiert, die die relevanten Informationen über einen Funkruf liefert, der an ein Paging-System weitergeleitet werden soll. Diese Datenquelle kann mit Hilfe der JSON-RPC Prozedur "select" abgefragt werden.

Die Datenquelle liefert die folgenden Felder:

m_id - M_ARCHIVE.ID

m_alarm_id - M_ARCHIVE.ALARM_ID

m_break_id - M_ARCHIVE.BREAK_ID

m_manual - M_ARCHIVE.MANUAL

m_messaging_state - M_ARCHIVE.MESSAGING_STATE (-1 oder -4, Zeilen mit anderen Werten werden herausgefiltert)

m_node - M_ARCHIVE.NODE

m_prod_group - M_ARCHIVE.PROD_GROUP

m_shift_group - M_ARCHIVE.SHIFT_GROUP

m_shift_id - M_ARCHIVE.SHIFT_ID

m_text - M_ARCHIVE.TEXT

m_time_end - M_ARCHIVE.TIME_END

m_time_start - M_ARCHIVE.TIME_START

r_rec_code - M_RECIPIENTS.REC_CODE

r_server_addr - M_RECIPIENTS.SERVER_ADDR

a_al_class - AL_ARCHIVE.AL_CLASS

a_al_type - AL_ARCHIVE.AL_TYPE

a_operand - AL_ARCHIVE.OPERAND

a_symbol - AL_ARCHIVE.SYMBOL

c_connection_symbol - P_CONNECTIONS.CONNECTION_SYMBOL

c_name - P_CONNECTIONS.NAME

n_description - L_NODES.DESCRIPTION

n_name - L_NODES.NAME

n_short_name - L_NODES.SHORT_NAME

g_group_symbol - T_SHIFT_GROUPS.GROUP_SYMBOL

p_name - T_PGROUPS.NAME

s_shift_type - T_SHIFTS.SHIFT_TYPE

b_break_type - T_BREAKS.BREAK_TYPE

t_model_pattern_symbol - T_MODEL_PATTERN.MODEL_PATTERN_SYMBOL

 

Clients können (und sollten) eine eingeschränkte Anzahl Felder für jeden Datensatz empfangen, indem sie eine Feldliste ("fields") mit ihrer Anfrage übergeben. Diese Datenquelle akzeptiert Strings mit einem oder mehreren Komma-getrennten Feldnamen oder einen Array solcher Strings als "fields"-Parameter.

Die Datenquelle unterstützt Paginierung durch Verwendung der Parameter "limit" und "offset".

Die aktuelle Implementierung hat folgende Einschränkungen:

Filter auf M_RECIPIENTS.SERVER_ADDR wird nicht unterstützt

Mehrsprachige Felder (wie Meldetext und die verschiedenen Namen und Beschreibungen) werden wie in allgemeinen Datenquellen behandelt, d.h., nur der Wert in der Sprachumgebung des Abfragenden wird zurückgegeben. Für Funkruf-Systeme, die Lokalisierung intern verarbeiten, müssen möglicherweise alle Übersetzungen übergeben werden.
