Einrichten eines Andon-Projekts

Für das Einrichten eines Andon-Projekts sind 6 Schritte zu beachten.

 

  1. Andon-Projekt erstellen (ähnlich wie ein HMI-Projekt)

  2. Projekt und Andon-Anzeigen/Clients registrieren

  3. Individuelle Zeichnungen für jede(n) Andon-Anzeige/Andon-Client einrichten

  4. An jeder Station den ID-Server einrichten

  5. Systemparameter für die Identifizierung der Station einrichten

  6. Voraussetzungen: Anzeige des Verbindungsstatus jedes Boards

1. Erstellen eines Andon-Projekts

Die Struktur eines Andon-Projekts ist der eines HMI-Projekts grundsätzlich sehr ähnlich. Es besteht aus einem Projektordner der die Projektdatei (andon.prj) und normalerweise eine Zeichnung pro Bereich enthält. Die Informationen zu jedem Bereich müssen auf einem oder mehreren Andon-Boards angezeigt werden. Es ist nicht nötig, einzelne Projekte für jedes Board oder jeden Bereich zu erstellen. Aus verwaltungstechnischen Gründen wird jedoch empfohlen, nur ein Andon-Projekt zu erstellen. Dieses Andon-Projekt umfasst für jeden Bereich mehrere Kunden, die eine Andon-Lösung nutzen (z. B. assembly, body shop …).

Beachten Sie, dass sich der Projektordner im Andon-Ordner im freigegebenen Verzeichnis befinden muss:

 

Struktur eines Andon-Projekts:

{

  "defaultLogin": {

    "userName": "andon_default",

    "password": "andon_default"

  },

  "parameters": {

    "debugMode": true

  },

  "theme": "sapient-dark",

  "header": {

    "hide": true

  },

  "boardlets": [

    {

      "component": "drawing-canvas-boardlet",

      "parameters": {

        "fileName": {

          "value": "andon/assembly/default.xml"

        },

        "showHeader": {

          "value": false

        }

      }

    }

  ],

  "footer": {

    "fileName": "andon/assembly/footer.xml",

    "height": "60px"

  },

  "navigation": {

    "items": []

  }

}

 

  • defaultLogin: Gibt den Nutzer an, der nach dem Hochfahren eingeloggt wird. Wenn der aktuelle Browser bereits eine Session mit einem aktiven Nutzer hat, wird dieser aktive Nutzer für den Login verwendet.

  • paramsters.debugMode: Stellt den Debug-Modus für das gesamte Projekt ein. Hierdurch wird mit gedrückter STRG-Taste und einem Klick an einer beliebigen Stelle in einer Visualisierungszeichnung die Debug-Seitenleiste geöffnet.

  • theme: Ermöglicht das Einrichten eines bestimmten Themes, das während der Laufzeit verwendet werden kann. Hierdurch wird das Theme ersetzt, welches in den momentanen Nutzerpräferenzen eingestellt ist. Diese Einstellung wird empfohlen, da die meisten Visualisierungszeichnungen für ein bestimmtes Thema optimiert sind.

  • header.hide: Ermöglicht das Ausblenden der Kopfzeile. Hierdurch wird die Visualisierung in voller Bildschirmgröße angezeigt.

  • boardlets: Richtet das/die Boardlet(s) so ein, dass sie ab Beginn angezeigt werden. Mithilfe des drawing-canvas-boardlet kann eine Visualisierung während der Laufzeit angezeigt werden. Beachten Sie, dass der Dateiname der wichtigste Teil dieser Komponenten ist. Die Kopfzeile des Boardlets wird in unserem Fall nicht benötigt. Die Kopfzeile wird mithilfe des showHeader-Parameters ausgeblendet.

  • footer: Optionale Fußzeilen-Visualisierung, die durchgehend angezeigt wird. Dies ist unabhängig vom Inhalt des Hauptbereichs und kann beispielsweise zur Darstellung von Datum und/oder Zeit, etc. verwendet werden. Um die Fußzeile auszublenden, muss der Dateiname zu einem leeren String geändert und die Höhe auf 0 gesetzt werden.

  • navigation: Ist in diesem Fall leer. navigation erlaubt das Festlegen einer Menüstruktur, die in der Kopfzeile angezeigt wird.

 

2. Registrieren des Projekts und der Andon-Anzeigen/Clients

Die Registrierung eines Andon-Projekts läuft über den LC2. Öffnen Sie hierzu die HMI-Projekte-Tabelle im Konfigurationsbaum (Frontend/HMI Projects):

 

Anschließend kann das Projekt im Browser geöffnet werden, indem eine Station in der URL festgelegt wird.

Geben Sie hierzu die Adresse und die Station ein:

http://***.**.**.**:****/app/andon?station=section1

(die Sternchen in der URL stellen Platzhalter dar)

 

3. Individuelle Zeichnungen für jede Anzeige einstellen

Die Projektdatei enthält den Bereich „Boardlets“, der das anfangs angezeigte Boardlet definiert und dessen Parameter definiert. In diesem Fall ist dies das Visualisierungs-Canvas-Boardlet.

Dieses Boardlet dient dem Darstellen einer Visualisierungszeichnung. Damit die gewünschte Visualisierungszeichnung für die einzelnen Stationen/Produktionsbereiche angezeigt werden, können Voreinstellungen durch Eingaben in der Spalte Projektkonfiguration überschrieben werden:

 

{

  "boardlets": [

    {

      "parameters": {

        "fileName": {

          "value": "andon/assembly/section1.xml"

        }

      }

    }

  ]

}

 

4. Einrichten des ID-Servers an jeder Station

Damit die Anwendung die Station identifizieren kann, muss der HMI-ID-Server auf jedem Stations-PC installiert werden.

Der Download und die Anleitung hierzu findet man unter https://github.com/Gefasoft/hmi-id-server/.

Rufen Sie localhost:8765/sapient/hmiterm.json auf, um zu prüfen, ob der lokale Server richtig funktioniert. Der Browser sollte ein JSON-Objekt anzeigen, welches Informationen zu der Maschine enthält.

 

5. Systemparameter für die Identifikation der Station einstellen

Um die Anwendung zu zwingen, die aktuelle Station zu identifizieren und den entsprechenden Stationsinhalt automatisch zu laden, muss ein Hostname für die einzelnen Stationen angegeben werden:

 

 

Die folgenden Systemparameter müssen angepasst werden:

 

• -330 – Wert 1: Wird benötigt, damit die WebApp den lokal installierten ID-Server nutzt.

• -320 – default 8765: Muss auf den Port eingestellt werden, der lokal installierte ID-Server überwacht.

• -310 – Wert 0: Ermöglicht es, jede Station auch direkt über die URL anzuzeigen (ID-Server kann ignoriert werden). Wenn der Wert auf 0 gesetzt ist, kann sichergestellt werden, dass ein bestimmter Client nur auf den gewünschten Host initialisiert werden kann.

Wenn die Anwendung ohne Angabe einer bestimmten Station mit der URL (http://hostname:8080/app/andon) aufgerufen wird, versucht die Applikation die Kundenstation mithilfe des Hostnamens zu bestimmen.

6. Voraussetzungen für die Anzeige des Verbindungsstatus jedes Clients

Jeder verbundene Client pingt den Server regelmäßig an. Hierbei wird die Uhrzeit des letzten Kundenfeedbacks in der hmi_client_states-Tabelle festgehalten, um den Verbindungsstatus jedes Clients anzugeben.

 

Anzeige einer Liste aller Clients (einschließlich ihres Verbindungsstatus in der WebApp)

Nutzen Sie den LC2, um ein Datenquellenmodell hinzuzufügen. Dies legt fest, wie diese Informationen abgefragt und wie auf die Anzeige einer Liste aller Clients in der WebApp zugegriffen werden kann. Außerdem wird hierdurch festgelegt, wie man die Informationen nutzen kann. Auf die Datenquellenmodelle kann über den Konfigurationsbaum unter Systemverwaltung und Datenquellenmodelle zugegriffen werden.

 

Fügen Sie ein neues Modell, z. B. „Client states“ hinzu und geben Sie die Anfragendefinition in das Feld Konfiguration ein:

 

{

  "resources": [

    { "resource": "hmi_clients", "as": "hmic" },

    { "resource": "hmi_client_states", "as": "hmics" }

  ],

  "query": {

    "fields": ["hmic.symbol", "hmic.name", "hmic.host_name", "hmics.last_sync"],

    "relations": [

      {

        "src": "hmic",

        "dst": "hmics",

        "how": "i",

        "on": { "left": "hmic.id", "op": "EQ", "right": "hmics.id" }

      }

    ]

  },

  "defaultFields": ["symbol", "name", "host_name", "last_sync"],

  "staticFilter": [

    {

      "left": "hmic.host_name",

      "op": "NN"

    }

  ]

}

 

Diese Definition kann je nach spezifischen Kundenanforderungen variieren.

 

Um das Ergebnis in der WebApp anzuzeigen, fügen Sie ein generisches Tabellenboardlet zu einem Dashboard hinzu. Wechseln Sie zum Tab Datenquellen Einstellungen in der Seitenleiste und wählen Sie anschließend das oben erwähnte Datenquellenmodell aus.

Damit sich die Tabelle automatisch aktualisiert, setzen Sie den Parameter „Aktualisieren“ und geben Sie ein passendes Intervall wie z. B. 10 Sekunden ein. Wählen Sie am besten kein unnötig kurzes Intervall. Dadurch wird überflüssiger Datenverkehr im Netzwerk vermieden.
