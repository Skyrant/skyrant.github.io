JSON

Java Script Object Notation

JSON ist ein standardmäßiges Ordnerformat zur Beschreibung von Datenobjekten. Im Kontext von Sapient Visualisierungszeichnungen, wird JSON für mehrere Fälle verwendet, wie Binding Definitionen eines Visualisierungsobjekts oder HMI Projektordnern.

Weitere Hintergrundinformationen können z. B. auf Wikipedia à JSON - Wikipedia gefunden werden. Im Folgenden wird lediglich eine kurze Einführung gegeben (teilweise von Wikipedia kopiert).

JSON wird verwendet um Schlüssel-Wert-Paare und Wertereihen zu definieren. Jeder Wert kann einer der folgenden Typen sein:

·         Number: ganzzahliger oder gleitender Wert

·         String: Sequenz von Unicode-Zeichen, in doppelte Anführungszeichen eingefasst. Doppelte Anführungszeichen innerhalb eines Strings können mit einem Backslash ausgeklammert werden.

·         Boolean: die Werte true oder false

·         Array: eine Liste von null oder mehreren Elementen, jedes Element kann jeglicher Typ sein

·         Object: eine Sammlung von Namen-Wert-Paaren (alle Namen sind Strings!)

·         Null: leerer Wert

 

Der Projektordner hmi.prj ist ein gutes Beispiel für die Verwendung unterschiedlicher Typen, da er z. B. ein Objekt mit zwei String-Werten für den Standard-Login, einen Boolean-Wert zum An-/Ausschalten des Debug-Modus oder ein Array für Menüpunkte enthalten.

 

{

  "defaultLogin": {

    "userName": "andon_default",

    "password": "andon_default"

  },

  "parameters": {

    "debugMode": true

  },

  "theme": "sapient-dark",

  "header": {

    "hide": false

  },

  "footer": {

    "fileName": "hmi/machine-terminal/footer.xml",

    "height": "250px"

  },

  "navigation": {

    "items": [

  {...},

  {...}

]

  },

  ...

  ...

}

 

Es ist nicht verpflichtend, aber wird dringend empfohlen, mit Einrückungen zu arbeiten, um die Struktur eines solchen Dokumentes übersichtlicher zu gestalten.

Schlagwörter unterscheiden zwischen Groß- und Kleinschreibung!
