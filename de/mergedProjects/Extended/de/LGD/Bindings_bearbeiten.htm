Bindings bearbeiten

Binding-Dialog

Der Binding-Dialog kann über das Kontextmenü Item Sapient Bindings bearbeiten (Alt+Shift+B) angezeigt werden.

Der Binding-Dialog ermöglicht das Bearbeiten von Bindings in einem JSON-Editor. Binding-Ausdrücke können komplett neu erstellt werden. Es ist allerdings effektiver, ein vorbereitetes Binding-Snippet auszuwählen und zu bearbeiten.

TIPP

• Halten Sie die STRG-Taste gedrückt und führen Sie einen Rechtsklick im Editor aus, um das Snippet-Menü zu öffnen. Wählen Sie ein Snippet, um es an der momentanen Position des Cursors einzufügen.

• Verwenden Sie den Shortcut STRG+Alt+K um einen Binding-Ausdruck zu formatieren.

 

 

Bereiche/Objekte können über das Dreieck-Symbol am Zeilenbeginn eingeklappt werden:

  • Es ist möglich, ein eingeklapptes Objekt auszuwählen, zu kopieren und wieder einzufügen!

  • Wenn der Cursor direkt hinter einer Klammer gesetzt wird, wird die jeweilige öffnende / schließende Klammer hervorgehoben.

Meist genutzte Bindings – Variablenwerte anzeigen und festlegen

Die folgenden Beispiele sollen einen Einblick in das Arbeiten mit Bindings geben. Dafür werden Bindings kreiert, um eine Legato Variable in einem Element anzuzeigen und ihren Wert ändern zu können.

Legato und HMI Variablen adressieren

Jede Variable in Legato kann über einen Symbolnamen adressiert werden. Legato Sapient ermöglicht es, Variablen zu definieren und übernimmt ihren Wert in die Datenbank („Legato Variablen“). Um die Beispiele in dieser Beschreibung direkt auszuprobieren, müssen womöglich die folgenden Variablen über LC2 konfiguriert werden:

 

 

  • Während des Verbindungsvorgangs zu SPSen werden HMI-Variablen normalerweise direkt während des Prozesses ausgetauscht. Diese Variablen können an dem Präfix # erkannt werden, z. B. #PLCXXYY.R2.TEMP33. Vor der Variablenkennung steht ein #, gefolgt von der SPS-Kennung und – getrennt durch einen Punkt – einem eindeutigen Symbolnamen des SPS-Elements.

  • Alle Variablenkennungen fangen normalerweise mit einer Verbindungskennung und einem beliebigen, aber eindeutigen Symbol innerhalb dieser Verbindung an. Die verschiedenen Abschnitte sind immer durch einen Punkt getrennt. Die Verbindungskennungen enthalten möglicherweise keine Punkte.

 

Beispiel: Eine Variable mit dem Text eines Elements verbinden

• Fügen Sie jedes beliebige Element zu einer Zeichnung hinzu, die über ein Anwendungsmenü aufrufbar ist. Öffnen Sie den Binding-Dialog.

• Öffnen Sie das Snippet-Menü (STRG + Rechtsklick).

• Wählen Sie Beispiele|Text aus.

• Ein voll funktionsfähiges Binding wird eingefügt, um eine Server- (oder HMI-)Variable an den Elementtext zu binden.

• Ersetzen Sie den Platzhalter für das Variablensymbol mit einem bereits vorhandenen, z. B. eine Reihe von Beispielvariablen wie oben abgebildet.

• Schließen Sie den Binding-Dialog (Anwenden). Speichern Sie die Änderungen in der Zeichnung.

• Wechseln Sie zu Laufzeit, öffnen Sie die Zeichnung, z. B. in einem Dashboard, das das Visualisierungslaufzeit Boardlet enthält, oder in der HMI-Anwendung  der Wert der Variablen wird angezeigt werden.

Beispiel: Eine Variable durch Anklicken eines Elements ändern

• Fügen Sie ein weiteres Element zur Zeichnung hinzu.

• Verwenden Sie das Snippet Beispiele|Legatovar Wert festlegen.

• Ersetzen Sie das Variablensymbol, um die gleiche Variable wie im vorherigen Beispiele zu adressieren. Ersetzen Sie den Platzhalter-Wert durch einen String. Der Platzhalter ist ein numerischer Wert – den String zu ersetzen bedeutet auch den Platzhalter 10 durch einen individuellen Textstring mit Anführungszeichen zu ersetzen, z. B. „HALLO!“.

• Apply binding. Save drawing. Update Runtime…

 

Beispiel: Dem Nutzer ermöglichen jeglichen Wert durch Anklicken eines Elements einzugeben

• Kopieren Sie den Button um die Variable vom letzten Beispiel zu setzen.

• Entfernen Sie das Attribut „Wert“ von der setValue Funktion.

• Wenden Sie das Binding an, speichern Sie die Zeichnung und testen Sie es…  Die Sidebar erscheint, sodass der Nutzer einen Wert eingeben kann.

Struktur von Binding-Ausdrücken

Bei genauerer Betrachtung der Beispiele wird bereits die grundlegende Struktur von Bindings ersichtlich.

Ein Binding besteht aus drei (optinalen) Haupt“blocks“:

• datasources: Eine Liste von Datenobjekten, die im Binding-Ausdruck verwendet werden.

• bindings: “Binding Ziele” = Elementeigenschaften oder -stile, die an jegliches Datenobject gebunden werden.

• events: Events wie Nutzerinteraktion mit Elementen, die Aktionen ausführen.

Datenquellen

Eine Liste von Datenobjekten, die alle über einen Eigenschaftsnamen (Key) benannt werden. Es gibt einige Datenanbieter / Datenquellen. Die Struktur von Datenobjekten ist einheitlich – jedes Datenobjekt hat die Eigenschaft source um den Anbieter zu spezifizieren (Legatovariable, Session/lokale Variable, Alarm, Berechtigungen, etc.). Das Param-Objekt ist abhängig vom Anbieter, da das Adressieren spezifischer Informationen auch vom Anbieter abhängt.

Beispiele mit einer Liste von definierten Datenobjekten, einige mit unterschiedlichen „Quellen“ und entsprechenden individuellen Parametern:

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        },

        "n2": {

            "source": "var",

            "params": {

                "sym": "examples.n2"

            }

        },

        "n3": {

            "source": "var",

            "params": {

                "sym": "examples.n3"

            }

        },

        "sum": {

            "source": "formula",

            "params": {

                "formula": "SUM(n1, n2, n3)"

            }

        },

        "alarm": {

            "source": "alarm",

            "params": {

                "conSym": "con",

                "extCode": -6

            }

        }

    },

    "bindings": {

        ...

    }

}

 

 

Die Referenz enthält alle verfügbaren Datenanbieter. Im Folgenden werden die üblichsten als Beispiele gezeigt.

Legato-Variable: Eine Variable befindet sich auf dem Server. Alle Klienten sehen den gleichen Wert. Im HMI-Kontext werden diese Variablen üblicherweise mit einer verbundenen SPS ausgetauscht.

"var": {

    "source": "var",

    "params": {

        "sym": "varSym"

    }

}

 

Session-Variable: Eine Variable ist in der gesamten App zugänglich (da der Wert mit allen Zeichnungen in der App ausgetauscht wird, ist er in der Session gleichbleibend).

"sessionvar": {

    "source": "sessionvar",

    "params": {

        "sym": "xyz"

    }

}

 

Lokale Variable: Eine Variable mit dem Umfang der Zeichnung.

"localvar": {

    "source": "localvar",

    "params": {

        "sym": "xxx"

    }

}

 

 

Alarm: Alarmstatus.

"alarm": {

    "source": "alarm",

    "params": {

        "conSym": "con",

        "extCode": -6

    }

}

 

Formeln: Der Wert wird mit einer Formel ausgerechnet.

"formula": {

    "source": "formula",

    "params": {

        "formula": "SUM(1,1)"

    }

}

 

TIP:

Verwenden Sie Snippets, um einzelne Datenobjekte einzufügen. Ändern Sie lediglich den Key des neuen Eintrags und die Params um das Datenobjekt zu adressieren.

 

Bindings

Die Bindingssektion (Objekt) enthält eine Liste mit allen Bindingzielen. Ziele definieren Elementeigenschaften oder -stile, die dynamische Daten während der Laufzeit erhalten. Die geläufigsten Bindingziele sind Text, Füllfarbe, Rahmenfarbe, Schriftfarbe oder Sichtbarkeit eines Elements. Einem Ziel kann ein fester Wert zugeschrieben werden oder sich auf ein Datenobjekt beziehen. Das einfachste, erwähnenswerte Binding:

{

    "bindings": {

        "text": {

            "value": "A static text"

        }

    }

}

 

Dem Binding-Objekt für den Text wird einfach ein statischer String zu seinem Wert hinzugefügt. Ein typischerer Anwendungsfall ist es, ein Datenobjekt zu referenzieren, um es als Wert zu verwenden. Die Struktur bleibt die gleiche, nur dass der Wert ein Objekt bekommt – in diesem Fall, eine Datenreferenz. Und es muss natürlich ein Datenobjekt existieren, das referenziert werden kann:

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "examples.str1"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "someVar"

                },

                "defaultValue": "/"

            }

        }

    }

}

 

 

Eine Datenreferenz hat die gleiche Basisstruktur wie jedes andere Datenobjekt.

 

Es ist sogar möglich, eine Berechnung oder Formel für einen Wert direkt zu definieren.

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        },

        "n2": {

            "source": "var",

            "params": {

                "sym": "examples.n2"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "formula",

                "params": {

                    "formula": "SUM(n1, n2)"

                }

            }

        }

    }

}

 

Das Konzept ist für alle Ziele genau gleich, z. B. eine statische Füllfarbe:

{

    "bindings": {

        "fillColor": {

            "value": "red"

        }

    }

}

 

Ein typisches Beispiel: eine Farbe, die vom Wert abhängig ist:

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        }

    },

    "bindings": {

        "fillColor": {

            "value": {

                "source": "formula",

                "params": {

                    "formula": "IF(n1 > 5, 'red', 'green')"

                }

            }

        }

    }

}

 

 

Das Bindings-Objekt kann jegliche Bindingziele enthalten, die durch Kommata voneinander abgegrenzt sind. Es gibt Snippets für die am meisten verwendeten Bindingziele, um sie in ein Bindings-Objekt einzufügen.

 

Beispiel:

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "n1"

                }

            }

        },

        "fillColor": {

            "value": {

                "source": "formula",

                "params": {

                    "formula": "IF(n1 > 5, 'red', 'green')"

                }

            }

        }

    }

}

 

TIPP:

Bei den meisten Eigenschaftswerten ist es möglich, einen statischen Wert in einem Binding durch eine Datenreferenz oder ein Datenobjekt zu ersetzen. Der Variablenname einer Definition für ein Variablendatenobjekt kann sogar an ein anderes Datenobjekt gebunden werden, z. B. dynamisches Erstellen eines Variablensymbols während der Laufzeit.

 

Events

Events können verwendet werden, um zu reagieren, z. B. wenn ein Element angeklickt wird. Jedes Event kann eine oder mehrere Aktionen ausführen. Wie bereits gezeigt – die Aktion Wert festlegen:

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "examples.str1"

            }

        }

    },

    "events": {

        "onClick": [

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "someVar"

                        }

                    },

                    "value": "HELLO!!!"

                }

            }

        ]

    }

}

 

Weitere Aktionen, die ausgeführt werden:

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        },

        "n2": {

            "source": "var",

            "params": {

                "sym": "examples.n2"

            }

        }

    },

    "events": {

        "onClick": [

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "n1"

                        }

                    },

                    "value": 7

                }

            },

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "n2"

                        }

                    },

                    "value":8

                }

            }

        ]

    }

}

 

Die folgenden Events werden zur Interaktion genutzt:

• onClick: Wird ausgeführt, wenn der Nutzer das Element mit der Maus anklickt.

• onMouseDown: Wird ausgeführt, wenn der Nutzer die Maustaste über einem Element klickt.

• onMouseUp: Wird ausgeführt, wenn der Nutzer die Maustaste über einem Element loslässt.

• onMouseEnter / onMouseLeave: Wird ausgeführt, wenn der Mauscursor in einen Elementbereich hinein-/hinausgeführt wird. Dies ist z. B. für Funktionalitäten wie Tooltips nützlich.

Die Events onLoad and onUnload werden verwendet um Aktionen aufzurufen, wenn eine Zeichnung geladen oder geschlossen wird. Dies wird z. B. verwendet, um während des Ladevorgangs Variablen für einen anfänglichen Wert zu setzen oder die Variablen zurückzusetzen, wenn die Zeichnung geschlossen wird.

 

Die Events onLoad und onUnload können nur für das Dokument selbst verwendet werden.

Das Event onChange kann verwendet werden, um auf Wertänderungen zu reagieren, und stellt die einzige Ausnahme mit einer etwas anderen Übermittlung dar. Details zur Nutzung können in der Referenz gefunden werden.

Actions

Die Einzelaktionen werden in der Referenz detaillierter erklärt. Eine wichtige Eigenschaft, die für jede Aktion genutzt werden kann, sollte hier allerdings bereits erwähnt werden: die enabled Eigenschaft, die die Ausführung von Aktionen verhindert, die auf irgendein Datenobjekt angewiesen sind. Ein typischer Anwendungsfall wäre es, bestimmte Aktionen abhängig von den Berechtigungen des momentanen Nutzers zu ermöglichen.

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "examples.str1"

            }

        },

        "isAdmin": {

            "source": "permission",

            "params": {

                "actions": [

                    "ADMIN"

                ]

            }

        }

    },

    "events": {

        "onClick": [

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "someVar"

                        }

                    }

                },

                "enabled": {

                    "source": "dataref",

                    "params": {

                        "ref": "isAdmin"

                    }

                }

            }

        ]

    }

}
