Standardwerte

Alle Datenobjekte (und Datenreferenzen) unterstützen die Eigenschaft defaultValue. Solange das Objekt während des Ladens der Zeichnung noch nicht initialisiert ist, wird der Standardwert verwendet.

Beispiel:

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "ADTest1.V1"

            },

            "defaultValue": "***"

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "someVar"

                },

                "defaultValue": "###" <-- ignored in this example!!

            }

        }

    }

}

 

Das exemplarische Binding hat einen Standardwert, der für das Variablenobjekt definiert ist, um eine Variable auf Serverseite zu lesen – der Standardwert wird zurückgegeben, überall dort wo auf das Objekt zugegriffen wird, vor allem in Formeln und in Datenreferenzen. Das bedeutet, dass der für die Referenz definierte Standardwert in diesem Fall nutzlos ist, da das Datenobjekt immer einen Wert zurückgeben wird – entweder den tatsächlichen Variablenwert oder den Standardwert, wenn die Variable noch nicht initialisiert wurde.

Um ein Objekt für verschiedenen Bindings zu nutzen, könnte es nötig sein, verschiedene Standardwerte zu benutzen – in diesem Fall sollte kein Standardwert für das Objekt selbst definiert werden, aber für jede individuelle Referenz.
