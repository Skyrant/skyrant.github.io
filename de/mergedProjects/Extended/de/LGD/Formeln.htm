Formeln

Überblick

Formeln sind ein hilfreiches Tool für flexible Berechnungen. Sie sind sehr ähnlich zu den Formeln in Excel. Die Anwendung basiert auf der Open Source Bibliothek formula.js. Das macht die Verwendung der Formeln in Binding-Ausdrücken sehr intuitiv und sollte alle Nutzern, die bereits mit Formeln in Excel gearbeitet haben, vertraut sein. Eine detaillierte Erklärung, wie Formeln erstellt werden können, sollte hier also nicht nötig sein – die umfangreiche Wissensgrundlage im Internet ist in diesem Fall viel hilfreicher!

Bindings mit Formeln

Formeln können als Datenobjekte definiert werden. Das Referenzieren von anderen Datenobjekten funktioniert, indem einfach der Name der Datenobjekte verwendet wird. Das folgende Beispiel hat die gleiche Funktionalität wie das zuvor für Berechnungen gezeigte.

{

    "datasources": {

        "n1": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        },

        "n2": {

            "source": "var",

            "params": {

                "sym": "examples.n2"

            }

        },

        "sum": {

            "source": "formula",

            "params": {

                "formula": "SUM(n1, n2)"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "sum"

                }

            }

        }

    }

}

 

Formeln können verschachteln sein (like in Excel), z. B.

IF(SUM(n1, n2) > 12, 'More than twelve!', 'Too less...')

 

Auf Eigenschaften eines Datenobjekts mit einem Objekt als Wert kann zugegriffen werden, indem die Eigenschaften in der Formel spezifiziert werden, z. B. nodestate.text

 

TIPP:

Eine hexadezimale Notation von Werten ist, im Vergleich zu Dezimalzahlen, einfacher zu lesen, vor allem wenn man mit Bitmasken etc. arbeitet. Hexadezimale Werte brauchen das Präfix 0x, z. B. könnte die Bitmaske 1111 0000 0001 0010 als 0xF012 geschrieben werden, was leichter zu lesen ist als das Pendant in Dezimalform 61458:

Wenn man dies verwendet, könnte eine Formel zur Maskierung eines Datenobjekts wie folgt aussehen: BITAND(val, 0xF012.

 

Besondere Funktionen

Es werden noch einige weitere Funktionen als die integrierten, die Sie auf der formula.js-Webseite zu finden sind, unterstützt.https://formulajs.info/functions/

Die ersten (Stand: Juni 2022) sind:

  • ISBITSET

  • BITMAP

 

Mit diesen Funktionen ist es möglich, mit einem kompakten Ausdruck zu prüfen, ob bestimmte Bits in einer Variablen gesetzt sind.

ISBITSET und BITMAP funktionieren nur mit 0 bis 31 Bits (nur für 32-Bit-Werte).

 

ISBITSET(value, bit, trueVal, falseVal)

 

  • value - value to test

  • bit - bit to test(starting from 0)

  • trueVal - value returned if specified bit is set (optional, default = true)

  • falseVal - value returned if specified bit is not set (optional, default = false)

BITMAP(value, defaultResult, bit, value, bit, value ...)

 

  • value - value to test

  • defaultResult - value returned if non of specified bit is set

  • bit, value - one or more pairs:

    • bit - bit to test

    • value - value returned if corresponding bit is set

Beispiel

 

{

    "datasources": {

        "value": {

            "source": "localvar",

            "params": {

                "sym": "xxx"

            }

        },

        "bit": {

            "source": "formula",

            "params": {

                "formula": "ISBITSET(value,0)"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "bit"

                },

                "defaultValue": "---"

            }

        }

    }

}
