Debugging

Prüfen von Binding-Informationen während der Laufzeit mit der Debug-Seitenleiste

Die Debug-Seitenleiste ist ein nützliches Tool um Binding-Ausdrücke während der Laufzeit zu prüfen. Die Nutzung der Debug-Seitenleiste muss über den jeweiligen Boardlet Parameter genehmigt werden oder – in der HMI Anwendung – über den Projektordner.

Durch gleichzeitiges Drücken der STRG-Taste und Rechtsklick auf ein Element wird die Debug-Seitenleiste angezeigt. Die Seitenleiste zeigt direkt das Binding für dieses Element an und ermöglicht direkten Zugang zum Designer. Gerade bei Elementgruppen ist es möglich, die Elterngruppe eines Elements über den Button Go to parent zu erreichen. Anschließend werden die enthaltene Ebene und das Dokument ausgewählt.

Der Button Open Currently Used Data Items ermöglicht es, eine Liste für jeden Variablen-, Knoten- und Alarmstatus zu öffnen, die in der Zeichnung aktuell verwendet werden.

In den einzelnen Tabs können unterschiedliche Debug-Informationen angezeigt werden.

• Bearbeitete Bindings: Bindings als Ergebnis der Vorbereitung für eine optimierte Ausführung während der Laufzeit.

• Metadaten: Name, Kommentar und Bibliothek-/Symbolinformationen im Fall von Bibliothekelementes.

• Datarefs: Der vermutlich bekannteste Debug Tab ist der Tab datarefs, der alle Datenobjekte anzeigt, die im Bindings des Elements referenziert sind. Die Liste enthält den aktuellen Wert und im Fall von Variablen den Zeitpunkt des letzten Wertewechsels. Im Falle von kalkulierten Datenobjekten (Formeln), werden hier auch die Rechenergebnisse angezeigt.

• Stile: Zeigt die aktuellen Stilwerte des Elements an.

• Leistung: Einige Leistungskennzahlen der Zeichnung.

Browser Protokoll

Falls Binding-Ausdrücke gravierende Fehler wie eine ungültige JSON-Syntax (z. B. fehlende Klammern) enthalten, ist es sinnvoll das Browser-Protokoll zu prüfen, um detaillierte Fehlerinformationen zu erhalten.

• Drücken Sie F12 um zu den Debug Tools des Browsers zu gelangen.

• Wechseln Sie zur Konsole – sie enthält möglicherweise Fehler wie …

• Die Nachricht zeigt deutlich, dass das Element mit der angezeigten ID einen Syntaxfehler enthält.

• Um dieses Element in der Zeichnung zu lokalisieren, kopieren Sie die ID des Elements.

• Wählen Sie Sapient|Find Element by ID… im Designer im Menü aus und fügen Sie die ID ein.

--> Das Element wird in der Zeichnung markiert.
