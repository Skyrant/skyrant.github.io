Konfiguration

Sämtliche Konfigurationen werden im root-SVG (Klick auf den Hintergrund oder die graue Umrandung) unter Inspector/Properties vorgenommen.

UserVars

Dieser Punkt erlaubt es, eigene Variablen zu definieren. Dies ist notwendig, um z.B. KPI- oder Schichtprotokollwerte in die Visualisierung zu integrieren.

Das Fenster ist zweigeteilt. Auf der linken Seite werden unter dem Oberpunkt "User-Defined-Variables" alle bereits parametrierten Variablen aufgelistet. Die rechte Seite zeigt weitere Informationen zu dem gewählten Punkt bzw. ermöglicht weitere Einstellungen.

Um eine neue Variable anzulegen, ist der Button "New" zu wählen.

Im folgenden Dialog können mögliche Arten von benutzerdefinierten Variablen ausgewählt werden. Die jeweiligen Arten können beliebig oft angelegt werden. Es stehen alle Arten zur Verfügung, die über das Data-Objects-File importiert wurden.

Diese Liste der abfragbaren Informationen ist durch das Projekt anpassbar und beliebig erweiterbar.

Die angelegten Variablen können im "Select Variable" Dialog unter "User-Defined Vars" bei den Dynamisierungen eingebunden werden.

Da die folgenden drei Arten am häufigsten genutzt werden, sollen sie hier detaillierter beschrieben werden:

KPIRel = KPI-Werte,

ShiftProtValRel = Schichtprotokollwerte und

ShiftInfoRel = Schichtinformationen.

 

KPI-Werte

Nach Wählen dieses Punktes wird eine neue Instanz angelegt.

In der rechten Fensterhälfte können die zugehörigen Daten geändert werden.

 

Parameter

Beschreibung

Name

Hier kann ein Name frei gewählt werden. Dieser dient zur Identifizierung bei der späteren Variablenreferenzierung. Er wird auch in der linken Ansicht genutzt.

Node

Auswahl eines Knotens der Fertigungsstruktur, dessen KPI-Werte angezeigt werden sollen.

ShiftOffset

Angabe einer relativen Schicht
1 = aktuelle oder letzte Schicht, wenn keine aktuell ist
2 = letzte Schicht
3 = vorletzte Schicht
…

Schichtprotokollwerte

Ebenfalls wird für Schichtprotokollwerte eine neue Instanz angelegt.

 

Parameter

Beschreibung

Name

Hier kann ein Name frei gewählt werden. Dieser dient zur Identifizierung bei der späteren Variablenreferenzierung. Er wird auch in der linken Ansicht genutzt.

Node

Auswahl eines Knotens der Fertigungsstruktur, dessen Schichtprotokoll-Werte angezeigt werden sollen.

ShiftOffset

Angabe einer relativen Schicht
1 = aktuelle oder letzte Schicht wenn keine aktuell ist
2 = letzte Schicht
3 = vorletzte Schicht
…

ValSym

Hier kann die Schichtprotokollvariable gewählt werden. Da leider keine direkte Zuordnung existiert, welche Variable auf welchem Knoten das Flag "protokolliert" hat, ist die korrekte Auswahl erschwert. Um Fehler zu vermeiden empfiehlt es sich, die Auswahl der Variable über den Tab "Nodes with Vars" vorzunehmen. Hier ist zumindest ersichtlich, welcher Knoten welche Variablen zugewiesen hat. Dennoch wird das Wissen aus der Admin benötigt, welche dieser Variablen mit dem Flag "protokolliert " versehen wurden.

Schichtinformationen

Zusätzlich zu den Werten der Schichten ist eine Klasse implementiert, die Informationen über die Schichten bietet.

 

Parameter

Beschreibung

Name

Hier kann ein Name frei gewählt werden. Dieser dient zur Identifizierung bei der späteren Variablenreferenzierung. Er wird auch in der linken Ansicht genutzt.

Node

Auswahl eines Knotens der Fertigungsstruktur, dessen Schichtinformationen angezeigt werden sollen.

ShiftOffset

Angabe einer relativen Schicht
1 = aktuelle oder letzte Schicht wenn keine aktuell ist
2 = letzte Schicht
3 = vorletzte Schicht
…

Localformat

Sprachspezifische Ausgabe der Schichtzeiten. Es stehen folgende Ausführlichkeits-Stufen zur Auswahl:
- FULL (0), z.B.: "Tuesday, April 12, 1952 AD" oder "3:30:42pm PST"
- LONG (1) z.B.: "January 12, 1997" oder "3:30:32pm"
- MEDIUM (2) z.B.: "Jan 12, 1997"
- SHORT (3) z.B.: "12.13.97" oder "3:30pm"

 

:

Werden die Namen der Instanzen nachträglich geändert, muss dies auch in der Referenzierung der Variablen in den Dynamiken nachgezogen werden. Eine Änderung wird nicht automatisch übernommen. Wird dies nicht berichtigt, kann das Bild nicht erfolgreich kompiliert werden.

ViewOpts

Mit den folgenden Einträgen kann das Aktualisierungsverhalten bzw. Aussehen der Grafik innerhalb von Legato spezifiziert werden. Die Einstellungen findet man im Properties-Fenster, wenn man auf den grauen Hintergrund außerhalb der Zeichenfläche klickt.

Dabei sollten die Standardeinstellungen nicht verändert werden bzw. nur mit Rücksprache des GEFASOFT Supports.

 

Parameter

Beschreibung

Varwait

Maximale Zeitspanne in Millisekunden, nach der der Server ein Lebenszeichen an den Browser schickt (empfohlener Standardwert: 110000).

Timeout

Zeitspanne in Millisekunden, nach der der Browser ohne Lebenszeichen vom Server eine Fehlermeldung anzeigt (empfohlener Standardwert: 120000).

Refresh

Verzögerung der Aktualisierung in Millisekunden, um eine Serverüberlastung bei Visualisierungen mit vielen Veränderungen zu vermeiden (empfohlener Standardwert: 2000).

PgWidth

Breite der Grafik (sollte auf 100% stehen).

PgHeight

Höhe der Grafik (sollte auf 99% stehen).
