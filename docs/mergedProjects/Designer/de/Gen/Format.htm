Das SVG-Format

Mit dem GefasoftDesigner können Visualisierungsansichten komfortabel durch Point & Klick-Tools erstellt werden. Erfahrungen haben jedoch ergeben, dass grundlegende Kenntnisse in SVG das Erstellen der Grafiken erheblich erleichtern und beschleunigen. Der folgende Abschnitt soll die wichtigsten Grundlagen kurz beleuchten. Für eine vertiefte Auseinandersetzung mit dem Thema SVG sei an dieser Stelle auf die Workshops der GEFASOFT GmbH zu diesem Thema verwiesen.

Allgemein

Dateiformat: *.svg

Vektororientiert - Positionierung über Koordinaten.

Quelltext in normalen Text, kein Binärcode.

Die Renderreihenfolge (Darstellung) beginnt am Anfang der Datei und geht von oben nach unten. Daher liegen Elemente, die später in der Datei folgen, über Elementen, die weiter vorn in der Datei stehen.

Standardmaßeinheit sind Pixel (Zahlenangaben ohne Zusatz). Es sind jedoch auch andere Angaben wie mm, cm, pt usw. möglich.

Die X-Achse des Koordinatensystems verläuft von links nach rechts, die Y-Achse von oben nach unten.

Der Bezugspunkt für die meisten Elemente (Rechteck, Gruppen, Images, Punkte, …) ist die linke obere Ecke des Objektes. Für Texte wird die Baseline als Bezugspunkt genutzt, bei Kreis und Ellipse der Mittelpunkt.

Jedes Element kann eine ID bekommen (der GefasoftDesigner vergibt diese automatisch beim Erzeugen des Elements). Diese ID muss dokumentenweit eindeutig sein, da es sonst zu Fehlern kommen kann.

SVG-Elemente werden sind in zwei Varianten aufgebaut:

<[Elementname] [Attribute] />

<[Elementname] [Attribute]>
                [Inhalt]
</[Elementname]>

Attribute / Positionierung

Das Aussehen der SVG-Elemente wird über Attribute (auch Styling) geregelt. Auch die Positionierung ist an die Angabe der Attribute angelehnt.

Die Attribute können auf zwei Arten angegeben werden. Die Positionierung kann nur auf die erste Art definiert werden.

[Attribut]="[Wert]"

style="[Attribut]:[Wert]"

Farben

Farben können auf drei Arten angegeben werden.

Farbwort (existiert nur für bestimmte typische Farben)

Bsp.: red, black, blue

Hexadezimale Angabe

Form: #RRGGBB*

Bsp.: #FF05AC

Dezimale Angabe

Form: RGB(red,green,blue)*

Bsp.: RGB(102,104,69)

Der GefasoftDesigner nutzt standardmäßig die dezimale Angabe. In den entsprechenden Dialogen können aber auch die anderen Arten genutzt werden. Diese werden anschließend automatisch auf den RGB-Wert umgerechnet.

*              RR = Wert für Rot-Anteil
                GG = Wert für Grün-Anteil
                BB = Wert für Blau-Anteil
                Jeder Anteil kann von 0 bis 255 bzw. FF gehen

Transformationen

Transformationen dienen dazu, Elemente oder ganze Gruppen weiter zu bearbeiten. Das meistgenutzte Beispiel ist eine nachträgliche Verschiebung. Für das Attribut "transform" gibt es verschiedene Optionen, die auch beliebig kombiniert werden können.

Zurzeit unterstützt der GefasoftDesigner nur die Matrix-Transformation, die jedoch alle anderen Transformationen abbilden kann.

Die Matrix wird in der Form matrix(x1,y1,x2,y2,x,y) geschrieben.

Bsp.: Für eine Verschiebung wird die Matrix in der Form matrix(1,0,0,1,40,-200) geschrieben. Diese Transformation bewirkt, dass das Element um 40 Pixel nach rechts und um 200 Pixel nach oben verschoben wird.

Skripting / Animation

Animationen dienen dazu, Dynamik in die ansonsten statischen Bilder zu bringen. Hierfür können zwei Wege genutzt werden: JavaScript oder Animationen. Da Animationen in technisch ausgerichteten Grafiken keine große Bedeutung haben, wurde dieser Aspekt im GefasoftDesigner nicht berücksichtigt.

JavaScript-Funktionen werden durch Events gesteuert. Beim Eintreffen dieser Events (z.B. Mausklick) für die einzelnen Objekte wird das parametrierte JavaScript ausgeführt.
