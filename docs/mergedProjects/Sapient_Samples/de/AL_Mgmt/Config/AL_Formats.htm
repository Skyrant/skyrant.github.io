Meldungsformatierungen

Zur Formatierung von Meldungen können bei den Meldeklassen in der Spalte "CSS Style" Vorgaben gemacht werden. Für jeden Status einer Meldung kann eine Schriftart, eine Farbe, ein Symbol etc. definiert werden.

Beispiel

[

{

"state": "default",

"style": ["color-error"],

"icon": "icon-attention"

},

{

"state": "acknowledged",

"style": ["font-italic"]

},

{

"state": "closed",

"style": ["font-small"]

}

]

 

States

Folgende States stehen zur Verfügung:

default

Standard, wird als Default verwendet.

acknowledged

Meldung wurde bereits quittiert, ist aber wegen fehlenden Gehend-Ereignisses noch anstehend.

recognized

Status "erkannt" wurde für die Meldung gesetzt; sie ist aber wegen fehlenden Gehend-Ereignisses noch anstehend.

closed

Meldung ist bereits gegangen (Ende-Zeitstempel existiert), steht aber noch an, weil sie z.B. noch quittiert werden muss.

archived

Meldung ist gegangen und wurde ins Archiv verschoben (d.h., auch alle anderen Aktionen sind abgeschlossen).

 

Styles

Folgende Styles stehen zur Verfügung. Die genauen Farben sind im jeweiligen Theme definiert.

Font-Farben

Jede dieser Farben kann normal (z.B. "style": ["color-error"]), light (z.B. "style": ["color-error-light"] oder dark (z.B. "style": ["color-error-dark"] verwendet werden.

&.color-error {

color: $alarm-error-color;

}

 

&.color-warning {

color: $alarm-warning-color;

}

 

&.color-success {

color: $alarm-success-color;

}

 

&.color-info {

color: $alarm-info-color;

}

 

&.color-misc {

color: $light-gray;

}

 

Font-Style

&.font-italic {

font-style: italic;

}

 

Font-Größe

&.font-small {

font-size: 85%;

}

 

&.font-large {

font-size: 120%;

}

 

Hintergrund-Farbe

&.background-error-color {

background-color: $alarm-error-color;

}
