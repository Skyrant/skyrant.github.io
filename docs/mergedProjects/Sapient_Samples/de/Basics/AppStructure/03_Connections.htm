Funktion

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer. Wird vom System vergeben. Schreibgeschützt.

Modul

Lookup-Liste zur Auswahl des zugehörigen Moduls. Wird automatisch befüllt und kann geändert werden.

Gerätetyp (optional)

Lookup-Liste zur Auswahl des Gerätetyps.

Die Auswahl des Gerätetyps dient standardmäßig nur Anzeigezwecken zur Strukturierung v.a. großer Projekte.

ID (SPS Identifier)

Eindeutige Bezeichnung der Steuerung (z.B. SPS) im Projekt, z.B. Ortskennzeichen oder Betriebsmittelkennzeichen. Standard-Gateway-Skripte verwenden diese ID für die Buchung von Verbindungszuständen diese Symbolik für die Adressierung der Verbindung in der Datenbank. Die Verbindungssymbolik im Gateway und hier sollten entsprechend gleichlautend sein.

Name

Verbindungsname. Der Name der Verbindung wird im WebClient an verschiedenen Stellen als Quelle visualisiert.

Hinweis: Im Namen darf die Zeichenfolge "##." nicht verwendet werden. Ansonsten wird eine Fehlermeldung angezeigt.

Beschreibung (optional)

Zusätzliche Beschreibung der Verbindung. Die Beschreibung der Verbindung wird im Webclient an verschiedenen Stellen als Quellenbeschreibung visualisiert.

Knoten (Default)

Lookup-Liste zur Auswahl eines Default-Knotens aus der Fertigungsstruktur. Diesem Knoten werden System-Meldung zur Verbindung und eventuell parametrierte Grenzwertverletzungen zugeordnet.

Beim Import von Meldungstexten für das Meldesystem werden Meldungen standardmäßig diesem Knoten zugeordnet.

Parent (optional)

Lookup-Liste zur Auswahl einer übergeordneten Verbindung zur hierarchischen Darstellung.

Sync (optional)

Checkbox. 0 = Verbindung ist noch nicht synchronisiert; 1 = Verbindung ist synchronisiert. Default ist 0.

Vorlageninstanz

Vorlageninstanz, von der der Wert stammt. Nur zur Information; schreibgeschützt.

Vorlagenreferenz

Vorlagenreferenz. Nur zur Information; schreibgeschützt.

Verbindungsstatus Nr.

Anzeige des aktuellen Verbindungsstatus als Nummer. Nur zur Information; schreibgeschützt.

Verbindungsstatus

Anzeige des aktuellen Verbindungsstatus als Text. Nur zur Information; schreibgeschützt.

Importeinstellungen (Meldungen)

Hier werden die Importeinstellungen für den Zuli-Import eingetragen. Für Details, siehe unten.

Auto Import

Die Datei wird für einen automatischen Import durch die Legato Sapient Engine vorgemerkt. Über Jobs wird die angegebene Datei auf Änderungen überwacht und der Import bei Bedarf mit den gewählten Einstellungen automatisch durchgeführt.

Letzter Zuli Import

Datum des letzten Imports von Zuordnungslisten (Zuli). Nur zur Information; schreibgeschützt.

Datum letzte Zuli

Datum der zuletzt importierten Zuli. Nur zur Information; schreibgeschützt.

Letzter Import-Fehler Text

Text des letzten Import-Fehlers. Nur zur Information; schreibgeschützt.

Letzter Import Variablenliste

Datum des letzten erfolgreichen Variablen-Imports. Nur zur Information; schreibgeschützt.

Datum letzte Datei Variablenliste

Datum der Datei des letzten Variablen-Imports. Nur zur Information; schreibgeschützt.

Letzter Import-Fehler Variablenliste

Text des letzten Fehlers eines fehlerhaften Variablen-Imports. Nur zur Information; schreibgeschützt.

Erweiterte Eigenschaften

Die unter Grundkonfiguration | Erweiterte Eigenschaften | Zuordnungen von Erweiterten Eigenschaften | Erw. Eigenschaften (Verbindungen) verwalteten erweiterten Verbindungseigenschaften werden hier angehängt (je Eigenschaft eine Spalte) und können hier mit Werten versehen werden.

Erw. Eigenschaften

Die definierten erweiterten Eigenschaften werden hier als eigene Spalten angehängt und können mit Werten versehen werden.

Die Parametrierung von erweiterten Eigenschaften ist in einem separaten Kapitel beschrieben, zu finden hier: Erweiterte Eigenschaften.

 

Erweiterte Eigenschaften

Wie in der Feldbeschreibung erwähnt, gibt es abhängig von der Verbindungsart verschiedene erweiterte Eigenschaften, die unter Grundkonfiguration | Erweiterte Eigenschaften | Zuordnungen von Erweiterten Eigenschaften | Erw. Eigenschaften (Verbindungen) gepflegt werden.

TIPP:

Da diese erweiterten Eigenschaften von der selektierten Verbindungsart abhängen, können einige von ihnen nicht im Tabellengrid gepflegt werden, sondern nur im Details-View.

Sollte eine erweiterte Eigenschaft für die gewählte Verbindungsart ein Pflichtfeld sein, so muss sie im Details View parametriert werden, um den neuen Eintrag in der Tabelle speichern zu können.

Beispiel

Die Steuerungen der Vormontage und Montage werden über ein Gateway, die der Logistikhalle über ein weiteres Gateway an die Datenbank angebunden. Entsprechend werden zwei Gateways (Gateway_1 und Gateway_2) angelegt. Gateway_1 erhält als Verbindungen zunächst die Bereiche Vormontage und Montage, unter denen jeweils die SPS-Systeme der Bereiche parametriert werden. Die Hierarchie erhält damit etwa folgendes Aussehen:

Interne Werte     …

Gateway_1         Vormontage SPS_VMON1

                             SPS_VMON2

                             SPS_VMON3

                  Montage    SPS_MONT1

                             SPS_MONT2

Gateway_2         Logistik   SPS_LOGI1

                             LOGI_PC_1

                             LOGI_PC_2

 

Importeinstellungen (Meldungen)

In der Spalte "Importeinstellungen (Meldungen)" der Verbindung werden die Einstellungen für den Import festgelegt.

:

Bei den Parametern ist auf Groß-/Kleinschreibung zu achten!

 

Folgende Einstellungen sind verbindlich in jedem Fall einzutragen:

Filename='Pfad zur und Dateiname mit Endung' (Beispiel: C:\Temp\Importdateien\GraphpicS5CSV.CSV)

FileType='Angabe des Typs der Import-Datei' (Beispiel: 5)

Default.AL_CLASS='ID der Default-Meldeklasse' Beispiel: 0)

Default.AL_TYPE='ID der Default-Meldeart' (Beispiel: 0)

 

Folgende Einstellungen können optional eingetragen werden:

Script='Name des auszuführenden Skripts'

:

Bei Verwendung des FileType 20 (Benutzerdefiniertes Skript) ist diese Eingabe verbindlich.

DeleteOldItems='Löschen alter Zuordnungen nach erneutem Import' (Beispiel: 1)

UpdateItems='Aktualisieren bestehender Zuordnungen' (Beispiel: 1)

Charset='Characterset der Import-Datei' (Beispiel: ISO8859_15)

 

Folgende optionale Einstellungen werden nur bei den Datentypen 1 (S5-SEQ-Datei), 2 (S7-SEQ-Datei) und 7 (S7-SDF-Datei) ausgewertet und verarbeitet:

ImportFilterOpType='ImportFilterOperationType' (mögliche Einträge: AB, EB, MB, SY); (Beispiel: SY)

:

SY ist nur bei FileType 1 (S5-SEQ_Datei) zulässig.

ImportFilterFirstByte='ImportFilter von (inklusive) erstem Byte' (Beispiel: 10)

ImportfilterLastByte='ImportFilter bis (inklusive) letztem Byte' (Beispiel: 1000)

 

Die beiden letzten dieser 3 Parameter werden nur ausgewertet, wenn der erste Parameter einen sinnvollen Eintrag enthält.

Mit obigen Beispielwerten werden nur Zuordnungen von SY 10 bis SY 1000 importiert.

 

Standardmäßig werden folgende Datenformate unterstützt:

S5-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S5-Programmes

S7-SDF-Dateien (*.sdf) Empfohlene Quelldatei bei Verwendung eines Simatic S7-Programmes

S7-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S7-Programmes (beschränkt auf 40 Zeichen)

P-DIAG (*.csv) aus Simatic PDiag exportierte Textdateien

Textliste – Meldungsnr. & Text (*.txt) mit tabulatorgetrennter Meldungsnummer und Text

P-DIAG XML (*.xml) Dateien

Kontextmenü

Im Kontextmenü der Verbindungen (rechte Maustaste auf die Verbindung im Navigationsbaum) werden verschiedene Optionen angeboten.

 

Alle Meldungen zurücksetzen

Setzt alle anstehenden Meldungen der selektierten Verbindung (ohne untergeordnete Verbindungen) auf "Gehend".

 

Alle Meldungen zurücksetzen (auch Unterverbindungen)

Setzt alle anstehenden Meldungen der selektierten Verbindung (inklusive aller untergeordneter Verbindungen) auf "Gehend".

 

Variablen: Import-Einstellungen…

Öffnet einen Dialog zur Auswahl einer Variablendatei mit Variablen, die für die selektierte Verbindung importiert werden sollen.

Über das Feld Zeichensatz" kann aus einer Drop-down-Liste der zu verwendende Zeichensatz (z.B. UTF8) gewählt werden. Dies ist nur relevant, wenn die Importdatei Zeichen eines speziellen Zeichensatzes verwendet.

Die folgenden Optionen stehen für den Import zur Verfügung:

Existierende Variablen aktualisieren:

Die Angaben von bereits vorhandenen Variablen werden beim Import ersetzt.

Automatisch importieren:

Die Datei wird für einen automatischen Import durch die Legato ApplicationEngine vorgemerkt. Über Jobs wird die angegebene Datei auf Änderungen überwacht und der Import bei Bedarf mit den gewählten Einstellungen automatisch durchgeführt.

 

Variablen: Import starten

Stößt den Import der Variablendatei mit den Einstellungen, die unter Import Settings… getroffen wurden, an.

Nach dem Anstoß des Importvorgangs wird dem Anwender direkt eine Meldung angezeigt, dass der Import gestartet wurde und asynchron durch die AE ausgeführt wird.

 

Meldungsdefinitionen: Import-Einstellungen…

Öffnet einen Dialog, in dem Import-Einstellungen für Zuordnungslisten getroffen werden können. In der Regel werden die Meldungen aus vorhandenen Datenquellen manuell (über den LC2) oder automatisch (Systemprogramm Legato ApplicationEngine) importiert. Die entsprechenden ApplicationEngine-Jobs sind in der separaten Dokumentation "Legato_ApplicationEngine"  genauer beschrieben. Die Scripting-Schnittstelle ermöglicht die Erstellung von Scripten zum Import spezieller Dateiformate sowie die automatische Festlegung von Meldungseigenschaften beim Import.

Es kann eine Zuordnungsdatei für den Import ausgewählt werden.

Unter "Type" legt man die Art der Zuordnungsdatei fest. Dies entscheidet darüber, wie die Meldungsnummer gebildet wird (das ist meist abhängig von der Erfassungsart der Meldung). Standardmäßig werden folgende Datenformate unterstützt:

S5-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S5-Programmes

S7-SDF-Dateien (*.sdf) Empfohlene Quelldatei bei Verwendung eines Simatic S7-Programmes

S7-SEQ-Dateien (*.seq) Sequenzielle Quelldatei eines Simatic S7-Programmes (beschränkt auf 40 Zeichen)

P-DIAG (*.csv) aus Simatic PDiag exportierte Textdateien

Textliste – Meldungsnr. & Text (*.txt) mit tabulatorgetrennter Meldungsnummer und Text

P-DIAG XML (*.xml) Dateien

 

Spezielle Formate für eine einheitliche Textbasis bei Verwendung von GEFASOFT GraphPic NT und Legato in einem Verbundprojekt sind:

S5/S7-GraphPic-Importdateien (*.txt oder *.csv), die von GraphPic NT erzeugt werden können.

Phönix CSV-Dateien

 

Über das Feld "Zeichensatz" kann aus einer Drop-down-Liste der zu verwendende Zeichensatz (z.B. UTF8) gewählt werden. Dies ist nur relevant, wenn die Importdatei Zeichen eines speziellen Zeichensatzes verwendet.

Das Feld "Scriptname" muss nur befüllt werden, wenn als Dateityp "Benutzerdefiniert" ausgewählt wurde. In diesem Fall wird hier der Scriptname der ApplicationEngine eingetragen, mit dem der Import durchgeführt werden soll. Diese Option wird verwendet, wenn für ein spezielles Dateiformat ein eigenes Importskript erstellt wurde.

Die folgenden Optionen stehen zur Verfügung:

Alte Zuordnungen löschen

Bereits vorhandene Meldungen, die beim Import nicht mehr in der Datei erhältlich sind, werden gelöscht.

Existierende Zuordnungen aktualisieren

Die Angaben für bereits vorhandene Meldungen werden beim Import ersetzt.

Nur Meldetexte aktualisieren

Zusätzliche Anwahl, um die Eigenschaften einer Meldung (z.B. Klasse und Art) beim Import beizubehalten und nur den Meldungstext zu aktualisieren.

Automatisch importieren

Die Datei wird für einen automatischen Import durch das Programm durch die Legato ApplicationEngine vorgemerkt. Über Jobs wird die angegebene Datei auf Änderungen überwacht und der Import bei Bedarf mit den gewählten Einstellungen automatisch durchgeführt.

 

Unter "Operandenbereich" können für umfangreiche Zuordnungsdateien Einschränkungen für den zu importierenden Operandenbereich festgelegt werden.

Bei "Vorgabewerte" können Standardwerte für die Felder "Klasse" und "Art" definiert werden. Beim Import wird dann der Defaultwert verwendet, sofern durch den Import keine meldungsspezifischen Angaben übergeben werden (möglich bei den Dateiformaten von GraphPic).

 

Meldungsdefinitionen: Import starten

Stößt den Import der Zuordnungsdatei mit den Einstellungen, die unter Import Settings… getroffen wurden, an.

Nach dem Anstoß des Importvorgangs wird dem Anwender direkt eine Meldung angezeigt, dass der Import gestartet wurde und asynchron durch die AE ausgeführt wird.

 

Meldungsdefinitionen: Import starten

Stößt den Import der Zuordnungsdatei an. Die Einstellungen werden im Feld "Importeinstellungen (Meldungen)" der Verbindung getroffen. Gibt es in diesem Feld keinen Eintrag, so wird eine entsprechende Fehlermeldung ausgegeben. Die entsprechenden Sapient Engine-Jobs sind in der separaten Dokumentation genauer beschrieben.

 

Synchronisationsfunktion der Steuerelemente

Für jede SPS-Verbindung gibt es im LC2 eine Synchronisationsfunktion, mit der Steuerungen synchronisiert/initialisiert werden können.

Diese Funktion können Sie im LC2 im Bereich Anwendungsstruktur und der Datenaustausch-Tabelle (P_CONNECTIONS) finden:

 

 

Durch das Auswählen des Kontrollkästchens in der Spalte Sync aktivieren Sie die Synchronisation.

 

:

Das Aktivieren der Synchronisationsfunktion hat nur eine Auswirkung, wenn diese im entsprechenden ITC-Projekt programmiert und somit festgelegt ist, wie mit der aktivierten Synchronisation weiter verfahren werden soll. Dies ist die Voraussetzung für das Funktionieren der Sync-Funktion.
