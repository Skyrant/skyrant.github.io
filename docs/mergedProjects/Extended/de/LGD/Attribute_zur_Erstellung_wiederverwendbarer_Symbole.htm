Attribute zur Erstellung wiederverwendbarer Symbole

Attribute sind ein nützliches Tool, wenn Symbole (einzelne Elemente oder Gruppen von Elementen) mit gleichem Aussehen und Verhalten / Bindings gebraucht werden, nur für unterschiedliche Daten. Attribute für jede Instanz solcher Elemente / Gruppen können einzeln festgelegt werden und können Attribut-Platzhalter in Binding-Ausdrücken während der Laufzeit ersetzen. Die Grundidee ist, dieses Symbol mehrmals in einer oder mehreren Zeichnungen zu verwenden – die einzelnen Instanzen sollten nicht verändert werden, z. B. Festlegen eines individuellen Textes oder Ändern von Bindings um spezifische Datenobjekte / -variablen zu adressieren. Stattdessen sollten individuelle Einstellungen einfach über Symbolattribute parametriert werden.

Das Speichern solch wiederverwendbarer Symbole zeigt den wichtigsten Vorteil: Die Möglichkeit, das Bibliothekssymbol später zu ändern (z. B. Bindings erweitern, zusätzliche Elemente / Details zu einer Gruppe hinzufügen) und diese Änderungen auf alle bereits in einer Zeichnung existierenden Symbolinstanzen anzuwenden, ohne die individuellen Attributwerte zu verlieren.

Um das Konzept der Attributverwendung zu verdeutlichen, wird im Folgenden ein einfaches Symbol für einen Puffer erstellt. Dieses Symbol sollte eine individuelle Legato Variable adressieren, die das aktuelle Fülllevel enthält. Für jede Symbolinstanz sollte individuell ein Titel für das Element sowie die maximale Kapazität festgelegt werden.

Das Symbol sollte nur ein Textfeld mit einem individuellen Titel und einem Fortschrittsbalken, der das aktuelle Fülllevel in Prozentzahlen anzeigt, enthalten:

Der Titel und der Fortschrittsbalken werden gruppiert um später als ein Symbol verwendet zu werden.

Definition eines Attributes

Attribute müssen zuerst für das gewünschte Element definiert werden – in unserem Beispiel, die ganze Gruppe. Wenn Attribute für eine Gruppe erklärt und festgelegt werden, können alle Elemente innerhalb dieser Gruppe diese Attribute verwenden. Auf den Dialog zur Bearbeitung von Attributdefinitionen kann über das Kontextmenü des Elements Sapient Attributdefinitionen bearbeiten zugegriffen werden. Attribute werden in JSON-Format definiert. Auf jedes Attribut wird über eine Kennung zugegriffen (der Objektname für jedes Attribut befindet sich in der Definition). Für jedes Attribut muss ein Anzeigename und ein Datentyp spezifiziert werden. Snippets stellen einen guten Startpunkt dar (STRG + rechter Mausklick) – vor allem das Snippet Beispiele / Längeres Beispiel ist ein mittelkomplexes, das eine typische Konfiguration anzeigt, die einige verschiedene Datentypen enthält:

Das Beispiel kann als anpassbare Vorlage verwendet werden. In diesem Fall stimmen der Titel und die Pufferkapazität, also wird das conSym Attribut nur umbenannt, um eine Spezifizierung des Symbols der Pufferlevel-Variable für jede Symbolinstanz zu ermöglichen:

{

         "title": {

                 "name": "Title",

                 "type": "string"

         },

         "varSym": {

                 "name": "Buffer (variable symbol)",

                 "type": "string"

         },

         "bufferCap": {

                 "name": "Buffer capacity",

                 "type": "int"

         }

}

 

 

Attribut Datentypen:

  • string: Ermöglicht die Eingabe jedes Strings als Attributwert.

  • int: Ermöglicht die Eingabe eines integren Werts – zeigt eine numerisches Eingabefeld im Attributdialog an.

  • bool: Ermöglicht die Eingabe eines booleschen Wertes – zeigt eine Checkbox im Attributdialog an.

 

Wichtig: Attributnamen müssen valide Kennungen ohne Punkte oder Sonderzeichen wie deutsche Umlaute etc. sein.

Symbole aus der Bibliothek zu einer Zeichnung hinzufügen

Indem Symbole mit Attributen in eine Zeichnung außerhalb der Bibliothek hinzugefügt werden, wird der Attributwertedialog automatisch für jede neue Symbolinstanz angezeigt, sodass die individuellen Symbolattribute direkt eingegeben werden können.

Nachdem ein Bibliotheksymbol später geändert wurde und alle Symbolinstanzen innerhalb einer Zeichnung aktualisiert wurden, werden die Symbole aktualisiert, während die individuellen Attributwerte wie Position oder Größe der Symbole in der Zeichnung beibehalten werden.
