Referenz: Datasources, Bindings und Actions

Bindings

Datasources

Variable

Liefert den Wert und zusätzliche Attribute einer Legato-Variablen auf Serverseite.

"var": {

"source": "var",

"params": {

"sym": "varSym"

}

}

 

Durch Referenzieren eines Datenobjekts wird der Wert der Variablen zurückgegeben. Das Ergebnisobjekt enthält außerdem den Zeitpunkt des letzten Updates der Variablen, zugänglich gemacht durch Referenzieren der Eigenschaft lastUpdate über die Datenreferenz.

Beispiel:

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "examples.n1"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "someVar",

                    "path": "lastUpdate"

                },

            "format":"datems YYYY-MM-DD HH:mm:ss"

            }

        }

    }

}

Für fortgeschrittenere Anwendungsfälle (z. B. Kalkulation der Zeitspanne seit der letzten Änderung an der Variable) ermöglicht die Eigenschaft updateTime den Zugriff auf ein Datums-/Zeitobjekt, das die Eigenschaften unterstützt.

• unix: Sekunden seit 1970

• milliseconds: Millisekunden seit 1970

• dateTime: Datum-/Zeitstring, der die aktuelle Einstellung verwendet

• date: Datumstring, der die aktuelle Einstellung verwendet

• time: Zeitstring

• timeMs: Zeitstring einschließlich Millisekunden

Beispiel (Zeitstempel des letzten Updates in Sekunden seit 1970 erhalten):

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "ADTest1.num"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "someVar",

                    "path": "updateTime.unix"

                }

            }

        }

    }

}

Berechtigung

Ermöglicht den Zugriff auf Berechtigungen und gibt einen boolischen Wert true zurück, wenn der aktuelle Nutzer die Berechtigung zur Durchführung mindestens einer bestimmten Aktion hat.

{

  "source": "permission",

  "params": {

    "actions": [

      "PV_SET_VALUE"

    ]

  }

}

 

 

 

Es können ausschließlich knotenbezogene Aktionen genutzt werden. Die Berechtigungen werden auf den logischen Strukturknoten geprüft, zu dem das HMI-Projekt gehört.

 

Client

Stellt die bestimmte Eigenschaft des aktuellen HMI-Klienten zur Verfügung (Stationinformation).

 

{

  "source": "client",

  "params": {

    "property": "host_name"

  }

}

 

 

 

Klienteninformation ist nur verfügbar, wenn die HMI-Anwendung als zugeordnete Station gestartet wird.

Skript

Stellt das Ergebnis eines serverseitigen Skripts zu Verfügung.

{

    "source": "script",

    "params": {

        "name": "con_config",

        "scriptParams": {

            "connectionSymbol": "AD_Test1"

        },

        "interval": 30

    }

}

 

Diese Skripte müssen als Nutzerskripte in Sapient Engine implementiert werden (Pfad customscripts/visu). Indem der Name und die Parameter des Skripts festgelegt werden, kann das vorgegebene Skript während der Laufzeit im gewünschten Abfrageintervall zyklisch abgerufen werden. Der Standardwert für das Intervall ist 10 Sekunden und ist auch auf 10 Sekunden beschränkt – schnellere Aufrufe sind nicht möglich, um zu kurze Abfrageintervalle zu vermeiden, die die Leistung des Systems beeinträchtigen könnten. Wenn sich ein Eingabeparameter ändert, wird das Skript umgehend ausgeführt.

 

Dataref

Um ein beliebiges Datenobjekt zu referenzieren, kann ein Dataref-Objekt verwendet werden. Die Struktur ist ähnlich zu der eines Datenobjekts:

{

    "source": "dataref",

    "params": {

        "ref": "lv"

    },

    "format": "d0.##"

}

 

Datarefs unterstützen das optionale Eigenschaftsformat, das ein numerisches Format festlegt, um z. B. eine bestimmte Anzahl von Dezimalstellen für Fließwerte anzuzeigen. Eine Beschreibung von Datenformaten kann in der Sapient Systemhilfe gefunden werden, unter dem Suchbegriff „Display Formats“. Zusätzlich zu diesen Standardformaten können spezielle Formatstrings verwendet werden, um einen unix Zeitstempel in einem speziellen Datum-Zeit Format anzuzeigen (entweder Sekunden oder Millisekunden seit 1970).

 

Beispiele:

• "datems YYYY-MM-DD HH:mm:ss"

--> z. B. 2021-11-29 17:23:55 (erwartet ms seit 1970)

 

• "date YYYY-MM-DD HH:mm:ss"

--> z. B. 2021-11-29 17:23:55 (erwartet sek seit 1970)

 

• "time"

--> z. B. 27:45:34 um eine Zeitspanne in Sekunden als einen Zeitstring anzuzeigen

 

• "timems"

--> ähnlich wie “time”, aber erwartet eine Zeitspanne in Millisekunden

 

Tipp: Formatierung kann entweder verwendet werden, um ein Datenobjekt an die Eigenschaft eines Textes zu knüpfen, oder sie kann als Berechnungsparameter verwendet werden.

Wenn Datenobjekte mit einem Objektwert aufgerufen werden, kann auf einzelne Eigenschaften innerhalb des Objekt zugegriffen werden, indem eine Pfadeigenschaft festgelegt wird. Ein typischer Anwendungsfall sind Knotenzustände, die den aktuell wichtigsten Alarm eines logischen Strukturknotens darstellen.

{

    "datasources": {

        "nodestate": {

            "source": "nodestate",

            "params": {

                "node": 5

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "nodestate",

                    "path": "text"  access the property text of the node state object

                }

            },

            "defaultValue": ""

        }

    }

}

Reihe

Die Reihenfunktionen prüfen ob some / any / every / not every der Eingabevariablen logisch wahr sind. Eingabewerte können Zahlen sein – Werte über 0 werden als logisch wahr interpretiert. Die Funktion gibt ein boolisches Ergebnis zurück. Es können optional ein Ergebnis und ein Standardwert festgelegt werden, die dann für „wahr“ oder „falsch“ Fälle zurückgegeben werden.

Das Binding sieht genau gleich aus, ändern Sie nur den Funktionsnamen in

• some: Einige Eingabewerte sind logisch wahr.

• any: Jeder beliebige Eingabewert ist logisch wahr.

• Every: Jeder Eingabewert ist logisch wahr.

• not every: Nicht jeder Eingabewert ist logisch wahr.

{

    "source": "function",

    "params": {

        "name": "some",

        "params": {

            "values": [

                {

                    "source": "dataref",

                    "params": {

                        "ref": "v1"

                    }

                },

                {

                    "source": "dataref",

                    "params": {

                        "ref": "v2"

                    }

                },

                {

                    "source": "dataref",

                    "params": {

                        "ref": "v3"

                    }

                }

            ],

            "result": "some"

        }

    },

    "defaultValue": "none"

}

Für bitweise Verfahren (or, and, xor, not) ist die Binding-Syntax äquivalent (ändern Sie einfach den Namen der jeweiligen Funktion). Der optionale Parameter kann verwendet werden, um die Anwendung „signed“ auszuführen (interpretieren sie das Bitfeld als vorzeichenbehaftet – wenn das höchste Bit gesetzt ist (Bit 31), würde das in einem negativen Wert resultieren.)

{

    "source": "function",

    "params": {

        "name": "bitor",

        "params": {

            "values": [

                1,

                4,

                8,

                16,

                32,

                64

            ],

            "signed": false

        }

    }

}

Datum

Numerischer Zeitstempel

Gibt den aktuellen Zeitstempel als eine Zahl zurück. Setzt die Einheit der Eigenschaft auf ms oder s um zu spezifizieren, ob der Zeitstempel als Wert in Millisekunden oder Sekunden zurückgegeben wird. Der Wert repräsentiert die Millisekunden oder Sekunden zwischen 1. Januar 1970 00:00:00 UTC und dem aktuellen Datum.

{

    "source": "function",

    "params": {

        "name": "currentDate",

        "params": {

            "unit": "ms"

        }

    }

}

 

 

Tipp: Die Formateigenschaft eines Textbindings kann verwendet werden, um Zeitstempel in einen Datum-Zeit-String zu formatieren.

 

Umschalten

Das Ergebnis der Umschaltfunktion schaltet im 1-Sekunden-Intervall dynamisch zwischen zwei oder mehreren Werten um. Wenn der Auslösewert logisch wahr ist, ändert sich das Ergebnis dynamisch, ansonsten wird der defaultValue zurückgegeben – falsch falls nicht explizit spezifiziert. Die Funktion kann verwendet werden, um Elemente blinken zu lassen, indem das Ergebnis an die Sichtbarkeit eines Elements gebunden wird, oder um einen Stil wie die Füllfarbe dynamisch zu ändern.

Schalten Sie einfach zwischen falsch --> wahr --> falsch --> wahr … um.

{

    "source": "function",

    "params": {

        "name": "toggle",

        "params": {

            "trigger": {

                "source": "dataref",

                "params": {

                    "ref": "someVar"

                }

            }

        }

    }

}

 

Zwischen mehreren Werten wechseln …

{

    "value": {

        "source": "function",

        "params": {

            "name": "toggle",

            "params": {

                "trigger": {

                    "source": "dataref",

                    "params": {

                        "ref": "state"

                    }

                },

                "values": [

                    "#FFFF00",

                    "#00FFFF",

                    "#FF00FF"

                ]

            }

        },

        "defaultValue": "none"

    }

}

HTTP Post an lokalen Server

Mit HTTP Post an einen lokalen Server ist der Aufruf eines externen Programms möglich.

Die URL hierzu ist

 

https://localhost:4748/run/ ...... { "ExeFileLocation": "C: Program Files\\XYZ app.exe", "RunAsAdmin": "0" }

 

Der Aufruf eines externen Programms ist nur lokal (-> localhost) möglich.

 

Beachten Sie:

  • Die Ergebnisse des Aufrufs durch das externe Programm müssen nicht an die HMI-Schnittstelle zurückgesendet werden.

  • Das Aktivieren/Deaktivieren von Aufrufen ist möglich (z. B. über Berechtigungen, die als "als dedizierte Station ausgeführt" werden).

  • Es wird sichergestellt, dass immer nur ein Post ausgeführt wird.

  • Der Option Post Body wird als Json oder Form/Url kodiert.

 

Es gibt zwei Visualisierungsaktionen hierzu: httpPost und cancelCurrenHttpRequest.
