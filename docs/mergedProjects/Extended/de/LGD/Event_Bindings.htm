Event Bindings

Events

onClick

Wird ausgeführt, wenn ein Element angeklickt wird.

onMouseDown

Wird ausgeführt, wenn die Maustaste auf einem Element gedrückt wird.

onMouseUp

Wird ausgeführt, wenn die Maus auf einem Element losgelassen wird.

onMouseEnter

Wird ausgeführt, wenn der Mauscursor in die Grenzen eines Elements hineinbewegt wird.

onMouseLeave

Wird ausgeführt, wenn der Mauscursor aus den Grenzen eines Elements hinausbewegt wird.

onLoad

Wird ausgeführt, wenn ein Dokument geladen wird. Dieses Event kann nur im Binding-Ausdruck eines Dokuments verwendet werden.

onUnload

Wird ausgeführt, wenn ein Dokument geschlossen wird. Dieses Event kann nur im Binding-Ausdruck eines Dokuments verwendet werden.

onChange

Wird ausgeführt, wenn sich ein Wert ändert.

Beispiel:

{

    "datasources": {

        "lv1": {

            "source": "localvar",

            "params": {

                "sym": "test"

            }

        },

        "lvtrigger": {

            "source": "localvar",

            "params": {

                "sym": "trigger"

            }

        }

    },

    "events": {

        "onChange": [

            {

                "trigger": {

                    "source": "dataref",

                    "params": {

                        "ref": "lvtrigger"

                    }

                },

                "actions": [

                    {

                        "action": "incrementValue",

                        "params": {

                            "target": {

                                "source": "dataref",

                                "params": {

                                    "ref": "lv1"

                                }

                            }

                        }

                    }

                ]

            }

        ]

    }

}

Angenommen, dass der Trigger, der umgeschaltet wird, ein boolischer Wert ist, wird er als Trigger für die onChange Funktion verwendet – das bedeutet, dass das Event ausgeführt wird, wann immer sich eine Variable ändert. Eine andere Variable (test) wird als Aktion inkrementiert.

Wie für alle Aktionen kann eine Liste von Aktionen festgelegt werden. Jede Aktion kann eine optionale, aktivierte Eigenschaft haben.

In vielen Fällen sollten einige Aktionen für einen Wertewechsel nur unter bestimmten Konditionen ausgeführt werden. Um das oben gezeigte Beispiel so zu ändern, dass nur die Aktion ausgeführt wird, wenn der Trigger-Wert auf wahr gesetzt ist, ist die einfachste Lösung, ausschließlich die Aktion zu aktivieren, wenn der Trigger gesetzt ist:

{

    "datasources": {

        "lv1": {

            "source": "localvar",

            "params": {

                "sym": "test"

            }

        },

        "lvtrigger": {

            "source": "localvar",

            "params": {

                "sym": "trigger"

            }

        }

    },

    "events": {

        "onChange": [

            {

                "trigger": {

                    "source": "dataref",

                    "params": {

                        "ref": "lvtrigger"

                    }

                },

                "actions": [

                    {

                        "action": "incrementValue",

                        "params": {

                            "target": {

                                "source": "dataref",

                                "params": {

                                    "ref": "lv1"

                                }

                            }

                        },

                        "enabled": {

                            "source": "dataref",

                            "params": {

                                "ref": "lvtrigger"

                            }

                        }

                    }

                ]

            }

        ]

    }

}

 

Anstatt direkt eine Variable für die aktivierte Eigenschaft zu verwenden, kann jedes andere Datenobjekt oder -ergebnis einer Berechnung oder Formel verwendet werden.

 

Aktionen

Navigation

loadSubView

Ermöglicht den Wechsel zu einer anderen Zeichnung.

{

    "action": "loadSubView",

    "params": {

        "fileName": "hmi/ws0/drawing2.xml"

    }

}

loadHomeView

Nach dem Wechsel zu einer anderen Zeichnung kann diese Aktion verwendet werden, um wieder zur ursprünglich geladenen Zeichnung zu wechseln.

{

"action": "loadHomeView"

}

prevView, nextView

Die Zeichnungen, die nacheinander mit der loadSubView-Aktion aufgerufen werden, werden in einer Liste gespeichert. Die prevView-Aktion kann die zuvor angezeigte Zeichnung laden, nextView geht wieder vorwärts in der Liste.

in der Liste weiter.{

"action": "prevView"

}

 

{

"action": "nextView"

}

 

changeDashboard

Wird meist für Zeichnungen verwendet, die in einem Dashboard der Standard-Sapient-App verwendet werden, um ein anderes Dashboard zu laden. Die Dashboard-ID wird verwendet, um das Dashboard zu identifizieren. Optional
kann ein bestimmtes Dashboard-Profil geladen werden. Beide Identifier können einfach aus der URL der Sapient-App kopiert werden, wenn das gewünschte Dashboard und Profil geöffnet wird:

{

    "action": "changeDashboard",

    "params": {

        "dashboardId": "6d50e746-552f-416c-ab79-de43350f86f3",

        "profileId": "753f0d6d-6e0c-442c-95a3-b3b7fcfea243",

        "dashboardParameters": {

            "node": 90,

            "nodeList": [

                90

            ],

            "RECIPE_ID": 80

        }

    }

}

 

openDashboardInNewTab

Öffnet die Sapient-Anwendung in einem separaten Browserfenster und lädt das angegebene Dashboard und optional das Profil.

{

    "action": "openDashboardInNewTab",

    "params": {

        "dashboardId": "6d50e746-552f-416c-ab79-de43350f86f3",

        "profileId": "753f0d6d-6e0c-442c-95a3-b3b7fcfea243",

        "parameters": {

            "node": 90,

            "nodeList": [

                90

            ],

            "RECIPE_ID": 80

        }

    }

}

 

openDrawingInWorkarea

Im Grunde ähnlich wie loadSubView, öffnet die Zeichnung aber immer im Hauptbereich einer HMI-Anwendung, unabhängig davon, wo sie aufgerufen wird. Ein typischer Anwendungsfall für diese Aktion ist das Hinzufügen von Verknüpfungen zu wichtigen Ansichten in der Fußzeile.

{

    "events": {

        "onClick": [

            {

                "action": "openDrawingInWorkarea",

                "params": {

                    "fileName": "hmi/ws0/detail/drawing2.xml"

                }

            }

        ]

    }

}

 

openLinkInWorkarea

Ermöglicht das Öffnen einer externen Website oder eines Dokuments im Hauptbereich der HMI-Anwendung. Über den optionalen Parameter newTab wird der Link in einem neuen Browser-Tab geöffnet.

{

    "action": "openlinkinworkarea",

    "params": {

        "url": "http://www.gefasoft.de",

        "newTab": true

    }

}

Values

setValue

Dient zum Setzen eines Variablenwertes (Legato-Variable, Session- oder lokale Variable). Wenn ein Wert direkt für die Aktion definiert ist, wird der Wert direkt gesetzt, andernfalls wird der Benutzer aufgefordert, einen Wert über die Seitenleiste einzugeben.

{

    "datasources": {

        "someVar": {

            "source": "var",

            "params": {

                "sym": "examples.str1"

            }

        }

    },

    "events": {

        "onClick": [

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "someVar"

                        }

                    },

                    "value": "HELLO!!!"

                }

            }

        ]

    }

}

 

Um die Erfolgsmeldung ("Toast") nach dem erfolgreichen Schreiben des Wertes zu deaktivieren, setzen Sie die Eigenschaft showUserFeedback auf false.

Weitere optionale Eigenschaften werden nur bei der Eingabe eines Wertes über die Seitenleiste verwendet, um z.B. die Eingabe von numerischen Werten innerhalb eines bestimmten Bereichs zu gewährleisten:

  • type: einen Datentyp (irrelevant für Legato-Variablen - für diese ist der Typ bereits serverseitig definiert) auf der Server-Seite definiert. Empfohlen für lokale und Session-Variablen).

o   str: String Wert

o   int: Integer Wert

o   float: Float Wert

o   bool: Boolean Wert

  • min: Untergrenze für den eingegebenen Wert

  • max: Obergrenze für den eingegebenen Wert

  • caption: Beschriftung für das Eingabefeld in der Seitenleiste

Beispiel:

{

    "action": "setValue",

    "params": {

        "target": {

            "source": "dataref",

            "params": {

                "ref": "somevar"

            }

        },

        "type": "int",

        "min": 3,

        "max": 10,

        "caption": "Value [3..10]",

        "showUserFeedback": false

    }

}

 

Damit der Benutzer einen Wert auswählen kann, indem er einen Eintrag aus einer Nachschlageliste wählt, kann eine Schlüssel-Wert-Liste

Liste an die Parameterelemente übergeben werden.

{

    "datasources": {

        "lookupList": {

            "source": "localvar",

            "params": {

                "sym": "lookuplist"

            }

        },

        "key": {

            "source": "localvar",

            "params": {

                "sym": "lookupkey"

            }

        }

    },

    "events": {

        "onClick": [

            {

                "action": "setValue",

                "params": {

                    "target": {

                        "source": "dataref",

                        "params": {

                            "ref": "key"

                        }

                    },

                    "items": {

                        "source": "dataref",

                        "params": {

                            "ref": "lookupList"

                        }

                    }

                }

            }

        ]

    }

}

 

Der Aufruf dieser Aktion zeigt eine Dropdown-Box in der Seitenleiste an, aus der ein Element ausgewählt werden kann. Intern wird der entsprechende Schlüssel gesetzt (Parameter target).

 

toggleValue

Schaltet den Wert der referenzierten Variablen um (false --> true, true --> false).

{

    "action": "toggleValue",

    "params": {

        "target": {

            "source": "dataref",

            "params": {

                "ref": "someVar"

            }

        }

    }

}

 

Diese Aktion funktioniert nur bei lokalen und Session-Var zuverlässig. Insbesondere bei der Verwendung dieser Aktion für Legato-Variablen in einem onLoad-Ereignis - die Variable könnte noch in einem undefinierten Zustand sein, wenn die Aktion ausgeführt wird, da neu abonnierte Variablen ihre Werte asynchron gesendet bekommen.

incrementValue, decrementValue

Erhöht den Wert einer numerischen Variablen.

{

    "action": "incrementvalue",

    "params": {

        "target": {

            "source": "dataref",

            "params": {

                "ref": "someVar"

            }

        }

    }

}

 

Optional können ein Bereich und ein Delta definiert werden:

  • delta: Das Inkrement (default: 1)

  • min: Startwert

  • max: Maximaler Wert - bei Erreichen des Wertes beginnt der Wert bei Null oder dem angegebenen Mindestwert

Diese Aktion funktioniert nur bei lokalen und Session-Var zuverlässig. Insbesondere bei der Verwendung dieser Aktion für Legato-Variablen in einem onLoad-Ereignis - die Variable könnte noch in einem undefinierten Zustand sein, wenn die Aktion ausgeführt wird, da neu abonnierte Variablen ihre Werte asynchron gesendet bekommen.

 

changeNode, changeNodes

Wird typischerweise für Zeichnungen in einem Dashboard in der Standard-Sapient-App verwendet, um die aktuelle nodeSelection zu ändern.

{

    "action": "changeNode",

    "params": {

        "node": 90

    }

}

 

{

    "action": "changeNode",

    "params": {

        "nodes": [

            1,

            3,

            90

        ]

    }

}

 

Sidebar

Zeichnungen können in der Seitenleiste angezeigt werden, um Vorgänge zu steuern oder einfach nur um bestimmte Detailinformationen. Es können mehrere Zeichnungen gleichzeitig in der Seitenleiste angezeigt werden - optional in separaten Registerkarten organisiert.

openDrawingInSidebar

Öffnet eine Zeichnung in der Sidebar.

{

    "action": "openDiagramInSidebar",

    "params": {

        "fileName": "hmi/examples/sidebar2_sb1.xml",

        "id": "SB1",

        "preventDuplicates": true,

        "tabId": 1,

        "tabIcon": "icon-view-list",

        "params": {

            "prop1": "xy",

            "prop2": 12

        }

    }

}

 

Die id kann übergeben werden, um der Zeichnung in der Seitenleiste eine ID zuzuweisen. Dies ist notwendig, um die Zeichnung zu identifizieren, um sie mit der Aktion removeDrawingFromSidebar zu entfernen, entfernt werden kann. Zusätzlich, um optional sicherzustellen, dass die Zeichnung nur einmal angezeigt wird (preventDuplicates = true).


Die folgenden Parameter fügen die Zeichnung optional zu einer bestimmten Registerkarte der Seitenleiste hinzu:

  • tabId: Identifier, um Zeichnungen in der Seitenleiste auf derselben Registerkarte hinzufügen zu können.

  • tabIcon: Icon, das für die Registerkarte angezeigt werden soll. Die verfügbaren Icons sind auf der Seite http://serverurl/app/assets/icons/demo.html aufgelistet. (Symbolnamen, die Unterstriche enthalten, können nicht verwendet werden)

Die Eigenschaften in params werden direkt an die Zeichnungsinstanz weitergegeben und können über den Pseudoparameter extra zugegriffen werden.

Beispiel für eine Bindung mit extra Parametern:

{

    "datasources": {

        "prop1": {

            "source": "parameter",

            "params": {

                "path": "extra.value.prop1"

            }

        }

    },

    "bindings": {

        "text": {

            "value": {

                "source": "dataref",

                "params": {

                    "ref": "prop1"

                },

                "defaultValue": "/"

            }

        }

    }

}

 

clearSidebar

Entfernt alle Zeichnungen aus der Seitenleiste und schließt die Seitenleiste.

{

    "action": "clearSidebar"

}

removeDrawingFromSidebar

Entfernt eine Zeichnung mit der angegebenen ID aus der Seitenleiste.

{

    "action": "removeDrawingFromSidebar",

    "params": {

        "id": "SB1"

    }

}

 

setSidebarProperties

Legt bestimmte Eigenschaften der Seitenleiste fest - vor allem, um die Seitenleiste auf eine Breite einzustellen, die für die anzuzeigende Zeichnung.

{

    "action": "setSidebarProperties",

    "params": {

        "width": "large"  small, medium, large, or spec. width e.g. 200px

    }

}

Die Breite kann eine bestimmte Breite in Pixel sein oder die vordefinierten Breiten klein, medium, groß.

resetSidebarProperties

Setzt alle Eigenschaften der Seitenleiste auf die für den aktuellen Benutzer eingerichteten Eigenschaften zurück.

{

    "action": "resetsidebarproperies"

}

 

Sonstige Aktionen

callRpc

Ermöglicht den Aufruf einer Prozedur der Sapient Engine. Parameter für den Prozeduraufruf werden über das params-Objekt übergeben.

{

    "action": "callRpc",

    "params": {

        "method": "de/gefasoft/sapient/db/values/Current/num_in",

        "params": {

            "ID": 7500,

            "VALUE_NUM": 13

        }

    }

 

confirmExecution

Ermöglicht es, die Ausführung von Aktionen durch den Benutzer bestätigen zu lassen. Wenn der Benutzer abbricht, werden die folgenden Aktionen übersprungen.

{

    "action": "confirmExecution",

    "params": {

        "message": "This action might have serious consequences!"

    }

}

 

Diese Aktion muss die erste in der Liste der Aktionen sein.

 

playAudio

 

Die Aktion playAudio akzeptiert zwei Parameter file und loop. Audiodateien werden im gemeinsamen Verzeichnis  detail-layout\audio abgelegt.

 

  • Ist der Parameter file leer, wird das Abspielen der aktuellen Audiodatei gestoppt.
  • loop legt fest, dass Audio in Schleife abgespielt werden soll. Der Standardwert ist false.

Example:

 {         

    "events": {                

        "onClick: [

            {

                "action": "playAudio", 

                "params": {

                    "file": "wilhelmscream.mp3",

                        "loop": false   

                    }

               }

           ]         

       }

  }

Browsers do not play sounds without user interaction by default. Please consider to set up your browser accordingly, e.g. for chrome use  "chrome.exe --autoplay-policy=no-user-gesture-required". Also, check if your browser is capable to do so.

 

httpPost

Sendet eine HTTP-Anfrage an eine bestimmte URL. Es kann jeweils nur eine Anfrage gestellt werden. Akzeptiert sind drei Parameter:

 

  • url

  • bodyType - json (Standard) oder Form

  • params -

 

{

    "events": {

        "onClick": [

            {

                "action": "httpPost",

                "params": {

                    "url": "http://localhost:808",

                    "params": {

                        "key":"value"

                    }

                }

            }

        ]

    }

}

 

 

cancelCurrentHttpRequest 

Storniert die aktuell anstehende Anfrage.
