Datenbank-Gateway (bis Version 4.7.2.4)

Installation auf einem Windows PC

Systemvoraussetzungen

Das Gateway ist eine 32 Bit Applikation.

 

:

Für den Betrieb unter Windows gilt die Einschränkung, dass aus Performancegründen und zur Gewährleistung eines stabilen Betriebsverhaltens, nicht mehr als 50 Verbindungen je Gateway-PC angelegt werden sollen. Das ist bei der Projektierung des Gesamtsystems und Festlegung der erforderlichen Gateway-PCs zu berücksichtigen. Für den Betrieb des Datenbankgateways auf UNIX-Plattformen gelten keine Einschränkungen.

Zusätzlich

Oracle Client und Provider for OLE DB ab Version 11.2.0.1.0. 32-bit.

:

Beim Oracle Client 11.2.0.3 kommt es gelegentlich zu Problemen, da der OLE DB Provider nicht korrekt installiert wird. Daher wird die Version 11.2.0.1 empfohlen.

:

Bei Gateways mit hohen Datendurchsatz wird empfohlen, die Datei sqlnet.ora vom Oracle Treiber zu suchen und dort den Parameter

Sqlnet.outbound_connect_timeout=5

zu setzen, damit auch im Fehlerfall der Ablauf gewährleistet bleibt.

Installation Datenbank-Gateway

Die Installation des Datenbankgateways erfolgt über das Installationsprogramm GatewaySetup.exe. Nach dem Start der Exe kann die Sprache der Installation gewählt werden.

Installation OPC-Server GOPS

Wird die Gateway-Installation zur Laufzeit (online) als Win32-Gateway betrieben, sind ggf. der OPC-Server GOPS und die zugehörigen Treiber ebenfalls am Gateway-PC zu installieren. Dies erfolgt über die Datei GopsSetup.exe. Es empfiehlt sich, die Installation von Gateway und GOPS gegebenenfalls in einem Schritt durchzuführen.

:

Wird im Projekt eine Kopplung über OPC gefordert, kann ausschließlich die Windows-Variante des Gateways eingesetzt werden. OPC wird von UNIX nicht unterstützt.

Installation für UNIX

Systemvoraussetzungen

Betriebssysteme

Das Gateway ist als 32 Bit und 64 Bit Applikation verfügbar.

 

Datenbanken

ORACLE11

 

Das Gateway muss auf der Zielmaschine kompiliert und gelinkt werden (eine entsprechende Entwicklungsumgebung mit GNU C++ Compiler ist erforderlich).

Zur Installation des Gateways müssen die Bibliotheken, die nicht auf dem System ohnehin schon vorhanden sind, inkl. dem Gateway Binary, in ein eigenes Verzeichnis kopiert werden (z.B. legato/bin) und die Ausführungsrechte gesetzt werden.

Der Bibliothekspfad muss das o. g. Installationsverzeichnis enthalten und der Oracle Client muss installiert sein.

Registrierung

Die Registrierung (und damit Freischaltung) des Gateways (und des OPC-Servers mit den verwendeten Treibern) erfolgt über den Menüpunkt "Legato|Lizenzinfo". Dort kann ein Dialog mit den aktuell vorhandenen Lizenzen aufgerufen werden. Mit dem Button "Registrierung" wird ein neuer Dialog für die Registrierung geöffnet:

Seriennummer

Eingabe der Seriennummer für das Produkt aus dem Original-Lieferschein.

Registrierungsschlüssel

Eingabe des Registrierungs-Schlüssels für das Produkt aus dem Original-Lieferschein.

 

:

Für die Registrierung des Gateways, des OPC-Servers und der Treiber sind unter Umständen mehrere Schlüssel erforderlich. Der Registrierungsvorgang muss auf jeden Fall für jeden übermittelten Schlüssel durchgeführt werden.
