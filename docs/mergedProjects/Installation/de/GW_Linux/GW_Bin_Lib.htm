Gateway Binary und Bibliotheken

Nach dem Compilieren/Linken muss in dem vom Kunden vorgegebenen Verzeichnis (dieses wird im Folgenden als <APP_DIR> bezeichnet) das Unterverzeichnis bin erstellt werden.

Gateway 32 Bit

In das Verzeichnis <APP_DIR>/bin müssen nun folgende Dateien kopiert werden:

<BASE_DIR>/GatewayIx/gateway

$ACE_ROOT/ace/libACE.so.6.2.0

<BASE_DIR>/3rdParty/SAP/RfcSdk.720.LINUX/lib/librfccm.so

 

Gateway 64 Bit

In das Verzeichnis <APP_DIR>/bin müssen nun folgende Dateien kopiert werden:

<BASE_DIR>/GatewayIx/gateway_64

$ACE_ROOT/ace/libACE.so.6.3.0

<BASE_DIR>/3rdParty/SAP/RfcSdk.720.LINUX_64/lib/librfccm.so

 

Der Pfad <APP_DIR>/bin muss in die Umgebungsvariable LD_LIBRARY_PATH aufgenommen werden.

Abhängig von der Kundenanforderung und des/der verwendeten Gatewayprojekts/Gatewayprojekte müssen weitere Verzeichnisse für Logdateien, Skripten, Gatewayprojekte… angelegt werden. Da dies z. T. sehr unterschiedlich ist, ist dies Teil des jeweiligen Projekts und nicht dieser Dokumentation.

Entsprechend der jeweiligen Projektanforderung können folgende Gatewayrechner als Beispiel dienen:

audibxug0010.bx.audi.prod.vwg (Audi Brüssel Montage)

audibxug0015.bx.audi.prod.vwg (Audi Brüssel KB)

vwsmmovmo305 (VW Zwickau KB)

vwskbaubav24 (VW Bratislava)

vwlegato001 (VWGTO Silao)
