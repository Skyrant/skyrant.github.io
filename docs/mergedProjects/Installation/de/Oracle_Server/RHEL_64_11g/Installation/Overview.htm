Übersicht über die Installation

Ziel der Installation des Oracle-Serversystems ist ein laufendes RDBMS inklusive Listener, aber ohne Datenbankinstanz. Dazu wird der Installationsprozess in den folgenden Einzelschritten durchgeführt:

1.  Pre-Installation (Anbindung des Oracle-Repository, Pre-Installation-Programm herunterladen und ausführen)

2.  Installation des RDBMS (Herunterladen der Installationsdateien, Vorbereitung von Verzeichnissen und Systemdateien, grafische Installation)

3.  Konfiguration der Login-Datei und Starten des Listeners

Unterstützt wird aktuell die Oracle-Version 11gR2.

In dieser Dokumentation wird vom Einsatz der Enterprise-Edition ausgegangen. Diese Version ist für den Betrieb von Legato empfohlen und kann kostenpflichtig bei Oracle erworben werden.

Prinzipiell ist es jedoch auch möglich, Legato mit einer Standard-Oracle-Version zu betreiben. Hier sind allerdings einige empfohlene Features nicht verfügbar.

Einige Oracle-Versionen können kostenlos heruntergeladen werden unter:

https://edelivery.oracle.com/

:

Hier sind nicht die neuesten Versionen vorhanden. So ist dort zum Beispiel die Version 11.2.0.1.0 vorhanden, allerdings nicht die von Oracle empfohlene Version 11.2.0.3.0 des Oracleservers. Diese Versionen findet man im kostenpflichtigen Oracle Metalink (siehe nächste Seite).

Im Oracle Metalink können Service Requests gestellt werden, neueste Software-Updates geholt werden oder auch nach Hilfen gesucht werden. Der Metalink ist jedoch nur für Oracle-Partner zugänglich und grundsätzlich kostenpflichtig. Wir empfehlen den Zugang zum Metalink, um stets die aktuellsten Versionen verfügbar zu haben. Für diese Anleitung wird ebenfalls von einem Zugang zum Metalink ausgegangen.

Der Oracle Metalink ist zu erreichen unter http://support.oracle.com
