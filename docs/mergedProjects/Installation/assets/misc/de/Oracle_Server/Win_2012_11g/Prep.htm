Vorbereitung zur Installation

Systemvoraussetzung

Als erste Voraussetzung muss das Windows Server System ordnungsgemäß installiert sein. Die folgenden Windows Server Systeme sind von Gefasoft freigegeben, jeweils in der 64-Bit Version:

Windows Server 2008

Windows Server 2008 R2

Windows Server 2012

 

Außerdem müssen die Netzwerkeinstellungen richtig konfiguriert sein.

Die Installation muss mit einem Benutzer erfolgen, der Administrationsrechte auf dem Server hat.

Weitere Hinweise zum benötigten Festplattenplatz und den Hardware- und Software-Voraussetzungen können der Oracle-Installationsdokumentation entnommen werden.

Version downloaden

Unterstützt wird aktuell die Oracle-Version 11gR2.

In dieser Dokumentation wird vom Einsatz der Enterprise-Edition ausgegangen. Diese Version ist für den Betrieb von Legato empfohlen und kann kostenpflichtig bei Oracle erworben werden.

Prinzipiell ist es jedoch auch möglich, Legato mit einer Standard-Oracle-Version zu betreiben. Hier sind allerdings einige empfohlene Features nicht verfügbar.

Einige Oracle-Versionen können kostenlos heruntergeladen werden unter:

https://edelivery.oracle.com/

:    

Hier sind nicht die neuesten Versionen vorhanden. So ist dort zum Beispiel die Version 11.2.0.1.0 vorhanden, allerdings nicht die von Oracle empfohlene Version 11.2.0.3.0 des Oracleservers. Diese Versionen findet man im kostenpflichtigen Oracle Metalink (siehe nächste Seite).

Im Oracle Metalink können Service Requests gestellt werden, neueste Software-Updates geholt werden oder auch nach Hilfen gesucht werden. Der Metalink ist jedoch nur für Oracle-Partner zugänglich und grundsätzlich kostenpflichtig. Wir empfehlen den Zugang zum Metalink, um stets die aktuellsten Versionen verfügbar zu haben. Für diese Anleitung wird ebenfalls von einem Zugang zum Metalink ausgegangen.

Der Oracle Metalink ist zu erreichen unter http://support.oracle.com

 

Installationspakete lokal entpacken

Alle Software-Installationspakete von Oracle sollten lokal auf einer internen Festplatte gespeichert und entpackt werden. Dabei ist es auch wichtig, dass alle Pakete in einem Verzeichnis entpackt werden, sollten mehrere Pakete zu einem Produkt vorhanden sein.

Es ist wichtig, dass der Verzeichnisname auch keine Sonderzeichen oder Leerzeichen enthält. Im gezeigten Beispiel ist dies das Verzeichnis "C:\Users\Administrator\Desktop\Oracle\Windows 64 Bit\11gR2\11_2_0_3_0_64bit".

Oft sind nicht alle Pakete von einem Produkt erforderlich. Man kann sich vorher in den Installationsdokumentationen von Oracle erkundigen, welche Pakete nötig sind, denn es macht keinen Sinn, unnötigerweise alle Pakete herunter zu laden, da diese zum Teil recht groß sein können.

In diesem Fall werden die Pakete 1, 2 und 7 von insgesamt sieben Paketen benötigt. Die Pakete 1 und 2 enthalten die Installationsprogramme für den Server und das Paket 7 ein Deinstallationspaket. Beim Entpacken der Pakete im gleichen Verzeichnis erstellen diese Pakete dann die Unterverzeichnisse "database" und "deinstall".

:

Die Windows Scheduler Jobs sollten alle deaktiviert sein, da diese während der Installation stören können.
