Installation des Legato Datenmodells mit J-LIW (für alle DB-Typen)

Systemvoraussetzungen

Java 8 und ein geeigneter JDBC-Treiber (wird mit der Application Engine bzw. Sapient Engine mit ausgeliefert oder kann beim DB-Update mit heruntergeladen werden). Gegebenenfalls muss die Umgebungsvariable JAVA_HOME noch erstellt und auf den korrekten Pfad gesetzt werden (Systemsteuerung - Systemumgebungsvariablen bearbeiten - neue Variable erstellen).

PC mit graphischer Oberfläche (X-Window), zum Beispiel Windows 10.

Die graphische Benutzeroberfläche (GUI) zum Installieren und Updaten der Datenbank kann über LDBsetup.exe gestartet werden.

:

Für alle Updates ab Sapient 2.3.001 muss zwingend die J-LIW Version 2.3.001 verwendet werden! Diese enthält einen neuen Befehl "WbDataCleanup", der JSON-Felder konvertiert.

Für alle Updates ab Sapient 2.4.2 muss zwingend die J-LIW Version 2.4.002 verwendet werden! Mit dieser werden Oracle-Constraints auf eine andere Weise erstellt.

Bei Oracle zusätzlich

Bei der Verwendung von Oracle müssen bei der Installation des DB-Servers bereits folgende Einstellungen getroffen worden sein:

1. DBTIMEZONE muss auf UTC (entspricht +00:00) gestellt sein.

Ist dies nicht der Fall, kommt es in J-LIW beim Schritt "Datenbank-Installation prüfen" zu einer Fehlermeldung: "[FAILURE] Your Database timezone setting is incorrect! It is '+01:00', but should be '00:00'". Führen Sie in diesem Fall folgenden Befehl als sysdba aus:

alter database set TIME_ZONE = '+00:00';

-- RESTART YOUR DATABASE !!

shutdown immediate;

startup;

 

2. COMPATIBLE MODE muss auf 12.1 gestellt sein.

Ist dies nicht der Fall, kommt es in J-LIW beim Schritt "Datenbank-Installation prüfen" zu einer Fehlermeldung: "[FAILURE] Your Database compatible mode is incorrect! It is '11.2.0.0.0', but should start with '12.1.'" Führen Sie in diesem Fall folgenden Befehl als sysdba aus:

alter system set COMPATIBLE = '12.1.0.2.0' scope=spfile;

-- RESTART YOUR DATABASE !!

shutdown immediate;

startup;

 

Überblick

Das Datenmodell für eine Instanz wird mit Hilfe der LDBsetup.exe eingerichtet.

Der Installer wird sowohl für eine Neuinstallation als auch für die Installation von Updates, unabhängig vom Datenbanktyp, verwendet.

INFO:

Ab J-LIW Version 2.5.0 können mehrere Updates in einem Durchlauf installiert werden.

Die wichtigsten Angaben für den Installer wie z.B. der Datenbanktyp und die Reihenfolge der auszuführenden Scripte sind in der Datei SetupLDB.ini festgehalten. Das J-LIW Command wandelt die Datei in das benötigte XML-Format um.

Beispiel für SetupLDB.ini

Ausführung des Setups

Aufruf

Zuerst wird die Datei LDBsetup.exe geöffnet. Darin wird dann die gewünschte Setup-Datei (setupldb.ini aus DB-Verzeichnis) sowie die zugehörige Properties-Datei geladen.

Treiber

Die benötigten Treiber werden üblicherweise automatisch geladen. Wird allerdings die Workbench gestartet, bevor LDBSetup gestartet wird, klappt dies unter Umständen nicht. Die Treiber können dann auch manuell eingetragen werden. Klick auf "File - Manage Drivers" öffnet folgenden Dialog:

Wählen Sie aus der Liste links Ihren Datenbank-Typ aus (Oracle oder PostgreSQL). Rechts werden dann die Treiber angezeigt und können durch Klick auf OK gespeichert werden.

INI-Datei laden

Die INI-Datei enthält Platzhalter (%???%), die zur Laufzeit vom Installer durch die vom Benutzer festgelegten Werte ersetzt werden.

:

Die Platzhalter für die Legato-Benutzer LEGATO_OWNER, LEGATO_READ, LEGATO_WRITE, LEGATO_LC2, LEGATO_GW, LEGATO_AE und LEGATO_WA dürfen nicht "Legato" genannt werden. Andernfalls kommt es zu Fehlern, da "Legato" auch der Name eines DB-Packages ist. Auch die Namen anderer DB-Packages dürfen nicht für Datenbank-Benutzer verwendet werden.

Die INI-Datei wird von GEFASOFT erstellt und ausgeliefert. Anpassungen sollten nur nach Rücksprache durchgeführt werden, um Fehler zu vermeiden.

Der eigentliche Installationsvorgang wird durch SQL-Skripte gesteuert, die durch den Installer interpretiert und ausgeführt werden. Auf diese Skripte wird im Abschnitt [MANUAL] der INI-Datei verwiesen.

Durch Laden der setupldb.ini wird automatisch das gewünschte Produkt (Legato Classic oder Sapient) gewählt. Bei Sapient sind bis zu 5 Sprachen möglich, zumindest die 1. Systemsprache muss festgelegt werden.

Dann müssen die jeweiligen Parameterwerte eingegeben werden. Nachdem die Installationsparameter einmal eingegeben wurden, kann durch Aktivieren der Checkbox "Save PRM-File" ein Parameter-File (Dateiendung .prm) lokal abgespeichert werden. Für weitere Installationen kann man das PRM-File dann durch Klick auf "File - Load PRM file ..." wieder laden und muss nicht alle Parameter erneut eingeben.

:

Es wird hier nicht geprüft, ob alle benötigten Parameterwerte eingegeben wurden. Dies wird erst beim Ausführen geprüft.

:

Wie im Screen erwähnt, müssen alle Clients, die auf die Datenbank zugreifen, vor Beginn der Installation deaktiviert werden.

Ausführung

Klick auf "Next" führt zum nächsten Dialog, in dem die auszuführenden Schritte festgelegt werden können.

Insbesondere die Aktion "Sprachtexte aktualisieren" sollte für jeden Patch aktiviert werden. Die Aktion "Synonyms für DB-User xxx erstellen" muss nur für den letzten (höchsten) Patch aktiviert werden und nicht jedes mal ausgeführt werden.

:

Wenn die Datenbankuser noch nicht angelegt sind, muss die Checkbox "Validate Connections" deaktiviert werden, sonst kommt es zu Fehlermeldungen.

:

Klick auf "Back" setzt bereits geänderte Checkboxen wieder auf den Default zurück.

Es empfiehlt sich, einen Testlauf durchzuführen, um mögliche Fehler vorab zu beheben. Dies kann durch Klick auf den Button "Write" erfolgen.

Validierung

Vor jeder Ausführung (beim Klick auf "Execute" oder "Write") wird das Setup validiert. Die Validierung kann auch mit folgendem Befehl separat aufgerufen werden:

LgtDbSetup -setup=C:/Projekte/Legato/WebApp/J-LIW/test/resources/TEST_SETUP_DIR_1

           -loadVars=C:/SVN/legatodb/env/DB_Modell_Install_Compare-56.prm

           -lang0=de

           -validate;

 

Bei der Validierung wird Folgendes geprüft:

1. Sind alle Dateien vorhanden?

2. Sind alle Platzhalter vorhanden?

3. Falls Übersetzungen benötigt werden (mindestens ein Schritt ist als "translate" angegeben):

a) Ist die Primärsprache (lang0) angegeben?

b) Ist die Sprachdatei vorhanden?

 

Wurden Fehler gefunden, so können diese nun behoben werden. Dies erfolgt in der Workbench:

Workbench

Zum Öffnen der Workbench, klicken Sie auf Tools - Open Workbench.

Dort muss zunächst die Verbindung angegeben werden, die analysiert / bearbeitet werden soll:

Klick auf "Test" prüft die Verbindungsdaten.

Um diese Verbindung zu speichern, klicken Sie oben in der Bedienzeile auf das Diskettensymbol.

Unter "Workspace - Load Workspace" kann nun der Arbeitsbereich geladen werden. Dieser ist üblicherweise unter C:\Users\Username\.sqlworkbench\Default.wksp gespeichert. Außerdem wurde mit J-LIW bereits ein vorbereiteter Workspace mit einer Installations-Checkliste ausgeliefert. Diesen finden Sie im entpackten J-LIW-Verzeichnis unter \workspaces\LegatoSystem.wksp. Diesen finden Sie im entpackten J-LIW-Verzeichnis unter \workspaces\SapientSystem.wksp.

Laden Sie den Workspace LegatoSystem.wksp und selektieren Sie Tab 1 "Update Checklist". Hier finden Sie vorgefertigte SQLs zum Beispiel zum Kompilieren von invaliden Objekten oder dem Stoppen der User-Jobs:

Durch Klick auf das "Play"-Symbol oben links werden die Befehle abgearbeitet. Im unteren Teil werden eventuell auftretende Meldungen angezeigt.

Installation

Danach kann man zur LDBSetup zurückkehren und dort durch Klick auf "Execute" die DB-Installation starten.

Es werden folgende Punkte abgearbeitet:

alle Platzhalter werden durch die konkreten Werte ersetzt

alle Übersetzungen werden eingesetzt

die einzelnen Schritte werden ausgeführt

 

Ein in der Spalte "Status" bezeichnet den Schritt, der gerade bearbeitet wird. zeigt einen Fehler an, einen Hinweis, eine erfolgreich durchgeführte Aktion. Bei einem Fehler wird die Installation abgebrochen. Nach der Fehlerbehebung kann die Installation an dieser Stelle fortgesetzt werden (vorherige Schritte können abgewählt werden).

Kam es bei einer Aktion zu einem Problem, so können genauere Informationen dem Logfile entnommen werden.

Die Installation ist ausgeführt, wenn alle Schritte erfolgreich bearbeitet wurden.

Passwort-Verschlüsselung für LC2-User

Bei einer Neuinstallation wird auch das verschlüsselte Passwort für den LC2-User angelegt.

Ab J-LIW Version 2.5.2 kann dies mit Hilfe von: 

LgtDbSetup -encryptLC2Key=yourkey

 

erfolgen. Dazu muss jedoch ein JRE mit 'Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files' installiert sein.

Bis J-LIW Version 2.5.2 ist die Verschlüsselung nicht mit Standard-Java-Mitteln möglich, es wird eine Extension dafür benötigt. J-LIW gibt dann folgenden Warnhinweis aus:

ERROR: Your Java cannot encrypt key used for LC2. Please update the 'Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files'. These files can be downloaded at Java SE download page (http://www.oracle.com/technetwork/java/javase/downloads/index.html) and extract them into folder '${java.home}/jre/lib/security/'.

 

In diesem Fall muss also die JCE wie in der Fehlermeldung beschrieben aktualisiert werden.

Ob J-LIW das LC2-Passwort verschlüsseln kann, kann mit folgender Anweisung geprüft werden:

LgtDbSetup -canEncryptLC2Key;

 

Log-Ausgabe

Über den Menüpunkt "Help > Show Log" kann die Log-Ausgabe der Workbench angezeigt werden. Das Log selbst liegt normalerweise im User-Verzeichnis unter ".sqlworkbench". Darunter legt der neue DB-Installer einen neuen Ordner jliw_log an, in dem sich die Logs pro Installation im CSV-Format befinden.

TIPP:

Das Log sollte am besten für jedes Update (d.h. wenn eine dbsetup.xml ausgewählt wurde) neu erstellt werden. So können Fehler besser nachvollzogen werden.

Sprachtexte verwalten

Für J-LIW wurde eine neue Extension namens J-PET (Java Property Editor for Translations) entwickelt, mit der die Sprachtexte angezeigt, editiert, exportiert und importiert werden können. Rufen Sie dazu die Datei Jpet.exe auf. Über "File - Open..." kann die gewünschte Properties-Datei gewählt werden. Die Datei configdata.properties enthält die Daten für alle Systemsprachen, die übrigen Dateien enthalten jeweils die in der Endung angegebene Sprache (z.B. configdata_de.properties Deutsch).

Umlaute werden manchmal bei den einzelnen Schritten nicht korrekt dargestellt --> kann ignoriert werden.

Nur Oracle-DB: Besonderheit für fremdsprachige Texte mit Sonderzeichen außerhalb dem jeweiligen Standardzeichensatz des DB-Servers

Um in LEGATO fremdsprachige Texte mit Sonderzeichen außerhalb dem jeweiligen Standardzeichensatz des DB-Servers installieren zu können, müssen bestimmte Einstellungen in der Datenbank vorgenommen werden. Andernfalls kommt es zu Problemen bei der Darstellung der Sonderzeichen. Beispielsweise muss für Installationen im westeuropäischem Standardzeichensatz (z.B. DE) für Sprachen aus demselben Sprachraum (z.B. Französisch), nichts beachtet werden. Soll nun aber z.B. eine deutsche Installation auf einem Server mit US-Einstellung installiert werden, so muss auch für DE der NLS_LANG Parameter gesetzt werden.

:

Fremdsprachige Texte werden nicht korrekt in die standardmäßig ausgelieferte Legato Datenbank übernommen, solange diese Einstellungen nicht vorgenommen wurden.

Der Oracle-Parameter NLS_LANG (für den Zeichencode) muss auf den entsprechenden Wert gesetzt werden, damit die Datenbank die fremden Sonderzeichen aufnehmen kann. Grundsätzlich gibt es dafür drei Möglichkeiten (siehe auch
http://www.oracle.com/technetwork/products/globalization/nls-lang-099431.html#_Toc110410543):

den Datenbank-Parameter NLS_LANG bzw. NLS_LANGUAGE setzen

den Datenbankinstanz-Parameter NLS_LANG bzw. NLS_LANGUAGE setzen

den Database Session-Parameter NLS_LANG setzen

 

Die Einstellung des Session-Parameters überlagert den Datenbankinstanz-Parameter und dieser wiederum den Parameter auf Datenbankebene.

Da weder die Datenbank insgesamt noch die eine Legato-Instanz dauerhaft umgestellt werden sollen, ist das (vorübergehende) Setzen des Session-Parameters das Mittel der Wahl. Der Session-Parameter wird auf Betriebssystem-Ebene gesetzt.

Am einfachsten wird dazu vor dem Starten der SetupLDB.ini in einer DOS-Box (auf Windows) die Umgebungsvariable NLS_LANG gesetzt:

C:\Aufrufverzeichnis\>set NLS_LANG=[gewünschter Zeichensatz]

C:\Aufrufverzeichnis\>setupldb

 

In den Parametern des Legato DB Setup-Tool wird dann als eine Sprache (z. B. "Language 1") die entsprechende Sprache (z.B. Chinesisch ("Chinese, Simplified")) eingestellt. Anschließend können die Texte eingespielt werden.

:

Diese Einstellung muss vor der Ausführung jeder einzelnen SetupLDB erfolgen, falls mehrere in Folge installiert werden müssen.

Wenn aus einem Tool (insbesondere SQL*Plus, PL/SQL Developer und Legato DB Setup) fremdsprachige Texte in die Datenbank geschrieben werden sollen, dann müssen sie in Umgebungen gestartet werden, in denen der Session-Parameter NLS_LANG gesetzt ist.

Folgende Werte müssen dem Parameter NLS_LANG gegebenenfalls zugewiesen werden:

für Chinesisch: SIMPLIFIED CHINESE_CHINA.UTF8 (sic!; inklusive Leerzeichen!)
für Polnisch: POLISH_POLAND.EE8MSWIN1250
für Ungarisch: HUNGARIAN_HUNGARY.EE8MSWIN1250
für Slowakisch: SLOVAK_SLOVAKIA.UTF8
für Deutsch: GERMAN_GERMANY.WE8ISO8859P15
