ThreadPool

Objekte dieses Typs erlauben die parallele Abarbeitung von Operationen innerhalb eines einzigen Scripts. Dies eignet sich besonders für Jobs, die mehrere voneinander unabhängige und potentiell langwierige Aufrufe von externen Servern durchführen müssen (z.B. Up-/Download von Dateien, Aufruf von Web-Services o.ä.), da dann alle diese Aufrufe parallel stattfinden können.

Der Typ ThreadPool hat folgende Eigenschaften und Funktionen:

Konstruktor ThreadPool(name, threadcount):

Dient zum Erzeugen von Threadpool-Objekten, wobei optional gleich dessen Name (siehe Eigenschaft "name") und die maximale Anzahl Threads für diesen Pool vorgegeben werden können. Beispiel: var tp = new ThreadPool('downloads', 3).

Eigenschaft name (readwrite):

Der Name des Thread-Pools. Wird zum Benennen der Threads benutzt (erscheint z.B. in Logmeldungen). Kann nur geändert werden, wenn der Thread-Pool nicht (oder nicht mehr) läuft (siehe Eigenschaft "running").

Eigenschaft threadcount (readwrite):

Maximale Anzahl zu erzeugender Threads (parallel ablaufende Operationen). Kann nur geändert werden, wenn der Thread-Pool nicht (oder nicht mehr) läuft (siehe Eigenschaft "running").

Eigenschaft running (readonly):

Gibt an, ob Threads im Pool laufen. Wird "true", wenn die erste Operation gestartet wird, wird "false", wenn waitTillDone() beendet wurde.

:

Die Threads im Pool laufen weiter, auch wenn alle Operationen beendet sind, es könnten ja jederzeit neue Operationen hinzukommen.

Funktion exec(operation):

Startet die angegebene Operation. Dies kann sein:

eine Zeichenkette: wird wie mittels "eval()" ausgewertet.

eine Funktion: wird in einem Thread aufgerufen.

ein Objekt: falls es eine Funktion "run()" enthält, wird diese aufgerufen.

ein Java-Runnable: seine Methode "run()" wird aufgerufen.

ein Java-Callable: seine Methode "call()" wird aufgerufen.

 

Der Aufruf von exec(operation) gibt ein Objekt vom Typ AsyncResult zurück (siehe unten).

Funktion waitTillDone():

Wartet, bis alle Operationen beendet sind und stoppt alle Threads im Pool

 

Das von exec(operation) zurückgegebene AsyncResult-Objekt hat folgende Eigenschaften und Methoden:

Funktion cancel(mayInterrupt):

Bricht die Ausführung der betreffenden Operation ab. Sofern sie noch nicht begonnen hat, wird sie aus der Warteschlange gelöscht. Bereits gestartete Operationen werden nur abgebrochen, wenn "mayInterrupt" als true angegeben ist.

Funktion awaitResult():

wartet auf Beendigung der Operation und gibt deren Ergebnis zurück:

bei Auswertung einer Zeichenkette das Ergebnis des letzten darin vorkommenden Ausdrucks

bei Auswertung einer Funktion deren Ergebnis

bei Auswertung eines Objekts oder Java-Runnable: das Objekt

bei Auswertung eines Java-Callable das Ergebnis der Methode "call()"

 

Falls bei der Abarbeitung der asynchronen Operation eine Ausnahme aufgetreten ist, gibt awaitResult() ein Java-ExecutionException-Objekt zurück, das die eigentliche Ausnahme über getCause() zur Verfügung stellt.

Eigenschaft done (readonly):

Gibt an, ob die Operation beendet ist.

Eigenschaft cancelled (readonly):

Gibt an, ob die Operation abgebrochen wurde.

 

Jeder ThreadPool enthält eine bestimmte Anzahl von Threads. Es können mehr Aufgaben an einen ThreadPool delegiert werden, als dieser Threads enthält. Überzählige Aufgaben werden in eine Warteschlange eingereiht und ausgeführt, sobald eine andere fertig wird.
