MailSender

Dient zusammen mit MailMessage dazu, E-Mails aus JavaScript-Jobs zu versenden.

Das Senden von E-Mails erfordert Kommunikation über Netzwerk, gegebenenfalls Kodieren von Anhängen und ähnlich aufwändige Operationen. Somit kann es sich als nützlich erweisen, das Senden asynchron über einen ThreadPool (s.u.) durchzuführen.

Ein MailSender enthält alle Konfigurationsdaten zum Versenden von Mails über einen gegebenen Mail-Server. Diese sind als Eigenschaften abgebildet:

host (String, readwrite):

Name oder IP-Adresse des Mail-Servers.

port (Number, readwrite):

TCP-Port, auf dem der Mail-Server Mails entgegennimmt. In der Regel entweder Port 25 (SMTP) oder 587 (SUBMISSION). Falls kleiner gleich 0 wird entsprechend der Konfiguration ein Standard-Port gewählt.

auth (Boolean, readwrite):

Ob der Mail-Server eine Anmeldung mit Benutzername und Kennwort erfordert.

user (String, readwrite):

Benutzername zum Anmelden beim Mail-Server.

password (String, readwrite):

Kennwort zum Anmelden beim Mail-Server.

security (String, readwrite):

Ob und wie die Verbindung zum Mail-Server verschlüsselt und gesichert werden soll. Einer der folgenden Werte:

none – Unverschlüsselte und ungesicherte Verbindung

tls_optional – Verbindung mittels TLS verschlüsseln, sofern der Server das unterstützt. Identität des Servers nicht prüfen, sondern jedes beliebiges vom Server präsentierte Zertifikat akzeptieren.

tls_trust_all – Nur TLS-verschlüsselte Verbindungen aufbauen, aber die Identität des Servers nicht prüfen, sondern jedes beliebige von ihm präsentierte Zertifikat akzeptieren.

tls_trust_host – Nur TLS-verschlüsselte Verbindungen aufbauen, aber jedes vom Server präsentierte Zertifikat akzeptieren, das auf den zum Verbindungsaufbau benutzten Rechnernamen (siehe Eigenschaft host) ausgestellt ist.

tls_required – Nur TLS-verschlüsselte Verbindungen aufbauen und die Identität des Mail-Servers mit den für die Java-VM konfigurierten Mitteln prüfen.

tls_strict – Nur TLS-verschlüsselte Verbindungen aufbauen und die Identität des Mail-Servers mit allen in RFC2595 definierten Mitteln überprüfen.

 

debug (Boolean, readwrite):

Kann auf true gesetzt werden, um Fehler beim Mail-Versand zu untersuchen. Die Klassen der JavaMail-Bibliothek geben dann Ablaufmeldungen und alle übertragenen Texte auf der Standard-Ausgabe (i.d.R. Konsole) aus.

 

MailSender bietet zwei Methoden zum Versenden von Mails:

boolean sendText(from, to, subject, text)

Versendet einfache Reintext-Mails mit Absender from an einen einzelnen Empfänger to. Die Mail hat den Betreff subject und enthält den Text text. Das Ergebnis gibt Erfolg (true) oder Misserfolg des Mail-Versands an.

boolean send(message)

Versendet eine als MailMessage-Objekt definierte Mail. Konnte die Nachricht an alle darin angegebenen Empfänger gesendet werden, ist das Ergebnis der Funktion true. Beim ersten Empfänger, an den die Nachricht nicht versendet werden kann, bricht die Funktion ab und liefert als Ergebnis false. Der Erfolg des Versendens bezieht sich nur auf die Übergabe an den im MailSender definierten Server. Später auftretende Fehler, z.B., wenn dieser Server die Nachricht nicht zustellen kann, werden nicht erfasst.
