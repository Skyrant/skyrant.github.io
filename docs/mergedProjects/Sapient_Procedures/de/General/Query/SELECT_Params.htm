SELECT Parameter

Beim Aufruf der "Abfrage-Prozedur" über JSON-RPC sind folgende Parameter anzugeben:

schema

(String) - Name der Repository-Verbindung / des Datenbank-Schemas (GP_R_CONNECTIONS), auf der die Abfrage durchgeführt werden soll.

query

(JSON-Objekt) - Die Definition, was abgefragt werden soll (siehe unten).

pre

(JSON-Array, optional) - Eine oder mehrere Voraussetzungen, die vor der Ausführung der Abfrage geprüft werden sollen. Ergeben sie NULL, FALSE, 0 oder einen Leerstring, wird die Abfrage nicht durchgeführt und ein Fehler-Ergebnis "failed precondition" zurückgegeben. Noch nicht implementiert.

newer

(optional) - Ein Zeitstempel, der benutzt wird, um nur Datensätze zu liefern, die neuer sind als dieser Zeitstempel. Ist dieser Parameter angegeben, fragt der Server von allen in der Abfrage vorkommenden Tabellen deren Feld mit dem letzten Änderungs-Zeitpunkt mit ab (sofern vorhanden). Er liefert dann nur Ergebnis-Zeilen in "data", wo mindestens ein solcher Zeitpunkt später oder gleich dem Angegebenen ist. Von den restlichen (unveränderten, aber nach wie vor vorhandenen) Datensätzen liefert er nur die IDs in einem Array "keep", das im Ergebnis auf gleicher Ebene wie "data" liegt. Noch nicht implementiert.

timeout

(Integer, optional) - Wenn vom Aufrufer angegeben, enthält dieser Parameter die ganzzahlige Anzahl Sekunden, die jede Datenbankabfrage dauern darf, die von dieser "Select"-Prozedur ausgeführt wird. Die WebApp übergibt ein Default-Timeout bei jedem Aufruf der JSON-RPC "Select"-Methode. Die "Select"-Prozedur verwendet dieses Timeout für alle generischen SQL-Abfragen, die sie selbst ausführt und gibt es auch an alle geskripteten Datenquellen weiter.

Ein spezieller Fehlercode SQL_TIMEOUT (-1002) wurde eingeführt für JSON-RPC Fehlermeldungen, die von Abfrage-Timeouts verursacht werden.

query

Eine query kann entweder

ein einzelnes query-Objekt sein (siehe unten) oder

ein Array von query-Objekten und Strings, der einen Ausdruck unter Verwendung von Unter-Abfragen, Operatoren ("UNION", "UNION ALL", "INTERSECT", or "MINUS") und Klammern definiert.

Ein query-Objekt besteht aus folgenden Elementen:

resources

(Array oder ein einzelnes resource-Objekte) - Die Datenquellen (wie im Sapient Repository angegeben), die abgefragt werden sollen. Handelt es sich um eine geskriptete Datenquelle, darf dies die einzige angefragte Datenquelle sein, da der Aufruf direkt an die entsprechende Java- oder JavaScript-Funktion weitergegeben wird. Bei Abfragen über mehrere Datenbank-Ressourcen (Tabellen oder Unterabfragen) erscheinen diese im SQL-Ergebnis in derselben Reihenfolge wie hier angegeben.

fields

(Array von field-Angaben oder String) - Welche "field(s)" der übergebenen "resources" abgefragt werden sollen. Felder können entweder als JSON-Objekte oder als Strings in der Form "<res-alias>.<field-name>" angegeben werden. String-Angaben können als Komma-getrennte Liste verknüpft werden.

relations

(Array von relation-Objekten) - Angaben, wie die übergebenen "resources" miteinander zu verknüpfen sind. Entfällt, wenn nur eine einzige "resource" abgefragt wird. Enthält resources mehrere Einträge (Tabellen oder Unterabfragen), dann muss für jeden dieser Einträge außer dem 1. eine relation angegeben werden, deren dst auf diesen Eintrag verweist. Jede dieser relations darf nur auf Quellen (src) verweisen, die vor ihrem Ziel (dst) im Ressourcen-Array stehen.

filter

(Array, optional) - Definiert Einschränkungen für die zurückgegebenen Ergebnis-Zeilen. Kann Folgendes enthalten:

filter-Objekte, die Bedingungen angeben, die diese Zeilen erfüllen müssen

Bool'sche Operatoren (Strings "AND" oder "OR"), um diese Bedingungen zu verknüpfen

Klammern (Strings "(" oder ")"), um diese Ausdrücke zu gruppieren und die Priorität festzulegen

sort

(Array von sort-Objekten, optional) - Wie die zurückgegebenen Ergebnis-Zeilen sortiert werden sollen. Wird nur ein einzelnes Kriterium angegeben, kann die Array-Notation weggelassen werden. Mehrere Sortier-Kriterien werden in absteigender Priorität behandelt, d.h. alle Zeilen werden nach dem ersten Kriterium sortiert, dann werden die Gruppen von Zeilen mit gleicher Sortierung gemäß diesem ersten Kriterium nach dem zweiten Kriterium sortiert und so weiter.

groupby

(Array oder String, optional) - Eine oder mehrere Feld-Referenzen, nach der die Datenbank-Zeilen gruppiert werden sollen.

offset

(Number, optional) - Anzahl der Datensätze, die am Anfang der Ergebnisse übersprungen (nicht im Ergebnis zurückgegeben) werden sollen. Wird für Paginierung benötigt. Offset 0 bedeutet "keine" und somit das gleiche, wie den Parameter wegzulassen.

limit

(Number, optional) - Maximale Anzahl zu liefernder Datensätze. Die restlichen Datensätze werden stillschweigend verworfen. Falls nicht angegeben, werden alle Datensätze geliefert.

distinct

(Boolean, optional) - Falls angegeben und true, werden nur eindeutige Datensätze abgefragt. Eindeutige Datensätze haben ungleiche Werte in mindestens einem Feld.

hierarchy

Noch nicht implementiert.

 

Statt einem einzelnen "query"-Objekt kann auch ein Array aus query-Objekten, Mengen-Operatoren (UNION, UNION ALL, INTERSECT und MINUS) und Klammern übergeben werden.

resource

Ein resource-Objekt kann entweder eine Datenbank-Tabelle sein (aus GP_R_DATASOURCES) oder eine Unterabfrage. Um dies zu unterstützen, kann ein resource JSON-Objekt folgende Elemente haben:

resource

(String) - Der Objektname (in Kleinbuchstaben), der in GP_R_DATASOURCES eingetragen ist.

query

(JSON-Objekt) - Die Unterabfrage.

as

(String) - Alias-Name, der in anderen Teilen der Abfrage verwendet wird, um auf diese Datenbank-Tabelle oder Unterabfrage zu verweisen. Für Unterabfragen verpflichtend, für Tabellen optional.

field

Ein field kann auf mehrere Arten angegeben werden:

Spalte von einer der Ressourcen. Diese kann spezifiziert werden durch:

Einen String-Wert in der Form [resource.][column][ AS name], z.B. "a.name as aname".

Ein JSON-Objekt mit den folgenden Elementen:

resource (String) - Alias oder Name der Tabelle oder Unter-Abfrage aus den für diese Abfrage definierten "resources".

field (String) - Name der entsprechenden Tabellenspalte. Falls aus einer Unter-Abfrage stammend und falls die Unter-Abfrage einen Alias für das fragliche Feld definiert hat, dieser Alias.

as (String, optional) - Alias, der verwendet werden soll, um dieses Feld in den Ergebnis-Zeilen oder in Abfragen von außerhalb zu referenzieren (falls diese Definition in einer Unter-Abfrage ist.

Unter-Abfrage - Ein JSON-Objekt mit den folgenden Elementen:

query (JSON-Objekt) - Ein Query-Objekt, das die Unter-Abfrage definiert.

as (String) - Alias, der verwendet werden soll, um dieses Feld in den Ergebnis-Zeilen oder in anderen Teilen der Abfrage zu referenzieren.

Aufruf einer SQL-Funktion - Ein JSON-Objekt mit folgenden Elementen:

function (String) - Name der SQL-Funktion. Aktuell werden folgende Funktionen unterstützt: ABS, MOD, CEIL, FLOOR, ROUND, EXP, LN, LOG, POWER, SIGN, SQRT, SIN, ASIN, COS, ACOS, TAN, ATAN, ATAN2, LENGTH, CONCAT, UPPER, LOWER, LTRIM, RTRIM, SUBSTR, TO_CHAR, TO_NUMBER, TO_DATE, TO_TIMESTAMP. Nicht alle diese Funktionen werden von allen Datenbank-Servertypen unterstützt und nicht alle nutzen dieselbe Semantik (Parameter und -typen, Default-Werte, Rückgabetypen etc.) Daher ist bei Verwendung dieses Features Vorsicht geboten.

params (Array) - Array von field-Referenzen oder Buchstabensymbolen, die als Argumente an die SQL-Funktion übergeben werden sollen.

as (String) - Alias, der verwendet werden soll, um dieses Feld in den Ergebnis-Zeilen oder in anderen Teilen der Abfrage zu referenzieren.

Expression:

expression (Array) - Ein Ausdruck mit den field-Referenzen, SQL-Funktionsaufrufen, Operatoren und Klammern, um den Wert zu berechnen, der für dieses Feld zurückgegeben werden soll. Field-Referenzen können als Strings oder als field-Objekte definiert werden. Operatoren oder Klammern als String-Elemente des Arrays. Die folgenden Operatoren werden unterstützt:

"+" - Addition

"-" - Subtraktion

"*" - Multiplikation

"/" - Division

"%" - Rest

"^" - Potenzierung

"||" - String-Verkettung

as (String) - Alias, der verwendet werden soll, um dieses Feld in den Ergebnis-Zeilen oder in anderen Teilen der Abfrage zu referenzieren.

relations

Ein relation-Objekt definiert, wie zwei "resources" miteinander verknüpft werden sollen. Das JSON-Objekt zum Definieren einer relation kann folgende Elemente haben:

src (String) - Name oder Alias der Quell-Tabelle, die verknüpft werden soll.

dst (String) - Name oder Alias der Ziel-Tabelle, die verknüpft werden soll.

how (String) - Art der Verknüpfung. Muss einer der folgenden Werte sein (unabhängig von Groß- / Kleinschreibung):

I - Für inner Join (alle Zeilenpaare von "src" und "dst", die die Join-Bedingung erfüllen. Zeilen einer Ressource, die keine Entsprechung in der anderen Ressource haben, werden vom Ergebnis ausgeschlossen).

L - Für left Join (alle Zeilen von "src" werden mit den entsprechenden Zeilen von "dst" gejoint).

R - Für right Join (alle Zeilen von "dst" werden mit den entsprechenden Zeilen von "src" gejoint).

F - Für full Join (alle Zeilenpaare von "src" und "dst", die die Join-Bedingung erfüllen und alle Zeilen von einer Ressource, die keine Entsprechung in der anderen Ressource haben).

C - Für cross Join (alle Zeilen von "src", jeweils gepaart mit jeder Zeile von "dst").

Eine (und nur genau eine) der folgenden Relations-Arten:

on - Ein onconstraint-Objekt oder ein Array von onconstraint-Objekten, boole'schen Verknüpfungen (AND oder OR) und runden Klammern.

rel - Ein field-Objekt aus einer der beiden Tabellen, das eine Relation zur anderen Tabelle darstellt (z.B. { "resource": "n", "field": "parent" }).

using - Ein oder mehrere Namen von Feldern, die in beiden Tabellen vorhanden sind und gleiche Werte haben sollen.

natural - Wert beliebig. Falls angegeben, werden alle gleichnamigen Felder in beiden Tabellen auf gleiche Werte geprüft.

filter

Ein "filter" JSON-Objekt definiert eine Bedingung, um Abfrage-Ergebnisse einzuschränken oder Paare von abgefragten Ressourcen zu verknüpfen (joinen). Es kann folgende Elemente haben:

left (String oder JSON-Objekt) - Entweder eine String-Referenz, ein Feld von einer Ressource, ein JSON-Objekt val, das ein Buchstabensymbol enthält oder ein JSON-Objekt query.

op (String) - Einer der folgenden Vergleichsoperatoren:

"nl" oder "nn" - Für "IS NULL" oder "IS NOT NULL". Für diesen Operator ist kein Wert für "right" erlaubt.

"eq" oder "ne" - Für "equal to" oder "not equal to". "right" muss ein einzelner Wert oder eine Feld-Referenz sein.

"lt" oder "le" - Für "less than" oder "less or equal". "right" muss ein einzelner Wert oder eine Feld-Referenz sein.

"gt" oder "ge" - Für "greater than" oder "greater or equal". "right" muss ein einzelner Wert oder eine Feld-Referenz sein.

"lk" oder "ul" - Für "like" oder "not like". Sowohl "left" als auch "right" müssen String-Werte oder Referenzen auf String-Felder sein.

"in" oder "ni" - Für "in" oder "not in" einer Liste von Werten. "right" muss ein Array von mindestens einem Wert oder einer Feld-Referenz sein.

"bw" oder "nb" - Für "between" oder "not between". "right" muss ein Array mit genau zwei Werten sein.

right (String, JSON-Objekt oder Array) - Rechte Seite des Vergleichs/der Bedingung. Abhängig vom verwendeten Operator (op oben), entweder kein, ein oder ein Array von Feld-Referenzen,  val- oder query-Objekten.

val

Ein Buchstabensymbol, das in einem Ausdruck (für Felder, Filter, etc.) oder als Parameter in einer Funktion verwendet werden soll. Hat ein einzelnes Element:

val (any) - das Buchstabensymbol, das verwendet werden soll.

sort

Ein sort-Objekt kann folgende Elemente spezifizieren:

field (String) - eine Feld-Referenz.

desc (Boolean, optional) - ob die Ergebnisse absteigend sortiert werden sollen. Defaultmäßig werden die Ergebnisse in aufsteigender Reihenfolge sortiert.

nullsfirst (Boolean, optional) - ob NULL-Werte zu Beginn der Ergebnisliste einsortiert werden sollen. Defaultmäßig werden Zeilen mit NULL-Werten für die abgefragten Felder ans Ende der Ergebnisliste sortiert.
