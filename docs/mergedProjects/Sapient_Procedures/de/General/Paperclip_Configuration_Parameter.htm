Paperclip Konfigurationsparameter

Service

Der Dienst hat eine strikte Trennung von Konfiguration und Code. Die Konfiguration kann von Installation zu Installation stark variieren, der Code jedoch nicht.

  • Erstellen Sie eine .env-Datei und fügen Sie umgebungsspezifische Variablen in neuen Zeilen in der Form von NAME=VALUE.

  • Zeilen, die mit # beginnen, sind Kommentare und werden zur Laufzeit ignoriert.

  • Wenn möglich, sollten Sie leere Werte vermeiden, da die Standardwerte in diesem Fall möglicherweise nicht nicht richtig angewendet werden. Um den Standardwert zu verwenden, entfernen Sie entweder die Zeile oder kommentieren Sie sie aus.

PASSWORD=

TARGET_HOST=

TARGET_PORT=

HTTPS=

LOCALE=

CHROME_FLAGS=

MAX_CONCURRENT_SESSIONS=

# DEBUG=

# QUEUE_MAX_WAIT_MS=

# QUEUE_MAX_WAITING_ITEMS=

# HTML_MIN_STABLE_SIZE_ITERATIONS=

# CHECK_ITERATIONS=

Begrenzen Sie die Anzahl der gleichzeitigen Sitzungen.

QUEUE_MAX_WAIT_MS Die Zeit (in Millisekunden), die eine Anfrage in der Warteschlange bleiben darf. Eine Anfrage wird
eine Anfrage in der Warteschlange, wenn die Anzahl der Anfragen, die gerade bearbeitet werden, mit
MAX_CONCURRENT_SESSIONS übersteigt. Wird die Anfrage nicht innerhalb der vorgegebenen Zeit abgebaut, wird die Anforderung mit dem Statuscode 503 (Service Unavailable) und dem Grund
"Aufgabe wurde nicht rechtzeitig aus der Warteschlange entfernt".		 Unlimited
QUEUE_MAX_WAITING_ITEMS

Die maximal zulässige Anzahl von Aufgaben, die gleichzeitig in die Warteschlange gestellt werden können. Berücksichtigt die Prioritäten der aktuell wartenden Tasks.


Wenn dieser Wert beispielsweise auf 5 gesetzt ist und sich bereits 5 Aufgaben mit der Priorität=0
in der Warteschlange stehen, dann werden neu eingehende Anfragen mit einer Priorität von 0 oder niedriger sofort verworfen.


In demselben Szenario würde jedoch eine eingehende Anfrage mit einer Priorität von z. B. 1 nicht abgelehnt werden.

 Unlimited
HTML_MIN_STABLE_SIZE_ITERATIONS

Minimale Iterationen, bei denen der Inhalt eine stabile Größe hat. Jede Iteration wird jede Sekunde ausgeführt und vergleicht die HTML-Größe mit der Größe der vorherigen Iteration. Der Druck wird gestartet, sobald der Inhalt für die angegebenen minStableSizeIterations in einer Zeile stabilen Inhalt erreicht.

 3
CHECK_ITERATIONS

Anzahl der Iterationen, in denen der Dienst nach dem Dashboard-Service sucht und den Status von isDashboardLoaded, bevor der Druck gestartet wird. Entschärft Nebeneffekte auf Systemen, bei denen das anfängliche Laden der Anwendung mehr Zeit als üblich in Anspruch nahm, so dass der Bericht bereits gedruckt wurde bevor die Anwendung vollständig geladen war.

 10

Wenn eine Anfrage erfolgreich ist, aber der Versuch, die E-Mail-Benachrichtigung zu senden, fehlschlägt (geplanter Druck), versuchen Sie, die E-Mail-Nachricht so oft zu senden.
