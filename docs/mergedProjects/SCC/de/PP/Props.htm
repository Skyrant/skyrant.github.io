Eigenschaften des Projektes

Ist das Projektelement selektiert, kann über das Kontextmenü - Eigenschaften der Eigenschaftsdialog des Projektes geöffnet werden:

 

Scripten bei Fehler abbrechen

Wenn angewählt, wird jedes Script, in dem ein Ablauffehler erkannt wird (z.B. Division durch Null, Blockgrenze überschritten etc.) sofort beendet. Ist "Scripten bei Fehlern abbrechen" abgewählt, wird das Script immer fortgesetzt. Funktionen im Script, die ein Ergebnis liefern sollen und fehlerhaft sind, geben dann immer einen Nullwert zurück.

Kommandoqueue bei Projektende löschen

Wenn angewählt, wird das Projekt sofort nach dem Ausführen des aktuell bearbeiteten Scripts beendet. Sind noch Wertänderungen in der Queue gespeichert, gehen diese verloren. Diese Option ist in Cloud Connectors sinnvoll, die mit der Datenbank über eine Netzwerkverbindung kommunizieren, um das Beenden bei gestörter Verbindung zu beschleunigen.

Reaktion bei NIL-Wert auslösen

Wenn angewählt, wird auch bei ungültigen Werten (z.B. bei Verbindungsfehlern) eine Reaktion ausgelöst. Wurde ein Script als Reaktion parametriert, kann diesem der Parameter @nilvalue übergeben werden, um eine Abfrage auf gültige Werte zu ermöglichen.

Alte Logklassen in Skripten

Wenn angewählt, werden beim Loggen aus einem Script heraus die alten Logklassen vom System automatisch in die neuen Klassen umgesetzt. Eine Anpassung dieser Scripten ist somit nicht notwendig. Neue Scripten müssen in diesem Fall aber auch die alten Logklassen verwenden.

Wartezeit bis StartDone

Gibt die Zeit an, die zwischen dem Starten des letzten Knotens bis zum Event "Ablaufstart fertig" vergehen soll. Dies gibt dem Anwender Zeit, um auf eventuell notwendige Startwerte vor weiteren Aktionen zu warten.

Verzögerung bei Projektende

Gibt die Zeit an, die zwischen dem Event "Ablaufende Vorbereitung" und dem Event "Ablaufende" vergehen soll. Dies gibt dem Anwender Zeit, um eventuell notwendige Aktionen vor dem tatsächlichen Projektende auszuführen.

Logikfelder der internen Datenbank

Hier kann gewählt werden, in welchem Format Bool-Felder in der internen Datenbank gehandhabt werden.
