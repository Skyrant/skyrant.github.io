Ereignisse

Der Cloud Connector funktioniert im Ablauf voll ereignisgesteuert. Solche Ereignisse werden z.B. von Wertänderungen der Variablen ausgelöst. Zusätzlich können jedoch auch Ereignisse wie Timer angelegt werden.

Anlegen eines Ereignisses

Selektieren Sie dazu das Grundelement "Event" im Projektbaum und wählen Sie im Kontextmenü den Befehl "Neues Ereignis". Der angezeigte Dialog hängt vom gewählten Typ ab.

Der Parameter "Typ" hat folgende Bedeutung:

 

Typ

Erklärung

Manuell

Das Ereignis kann nur manuell ausgelöst werden. Zum Auslösen im Kontextmenü des Events (rechte Maustaste auf das Event) den Eintrag "Ausführen" anklicken.

Zyklisch

Das Ereignis wird zyklisch ausgelöst. Das Zeitraster wird in Millisekunden angegeben, die dann ggf. in Std und Min gewandelt werden, um die Anzeige zu vereinfachen.

Ist zusätzlich die Checkbox "Mehrfacheintrag sperren" aktiviert, so wird das Event nur einmal getriggert und danach solange gesperrt, bis das zugehörige Script abgearbeitet wurde. Der Timer wir dann neu aufgezogen und wieder freigegen.

Zeittrigger

Das Ereignis wird zu einem bestimmten Zeitpunkt ausgelöst. Es kann eine Uhrzeit und einer oder mehrere Wochentage angegeben werden.

Zeitpunkt

Das Ereignis wird zu einem einmaligen Zeitpunkt ausgeführt. Dieser wird mit Datum und Uhrzeit festgelegt.

Ablaufstart

Das Ereignis wird einmalig nach dem Projektstart ausgelöst. Es werden zuerst alle Variablen gestartet, danach wird das Ereignis "Projektstart" ausgelöst.

Ablaufende Vorbereitung

Das Ereignis wird einmalig vor dem Ereignis "Ablaufende" ausgelöst.

Ablaufende

Das Ereignis wird einmalig vor dem Projektende ausgelöst. Es wird zuerst das Ereignis "Ablaufende" ausgelöst, danach werden alle Variablen gestoppt.

Grenze Kommandoqueue

Ab einer bestimmten Anzahl von Kommandos in der Queue wird ein Script gestartet, das eine Fehlermeldung generiert. Bei Kommend wird das Event bei Überschreitung des eingegebenen Schwellenwerts ausgelöst, bei Gehend bei Unterschreitung.

Um ein zu häufiges Generieren des Ereignisses zu verhindern, kann eine Hysterese eingestellt werden. Im Bild oben wird beim Überschreiten von 100 Einträgen das Script ausgeführt. Wenn die Queue danach kleiner als 90 Einträge wird und im Anschluss wieder auf über 100 steigt, wird das Script erneut ausgeführt. Dieser Mechanismus kann auch in die andere Richtung verwendet werden (Gehend).

 

Als auszulösende Reaktion bei Auftreten eines Events kann ein Script vom Typ OnTrigger aufgerufen werden.

Ist bei der Aufrufart "Zyklisch" als Aktion der Aufruf eines Scripts eingestellt, so können an diesem Script dennoch Änderungen vorgenommen werden. Diese werden dann beim nächsten Aufruf des Scripts nach dem Kompilieren übernommen.
