Funktionen der Publish/Subscribe-Schnittstelle

Subscribe

Deklaration: Subscribe(string topic, int qos)

Abonniert beim Broker Nachrichten zum Thema topic. qos definiert die Qualität des Services.

qos:

0 höchstens einmal

1 mindestens einmal

2 genau einmal

Beispiel:

PubSub.Mosquitto.Subscribe('temp/random',0)

 

Unsubscribe

Deklaration: Unsubscribe(string topic)

Beendet das Abonnement der Nachrichten zum Thema topic.

Beispiel:

PubSub.Mosquitto.Unsubscribe('temp/random',0)

 

Publish

Deklaration: Publish(string topic, string message, int qos)

Sendet eine Nachricht message zum Thema topic. qos definiert die Qualität des Services.

qos:

0 höchstens einmal

1 mindestens einmal

2 genau einmal

Beispiel:

PubSub.Mosquitto.Publish('temp/random', '12.0',0)

 

PublishEx

Deklaration: PublishEx(string topic, string message, int qos, bool dup, bool retain)

Sendet eine Nachricht message zum Thema topic. qos definiert die Qualität des Services.

qos:

0 höchstens einmal

1 mindestens einmal

2 genau einmal

dup gibt an, ob es sich um eine Wiederholung der Nachricht handelt (dup = true). Bei qos = 0 muss dup false sein!

retain = true veranlasst den Server, die Nachricht zu speichern, um den Wert auch zukünftigen Clients senden zu können.

Beispiel:

PubSub.Mosquitto.Publish('temp/random', '12.0',0,false,true)
