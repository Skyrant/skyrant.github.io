Debug-Funktionen über das Diagnose-Menü

Über das "Diagnose"-Menü können folgende "Debug"-Fenster geöffnet werden:

Performance-Monitor (Diagnose | Performance-Monitor)

Zeigt für bestimmte Prozesse (z.B. Befehle, Scripten) den Wert bzw. die Dauer in ms an. Folgende Werte werden angeboten: minimaler Wert, maximaler Wert, arithmetisches Mittel, aktueller Wert. Zu jedem Wert wird zusätzlich ein Zeitstempel erfasst, wann der Wert aufgetreten ist. Die Werte werden zur Laufzeit aktualisiert.

 

Beobachten (Diagnose | Beobachten neu bzw. Beobachten öffnen)

Zeigt den Status von ausgewählten Variablen. Der Eintrag der Informationen in das Fenster erfolgt, wenn "Beobachten" im Kontextmenü des Elements aktiviert wird. Wurde die Liste geändert, öffnet sich beim Schließen ein Dialog, um die Datei getrennt vom Projekt zu speichern. Dafür kann ein beliebiger Dateiname für die Debug-Datei (Endung .DBG) angegeben werden. Diese Datei wird im Projekt-Ordner des Cloud Connector Projekts abgelegt und kann jederzeit wieder über das Menü "Diagnose | Beobachten öffnen" geladen werden.

Debuggen von Scripten

Ein beliebiges Script kann über den Eintrag "Script debuggen" im Kontextmenü in ein Debug-Fenster geladen werden. Das Fenster erscheint erst, wenn das Script erstmals bearbeitet wird. Mit dem Start des Debug-Modus wird das Script (und die gesamte Bearbeitung aller Scripten) an der ersten Position angehalten und kann nun in Einzelschritten oder auch bis zum Ende des Scripts durch den Bediener fortgesetzt werden.

:

Das Laden eines Scripts in den Debug-Modus bewirkt einen generellen Stopp der Scriptbearbeitung!

Mit Start des Debug-Modus wird die Symbolleiste des Cloud Connectors um einige Schaltflächen erweitert:

Nächste Anweisung ausführen

Die Bearbeitung des Scripts wird in der nächsten Zeile fortgesetzt. Das Script (und damit die gesamte Scriptbearbeitung) verbleibt im Debug-Modus.

Ausführung fortsetzen

Das Script wird bis zum Ende durchlaufen. Das Script (und damit die gesamte Scriptbearbeitung) verbleibt aber im Debug-Modus, bis das Debug-Fenster durch den Anwender geschlossen wird.

Debugger beenden

Beendet den Debugger und schließt das Debug-Fenster.

Im oberen Teil des Fensters wird der Sourcecode des Scripts mit einem Balken für die aktuelle Bearbeitungsposition angezeigt. Im unteren Teil wird der Zustand der vom Script verwendeten Variablen zur aktuellen Position angezeigt.
