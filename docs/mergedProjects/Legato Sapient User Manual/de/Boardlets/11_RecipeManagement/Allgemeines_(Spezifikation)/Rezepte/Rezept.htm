Rezept anlegen

Rezepte werden über den LC2 angelegt.

 

Gehen Sie wie folgt vor, um ein Rezept im LC2 anzulegen.

 

  1. Navigieren Sie im LC2 unter Produktionsmanagement auf Rezeptverwaltung und Rezeptvorlagen (1).

  2. Fügen Sie im Hauptfenster entweder eine neue Rezeptvorlage hinzu oder wählen Sie eine bereits bestehende Rezeptvorlage aus. Klicken Sie hierzu auf die weiße Fläche im Hauptfenster (2).
    Weitere Informationen zum Thema Rezeptvorlagen finden Sie unter Rezeptvorlagen.

  3. Klicken Sie auf die unten angeordnete Plus-Schaltfläche (3).

    Hierdurch können Sie eine neue Rezeptvorlage anlegen.

  4. Navigieren Sie anschließend in der Baumstruktur im linken Teil der LC2-Oberfläche unter Produktionsmanagement, Rezeptverwaltung, Rezeptvorlage auf die jeweilige Rezeptvorlage (4) und auf Rezepte (5).




    Der beste Weg ist jedoch das Öffnen des angelegte Rezept über das Hauptfenster im LC2 durch Anklicken des jeweiligen Rezeptes.

  5. Wenn Sie im Hauptfenster des LC2 die angelegte Rezeptvorlage öffnen möchten, ziehen Sie an dem orangenen Balken das untere Element nach oben.
    In der Rezeptvorlage im unteren Fenster sehen Sie einzelne Parameter der Rezeptvorlage (6).

  6. Klicken Sie anschließend auf den Reiter Rezepte (7).
    Hier können Sie die Parameter für die jeweiligen Rezepte eingeben.
    Falls noch keine Parameter hinterlegt sind, können Sie über die Plus-Schaltfläche Parameter hinzufügen.





    Beachten Sie, dass sich die Parameter im Reiter Elemente (für die Rezeptvorlage) von den Parametern des Reiters Rezepte unterscheiden.
    Beachten Sie die relevanten Parameter:

Rezeptfunktion

Rezepte sind auf der jeweiligen SPS und auf Legato Sapient gespeichert und sind inhaltlich identisch.

Grundsätzliche Aufgaben beim Rezeptmanagement

Es gibt zwei grundsätzliche Aufgaben:

  • Abgleich der Rezepte zwischen SPS und Legato Sapient und

  • aktivieren eines Rezepts in der SPS (mit erfolgreicher Übertragung ist dies sofort in der Steuerung aktiv).

HMI und Laden eines Rezepts in die Steuerung

Das HMI stellt bestimmte Anforderungen an ein Rezept.

Beachten Sie die folgenden Punkte zum Laden eines Rezepts in die Steuerung.

 

  • Am HMI wird über eine Schaltfläche der Link der Rezeptauswahl in Legato Sapient aufgerufen.

  • Dabei kann der Stationsname (Knoten) übergeben werden, damit Legato Sapient bereits eine Liste von Rezepten einblendet, die nur für diesen Anlagenbereich auswählbar ist.

  • Der autorisierte User wählt das gewünschte Rezept aus.

  • Parameter mit Sollwerten werden angezeigt und mit dem Kommando Senden wird das identische Rezept in der SPS als Arbeitsprogramm aktiviert. Diese Funktion ist natürlich auch direkt in Sapient erreichbar.

Initialisierung der SPS

  • Nach einem Neustart der SPS (Neuanlauf der Steuerung), z.B. nach Wochenenden, Wartungs- oder Umbauarbeiten, werden alle Rezepte zwischen SPS und Legato Sapient ausgetauscht.

  • Legato Sapient überträgt, sofern das Masterflag gesetzt ist, den aktuellen Rezeptstand an die SPS.

  • Ist das Masterflag nicht gesetzt, werden die Rezeptdaten von der Steuerung übernommen und die in Sapient gespeicherten Werte automatisch überschrieben (dies wird in Sapient im Änderungsjournal festgehalten).

  • Wenn das Rezept in Legato Sapient nicht bekannt ist, muss zuerst eine entsprechende Struktur für das Rezept in Legato Sapient erstellt werden.

Rezeptwerte bearbeiten

  • Über einen HMI-Link wird in Sapient das Rezeptmodul ausgewählt und dort in die Rezeptauswahl gesprungen.

  • Mit entsprechender Berechtigung kann der Bediener hier Parameterwerte anpassen und an die Steuerung übertragen oder die Anpassungen verwerfen.

Rezeptdefinitionen kopieren und editieren

  • Da Rezepte zwischen verschiedenen Steuerungen ähnlich sein können, müssen sie kopierbar sein.

  • Im LC2 kann ein Rezept ausgewählt werden. Dabei wird die Editier-Funktion aktiviert. Hierdurch ist es beispielsweise möglich, einzelne Werte über die Copy & Paste-Funktion zu kopieren.
    Es kann aber auch die komplette Zeile mit Zeilen duplizieren kopiert und eingefügt werden.

  • Hier kann der Fertigungsknoten ausgewählt werden, in die das Rezept kopiert werden soll. Es erfolgt ein Namensvorschlag auf Basis des Ausgangsrezeptes. Dieser Namensvorschlag ist editierbar.

  • Ein so erzeugtes Rezept wird nicht automatisch an die SPS übertragen, hierzu müssen im Sapient-Client die Parameterwerte überprüft und die Übertragung aktiviert werden.

Rezept löschen

  • Es gibt auch die Möglichkeit, in Legato Sapient Rezepte zu löschen.

  • Dieser Löschvorgang wird in der SPS ausgelöst werden, d.h. Datenbereiche werden mit 0 überschreiben.

Rezept-Trigger

  • Rezepte werden über Ereignisse aktiviert.

  • Ein Ereignis kann der Senden-Befehl aus dem Rezeptmenü von Legato Sapient sein.

  • Kommt der Trigger von Legato Sapient, wird der Rezeptname an die SPS übertragen.

  • Die SPS aktiviert anschließend das intern gespeicherte Rezept und arbeitet fortan mit diesen Sollwerten.

Rezeptauswahl

  • Die Rezepte sind je SPS-Bereich individuell.

  • Für die Rezeptauswahl ist daher eine Baumstruktur hinterlegt. Eine flache Liste ist hier ungeeignet.

  • Deshalb sind Rezepte an Fertigungsknoten verknüpft, um hierüber eine Hierarchie abzubilden.
