Rezeptvorlagen

 

Allgemeines

Im LC2 können für Rezepte Vorlagen angelegt werden.

 

Beachten Sie folgende Punkte:

  • Eine Rezeptvorlage deckt die gesamte Struktur einer Rezeptur ab, ohne eine andere Verbindung zu anderen Objekten (P_CONNECTIONS, L_NODES, VALUES etc.) zu haben.

  • Die Rezeptvorlage ist somit unabhängig von den Objekten, die den Shopfloor abdecken. Rezepturvorlagen und Rezepte werden als Baum mit flexiblen Ebenen strukturiert.

  • Es wird beispielsweise eine Gruppe (Array) von Parametern verwendet, um ein bestimmtes Produkt abzudecken. Hierbei erhält die SPS die Parameter für alle möglichen Produkte in einem Rezept. Für diesen Anwendungsfall ist das Rezept mit Hilfe von Gruppen (Arrays) von Parametern für jedes mögliche Produkt strukturiert. Die Anzahl der Produkte (Gruppen im Rezept) kann dabei sehr hoch sein. Daher gibt es eine Kopierfunktion für Gruppen (Arrays), die dies vereinfachen.

  • Neue Rezepte werden mit Hilfe einer Rezepturvorlage oder direkt ohne Vorlage erstellt. In beiden Fällen werden die entsprechenden Werte (Records in der Tabelle RM_ITEM_VALUES [siehe Kapitel Datenmodell]) automatisch angelegt. Wenn ein Element in der Rezepturstruktur entfernt wird, wird der entsprechende Wert (Datensatz in der Tabelle RM_ITEM_VALUES [siehe Kapitel Datenmodell]) ebenfalls automatisch gelöscht.

  • Wenn ein neues Rezept mit Hilfe einer Rezepturvorlage erstellt wird, können in diesem Rezept "Standardwerte" gesetzt werden, die auf den Standardwerten der Vorlage basieren.

  • Generell wird ein Rezept als JSON-Objekt gespeichert.

  • Mit Hilfe des SCC werden alle Parameter eines Rezeptes an die SPS übertragen.

  • Durch Skripte kann ein Rezept auf Richtigkeit und Gültigkeit validiert werden.

 

Im LC2 können Sie:

  • Die Grundstruktur eines Rezeptes definieren.

  • SPS-Datentypen verwalten, die in Rezepten verwendet werden (Inhalt der Tabelle P_DATA_TYPES [siehe Kapitel Datenmodell]),

  • "Rezepturmodi" verwalten, die das Betriebsverhalten von Rezepturen steuern (Inhalt der Tabelle RM_RECIPE_MODES [siehe Kapitel Datenmodell]),

  • die "Up-/Download Results" verwalten, die in der Web-GUI angezeigt und optional in die SPS-Ebene übertragen werden
    (Inhalt der Tabelle     RM_UP_DOWNLOAD_RESULTS [siehe Kapitel Datenmodell]),

  • Rezepturvorlagen verwalten (anlegen, bearbeiten, löschen),

  • die Erstellung von Rezepturvorlagen durch eine Kopierfunktion für Gruppen (Arrays) vereinfachen,

  • neue Rezepte auf Basis einer gespeicherten Rezepturvorlage inkl. der Einstellung für "Standardwerte" erstellen,

  • das Anlegen von Rezepten durch eine Kopierfunktion für Gruppen (Arrays) vereinfachen,

  • Rezepte verwalten (anlegen, bearbeiten, löschen).

  • Standardrezeptdaten erstellen.

 

Anlegen einer Rezeptvorlage

Es gibt zwei Wege, um eine Rezeptvorlage anzulegen:

  1. Durch das Hinzufügen Rezeptvorlage-Dialogfenster.

  2. Durch das Plus-Symbol (1) im Rezeptvorlagen-Menü im LC2-Hauptbildschirm.

 

Der bevorzugte Weg, eine Rezeptvorlage anzulegen ist durch das Erstellen einer neuen Zeile im Rezeptvorlagen-Reiter:

  1. Navigieren Sie zu Produktionsmanagement Rezeptverwaltung Rezeptvorlagen und klicken Sie in die leere, weiße Fläche im Rezeptvorlagen-Reiter.

  2. Klicken Sie anschließend auf das Plus-Symbol (1).

Hierdurch wird eine neue Zeile mit ID (2), Sortiernummer (3), usw. generiert:

 

 

 

Im Rezeptvorlagen-Reiter des LC2 können Sie auf die Elemente der Rezeptvorlage zugreifen, in dem Sie die orangenen Balken anklicken und mit gedrückter Maustaste das Fenster nach oben ziehen.

Rezept abrufen und bearbeiten

Die einzelnen Rezepte, die einer Rezeptvorlage angehören, können im Hauptfenster des LC2 abgerufen und bearbeitet werden:

 

  1. Wählen Sie hierzu im Hauptfenster des LC2 eine Rezeptvorlage (1) aus.

  2. Wählen Sie im Fenster des unteren Bereichs (großgezogen durch den orangenen Balken) zunächst den Reiter Elemente (2) aus.

  3. Wählen Sie in diesem Reiter anschließend das jeweilige Element aus.

  4. Wählen Sie im Reiter Rezepte das gewünschte Rezept (3) aus.

  5. Sie können das markierte Rezept abschließend bearbeiten (4).

 

 

 

Beachten Sie das Kapitel Rezepte, um für das Anlegen von Rezepten weitere Details zu erhalten. Die Definition der einzelnen Parameter des Reiters Rezepte können Sie ebenfalls im Kapitel Rezepte abrufen.

Datentypen

Die Elemente der Rezeptvorlagen entsprechen verschiedenen Datentypen. Durch die Eingrenzung des Datentyps wird sichergestellt, dass die richtige Eingabe erfolgt, d.h. bei bestimmten Rezeptparametern darf nur eine Ganzzahl oder eine Zeichenkette angegeben werden.

Generell bilden die Datentypen die Grundlage für die Informationen aus der Steuerung. Für die Steuerung müssen die Datentypen angepasst sein, da diese die Grundstruktur der Datenübertragung darstellen. Der SCC überprüft, ob die Datentypen in der SPS und in Legato Sapient gleich sind.

Bearbeiten einer Rezeptvorlage

Im LC2 kann eine Rezeptvorlage wie folgt abgerufen und bearbeitet werden:

  1. Navigieren Sie im Navigationsbaum auf Rezeptverwaltung und auf Rezeptvorlagen (1).

  2. Klicken Sie auf eine Rezeptvorlage (2).

  3. Im sich öffnenden Baummenü können Sie auf Elemente (3) klicken.

  4. Dort können Sie die einzelnen Parameter (4) anwählen.

  5. Unter Rezepte (5) können Sie auf die jeweiligen Rezepte innerhalb der Rezeptvorlage (2) zugreifen.

  6. Bearbeiten Sie einzelne Werte im Hauptbildschirm des LC2 (6).

 

 

  • Ein Rezept wird im LC2 in den Rezeptvorlagen (1) in einer Baumstruktur mit flexiblen Ebenen strukturiert.

  • Für die Struktur eines Rezepts (2) gilt Folgendes (siehe Rezeptprozedur RM_RECIPE_ITEMS):

    • Die Struktur eines Rezeptes ist eine Liste von Rezeptelementen (auch "Zutaten" genannt).

    • Diese Struktur ist geordnet.

    • Jedes Rezept hat einen Typ als "SPS-Datentypen" definiert, welche in den Rezeptelementen verwendet werden (Inhalt der Tabelle P_DATA_TYPES).

    • Rezeptelemente können in Gruppen angeordnet werden, die man zwischen den Rezepten kopieren kann.

  • Ein Rezept hat einen Modus "Recipe Mode", der das Betriebsverhalten von Rezepten steuert (Inhalt der Tabelle RM_RECIPE_MODES).

  • Dieser "Recipe Modus" für ein neues Rezept wird auf "0" gesetzt (Upload SPS > MES), und kann vom Anwender geändert werden.

  • Ein Rezept ist unabhängig von den Objekten, die den Shopfloor abdecken.

 

Die Werte aller Rezeptelemente werden aus den SPSen importiert, wenn sie von Sapient ausgelöst werden. Sie werden daher nicht in LC2 konfiguriert.
