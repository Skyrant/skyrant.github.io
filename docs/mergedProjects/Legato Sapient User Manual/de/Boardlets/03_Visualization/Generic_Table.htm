Extern - Generische Tabelle

Allgemein

Das Boardlet Generische Tabelle dient vor allem dazu, individuelle Abfragen zu verwirklichen und die Ergebnisse gegebenenfalls als Parameter an andere Boardlets weiterzugeben. Es ist möglich, eine beliebige Datenquelle abzufragen und anzuzeigen, egal ob aus dem Sapient Datenmodell oder einer anderen Datenbank.

Bei der Abfrage muss bei Datasource der Name der DB-Tabelle eingetragen werden (z.B. sys_units).

Für das Boardlet Generische Tabelle stehen außerdem folgende Einstellungen zur Verfügung.

Reiter Einstellungen

Zeile bei Klick senden:

Ist dieser Schieberegler aktiv, wird die selektierte Spalte in der Tabelle bei Klick an eine Empfänger-Tabelle mit dem Platzhalter $detailinfo$ als Filterkriterium gesendet.

Für Details zu Filterstrings, siehe bitte Filterstrings, für Details zu Platzhaltern, siehe bitte Platzhalter.

Sortieren nach:

Name der Spalte, nach der sortiert werden soll. Spaltenname wie in Datenbank, klein geschrieben. Sofern die Tabelle im Client sortierbar ist, wird dieses Feld beim Sortieren auch automatisch befüllt.

Verbindung:

Eingabe der Verbindung, aus der die Datenquelle stammt, die angezeigt werden soll, z.B. Legato oder Repository.

Seitenlimit:

Anzahl der Zeilen, die auf einmal geholt werden. Wird nur ausgewertet, wenn der Schieberegler "Paging" aktiv ist. Sonst werden alle Datensätze geladen.

Datenquelle:

Eingabe der Datenquelle, die angezeigt werden soll.

Spalten:

Hier können die anzuzeigenden Spalten eingeschränkt werden. Keine Eingabe = alle Spalten werden angezeigt. Ansonsten Eingabe der Spaltennamen aus der Datenbank als komma-separierte Liste ohne Leerzeichen, nur Kleinbuchstaben.

Alias:

Hier kann ein Alias für jede Tabelle definiert werden, z.B. "gp_r_datasources as source". Dieser Alias kann dann verwendet werden, um die Tabelle in den Spalten-Definitionen zu referenzieren oder um Joins zu erstellen.

:

Der erste Alias muss der der Haupttabelle sein!

Mehr Details zu diesem Thema finden Sie im Kapitel Joins.

Relationen:

Hier können Relationen/Beziehungen definiert werden, um Daten von mehreren Tabellen abzufragen. Zum detaillierten Vorgehen, lesen Sie bitte das Kapitel Joins.

Tabelle ist sortierbar in ...:

Auswahlliste, wo die Tabelle sortiert werden soll. Ist von Bedeutung, wenn Paging aktiviert ist.

  • DB: Das Sortierkriterium wird in der Abfrage an die DB mitgegeben, es wird in der DB sortiert und dann die festgelegte Anzahl Resultate zurückgeliefert. Beim Sortieren über den Spaltenheader im Client wird keine neue Anfrage mehr geschickt, sondern die bestehenden Ergebnisse lediglich anders sortiert. Beispiel: Sort by ID liefert die ersten 50 Treffer aus der DB (IDs 1-50). Wird danach im Client nach Name sortiert, werden trotzdem die Einträge mit den IDs 1-50 angezeigt.

  • Client: Die Sortierung erfolgt im Client, d.h. jedes Sortieren über den Spaltenheader stößt eine neue Query an und liefert andere Ergebnisse.

  • False: Es erfolgt keine Sortierung, z.B. weil alle Ergebnisse geholt werden.

Paginierung:

Über diesen Schieberegler wird definiert, ob nur die oben eingetragene Anzahl Zeilen geholt werden sollen oder alle Ergebnisse. Wenn aktiv, wird die festgelegte Anzahl Zeilen so oft geholt, bis die sichtbare Tabelle voll ist. Danach werden weitere Ergebnisse geholt, wenn man an den unteren Rand scrollt. Wenn nicht aktiv, werden immer alle Ergebnisse geholt. Insbesondere bei Tabellen mit vielen Datensätzen empfiehlt sich hier eine Einschränkung, um eine gute Performance zu gewährleisten.

Spalten-Definitionen:

Definition, wie die Spalten aussehen sollen. Hierfür steht ein Editor zur Verfügung:

Klick auf + fügt eine weitere Spalte hinzu. Der Eintrag besteht aus Wertepaaren mit folgender Bedeutung:

  • label = Beschriftung der Spalte

  • valuePath = Spaltenname in DB

  • width = Spaltenbreite

  • sortable = sortierbar? Kann true oder false sein

  • component = hier muss "sortable-table-header-component" eingetragen werden, damit zum Sortieren die Pfeile angezeigt werden.

Klick auf das Listen-Symbol öffnet Optionen, z.B. zum Löschen des Eintrags oder zur An- und Abwahl weiterer Parameter:

Reiter Layout

Tabellenbreite bestimmen:

Hier kann festgelegt werden, wie die Tabellenbreite bestimmt wird; feste Breite oder automatisch.

Reiter Filter

Filtern nach ...:

Möglichkeit, die Ausgabe mittels eines Filterstrings mit Parametern zu filtern. Es können die Platzhalter $node$, $detailinfo$, $timeStart$ und $timeEnd$ verwendet werden.

Für Details zur Formatierung siehe bitte Filterstrings.

Für Details zur Bedeutung der einzelnen Platzhalter, siehe bitte Platzhalter.

Feld der Detailinformation:

Soll das Boardlet Empfänger von Datein aus einer anderen Generischen Tabelle sein, kann hier festgelegt werden, mit welchem Wert ein im Filter verwendeter Platzhalter $detailinfo$ aus der sendenden Tabelle ersetzt werden soll, z.B. name. Der Parametertyp sollte hier immer auf local gesetzt sein.

Für weitere Infos zu Filterstrings, siehe bitte Filterstrings.

Für Details zur Bedeutung der einzelnen Platzhalter, siehe bitte Platzhalter.
