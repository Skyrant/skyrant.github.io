Extern - iFrame

Allgemein

Dieses Boardlet kann verwendet werden, um eine externe Webseite oder ein SVG einzubetten. Es kann auch ein Knotenbezug hergestellt werden und so je nach Knoten ein anderes Bild / eine andere Seite angezeigt werden.

Im Beispiel ist die Webseite www.w3schools.com eingebettet.

Einstellungen

Die häufigsten Einstellungen auf den Reitern Einstellungen, Layout und Filter sind unter Boardlet-Einstellungen beschrieben:

  • Reiter Einstellungen

  • Reiter Layout

  • Reiter Filter

Für das iFrame-Boardlet stehen außerdem folgende Einstellungen zur Verfügung.

Reiter Einstellungen

URL:

URL der anzuzeigenden Webseite oder Pfad zum SVG.

Enthält die URL den Platzhalter $loginToken$, wird die Prozedur User_getLoginToken aufgerufen und der zurückgegebene Wert eingefügt. Gibt die Prozedur einen Fehler zurück, wird ein Leerstring in den Platzhalter eingesetzt. Dies ermöglicht die Anmeldung an einer anderen Applikation mit dem User, der an der Sapient Web App angemeldet ist.

Um beispielsweise ein SVG aus Legato Classic anzuzeigen, muss der Pfad zunächst die Adresse der Classic-Webapplikation enthalten und kann zusätzlich um die Angabe des Views, eines Benutzers und ggf. einer Knoten-ID erweitert werden. Beispiel: http://legato.classic.de/Legato/visu.html?l_username=xxx&l_password=xxx

Welche Parameter zum Aufruf von Legato Classic übergeben werden können, kann in der Onlinehilfe der Classic-Anwendung nachgeschlagen werden ("Direktaufruf von Ansichten über eine URL).

Zusätzlich können die Platzhalter $node$ und $detailInfo$ verwendet werden. Diese dienen dazu, um abhängig von den Werten dieser Parameter andere Informationen anzuzeigen. Befüllt werden die Parameter durch die Boardlets Knotenlayout / Knotenbaum und Generische Tabelle.

Für Details zu den Platzhaltern, siehe bitte Platzhalter.

Reiter Filter

Knoten-ID:

ID des Knotens, der zuletzt im Layout-Boardlet / Knotenbaum selektiert wurde. Dieser Wert wird in den Parameter $node$ eingetragen und für die Abfrage verwendet.

Detailinfo:

Inhalt der selektierten Spalte, die in der Generische Tabelle als Detail-Info festgelegt wurde. Dieser Wert wird in den Parameter $detailInfo$ eingetragen und für die Abfrage verwendet.

Für Details zu Filterstrings, siehe bitte Filterstrings.

Feld der Detailinformation:

Soll das Boardlet Empfänger und nicht Sender sein, kann hier festgelegt werden, mit welchem Wert ein im Filter verwendeter Platzhalter $detailinfo$ aus der sendenden Tabelle ersetzt werden soll, z.B. name. Der Parametertyp sollte hier immer auf local gesetzt sein.

Für weitere Infos zu Filterstrings, siehe bitte Filterstrings.

Für Details zur Bedeutung der einzelnen Platzhalter, siehe bitte Platzhalter.

Benötigte Parametrierung im LC2

Für dieses Boardlet müssen keine Daten im LC2 konfiguriert werden.
