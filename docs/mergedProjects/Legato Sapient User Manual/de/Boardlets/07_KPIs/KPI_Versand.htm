Vorlagen

Vorlagen definieren die Parameter und den Zeitplan des zu konfigurierenden kpi-Versandauftrags.

Wählen Sie + Neue Vorlage hinzufügen, um den Konfigurationsdialog für Vorlagen zu öffnen.

Vorlagensymbol
Eindeutiges Symbol für den KPI-Versandauftrag.

Vorlagenname
Der Name der Vorlage in der Sprache der Vorlage. Sprachindizes können definiert und hinterlegt werden.

Vorlagenbeschreibung
Fügen Sie dem ausgewählten kpi-Mailing-Auftrag relevante Notizen für Empfänger hinzu.

Knoten
ID des abzufragenden Anlagenknotens.

SE Module
Ausführende Instanz des ausgewählten SE-Auftrags.

Sprachgebietsschema
Legt die Parameter Datum, Sprache und Einheit fest. Nur für das installierte System konfigurierte Sprachen sind auswählbar.

E-Mail-Thema
Vorlage für den Betreff der generierten Mails.

E-Mail-Text
Vorlage für den Inhalt der generierten Mails. 

Content-Typ
MIME-Inhaltstyp, der für die generierte E-Mail festgelegt werden soll. Muss gesetzt werden, wenn der Mail-Inhalt nicht reiner Text ist, sondern ein anderes Format wie z.B. HTML.

Aktiv
Aktiviert/ deaktiviert den Auftrag.

Test 
Prüft, ob eingefügte Parameter logisch sind.

Nutzbare Platzhalter für E-Mail Thema und Text:

  • ${TEMPLATE!field!langix} Wird durch das angegebene Feld der Vorlage ersetzt. Das Feld kann sein:
    • SYM - Symbol der Vorlage (Sprache wird ignoriert).
    • NAME - Der Name der Vorlage in der Sprache der Vorlage, sofern kein spezifischer Sprachindex angegeben ist.
    • DESC - Die Beschreibung der Vorlage in der Sprache der Vorlage. Es können spezifische Sprachindizes angegeben werden.
  • ${SCHEDULE!field!langix} Wird durch das angegebene Feld des Zeitplans ersetzt:
    • SYM - Symbol der Vorlage (Sprache wird ignoriert).
    • NAME - Der Name des Zeitplans in der Sprache der Vorlage. Zusätzliche Sprachindizes können definiert werden.
    • DESC - Die Zeitplanbeschreibung in der Vorlagensprache, sofern kein spezifischer Sprachindex angegeben ist.
  • ${NODE!field!langix} Wird durch das angegebene Feld des für die Vorlage eingerichteten Anlagenknotens ersetzt:
    • ID - Die Knoten ID (Sprache wird ignoriert)
    • SYM - Kurzbezeichnung des Knotens (Sprache wird ignoriert)
    • NAME - Der Knotenname in der Vorlagensprache, sofern kein spezifischer Sprachindex angegeben ist.
    • DESC - Die Knotenbeschreibung in der Vorlagensprache, sofern kein spezifischer Sprachindex angegeben ist.
  • ${DATE!format!langix} - Formatiert den aktuellen Zeitstempel entsprechend dem angegebenen Format.
  • ${SHIFT!offset!...} - Informationen über die abgeschlossene Schicht:
    • ${SHIFT!offset!ID} - ID der Schicht.
    • ${SHIFT!offset!NAME!langix} - Name der Schicht in der Vorlagensprache, sofern kein spezifischer Sprachindex angegeben ist.
    • ${SHIFT!offset!START!format!langix} - Startzeit der Schicht wie angegeben formatiert.
    • ${SHIFT!offset!END!format!langix} - Ende der Schicht, formatiert wie angegeben.
    • ${SHIFT!offset!PDAY!format!langix} - Produktionstag der Schicht, wie angegeben formatiert.

Nutzbare Platzhalter für E-Mail Text

  • KPI values: [!K:node:offset:kpi!] - fügt den angegebenen KPI des angegebenen Knotens aus der n-ten abgeschlossenen Schicht ein.
    • node - Die ID des Knotens oder sein Kurzname (einer der Platzhalter des 1. Durchgangs ${NODE!ID} oder ${NODE:SYM} kann verwendet werden)
    • offset - gibt den Knotenpunkt der Schicht an. Z.B. 1 ist die letzte (neueste) abgeschlossene Schicht, 2 die davor usw.
    • kpi - Das Symbol für den KPI-Typ
  • KPI Limits: Die für eine Kennzahl konfigurierten Ober- und Untergrenzen, verwenden Sie KU für die Ober- oder KL für die Untergrenze.
  • Shift protocol values: [!P:node:offset:varsym!] wird durch Werte aus dem Schichtprotokoll ersetzt.
  • Aktuelle Werte: [!V:varsym!]

Auslöser

KPI Versand Jobs können über Auslöser automatisiert ausgeführt werden. Der Dialog Auslöser hinzufügen hilft dabei einem Mailing job einen bestimmten oder mehrere automatische Auslöser hinzuzufügen

Wählen Sie Auslöser hinzufügen ,um den Dialog Auslöser hinzufügen zu öffnen.

Name
Geben Sie einen Namen für den Trigger ein.

Beschreibung
Geben Sie eine kurze Beschreibung für den Trigger ein.

Auslöser-Typ
Wählen Sie einen Auslösertyp aus dem Dropdown-Menü.

Zeitzone (Optional)
Geben Sie die gewünschte Zeitzone aus dem Dropdown-Menü ein.

Knoten
Geben Sie den Knoten an, auf den der Trigger zugreifen soll, indem Sie den Knotennamen eingeben. (Für den Auslösertyp KPI-Berechnung)

Versatz
Definiert den Versatz von Schichten, deren Werte abgefragt werden sollen. Standardwert ist 1 für die letzte abgeschlossene Schicht. Der Wert 2 fragt die Schicht vor der letzten abgeschlossenen Schicht ab. (Für den Auslösertyp KPI-Berechnung)

Schichtgruppe
Geben Sie die Schichtgruppe an, die von dem Trigger betroffen ist. (Für den Auslösertyp KPI-Berechnung)

Produktionsgruppe
Geben Sie die Produktionsgruppe ein, die von dem Auslöser betroffen ist. (Für den Auslösertyp KPI-Berechnung)

Klicken Sie auf Absenden, um die Konfiguration des ausgewählten Parameters zu bestätigen.

Abonnements

KPI-Mailingaufträge können an ausgewählte Abonnenten gesendet werden. Der Dialog Abonnenten hinzufügen hilft Ihnen, eine Auswahl von Empfängern für einen ausgewählten Mailing-Job zu treffen.


Klicken Sie auf Abonnenten hinzufügen, um das Dialogfeld Abonnenten hinzufügen zu öffnen.


Abonnenten
Zeigt die Liste der eingestellten Abonnenten an, die eine E-Mail aus dem ausgewählten Mailing-Job erhalten können.


Neue Abonnenten hinzufügen
Fügen Sie der Abonnentenliste aus dem Dropdown-Menü Empfänger hinzu, indem Sie die Sapient personID des Empfängers eingeben oder auswählen.

Aktiv
Aktiviert/deaktiviert den ausgewählten Empfänger für den Empfang von E-Mails des ausgewählten Jobs.


Anzahl und ausgewählte Empfänger eines Jobs werden in den Spalten Abonnements Liste und Abonnements Anzahl im KPI Mailing Boardlet angezeigt.
