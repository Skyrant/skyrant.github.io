Aufgabenverwaltung

Allgemein

Das Ziel der "Aufgabenverwaltung" ist es, eine Möglichkeit zu bieten, Aufgaben zu verwalten, die im Rahmen der täglichen Arbeitsroutine auftreten. Eine Aufgabe kann über eine "Aufgabe hinzufügen"-Funktion bei dem Boardlet, bei dem der Bedarf entdeckt wurde, manuell erstellt werden. Dabei kann es sich um ein beliebiges Boardlet handeln, die Funktion ist nicht auf SFM-Boardlets eingeschränkt.

Um präzise und einfach auswert- und durchsuchbar zu sein, sollte eine Aufgabe mindestens die folgenden Arten von Datenattributen enthalten: Titel, Beschreibung, verantwortliche Person, Fälligkeitsdatum, Priorität, URL. Wenn eine Aufgabe angelegt wird, müssen wichtige Referenzen mit der Aufgabe zusammen gespeichert werden. Diese sind: Beziehung zu einem Knoten, Verweis auf ein SFM-Ereignis (wenn verfügbar), Erstellungsdatum und -zeit, Ersteller.

Außerdem ist es möglich, Aufgaben mit Tags zu versehen. Verfügbare Tags werden in einer Dropdown-Liste angeboten.

Der Arbeitsfortschritt einer Aufgabe wird durch das Attribut "Zustand" angegeben. Die verschiedenen Zustände können im LC2 parametriert werden.

Aufgaben werden als Liste im Boardlet angezeigt. Üblicherweise werden pro Boardlet alle Aufgaben angezeigt. Über den Filter kann aber auch eingeschränkt werden, für welche Zustände Aufgaben angezeigt werden sollen.

Beim Bearbeiten des Boardlets im Bearbeitungsmodus kann definiert werden, welche Attribute in der zusammengeklappten Ansicht einer Aufgabe angezeigt werden (Fälligkeit, Beschreibung, Tags, Externe Tool-Links oder Anhänge). Hier ist eine Mehrfach-Auswahl möglich.

Es ist möglich, die Aufgaben nach allen Attributen zu filtern, z.B. "fällig in den nächsten xyz Tagen" oder "Aufgaben, die mir zugeordnet sind".

Ausrufezeichen am Anfang einer Zeile geben Aufschluss über die Priorität der Aufgabe (keine, ! = niedrig, !! = mittel, !!! = hoch).

Es ist möglich, Aufgaben in Listen zu gruppieren, die frei in der WebApp konfiguriert werden können. Eine farbige Markierung zeigt an, zu welcher Liste die Aufgabe gehört.

Durch Klick auf v kann die Aufgabe aufgeklappt werden, um mehr Informationen anzuzeigen. Sofern das Fälligkeitsdatum nicht ohnehin im zusammengeklappten Modus angezeigt wird, wird es nun in der Kopfzeile eingeblendet.

In der Detailansicht wird der Verantwortliche für die Aufgabe angezeigt, der Status, das Fälligkeitsdatum, die Beschreibung, Kommentare, Tags, Anhänge und Links zu externen Tools.

Über die Dropdown-Liste kann der Status der Aufgabe verändert werden, wodurch diese in das entsprechende Boardlet verschoben wird. Außerdem kann durch Klick auf ein Kommentar erfasst werden. Weitere Bearbeitungsmöglichkeiten öffnen sich durch Klick auf das Stift-Symbol in der Seitenleiste. Durch Klick auf kann die Aufgabe gelöscht werden.

Edit-Modus

Durch Klick auf das Stift-Icon bei einer Aufgabe kann diese bearbeitet werden. Es öffnet sich folgender Dialog:

Die folgenden Daten können erfasst bzw. geändert werden: Name der Aufgabe, zugeordnete Liste (Auswahl aus bestehenden Listen), Verantwortlicher, Fälligkeitsdatum, Priorität, Knotenzuordnung (sofern gewünscht) Beschreibung, Tags, Anhänge, Links zu externen Tools.

Bei den Tags kann aus einer Dropdown-Liste ein bereits existierender Tag selektiert werden.

Neue Liste anlegen

Zum Anlegen einer neuen Liste muss man im Edit-Modus in den Reiter "Liste" wechseln ().

Es müssen eine Farbe selektiert werden sowie ein Name und eine Beschreibung vergeben werden.

Für bereits existierende Listen kann die Farbe geändert werden und sie können gelöscht werden.

Neue Aufgabe anlegen

Durch Klick auf + in der Kopfzeile des Boardlets kann eine neue Aufgabe mit dem für das Boardlet definierten Status angelegt werden. Werden im Boardlet mehrere Zustände angezeigt, erhält die neue Aufgabe den, der als erstes dem Filter hinzugefügt wurde. Am Anfang der Liste öffnet sich folgende Eingabemaske:

Es muss ein Name eingegeben werden. Sonst kann die Aufgabe nicht angelegt werden. Verantwortlicher und Fälligkeit können direkt hier eingegeben werden oder später beim Bearbeiten in der Seitenleiste zugewiesen werden.

Für das Aufgabenverwaltung Boardlet stehen folgende weitere Einstellungen zur Verfügung.

Reiter Einstellungen

Fälligkeitsdatum:

Über diesen Schieberegler kann definiert werden, ob im eingeklappten Modus einer Aufgabe zusätzlich das Fälligkeitsdatum angezeigt werden soll.

Beschreibung:

Über diesen Schieberegler kann definiert werden, ob im eingeklappten Modus einer Aufgabe zusätzlich die Beschreibung angezeigt werden soll.

Tags:

Über diesen Schieberegler kann definiert werden, ob im eingeklappten Modus einer Aufgabe zusätzlich die Tags angezeigt werden sollen.

Anhänge:

Über diesen Schieberegler kann definiert werden, ob im eingeklappten Modus einer Aufgabe zusätzlich die Anhänge angezeigt werden sollen.

Links zu externen Tools:

Über diesen Schieberegler kann definiert werden, ob im eingeklappten Modus einer Aufgabe zusätzlich die Links zu externen Tools angezeigt werden sollen.

Reiter Filter

Knoten-ID:

ID des Knotens, für den Werte angezeigt werden. Kann entweder hier geändert werden oder durch Auswahl eines anderen Knotens im Boardlet Knotenlayout oder im Boardlet Knotenbaum (außerhalb des Bearbeitungsmodus).

Nach Zeit filtern:

Über diesen Schieberegler kann definiert werden, ob ein Zeitfilter auf das Fälligkeitsdatum angewendet werden soll oder nicht.

Fälligkeitsdatum:

Hier kann ein Zeitraum eingegeben werden, in dem sich das Fälligkeitsdatum befinden soll.

Name des Verantwortlichen:

Dropdown-Liste zur Auswahl des Verantwortlichen, für den Aufgaben angezeigt werden sollen.

Liste:

Dropdown-Liste zur Auswahl der Liste, der die angezeigten Aufgaben zugeordnet sein sollen.

Tags:

Dropdown-Liste zur Auswahl eines oder mehrerer Tags, die den angezeigten Aufgaben zugewiesen sein müssen.

Priorität:

Dropdown-Liste zur Auswahl der Priorität, die den angezeigten Aufgaben zugewiesen sein muss.

Fälligkeit erreicht:

Über diesen Schieberegler wird definiert, ob nur Aufgaben angezeigt werden sollen, deren Fälligkeitsdatum bereits erreicht ist oder alle.

Sortieren nach:

Dropdown-Liste zur Auswahl des Felds, nach dem die Aufgaben sortiert werden sollen.

Aufgaben-Zustand:

Über die Dropdown-Liste kann der Zustand gewählt werden, der die angezeigten Aufgaben zugeordnet sein soll. Mehrfachauswahl ist möglich.
