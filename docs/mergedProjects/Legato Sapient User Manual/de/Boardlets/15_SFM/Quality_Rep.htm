Qualitätsbericht

Allgemein

Das Boardlet Qualitätsbericht dient dazu, Qualitätsprobleme zu erfassen. Der selektierte Monat wird für den selektierten Knoten und alle Unterknoten dargestellt und jeder Tag gemäß Vorgabe eingefärbt (die Bedeutung der Farben kann im LC2 definiert werden, der Status mit der Sortiernummer 0 wird als Default vorbefüllt). Dabei hat der Eintrag mit der höchsten Priorität Vorrang. Es wird immer ein kompletter Monat angezeigt, unabhängig vom gewählten Zeitraum. Wird ein Zeitraum > 1 Monat selektiert, so wird der letzte Monat im Zeitraum angezeigt (bei 2016 also z.B. Dezember, beim aktuellen Jahr der aktuelle Monat).

Erfassungs-Dialog

Standardmäßig befindet sich das Boardlet im Edit-Modus, d.h., es können durch Klick auf einen Tag neue Vorkommnisse erfasst werden. Nach Auswahl einer Farbe öffnet sich in der rechten Seitenleiste ein neuer Dialog:

Hier kann direkt eine Aufgabe für das Ereignis erfasst werden. Der Tag "Sicherheit" ist dabei automatisch befüllt. Siehe auch Aufgabenverwaltung.

Durch Klick auf das Stiftsymbol oben wechselt der Dialog zur Erfassung des eigentlichen Ereignisses.

Hier kann das Qualitätsproblem beschrieben werden.

Analyse-Modus

In der Menüleiste des Boardlets Qualitätsbericht kann durch Klick auf in den Analyse-Modus geschaltet werden.

Hier sieht man für die einzelnen Tage, welche Qualitätsprobleme in welcher Anzahl erfasst wurden. Beispielsweise wurden am 23.11. zwei Reklamationen und zwei Qualitätsprobleme auf eigener Kostenstelle erfasst.

Durch Klick auf kann wieder in den Edit-Modus geschaltet werden.
