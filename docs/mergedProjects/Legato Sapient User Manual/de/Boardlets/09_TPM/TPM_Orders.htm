TPM Wartungsaufträge

Allgemein

Das Boardlet TPM Wartungsaufträge zeigt alle offenen Wartungsaufträge zum selektierten Knoten an. Es werden verschiedene Informationen angeboten, beispielsweise die geplante Fälligkeit, der Auslöser und der aktuelle Status.

Mittels Drag & Drop lassen sich die Breite oder Position der Spalten einfach verändern. Durch Klick auf eine Zeilenüberschrift wird nach dieser Spalte sortiert.
Nach welcher Spalte sortiert ist, und ob auf- oder absteigend sortiert ist, wird durch die dreieckigen Symbole angezeigt:

Nach Spalten, die mehr als einen Eintrag enthalten können, kann nicht sortiert werden, z.B. Auslöser oder Instandhaltungs-Gruppe (IH-Gruppe).

Die Symbole in der ersten Spalte zeigen an, wodurch der Wartungsauftrag angefordert wurde:

durch Wartungsplan angefordert

durch Benutzer angefordert

Status ändern

Der aktuelle Status des Wartungsauftrags wird in der Spalte State des Boardlets TPM Wartungsaufträge angezeigt. Durch Klick auf die jeweiligen Symbole in der Boardlet-Menüleiste wird der ausgewählte Wartungsauftrag in den nächsten Status gesetzt.

Es gibt folgende Statusarten:

  • zu planen

  • geplant

  • übernommen

  • erledigt

  • abgelehnt

zu planen:


Der Auftrag muss einem Benutzer zugewiesen werden, der ihn ausführen soll. Durch Klick auf das Symbol in der Boardlet-Menüleiste öffnet sich der Dialog Order zuordnen in der Seitenleiste: ------>>

Nachdem Sie die Felder bearbeitet und auf Bestätigen geklickt haben, zeigt die Zelle in der Spalte State den nächsten Status an: geplant.

Sie können auch mehrere Wartungsaufträge im Boardlet auswählen und einem Verantwortlichen zuweisen, wenn folgende Bedingungen erfüllt sind:

  • Bedingung 1: Nur der jüngste Wartungsauftrag eines Wartungsplans ist ausgewählt.
    Beispiel: Wenn Plan A die Aufträge X (10.05.2020), Y (30.05.2020) und Z (21.06.2020) hat, dann ist die Zuweisung nur für Auftrag Z möglich.

  • Bedingung 2: Die markierten Aufträge haben dieselbe IH-Gruppe.

Eine Fehlermeldung erscheint, wenn die Mehrfachauswahl die Bedingungen für die Zuweisung an einen Verantwortlichen nicht erfüllt.

 

 

Beispiel zur Veranschaulichung von Bedingung 1 und 2:

A

Jede Reihe steht für einen Wartungsauftrag.

B

Diese Spalte zeigt die Namen der Wartungspläne, zu denen die Wartungsaufträge gehören.

C

Zwei Wartungsaufträge für denselben Plan "Ölwechsel", aber nur der jüngste Auftrag ist ausgewählt.

D

Alle ausgewählten Wartungsaufträge gehören zur selben IH-Gruppe "IHGruppe1".

 

Ergebnis von B, C und D: Sie können mit der Zuweisung fortfahren.

geplant:


Der Auftrag wurde einem Benutzer zugewiesen und wartet auf Erledigung. Der Name des zugewiesenen Benutzers wird in der Spalte Responsible angezeigt.
Nachdem der zugewiesene Benutzer das Symbol in der Boardlet-Menüleiste geklickt hat, zeigt die Zelle in der Spalte State den nächsten Status: übernommen.

 

INFO:

Das Symbol ist für alle anderen Benutzer ausgeblendet, die diesen speziellen Wartungsauftrag anklicken.

 

 

übernommen:


Der zugewiesene Benutzer hat den Wartungsauftrag angenommen und arbeitet daran.

Durch Klick auf das Symbol in der Boardlet-Menüleiste öffnet sich der Dialog Order beenden in der Seitenleiste: ------>>

Nachdem Sie eine Beschreibung eingegeben und auf Bestätigen geklickt haben, zeigt die Zelle in der Spalte State den Status an: erledigt.

 

 

erledigt:


Der Wartungsauftrag wurde vom zugewiesenen Benutzer durchgeführt.

 

 

abgelehnt:


Solange der Wartungsauftrag im Status zu planen, geplant oder übernommen vorliegt, kann der Wartungsauftrag vom Benutzer abgelehnt werden.

Durch Klick auf das Symbol in der Boardlet-Menüleiste öffnet sich der Dialog Refuse Order in der Seitenleiste: ------>>

Nachdem Sie eine Beschreibung eingegeben und auf Bestätigen geklickt haben, zeigt die Zelle in der Spalte State den Status an: abgelehnt.

 

 

Neuen Wartungsauftrag anlegen

Durch Klick auf in der Menüleiste des Boardlet TPM Wartungsaufträge kann ein neuer Wartungsauftrag angelegt werden. Es erscheint der Dialog Neuer Wartungsauftrag in der Seitenleiste: -->>

So liegen Sie einen neuen Wartungsauftrag an:

1

Wählen Sie den Knoten, für den der Auftrag angelegt wird.

2

Führen Sie einen der beiden Schritte aus:

  • Wenn es sich bei dem Auftrag um einen einmaligen Wartungsauftrag handelt, dann geben Sie einen Namen für den Auftrag ein. Das Ausrufezeichen kennzeichnet das Feld Name als Pflichtfeld. Ignorieren Sie für einmalige Wartungsaufträge die Dropdown-Liste Wartungsplan.

  • Wenn Sie den Auftrag einem Wartungsplan zuweisen wollen, dann wählen Sie den gewünschten Plan aus der Dropdown-Liste Wartungsplan. Daraufhin wird das Feld Name ausgeblendet und ist kein Pflichtfeld mehr.
     

3

Restliche Schritte:

  • Legen Sie Plandatum und Uhrzeit fest.

  • Wählen Sie eine oder mehrere IH-Gruppen. Die Namensliste in der Dropdown-Liste Verwantwortlich passen sich automatisch an. Sie können jedoch nur einen Verantwortlichen für den Wartungsauftrag auswählen.

  • Legen Sie die Priorität das Wartungsauftrags fest: hoch, mittel, niedrig.
     

4

Wenn Sie fertig sind, klicken Sie Bestätigen.

Tätigkeitsverwaltung: Tätigkeit hinzufügen/entfernen

Ist ein Wartungsauftrag im Boadlet TPM Wartungsaufträge markiert, erscheinen in der Boardlet-Menüleiste weitere Symbole.

Durch Klick auf haben Sie Zugriff auf die Tätigkeitsverwaltung des Wartungsauftrags. Der Dialog Aktivitätsmanagement erscheint in der Seitenleiste, siehe Reiter [A]: ------->>

Wenn der Wartungsauftrag auf einem Wartungsplan beruht, für den die Tätigkeiten bereits festgelegt sind, dann werden diese Tätigkeiten hier angezeigt.

Wenn Sie die Tätigkeitsverwaltung für einen neu angelegten Wartungsauftrag geöffnet haben, dann ist noch keine Checkbox ausgewählt.

Auf Reiter [A] können Sie die folgenden Aktionen ausführen:

  • Hinzufügen von Tätigkeiten zum Wartungsauftrag, indem Sie die Checkboxen neben den erforderlichen Tätigkeiten aktivieren.

  • Entfernen von Tätigkeiten aus dem Wartungsauftrag, indem Sie die Checkboxen neben den erforderlichen Tätigkeiten deaktivieren.

Eine Beschreibung von Reiter [B] finden Sie in Tätigkeitsverwaltung: Tätigkeit hinzufügen/bearbeiten below.

 

Tätigkeitsverwaltung: Tätigkeitsbeschreibung hinzufügen

Ist ein Wartungsauftrag im Boardlet TPM Plans markiert, erscheinen oben in der Boardlet-Menüleiste weitere Symbole.

Mit Klick auf und auf den zweiten Reiter [B] können Sie den Dialog Aktivitätsbeschreibung hinzufügen in der Seitenleiste öffnen. Dort können Sie neue Tätigkeitsbeschreibungen anlegen.

Die neuen Tätigkeitsbeschreibungen stehen danach auf dem Reiter [A] zur Auswahl bereit.

So fügen Sie eine neue Tätigkeitsbeschreibung zur Liste der Tätigkeiten hinzu:

1

Geben Sie eine Beschreibung für die neue Tätigkeit ein.

2

Weisen Sie die neue Tätigkeitsbeschreibung einer Kategorie zu.

3

Wenn Sie fertig sind, klicken Sie Bestätigen.

Die neue Tätigkeit steht nun auf dem Reiter [A] zur Auswahl bereit.

Filter "Aktuelle Person"

Im Boardlet TPM Wartungsaufträge kann nach dem Benutzer, der gerade angemeldet ist ("aktuelle Person") gefiltert werden. Der Filter zeigt dem die jeweiligen Benutzer zugeordneten TPM Wartungsaufträge. Wenn die TPM Wartungsaufträge mehreren Personen zugeordnet sind, kann jede Person, die zu dieser Personengruppe gehört, die TPM Wartungsaufträge sehen.

 

Beim Ad-hoc-Druck übergibt das TPM Wartungsaufträge Boardlet den aktuellen Benutzer (Auftraggeber) an den Print Task.

 

Der zugehörige Boardlet-Filter sieht wie folgt aus:

Der zugehörige Filtername lautet <Aktuelle Person>

 

Die zusätzliche Option zum Dropdown-Filter "Verantwortliche Person für Wartungsaufträge" ermöglicht es, den Filter für die aktuelle Person festzulegen.

Entfernen von verantwortlichen Personen aus den TPM Wartungstätigkeiten

Wenn die verantwortliche Person aus dem Instandhaltungsauftrag gestrichen wird, wird diese Person auch aus jeder aktiven Instandhaltungstätigkeit (nicht abgelehnt und nicht abgeschlossen) gestrichen.  Der Status dieser TPM-Aktivitäten wird ebenfalls zurückgesetzt. Der Status der betroffenen Tätigkeiten wird auf "zuzuordnen" geändert.

 

Wie man die Dateien mit dem TPM-Boardlet verknüpft

Das Boardlet TPM-Pläne (siehe TPM Plans) verfügt über die Funktion "Dateien verknüpfen", mit der der Benutzer Dateien aus dem TPM-Verzeichnis mit den jeweiligen TPM-Plänen verknüpfen und auch wieder entknüpfen kann. Diese Funktion stellt somit eine Möglichkeit zur Auswahl von Dateien/Verzeichnissen aus einem speziellen tpm-Dokumentenverzeichnis dar.

Beim Klicken auf (1) öffnet sich die Seitenleiste "Dateien verknüpfen". Hier kann der Benutzer im oberen Teil (2) durch die verfügbaren Dateien navigieren und die Dateien auswählen, die verknüpft/entknüpft werden sollen.

 

Im unteren Teil der Seitenleiste wird Folgendes angezeigt:

(3) Die verknüpften, aber im Dateisystem fehlenden Dateien (graue Schriftfarbe).

(4) Die zu entknüpfenden Dateien (rote Schriftfarbe).

(5) Die Liste der bereits verknüpften Dateien (normale Schriftfarbe).

(6) Die Liste der zu verknüpfenden Dateien (grüne Schriftfarbe).

 

Der Benutzer kann auch eine Datei in einer neuen Registerkarte öffnen und die Beschreibung für verknüpfte Dateien festlegen (die Beschreibung wird immer der TPM-Datei zugewiesen). Wenn der Benutzer eine Datei verknüpft, der bereits eine Beschreibung zugewiesen wurde, enthält die Datei diese Beschreibung im Beschreibungsfeld.

Nach dem Klicken auf die Schaltfläche "Absenden" werden alle Änderungen gespeichert.

 

Beachten Sie außerdem

Der Benutzer muss über die Berechtigung "edit" für die Datenquellen

  • tpm_linked_files und

  • tpm_file_plan_rel

verfügen, um Dateien verlinken oder die Verlinkung aufheben oder deren Beschreibung ändern zu können. Wenn "edit"-Rechte fehlen, der Benutzer aber "view"-Rechte auf tpm_linked_files- und tpm_file_plan_rel-Datenquellen hat, kann er nur verknüpfte Dateien und Dateibeschreibungen ansehen und Dateien in einem neuen Tab öffnen.

In der Dateiliste in der Seitenleiste erscheint bei fehlenden Dateien ein Hinweis.

 

Notwendige Parametrierung

  • Das "TPM-Verzeichnis", in dem die TPM-Dateien gespeichert werden und auf das in der Webanwendung zugegriffen werden kann, kann über die Eigenschaft tpmrefsPath konfiguriert werden.

  • Diese sollte in der Konfigurationsdatei sapieng.properties angegeben werden. Wenn diese Eigenschaft nicht angegeben wird, wird das Standardverzeichnis "tpmrefs" im Verzeichnis "shared" erstellt.

Abrufen von Plan-Dateien

Es ist möglich, die verlinkten Dateien des entsprechenden Plans im TPM Wartungsaufträge Boardlet zu sehen. Dies gilt für TPM-Benutzer, die nur TPM Wartungsaufträge und TPM Aktivitäten verwenden.

Dateiverknüpfungen (verlinkte Dateien) in TPM-Plänen gehören zu jedem TPM Wartungsauftrag. Jedoch ist es nicht notwendig, die Dateien mit jedem Auftrag zu verknüpfen.

 

Beachten Sie zusätzlich folgende Informationen:

  • Die verknüpften Dateien werden aufgelistet und können schreibgeschützt geöffnet werden. Dies ist in der Seitenleiste möglich, die sich öffnet, wenn angehängte Dateien angezeigt werden.

  • Es kann die Beschreibung einer Datei eingesehen werden (schreibgeschützt).

  • Es werden die gleichen Informationen angezeigt (schreibgeschützt und im gleichen Stil) wie bei der Anzeige der verknüpften Dateien für einen Plan.

  • Die Spalte im TPM Wartungsaufträge Boardlet heißt "Links/Anhänge".

  • In der entsprechenden Zelle des TPM Wartungsaufträge Boardlets steht z.B. "3 Datei(en) angehängt, 2 Datei(en) verlinkt".

  • Die Spalte TPM-Pläne heißt "Links".

  • Der entsprechende Zelleninhalt ist bei "2 Datei(en) verlinkt".

Deaktivieren des manuellen Änderns des Auftragsstatus

Es ist nicht möglich, den Status eines TPM Wartungsauftrags direkt (manuell) über das TPM Wartungsaufträge Boardlet zu ändern. Falls der Auftrag abgelehnt wird, ist dies dennoch möglich. Der Status eines TPM Wartungsauftrags basiert auf dem Status der zugehörigen Instandhaltungsaktivitäten.

 

Außerdem wirkt sich das Ändern des Pools der verantwortlichen Personen eines Auftrags nicht auf den Status des Auftrags aus. Das TPM Wartungsaufträge Boardlet wird regelmäßig aktualisiert, damit neue Zustände angezeigt werden.

Priorität von TPM Status

Der Status des TPM Wartungsauftrags basiert auf dem Status seiner TPM Maintenance Activities (Instandhaltungstätigkeiten). Wenn eine Aktivität eines Auftrages einen bestimmten Zustand hat, hat der Auftrag den gleichen Zustand (ein Auftrag hat mehrere Aktivitäten mit unterschiedlichen Zuständen).

Benutzer können Aktivitäten zugewiesen werden, ohne dass der Status der Aktivität auf "in Bearbeitung" wechselt.

Es gibt folgende Prioritäten von TPM Status:

  • "zu planen" --> "zuzuordnen"

  • "übernommen" --> "in Bearbeitung"

  • "geplant" --> "zugewiesen"

  • "abgelehnt" --> "abgelehnt"

  • "erledigt" --> "Erledigt"

 

Es gilt:

  • Wenn mindestens eine Tätigkeit den Status "zuzuordnen" hat, hat der Auftrag auch den Status "zuzuordnen".

  • Wenn keine Tätigkeit den Status "zuzuordnen" hat und mindestens eine Tätigkeit den Status "in Bearbeitung" hat, hat der Auftrag den Status "in Bearbeitung".

  • Wenn jeder Vorgang den Status "zuordnen" hat, hat der Auftrag den Status "zuzuordnen".

  • Wenn ein Vorgang eines Auftrags den Status "abgelehnt" hat und alle anderen haben den Status "erledigt", dann hat der Auftrag den Status"abgelehnt".

  • Wenn alle Tätigkeiten eines Auftrags den Status "erledigt" hat, hat der Auftrag ebenfalls den Status"erledigt".

Reiter Filter

Selektierte Knoten:

Hier können Sie nach einer Knoten ID oder mehrerer Knoten IDs filtern. Geben Sie die Knoten IDs als Komma-separierte Liste ein.

Überfällig:

Über diesen Schieberegler wird definiert, ob nur Wartungsaufträge angezeigt werden sollen, die bereits überfällig sind oder alle.

Wartungsauftrag:

Hier kann auf den Namen eines Wartungsauftrags gefiltert werden.

Zustand:

Über diese Dropdown-Liste kann ein Zustand ausgewählt werden, in dem sich die angezeigten Wartungsaufträge befinden sollen.

Priorität:

Über diese Dropdown-Liste kann die Priorität der angezeigten Wartungsaufträge ausgewählt werden.

Auslöser:

Über diese Dropdown-Liste kann ausgewählt werden, was den Wartungsauftrag ausgelöst hat.

Verantw. für Wartungsauftrag:

Über diese Dropdown-Liste kann ein Verantwortlicher für den Wartungsauftrag selektiert werden, auf den gefiltert wird.

Verantw. für Tätigkeiten:

Über diese Dropdown-Liste kann ein Verantwortlicher für Tätigkeiten des Wartungsauftrags selektiert werden, auf den gefiltert wird.

Zugeordnete IH-Gruppe:

Über diese Dropdown-Liste kann eine IH-Gruppe selektiert werden, die dem Wartungsauftrag zugeordnet ist.

Plan-id:

Hier kann auf die ID eines Wartungsauftrags gefiltert werden.

Benötigte Parametrierung im LC2

  • Tätigkeiten werden im LC2 unter Werksverwaltung TPM Tätigkeiten angelegt.

  • Typische Begründungen werden im LC2 unter Werksverwaltung TPM Tätigkeiten Bez. Tätigkeit Begründung angelegt.

  • Komponenten werden im LC2 unter Werksverwaltung TPM Komponentenarten angelegt und ein Knotenbezug hergestellt.

  • TPM-Gruppen werden im LC2 unter Werksverwaltung TPM TPM-Gruppen angelegt. Hier können auch Personen den Gruppen zugeordnet werden.
