Beispiele zur Parametrierung von Detail-Boardlets

Wie bereits beschrieben, können für das Knotenlayout Detail-Boardlets eingeblendet werden, die pro Knoten bestimmte Informationen wie beispielsweise die aktuellen Stückzähler oder überfällige Wartungsaufträge visualisieren. Im Folgenden werden einige Fälle beispielhaft parametriert.

Zu beachten ist, dass die Eingaben bzgl. Knotenstatus und die Eingaben bzgl. Detail-Boardlets voneinander unabhängig sind. Somit können sie auch parallel verwendet werden.

Anzeige von Knotenstatus basierend auf Meldungen

Das folgende Beispiel zeigt den Knotenstatus basierend auf Meldungen als Umrandung an. Dabei wird der Knoten in der Farbe der höchsten anstehenden Meldung umrandet (z.B. rot = Störung, gelb = Meldung).

Die folgenden Einstellungen müssen vorgenommen werden:

Bearbeiten - Boardlet Layout:

Anzeige Knotenstatus:

Auswahl "Border"

Server-Datenquelle zur Statusermittlung:

al-node-states

Anzeige von Aktuellstückzahl auf Level 1

Das folgende Beispiel zeigt die nötigen Eingaben, um die Aktuellstückzahl der einzelnen Knoten auf Level 1 einzublenden.

Die folgenden Einstellungen müssen vorgenommen werden:

Bearbeiten - Boardlet Layout:

Detaillevel:

1

Detail-Boardlet:

node-values-boardlet

Detail-Boardlet-Parameter:

{"valueTypes":[2]}

Anzeige von Ist- und Sollstückzahl für Knoten vom Typ 2

Es ist möglich, Parameter einzugeben, um abhängig vom Knotentyp andere Informationen im Details-Bereich einzublenden. Im Beispiel werden nur für Knoten des Knotentyps 2 Details angezeigt, und zwar das Node-Info Boardlet mit Variablen der Arten 2 und 3.

Dazu ist Folgendes einzutragen:

Bearbeiten - Boardlet Layout:

Knotenstatus anzeigen:

Aus

Knotentyp-Details:

[{"nodeType":2,"detailBoardlet":"node-info-boardlet","detailBoardletParameters":{"valueTypes":[2,3,40]}}]

Detaillevel:

1

Tortendiagramm für einzelne Variablenarten mit festen Farben

Es ist auch möglich, das Aktuelle Werte Boardlet als Tortendiagramm mit fest vergebenen Farben je Variablenart anzuzeigen, um beispielsweise einen Status abzubilden.

Dazu ist Folgendes einzutragen:

Bearbeiten - Boardlet Layout:

Detaillevel:

1

Detail-Boardlet:

node-values-boardlet

Detail-Boardlet-Parameter:

{"viewMode":"piechart", "typeColors":{"2":"success","40":"error"}}

 

Es wird also der Variablentyp 2 mit der im Theme definierten "Success"-Farbe dargestellt, der Variablentyp 40 mit der Farbe "Error". Für die weiteren Variablenarten werden die Standardfarben des Themes verwendet.

Die Parameter können vom Knoteninfo Boardlet und vom Knotenstatus Boardlet nicht interpretiert werden, dort sind sie fest vorgegeben.

Detail-Boardlet-Breite:

100

Detail-Boardlet-Höhe:

100

Anzeige von überfälligen Wartungsaufträgen

Das folgende Beispiel zeigt ein Icon an für Knoten, an denen an einem Wartungsauftrag gearbeitet wird (für jeden Auftrag mit Status "work on" ein Icon). Zusätzlich kann der Hintergrund der betroffenen Knoten für überfällige Wartungsaufträge rot eingefärbt werden (es gibt mindestens einen Auftrag, der überfällig ist und niemandem zugeordnet ist).

Die folgenden Einstellungen müssen vorgenommen werden:

Bearbeiten - Boardlet Layout:

Anzeige Knotenstatus:

Auswahl "Background" (wenn rote Einfärbung gewünscht ist)

Server-Datenquelle zur Statusermittlung:

tpm_maint_order_overdue

Detaillevel:

1

Detail-Boardlet:

maint-orders-status-boardlet

Um eine zentrierte Darstellung zu ermöglichen, muss außerdem beim das Detail-Bereich auf die Maße des Knotens gesetzt werden (Layout editieren, Details Breite & Höhe auf die entsprechenden Werte setzen).
