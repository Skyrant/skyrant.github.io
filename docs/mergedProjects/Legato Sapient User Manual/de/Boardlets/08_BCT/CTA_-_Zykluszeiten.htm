Analyse der Zykluszeiten mit dem Cycle Time Analyzer (CTA)

Mit dem CTA Zykluszeiten Standard-Dashboard ist Folgendes möglich:

  • Das CTA Zykluszeiten Dashboard gibt einen Überblick über die Taktzeiten einer Anlage.

  • Durch das Standarddashboard ist es möglich, den Verlauf der verfügbaren Zykluszeiten, d.h. z.B. aller Prozessgeräte, über eine ausgewählte Zeitspanne zu sehen.

  • Man kann nicht relevante Zykluszeitvariablen ausblenden, damit nur die persönlich-relevanten Zykluszeiten analysiert werden.

  • Neben dem Liniendiagramm gibt es einen Reiter Cycle Time Histogram, in dem die Daten in einem Histogram ausgewertet werden.

  • Um einen Eindruck über mögliche Ursachen für zu hohe Zykluszeiten zu bekommen, ist ein direkter Zugriff auf die entsprechenden Alarme (im Reiter Archived Alarms) für die gleichen Zeitspanne abrufbar.

 

In der unten stehenden Abbildung werden die verschiedenen Reiter angezeigt: (1) Taktzeiten-Diagramm, Taktzeiten-Histogramm und Archivierte Meldungen..

Um Daten zur Anzeige der (2) Taktzeiten-Diagramm (Liniendiagramm) zu erhalten, muss eine bestimmten Zeitspanne über die (3) Filter ausgewählt werden. Dies gilt auch für den im 2. Reiter dargestellte Taktzeiten-Histogram.

 

(4) Die einzelnen Variablen zur Darstellung der Zyklen werden in der Legende dargestellt.

 

 

Mit der Maus können einzelne Abschnitte ausgewählt werden.

 

Im Reiter Taktzeiten-Histogramm werden die gleichen Daten als Quelldaten verwendet, jedoch als Histogramm dargestellt. In der Tabelle unterhalb des Histogramms werden z.B. verschiedene Angaben aufgezählt, die auch analysiert wurden:

 

  • Anzahl der Zyklen ohne Alarm

  • Anzahl der Zyklus mit Alarm

  • Ziel-Zyklus-Zeit (basiert auf "value_setpoint" aus der Tabelle "p_values_config")

  • Standardabweichung

  • Prozess-Capability Prozess Index cp

  • Prozess Capability index cpk
