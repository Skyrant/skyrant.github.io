Knoten und KPIs in Sapient konfigurieren

Die notwendige Konfiguration innerhalb von Sapient wird für den EpiK-Report wie in den folgenden Abschnitten beschrieben, eingerichtet:

Lieferung der Datenbasis in Sapient

Die Daten müssen gesammelt und die KPIs für eine Engpassidentifikation berechnet werden, so dass die Datenbasis für den späteren Bericht konfiguriert werden kann.

Datenbasis für die Leerlaufzeiten

  • Es gibt eine Funktionalität, die für jede Schicht berechnet, für wie viele Minuten ein Pufferwert unter einem definierten Schwellenwert lag. Dieser Schwellenwert ist für jede Puffervariable spezifisch.

 

Verwenden Sie nicht den Wert min_value, da dieser Wert an anderer Stelle für eine andere Funktionalität benötigt wird.

 

  • Es wird zwischen Puffertyp 1 und Puffertyp 2 unterschieden.
  • Es wird nicht der Variablentyp verwendet, sondern eine erweiterte Eigenschaft (vergleichbar mit dem VEMA-Flag). Diese Funktionalität muss automatisch ausgeführt werden, wenn eine Schicht abgeschlossen ist.
  • Das Ergebnis wird für diese Verschiebung gespeichert. Wenn der Schwellenwert geändert wird, muss die Berechnung für jede Schicht der letzten 6 Monate neu durchgeführt werden.


Vorgänger/Nachfolger-Beziehungen

  • Vorgänger-Nachfolger-Beziehungen zwischen Knoten können m mal n (m x n) modelliert werden.
  • Beziehungen unabhängig von der Knotenhierarchie mit l_node_workflow_relations
  • Für Knoten ist eine Klassifizierung möglich, falls sie Teil einer Hauptleitung oder einer Nebenleitung sind (mehrere Unterzweige sind möglich): Mit l_workflow_relation_types
  • Puffervariablen sind immer mit dem Knoten verknüpft, aus dem der Puffer gefüllt wird.

Anpassung der KPI-Berechnung "Pufferleerzeit" an neue Anforderungen

Wenn ein Puffer Produktionseinheiten verschiedener Typen enthält, wird die Anzahl der enthaltenen Einheiten von der SPS als Werte separater "Pufferebenen"-Variablen gemeldet. Alle diese Variablen werden in Sapient demselben Pufferknoten zugewiesen.

Wenn einem Pufferknoten mehr als eine "buffer level"-Variable zugewiesen ist, sollte die Auswertung wie folgt erfolgen:

 

  • Für jede Variable wird einzeln eine Menge von "Leerzeiten" berechnet.
  • Neu: Wenn eine der oben genannten Mengen die gesamte Schicht abdeckt, wird sie verworfen.
  • Die verbleibenden leeren Zeiten werden addiert, wobei sich überschneidende Zeiten nur einmal gezählt werden.
  • Es kann eine einfache Addition der Zeiten ohne Berücksichtigung von Überschneidungen geändert werden und (optional) die resultierende Summe auf die Schichtdauer begrenzt werden.

 

Außerdem gilt:

Die Berechnung für den KPI-Typ "BET" (Buffer Empty Time) wird geändert:

  • Wenn mehrere Pufferfüllstandsvariablen für einen Puffer eingerichtet sind, werden sie einzeln ausgewertet und die sich daraus ergebenden Leerzeiten unter Berücksichtigung von Überschneidungszeiten addiert.
  • Wenn die aus den Werten einer Pufferebenenvariablen berechnete Leerzeit die gesamte Schicht abdeckt, wird sie ignoriert.
  • Wenn die Leerzeiten aller Pufferebenenvariablen ignoriert werden. Dies zeigt, dass der Puffer nicht genügend Teile irgendeines Typs enthält. In diesem Fall wird der Puffer für die gesamte Schicht als leer dargestellt.
