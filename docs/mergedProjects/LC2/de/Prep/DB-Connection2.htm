Herstellen der Datenbankverbindung zu PostgreSQL

Verbindung konfigurieren

Nach Start des LC2 erscheint ein Dialog zum Einrichten beliebig vieler Datenbankverbindungen / Repositories. Die Verbindungsherstellung erfolgt in zwei Schritten.

Zunächst wird ein neues Repository angelegt (Create New …).

Es öffnet sich der folgende Dialog:

 

Name

Der Name kann frei gewählt werden, muss jedoch eindeutig sein.

Sortiernummer

Der Sortiernummer kann frei gewählt werden, muss jedoch eindeutig sein.

Provider

Datenbank-Provider, z.B. Npgsql. Wird nach Eingabe des Legato Connection Strings automatisch befüllt.

Connection String

Das Feld "Provider" wird automatisch nach Definition der Datenbankverbindung befüllt. Klicken Sie dazu auf die drei Punkte (…) neben "Legato Verbindungs-String". Es erscheint ein Dialog zur Eingabe der Verbindungseinstellungen.

Zunächst kann hier die Datenquelle oder der gewünschte Treiber (Provider) durch Klick auf "Ändern..." selektiert werden:

Bei Auswahl des "Managed Data Providers" muss kein ODAC mehr installiert werden. Der Eintrag "Unmanaged Data Provider" wird ohnehin nur angeboten, wenn ODAC auf dem PC installiert ist.

Nach Übernehmen der Auswahl kann der Servername sowie ein Datenbankbenutzer eingegeben werden. Die notwendigen Einstellungen sind abhängig von Datenbanksystem und –version. In der Regel besteht ein Servername aus der IP-Adresse, einem Port und dem Namen der Datenbank (z.B. 172.21.6.36, Port: 5432, Datenbank: SAPIENT).

Bei der Verbindungskonfiguration muss der Benutzername und Kennwort angegeben werden. Dabei ist ein Datenbankbenutzer zu wählen, der vom Systemadministrator der Datenbank angelegt wurde. Dieser Benutzer kann mit minimalen Rechten auf Datenbankseite ausgestattet sein. Die eigentliche Benutzeranmeldung an Sapient erfolgt dann einmalig mit dem im LC2 angelegten Benutzernamen und Kennwort. Die Rechte dieses spezifischen Legato-Benutzers sind in der Benutzerverwaltung des LC2 festgelegt. Nach der Einrichtung der Verbindung kann die Verbindung getestet werden.

Developer Mode

Wird diese Checkbox aktiviert, so wird der LC2 im Entwicklermodus geöffnet. Hier stehen dem Anwender weitere Entwickler-Tools und Customizing-Optionen zur Verfügung. Hat der Benutzer nicht das benötigte Recht, wird der LC2 mit dieser Verbindung nicht gestartet und es wird eine entsprechende Fehlermeldung ausgegeben. Die Checkbox muss auch zum Verwenden des System Developer Modes gesetzt werden. Die Checkbox kann auch beim späteren Editieren von Verbindungen gesetzt werden.

Lokales Repository erzwingen

:

Die Checkbox "Force Local Repository" darf für Sapient nicht aktiviert sein! Diese Funktion verwendet immer das aktuelle Legato Classic Repository, das für Sapient nicht geeignet ist.

Verbindung zur Sapient Engine eintragen

HTTP

Der LC2 benötigt zusätzlich die Verbindungsdaten zur Sapient Engine. Diese müssen in der Datei "Global.config" beim Repository eingetragen werden. Die Datei liegt unter %AppData% - Roaming\Gefasoft\GraphPic10.

Darin sind alle angelegten Repositories eingetragen. Suchen Sie den Eintrag für Sapient und ergänzen Sie in der letzten Zeile die URL der Sapient Engine, z.B.:

sapientEngineUrl="http://localhost:8080/"

 

Der gesamte Repository-Eintrag könnte also folgendermaßen aussehen:

<add name="Sapient_PostgreSQL" sortId="22" provider="Npgsql"

  connectionStringEncrypted=
"0C1AF0C604CEF73F34EEE3BA9757DC0012B29942DDFF11FA611CDD07
C31873659B579BAF0F9B995C84F44C1C5CE985DDC1FE11B4CCD354597
637B1CF5AEE833BB2FC1785B57A9A6A42A79D8185E7635511E0187B82
C5BC78B0C0D501C0418649"

  authenticationType="GetTypeFromSysParams" sapientEngineUrl="http://localhost:8080/"/>

 

Alternativ kann die URL der Sapient Engine auch über den Systemparameter -260 WebApp URL geholt werden.

HTTPS

Es ist auch möglich, die Verbindung zur Sapient Engine über HTTPS herzustellen. In diesem Fall werden standardmäßig nur sichere Verbindungen akzeptiert. Dazu ist der Systemparameter 261 Web-App login secure auf 1 zu setzen. Bei Zertifikatsfehlern ist eine Verbindung dann nicht möglich.

Sollen, z.B. für Testzwecke, selbst erstelle Zertifikate verwendet werden, kann im der global.config-Datei eingestellt werden, dass Zertifikatsfehler ignoriert werden sollen (ignoreCertificateErrors="true").

Der gesamte Repository-Eintrag könnte also folgendermaßen aussehen:

<add name="Sapient_PostgreSQL" sortId="22" provider="Npgsql"

  connectionStringEncrypted=
"0C1AF0C604CEF73F34EEE3BA9757DC0012B29942DDFF11FA611CDD07
C31873659B579BAF0F9B995C84F44C1C5CE985DDC1FE11B4CCD354597
637B1CF5AEE833BB2FC1785B57A9A6A42A79D8185E7635511E0187B82
C5BC78B0C0D501C0418649"

  authenticationType="GetTypeFromSysParams" sapientEngineUrl="https://LegatoSapient:8080/"
ignoreCertificateErrors="true"/>

 

System Developer Mode

In manchen Fällen ist es nötig, den LC2 mit stark erweiterten Rechten zu starten. Dazu dient der System Developer Mode. Einige Tabellen und Funktionen stehen nur in diesem Modus zur Verfügung.

Zum Einrichten des System Developer Modes müssen folgende Schritte durchgeführt werden:

1. Gegebenenfalls eine neue Verbindung im Repositories-Dialog anlegen und die Checkbox "Developer Mode" aktivieren. Die Checkbox kann auch für eine bereits bestehende Verbindung aktiviert werden.

2. Öffnen Sie die Datei "Global.config". Die Datei liegt unter %AppData% - Roaming\Gefasoft\GraphPic10. Darin sind alle angelegten Repositories eingetragen. Suchen Sie den Eintrag für die gewünschte Verbindung und ergänzen Sie den Eintrag sysDeveloperMode="true", z.B.:

<add name="Legato" sortId="17" provider="Oracle.DataAccess.Client" connectionString="...", password="..." authenticationType="GetTypeFromSysParams"

developerMode="true" sysDeveloperMode="true" />

 

3. Der Benutzer muss die Modul-Berechtigung System Developer Mode haben. Hat der Benutzer nicht das benötigte Recht, wird der LC2 mit dieser Verbindung nicht gestartet und es wird eine entsprechende Fehlermeldung ausgegeben.

Emergency Access

In vielen Fällen erfolgt die Benutzerprüfung im LC2 über einen externen Authentifizierungs-Service (LDAP, Authentifizierung via Webapp, Authentifizierung via SapientEngine). Für den Fall, dass der verwendete Authentifizierungs-Service nicht erreichbar ist, wird ein Notfall-Login auf Datenbankebene ermöglicht. Dabei erfolgt die Anmeldung nicht mit einer spezifischen Benutzerkennung, sondern mit einem geeigneten Datenbankbenutzer.

Zum Einrichten des Emergency-Access müssen folgende Schritte durchgeführt werden:

1. Gegebenenfalls eine neue Verbindung im Repositories-Dialog anlegen. Dieser Schritt kann entfallen; der Emergency-Access kann auch für eine bestehende Verbindung erstellt werden.

2. Öffnen Sie die Datei "Global.config". Die Datei liegt unter %AppData% - Roaming\Gefasoft\GraphPic10. Darin sind alle angelegten Repositories eingetragen. Suchen Sie den Eintrag für die gewünschte Verbindung und ändern Sie den Eintrag bei "Authentication Type" von "getTypeFromSysParams" in "EmergencyAccess", z.B.:

<add name="Legato" sortId="17" provider="Oracle.DataAccess.Client" connectionString="...", password="..." authenticationType="EmergencyAccess" />

 

3. Starten Sie den LC2. Bei Auswahl der geänderten Verbindung wird nun nicht mehr der gewohnte Dialog zum Anmelden angezeigt. Stattdessen muss die Anmeldung mit einem entsprechend privilegierten Datenbank-Benutzer (z.B. OWNER) erfolgen.

:

Normalerweise erfolgt im LC2 nach erfolgreicher Benutzerprüfung intern ein Upgrade des DB-Benutzers auf einen Benutzer mit unbeschränkten Datenbank-Schreibrechten. Dieses Berechtigungsupgrade wird bei dem "Emergency Login" nicht ausgeführt! Damit wird erreicht, dass insbesondere Konfigurationsänderungen nur möglich sind, sofern der Login mit einem Datenbankbenutzer mit ausreichend hohen Datenbankberechtigungen erfolgt.

Festlegen der DB-Verbindung als Startparameter

Nachdem die benötigten Datenbankverbindungen einmalig angelegt wurden, können diese auch direkt als Startparameter der LC2-Verknüpfung übergeben werden. Dazu geht man folgendermaßen vor:

benötigte Anzahl Verknüpfungen der LC2.exe (Gefasoft.Legato.LegatoConfigurationCenter.exe) auf dem Desktop ablegen (pro Verbindung wird eine Verknüpfung benötigt)

Rechtsklick auf die Verknüpfung – Eigenschaften

Im Reiter "Verknüpfung" in der Zeile "Ziel" am Ende des dort eingetragenen Pfads folgende Eingabe machen: Leerzeichen/repository="Name der Datenbankverbindung"

 

So könnte der Eintrag unter "Ziel" jetzt aussehen:

"C:\Program Files (x86)\Gefasoft\LC2\Gefasoft.Legato.LegatoConfigurationCenter.exe" /repository="Sapient_PostgreSQL"

 

Jetzt wird diese Datenbankverbindung automatisch geladen, wenn das LC2 über die Verknüpfung geöffnet wird.

Ablage der Verbindungseinstellungen auf Netzlaufwerk

Die im Netzwerk liegende Config wird als Fallback für die lokale Config verwendet. Somit sind die dort abgelegten Verbindungen beim Start sichtbar. Wenn diese jedoch geändert oder gelöscht werden, wirkt sich das nicht auf die im Netzwerk liegende Config aus, sondern wird nur lokal persistiert.

:

Die Konfiguration dieses Features erfordert fortgeschrittene Kenntnisse und darf daher nur von Administratoren oder Power Usern durchgeführt werden.

Zuerst muss in das gewünschte Netzwerkverzeichnis eine Config mit Verbindungseinstellungen abgelegt werden. Als Vorlage hierfür eignet sich die lokale Config, die unter

%APPDATA%\Gefasoft\GraphPic10\Global.config

 

liegt. Diese kann zuerst über den Repository-Dialog des LC2 entsprechend gepflegt werden (siehe Kapitel "Verbindung konfigurieren").

Dann muss auf diese neue Netzwerk-Config verwiesen werden. Das kann entweder für den aktuellen User erfolgen

%APPDATA%\Gefasoft\GraphPic10\Global.config

 

oder für den lokalen Rechner

%ProgramFiles(x86)%\Gefasoft\LC2\Gefasoft.Legato.LegatoConfigurationCenter.exe.config

 

In dieser Datei muss das XML-Element "graphPic" um das Attribute "sharedConfigFilePath" erweitert werden, dessen Wert den Pfad zur Netzwerk-Config beinhaltet.

Beispiel:

<graphPic sharedConfigFilePath="\\ar-datastore\try\LC2\Shared.config">

 

:

In der Netzwerk-Config dürfen sich nur Verbindungen zu zentralen Repositories befinden, die in der Legato Datenbank abgelegt sind. Verbindungen zu SQL Compact Repositories (.sdf-Datei) werden nicht  unterstützt.

Wenn ein Benutzer eine Verbindung, die aus einer Netzwerk-Config stammt, abändert, so wird er von diesem Zeitpunkt an eine lokale Kopie dieser Verbindung sehen.
