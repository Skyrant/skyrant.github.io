Funktion

Hier kann pro Variable die Sollwert-Vererbung parametriert werden. Für weitere Informationen bzgl. Sollwerten, siehe bitte Sollwertvorlagen.

Rundungs-Art

Lookup-Liste zur Auswahl einer Rundungs-Art.

Rundungs-Genauigkeit

Numerischer Wert zwischen 0 und 10, der die Rundungs-Genauigkeit angibt (0 = auf Integer runden, 1 = auf 1 Nachkommastelle runden, 2 = Nachkommastellen runden, etc.)

Sortiernummer

Sortiernummer, die die relative Reihenfolge von Variablen mit dem gleichen Parent angibt.

 

Berechnungs-Grundlagen

Berechnung nach Faktor

  • Diese Methode wird verwendet, wenn die Spalte "Berechn. nach Anteil?" auf 0 gesetzt ist. Zur Berechnung der Werte, die vererbt werden, wird ein Faktor und zwei Offsets verwendet. Die Formel gestaltet sich wie folgt:

  • Variable = (Parent-Variable + Parent-Offset) * Faktor + Offset

  • Soll der Wert der Parent-Variable auf mehrere Variablen verteilt werden und ist eine Rundungs-Art selektiert (z.B. für diskrete Stückfertigung), kann dies zu fehlerhaften Ergebnissen führen. Beispiel: Ein Wert soll in drei gleiche Teile geteilt werden, die alle auf Ganzzahl gerundet werden sollen. Die Summe dieser Teile kann dann höher oder niedriger als der Wert der Parent-Variable sein (10/3 = 3.33, gerundet auf 3, 3*3 = 9.  11/3 = 3,67, gerundet auf 4, 3*4 = 12). Um diese Situation abzubilden, wird der folgende Algorithmus zur Verhinderung der Fehler-Verbreitung verwendet:

abhängige Werte werden in einer vorgegebenen Reihenfolge berechnet (Sortiernummer)

nach jeder Berechnung wird dieselbe Berechnung "rückwärts" auf den Rundungsfehler durchgeführt und das Ergebnis wird dem Wert der Parent-Variable hinzugefügt

dieser angepasste Wert der Parent-Variable wird dann verwendet, um den nächsten abhängigen Wert zu berechnen.

 

Berechnung nach Anteil

Diese Methode wird verwendet, wenn die Spalte "Berechn. nach Anteil?" auf 1 gesetzt ist. Hier kann der Benutzer Werte wie "10-10-15" eingeben, um ein Ziel auf 3 Maschinen zu verteilen, von denen eine 1,5 mal so schnell arbeiten kann wie die anderen, anstatt die entsprechenden Faktoren von "0.2857-0,2857-0.4286" eingeben zu müssen.

Damit dies als vollumfänglicher Ersatz für die Verteilung via Faktor dient, wurde ein zusätzlicher Parent-Faktor eingeführt, mit dem Faktor der Parent-Wert multipliziert wird. Andernfalls könnte der Benutzer keine Ziele übergeben, bei denen die Summe der benötigten Teile ein Vielfaches des Parent-Werts ist (z.B. "produziere 2 Ohren pro Kopf").

Die Berechnung nach Anteil/Parent-Faktor erfolgt so:

alle Anteils-Werte der abhängigen Variablen einer Vererbungskette summieren, um einen Gesamtanteil zu erhalten.

für jeden abhängigen Wert wird (((Parent-Variable * Parent-Faktor + Parent-Offset) * Anteil) / Gesamtanteil) + Offset berechnet

wenn alle abhängigen Variablen denselben Parent-Faktor und Parent-Offset haben oder wenn sie gar keinen haben, und wenn alle abhängigen Variablen dieselbe Rundungs-Genauigkeit haben, dann wird die letzte abhängige Variable als Restwert berechnet, um Parent-Variable * Parent-Faktor + Parent-Offset zu vervollständigen. Dabei werden etwaige depd_offsets von vorangegangenen abhängigen Variablen nicht berücksichtigt. Ein einfaches Beispiel: Mit Anteilen von "1-1-1", kein Parent-Faktor, keine Offsets jedweder Art und Integer-Rundung würde ein Parent-Wert von 100 folgendermaßen verteilt "33-33-34".
