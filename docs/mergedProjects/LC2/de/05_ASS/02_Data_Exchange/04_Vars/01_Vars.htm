Funktion

Wertgebundene Daten (Variablen) werden in dieser Tabelle parametriert. Variablen können von einem der Datentypen numerisch, String oder Bool sein.

Für jede parametrierte Variable wird ein Eintrag in der Datenbank vorgenommen. Zur Laufzeit können dann für diese Variablen Werte aus dem Gateway übergeben und in der Datenbank abgelegt werden. Dabei können jeweils nur der letzte übergebene Wert (Versorgung der Übersicht) oder alle übergebenen Werte mit Zeitstempel (für eine Messwertanzeige) gespeichert werden. Umgekehrt ist es möglich Variablenwerte auch über das Gateway an den Prozess zu übergeben (Sollwerte).

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer. Wird vom System vergeben. Schreibgeschützt.

Verbindung

Lookup-Liste zur Auswahl der Verbindung. Wird automatisch befüllt und kann geändert werden.

Variablensymbol (wie im GW)

Nur bei Modultyp <Gateway> relevant.

Vollständiger Variablenname aus dem Gateway-Projekt. Das Buchen von Werten vom Gateway in die Datenbank bzw. das Übergeben von (Soll-)Werten von der Datenbank an ein Gateway erfolgt standardmäßig über das Variablensymbol.

Das Variablensymbol ist mit dem vollständigen Pfad der Variable im Gateway-Projekt anzugeben bzw. zu importieren

Beispiel einer OPC-Variable: OPC.GOPS.SPS_VMONT1.Istwert

Name

Frei wählbarer Name der Variablen. Dieser Name wird für die Anzeige des Wertes im Webclient verwendet.

Beschreibung (optional)

Zusätzliche Beschreibung der Variablen.

Parent

Lookup-Liste zur Auswahl eines übergeordneten Datensatzes. Wird zur Parametrierung von abhängigen Werten verwendet (z.B. um eine Variable "Taktzeit" in "Anlaufzeit", "Rüstzeit", "Durchlaufzeit" zu splitten).

SPS-Operand (optional)

SPS-Operand zur Adressierung der Variable in der angebundenen Steuerung (wird abhängig vom verwendeten Gateway-Projekt berücksichtigt).

Datentyp

Datentyp der Variable. Zulässige Typen sind Numerisch, Boolean oder String.

Variablenart

Auswahlfeld für die Art der Variablen. Die Art dient der Definition der Verwendung in der Applikation.
Die Art der Definition hat direkte Auswirkung auf die KPI-Berechnung (Verfügbarkeit) !
statistikrelevant (KPI)     Iststückzahl
                                                Sollstückzahl
                                                Soll-Taktzeit

nicht statistikrelevant     Ist-Taktzeit
                                                Grenzwert
                                                Pufferzähler
                                                Geplante Verfügbarkeit
                                                Trendstückzahl
                                                Sollstückzahl aktuell
                                                Wartungszähler
                                                etc.

Die wählbaren Variablenarten sind unter Grundkonfiguration | Allgemeine Anwendungen | Variablenarten festgelegt (siehe auch dort).

Einheit (optional)

Lookup-Liste zur Auswahl einer Einheit, die im WebClient mit dem Variablenwert angezeigt wird. Dargestellt wird dabei jeweils die gewählte Kurzbezeichnung, z.B. °C, min, s, Stk., etc.

Darstellungsformat (optional)

Normierung der Darstellung des Variablenwertes im WebClient für numerische Variablen. Siehe auch das Unterkapitel Darstellungsformate.

Hinweis: Bei numerischen Variablen sollte hier immer ein Format vergeben werden, damit auch andere Anwendungen, die mit den Variablen arbeiten korrekt funktionieren.

Weitergabe an GW (optional)

Checkbox. Wenn aktiv, soll die Variable bei Wertänderung dem Gateway zur Verfügung gestellt werden. Die Variable wird dann in die Schnittstellentabelle P_OUT_VALUES der Datenbank gebucht, statt in die Tabelle P_VALUES_CURRENT. Von dort wird der Wert von der SPS abgeholt und in P_VALUES_CURRENT geschrieben.

siehe auch Kapitel Gateway und Schnittstellentabelle.

Weitergabe optimieren (optional)

Nur bei Variablen mit Weitergabe an GW = aktiv relevant.

Checkbox nicht aktiv: Jede Wertänderung der Variable wird in die Schnittstellentabelle eingetragen.

Checkbox aktiv: Es wird eine Warteschlangenoptimierung für die Weitergabe an das Gateway durchgeführt. Ist vor dem Eintrag eines Wertes noch ein älterer Wert der gleichen Variablen im Übergabebereich (Wert wurde noch nicht vom Gateway abgeholt), so wird kein neuer Eintrag erstellt, sondern lediglich der Wert des vorhandenen Eintrags aktualisiert.

Faktor

Faktor zur Berechnung der Variablen. Wird vom Gateway für Berechnungen verwendet. Falls kein Gateway angebunden ist, wird der Faktor nicht berücksichtigt (z.B. für Verbindung "Interne Werte). Default = 1.

Offset

Gibt das Wertbuchungs-Intervall der Variablen in Sekunden an. Mögliche Werte sind 86.400 = Tag, 28.800 = Schicht (8 Stunden), 3.600 = Stunde. Variablen, die hier einen Eintrag haben, werden im Boardlet "Periodenbezogene Wertbuchung" angezeigt.

Untere/Obere Warngrenze

Untere/Obere Alarmgrenze (optional)

Für die Alarm- und Warngrenzen eines Wertes kann jeweils eine Variable (mit numerischen Wert) festgelegt werden. Bei Erreichung einer dieser Wertegrenzen erhält der Prozesswert in der Übersicht des Clientprogramms einen parametrierbaren Farbumschlag und gleichzeitig wird vom System eine Störmeldung erzeugt, die automatisch in das Störmeldesystem eingebucht wird.
Siehe auch Werksverwaltung | Meldungsmanagement | Meldeklassen.

Siehe auch VALUE_ALARMS.

Archiv (optional)

Checkbox. Wenn aktiv, wird die Variable mit Historie im Wertarchiv gespeichert werden und steht somit als Messwert zur Verfügung. Wertverlauf der Variable wird gespeichert, jede Wertänderung wird mit dem Zeitstempel der Änderung archiviert.

Checkbox nicht aktiv: Variable wird mit ihrem aktuellen Wert und Zeitstempel der letzten Änderung gespeichert.

Auf die Berechnung der KPIs hat diese Checkbox keine Auswirkung.

Diskret?

Checkbox: 1 = der Variablenwert soll diskret in der Messwertanzeige interpretiert werden (z.B. geeignet für Stückzahlen mit ganzzahligen Werten). 0 = der Variablenwert soll als kontinuierlicher Verlauf angezeigt werden (z.B. geeignet für Temperatur-Kurven).

Delta (optional)

Nur bei Variablen mit Archivieren relevant.

Bei Angabe eines Wertes ungleich 0 wird für die Messwerterfassung nur ein Wert gespeichert, wenn die Differenz zum letzten gespeicherten Wert größer / gleich dem Delta ist. Bei Delta = 0 oder <leer> werden alle übergebenen Werte gespeichert. Die Voreinstellung dieses Parameters ist 0.

Aktueller Wert

Hier wird der aktuelle Wert der Variablen angezeigt. Nur zur Information; schreibgeschützt.

Letztes Update

Hier wird der Zeitstempel der letzten Wertänderung der Variablen angezeigt. Nur zur Information; schreibgeschützt.

Aus Import

Checkbox: 1 = Variable wurde durch den letzten Variablen-Import der AE angelegt/geändert.

0 / NULL = Variable war nicht Bestandteil des letzten Variablen-Imports.

Die Spalte wird automatisch durch die AE befüllt (ab Version 4.6.0.2). Sie dient lediglich zur Information und ist schreibgeschützt.

Max. Wert (optional)

Lookup-Liste zur Auswahl einer anderen Variable, die den maximalen Wert vorgibt.

Min. Wert (optional)

Lookup-Liste zur Auswahl einer anderen Variable, die den minimalen Wert vorgibt.

Sollwert (optional)

Lookup-Liste zur Auswahl einer anderen Variable, die den Sollwert vorgibt.

Farb-Klasse

FK - Verweis auf eine Farbe, in der dieser Wert in Charts dargestellt werden soll. Die Farbcodes selbst sind im jeweiligen CSS-File definiert.

Wenn hier nicht spezifiziert, wird die beim Wert-Typ definierte Farbe verwendet.

Kontextmenü

Durch einen Klick mit der rechten Maustaste auf eine Variable entweder im Navigationsbaum oder am Anfang der Tabellenzeile öffnet sich ein Kontextmenü. Darüber kann der Wert der Variable gesetzt werden.

 

Wert setzen

Für Testzwecke kann der Wert einer Variablen gesetzt werden, damit er im WebClient sichtbar wird.

In dem erscheinenden Dialog kann der neue Wert der Variable eingegeben werden, die Syntax der Eingabe wird überwacht (z.B. keine Buchstaben bei num. Variablen). Durch Aktivieren der Checkbox "Set Value to NULL" wird der Wert der Variablen auf NULL gesetzt. Der derzeitige Wert der Variable ist im Eingabefeld vorbelegt.

Wenn die Variable mit Weitergabe an Gateway parametriert ist, wird der geänderte Wert erst dann eingetragen, wenn die Übergabe des Wertes vom Gateway mit gw_val_transmitted quittiert wurde! (siehe dazu auch weiter unten den Abschnitt P_OUT_VALUES).

Wert setzen (Multi)

Hiermit kann der Wert mehrerer Variablen gleichen Datentyps gleichzeitig gesetzt werden. Haben die selektierten Variablen unterschiedliche Datentypen, wird eine Fehlermeldung ausgegeben.

In dem erscheinenden Dialog kann der neue Wert der Variablen eingegeben werden, die Syntax der Eingabe wird überwacht (z.B. keine Buchstaben bei num. Variablen). Durch Aktivieren der Checkbox "Set Value to NULL" wird der Wert aller Variablen auf NULL gesetzt. Der derzeitige Wert der zuletzt markierten Variable ist im Eingabefeld vorbelegt.

Dem Default-Knoten zuordnen

Die Variable wird dem Default-Knoten der Verbindung direkt zugeordnet. Dieser Menüeintrag steht auch bei Mehrfachauswahl zur Verfügung.

Layout-Snippet erstellen (Liste)

Erstellt ein Layout-Snippet mit den Databinding-Informationen der selektierten Variablen (Mehrfachauswahl möglich) und kopiert diese Info in die Zwischenablage. Es wird eine Zeile je selektierter Variable erstellt. Aus der Zwischenablage kann die Information in eine Visualisierung eingefügt werden, um Variablen anzubinden. Für weitere Informationen, siehe bitte Kapitel Erweiterte Bedienung - Legato Graphic Designer.

Layout-Snippet erstellen (Summe)

Erstellt ein Layout-Snippet mit den Databinding-Informationen der selektierten Variablen (Mehrfachauswahl möglich) und kopiert diese Info in die Zwischenablage. Es wird die Summe der selektierten Variablen ausgegeben. Aus der Zwischenablage kann die Information in eine Visualisierung eingefügt werden, um Variablen anzubinden. Für weitere Informationen, siehe bitte Kapitel Erweiterte Bedienung - Legato Graphic Designer.
