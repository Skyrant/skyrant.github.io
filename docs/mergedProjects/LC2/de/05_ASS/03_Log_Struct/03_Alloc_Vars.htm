Funktion

Jedem Knoten können Variablen einer Verbindung zugeordnet werden. Zugeordnete Variablen können im WebClient angezeigt werden und mit Statistiken zu diesem Knoten verknüpft werden.

Feldbeschreibung

 

Feld

Beschreibung

ID

Eindeutige Identifikationsnummer. Wird vom System vergeben. Schreibgeschützt.

Knoten

Lookup-Liste zur Auswahl des Knotens. Wird automatisch befüllt und kann geändert werden.

Variable

Lookup-Liste zur Auswahl einer Variablen.

Anzeigen (optional)

Checkbox. Wenn aktiv, wird die Variable im WebClient in der Übersicht angezeigt; sonst nicht.

Änderbar (optional)

Checkbox. Wenn aktiv, kann die Variable im WebClient in der Übersicht editiert werden, wenn der Benutzer die entsprechenden Rechte für diesen Knoten besitzt.

Sortiernummer (optional)

Mehrere Variablen können durch Angabe einer Sortiernummer aufsteigend sortiert werden.

Protokoll (optional)

Checkbox. Wenn aktiv, wird die Variable mit dem letzten Wert ins Schichtprotokoll übernommen. Wenn nicht aktiv, wird die Variable nicht im Schichtprotokoll gespeichert.

Das Flag "Statistik-relevant" sorgt auch dafür, dass für die KPI-Berechnung benötigte Werte ins Schichtarchiv geschrieben werden. Es ist also nicht nötig, das Flag doppelt zu setzen.

: Handelt es sich um eine Variable vom Typ Sollstückzahl oder Soll-Taktzeit, die über Sollwertvorgaben gepflegt werden soll, darf diese Checkbox nicht aktiv sein. Die Checkbox muss aktiv sein, wenn es sich um statistikrelevante Variablen vom Typ Sollstückzahl oder Soll-Taktzeit handelt, die für KPI-Auswertungen ohne Sollwertvorgaben oder für vererbte Sollwertvorgaben (Spalte Automatisch berechnen) herangezogen werden sollen. Siehe auch das eigene Kapitel "Konfiguration von Sollwerten".

Statistik-relevant (optional)

Checkbox. Wenn aktiv, wird die Variable für statistische Berechnungen verwendet. Variablen vom Typ Aktuellstückzahl / Sollstückzahl oder Soll-Taktzeit können für den gewählten Knoten statistikrelevant sein und bilden damit entsprechend ihres Typs die Basis für statistische Berechnungen (die Beschreibung der KPIs PREAL, PTARG, CTARG).

: Variablen für Online-KPIs dürfen nicht statistik-relevant markiert sein.

Autom. berechnen (optional)

Checkbox. Wenn aktiv, wird die Variable über die automatische Sollwertberechnung versorgt.

: Werden solche Werte für die KPI-Berechnung herangezogen, müssen die Spalten "Schichtprotokoll" und "Statistik-relevant" auf 1 gesetzt sein.

Siehe auch Unterkapitel "Automatische Sollwertberechnung".

Bauteilbez. speichern (optional)

Checkbox. Wenn aktiv, wird die Variable inklusive Bauteildetails gespeichert (relevant für das Modul Bauteilverfolgung).

KPI-Typ

Verweis auf einen KPI-Typ. Dieses Feld wird bei Variablen für Online-KPIs automatisch mit dem entsprechenden KPI-Typ befüllt. Nur zur Information, schreibgeschützt.

KPI Delta

Dieser Wert ist wichtig für Online-KPIs, danach richtet sich die AppEngine bei der Berechnung. Dieser Eintrag kann beim jeweiligen KPI gepflegt werden. Hier dient es nur der Information, schreibgeschützt.

Aktueller Wert

Hier wird der aktuelle Wert der Variablen angezeigt. Nur zur Information; schreibgeschützt.

Letztes Update

Hier wird der Zeitstempel der letzten Wertänderung der Variablen angezeigt. Nur zur Information; schreibgeschützt.

 

Kontextmenü

Wert setzen

Durch einen Klick mit der rechten Maustaste auf eine Variable entweder im Navigationsbaum oder am Anfang der Tabellenzeile öffnet sich ein Kontextmenü. Darüber kann der Wert der Variable gesetzt werden.

Für Testzwecke kann der Wert einer Variablen gesetzt werden, damit er im WebClient sichtbar wird.

In dem erscheinenden Dialog kann der neue Wert der Variable eingegeben werden, die Syntax der Eingabe wird überwacht (z.B. keine Buchstaben bei num. Variablen). Durch Aktivieren der Checkbox "Set Value to NULL" wird der Wert der Variablen auf NULL gesetzt. Der derzeitige Wert der Variable ist im Eingabefeld vorbelegt.

Wenn die Variable mit Weitergabe an Gateway parametriert ist, wird der geänderte Wert erst dann eingetragen, wenn die Übergabe des Wertes vom Gateway mit gw_val_transmitted quittiert wurde!
